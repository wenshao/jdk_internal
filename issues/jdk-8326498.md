# JDK-8326498: HttpClient HTTP/2 连接泄漏修复

> **严重程度**: 🔴 高
> **影响版本**: JDK 21-25
> **修复版本**: JDK 26
> **Commit**: `c19b12927d2`
> **作者**: Jaikiran Pai

---

## 1. 概述

这是一个严重的 bug，导致 HTTP/2 连接在使用后未正确释放回连接池，最终导致连接池耗尽和服务不可用。

---

## 2. 问题描述

### 症状

```
1. 应用长时间运行后，HTTP 请求开始超时
2. 日志显示 "connection pool exhausted"
3. 重启后恢复正常，但问题会再次出现
4. 监控显示连接数持续增长
```

### 复现场景

```java
HttpClient client = HttpClient.newBuilder()
    .version(HttpClient.Version.HTTP_2)
    .build();

// 循环发送请求
for (int i = 0; i < 10000; i++) {
    HttpRequest request = HttpRequest.newBuilder()
        .uri(URI.create("https://api.example.com/data"))
        .build();

    HttpResponse<String> response = client.send(
        request,
        HttpResponse.BodyHandlers.ofString()
    );

    // 问题: 某些情况下连接未释放
    Thread.sleep(100);
}

// 最终: 连接池耗尽，新请求阻塞
```

### 影响范围

| 场景 | 影响 |
|------|------|
| 微服务调用 | 高频调用导致连接泄漏 |
| API 网关 | 请求转发失败 |
| 长时间运行服务 | 最终 OOM 或超时 |

## 3. 根因分析

### 问题代码

**文件**: `src/java.net.http/share/classes/jdk/internal/net/http/Http2Connection.java`

```java
// 问题代码 (简化)
class Http2Connection {
    private volatile State state;

    void close() {
        if (state == ACTIVE) {
            sendGoAway();
            // 问题: 某些异常路径未调用 close
            // 问题: 未从连接池移除
        }
    }

    // 异常处理路径
    void handleStreamError(Throwable t) {
        // 问题: 只关闭流，未处理连接状态
        stream.close();
        // 缺少: 连接池清理
    }
}
```

### 泄漏路径分析

```
正常路径:
请求 → 获取连接 → 发送请求 → 收到响应 → 释放连接 ✓

泄漏路径 1:
请求 → 获取连接 → 发送请求 → 流错误 → 连接未释放 ✗

泄漏路径 2:
请求 → 获取连接 → 超时 → 连接未正确关闭 ✗

泄漏路径 3:
请求 → 获取连接 → 服务器关闭连接 → 客户端未感知 ✗
```

## 4. 解决方案

### 新增终止原因枚举

**文件**: `src/java.net.http/share/classes/jdk/internal/net/http/Http2TerminationCause.java`

```java
/**
 * HTTP/2 连接终止原因
 */
public enum Http2TerminationCause {
    /** 正常关闭 */
    GRACEFUL_SHUTDOWN,

    /** 流错误导致关闭 */
    STREAM_ERROR,

    /** 连接错误 */
    CONNECTION_ERROR,

    /** 空闲超时 */
    IDLE_TIMEOUT,

    /** 收到 GOAWAY 帧 */
    GOAWAY_RECEIVED,

    /** 协议错误 */
    PROTOCOL_ERROR,

    /** 流被拒绝 */
    REFUSED_STREAM,

    /** 内部错误 */
    INTERNAL_ERROR;

    /**
     * 是否需要发送 GOAWAY 帧
     */
    public boolean shouldSendGoAway() {
        return this != GOAWAY_RECEIVED && this != IDLE_TIMEOUT;
    }

    /**
     * 是否为错误
     */
    public boolean isError() {
        return this != GRACEFUL_SHUTDOWN && this != IDLE_TIMEOUT;
    }
}
```

### 修复连接关闭逻辑

**文件**: `src/java.net.http/share/classes/jdk/internal/net/http/Http2Connection.java`

```java
class Http2Connection {

    /**
     * 统一的连接终止方法
     */
    private void terminate(Http2TerminationCause cause) {
        synchronized (this) {
            if (state == CLOSED) {
                return;  // 已关闭
            }
            state = CLOSED;
        }

        try {
            // 1. 发送 GOAWAY 帧 (如果需要)
            if (cause.shouldSendGoAway()) {
                sendGoAway(cause);
            }

            // 2. 关闭所有活动流
            closeAllStreams(cause);

            // 3. 从连接池移除 (关键修复!)
            connectionPool().remove(this);

            // 4. 关闭底层 TCP 连接
            closeUnderlyingConnection();

            // 5. 记录日志
            if (logger.isLoggable(Level.FINE)) {
                logger.fine("Connection terminated: " + cause);
            }

        } catch (Exception e) {
            logger.log(Level.WARNING, "Error during connection termination", e);
        }
    }

    /**
     * 处理流错误
     */
    void handleStreamError(Http2Stream stream, Throwable error) {
        // 关闭单个流
        stream.close();

        // 检查是否需要关闭整个连接
        if (shouldCloseConnection(stream, error)) {
            terminate(Http2TerminationCause.STREAM_ERROR);
        }
    }

    /**
     * 处理连接错误
     */
    void handleConnectionError(Throwable error) {
        terminate(Http2TerminationCause.CONNECTION_ERROR);
    }

    /**
     * 空闲超时处理
     */
    void onIdleTimeout() {
        terminate(Http2TerminationCause.IDLE_TIMEOUT);
    }

    /**
     * 收到 GOAWAY 帧
     */
    void onGoAwayReceived(GoAwayFrame frame) {
        terminate(Http2TerminationCause.GOAWAY_RECEIVED);
    }
}
```

### 连接池管理改进

**文件**: `src/java.net.http/share/classes/jdk/internal/net/http/ConnectionPool.java`

```java
class ConnectionPool {

    /**
     * 移除连接 (新增方法)
     */
    void remove(Http2Connection conn) {
        synchronized (pool) {
            PoolKey key = conn.poolKey();
            Set<Http2Connection> connections = pool.get(key);

            if (connections != null) {
                connections.remove(conn);

                // 清理空集合
                if (connections.isEmpty()) {
                    pool.remove(key);
                }
            }
        }

        // 更新统计
        metrics.recordConnectionRemoved(conn);
    }

    /**
     * 清理泄漏连接 (新增方法)
     */
    void cleanupLeakedConnections() {
        synchronized (pool) {
            for (Set<Http2Connection> connections : pool.values()) {
                connections.removeIf(conn -> {
                    if (conn.isClosed() || conn.isExpired()) {
                        metrics.recordConnectionLeaked(conn);
                        return true;
                    }
                    return false;
                });
            }
        }
    }
}
```

### 新增测试用例

**文件**: `test/jdk/java/net/http/Http2ConnectionLeakTest.java`

```java
/**
 * @test
 * @bug 8326498
 * @summary Verify HTTP/2 connections are properly released
 */
public class Http2ConnectionLeakTest {

    @Test
    public void testConnectionNotLeaked() throws Exception {
        // 创建 HTTP/2 服务器
        try (var server = SimpleHttp2Server.create()) {
            HttpClient client = HttpClient.newBuilder()
                .version(HttpClient.Version.HTTP_2)
                .build();

            // 发送大量请求
            for (int i = 0; i < 1000; i++) {
                HttpRequest request = HttpRequest.newBuilder()
                    .uri(server.uri("/test"))
                    .build();

                HttpResponse<String> response = client.send(
                    request,
                    HttpResponse.BodyHandlers.ofString()
                );

                assertEquals(200, response.statusCode());
            }

            // 验证连接池状态
            // 应该只有一个连接被复用
            assertEquals(1, getConnectionPoolSize(client));
        }
    }

    @Test
    public void testConnectionReleasedOnError() throws Exception {
        try (var server = SimpleHttp2Server.create()) {
            // 配置服务器返回错误
            server.setResponseCode(500);

            HttpClient client = HttpClient.newBuilder()
                .version(HttpClient.Version.HTTP_2)
                .build();

            // 发送请求 (会失败)
            for (int i = 0; i < 100; i++) {
                try {
                    HttpRequest request = HttpRequest.newBuilder()
                        .uri(server.uri("/error"))
                        .build();
                    client.send(request, HttpResponse.BodyHandlers.ofString());
                } catch (Exception expected) {
                    // 预期失败
                }
            }

            // 验证连接未泄漏
            assertTrue(getConnectionPoolSize(client) <= 1);
        }
    }

    @Test
    public void testConnectionReleasedOnTimeout() throws Exception {
        try (var server = SimpleHttp2Server.create()) {
            // 配置服务器延迟响应
            server.setDelay(Duration.ofSeconds(30));

            HttpClient client = HttpClient.newBuilder()
                .version(HttpClient.Version.HTTP_2)
                .connectTimeout(Duration.ofSeconds(1))
                .build();

            // 发送请求 (会超时)
            for (int i = 0; i < 10; i++) {
                try {
                    HttpRequest request = HttpRequest.newBuilder()
                        .uri(server.uri("/slow"))
                        .timeout(Duration.ofSeconds(1))
                        .build();
                    client.send(request, HttpResponse.BodyHandlers.ofString());
                } catch (HttpTimeoutException expected) {
                    // 预期超时
                }
            }

            // 验证连接未泄漏
            assertTrue(getConnectionPoolSize(client) <= 1);
        }
    }
}
```

### 压力测试

**文件**: `test/jdk/java/net/http/BurstyRequestsTest.java`

```java
/**
 * @test
 * @bug 8326498
 * @summary Stress test for connection pool
 */
public class BurstyRequestsTest {

    @Test
    public void testBurstyRequests() throws Exception {
        try (var server = SimpleHttp2Server.create()) {
            HttpClient client = HttpClient.newBuilder()
                .version(HttpClient.Version.HTTP_2)
                .build();

            // 模拟突发请求
            ExecutorService executor = Executors.newVirtualThreadPerTaskExecutor();

            for (int burst = 0; burst < 10; burst++) {
                // 突发 100 个并发请求
                List<Future<Void>> futures = new ArrayList<>();
                for (int i = 0; i < 100; i++) {
                    futures.add(executor.submit(() -> {
                        HttpRequest request = HttpRequest.newBuilder()
                            .uri(server.uri("/test"))
                            .build();
                        client.send(request, HttpResponse.BodyHandlers.ofString());
                        return null;
                    }));
                }

                // 等待完成
                for (Future<Void> future : futures) {
                    future.get();
                }

                // 间隔
                Thread.sleep(1000);

                // 验证连接池
                int poolSize = getConnectionPoolSize(client);
                System.out.println("Burst " + burst + " pool size: " + poolSize);
                assertTrue(poolSize <= 10, "Connection pool should not grow unbounded");
            }
        }
    }
}
```

## 6. 影响评估

### 修复前 vs 修复后

| 指标 | 修复前 | 修复后 |
|------|--------|--------|
| 长时间运行 | 连接泄漏 | 正常 |
| 高并发请求 | 可能阻塞 | 正常 |
| 错误处理 | 连接残留 | 正确清理 |
| 内存占用 | 持续增长 | 稳定 |

### 升级建议

| 当前版本 | 建议 |
|----------|------|
| JDK 21-25 | 升级到 JDK 26 或应用补丁 |
| JDK 26+ | 已修复 |

### 临时解决方案

如果无法立即升级：

```java
// 方案 1: 定期重建 HttpClient
ScheduledExecutorService scheduler = Executors.newScheduledThreadPool(1);
scheduler.scheduleAtFixedRate(() -> {
    oldClient.close();
    client = HttpClient.newBuilder()
        .version(HttpClient.Version.HTTP_2)
        .build();
}, 1, 1, TimeUnit.HOURS);

// 方案 2: 限制连接池大小
HttpClient client = HttpClient.newBuilder()
    .version(HttpClient.Version.HTTP_2)
    .executor(Executors.newFixedThreadPool(100))  // 限制并发
    .build();
```
