# JDK-8359683: ZGC NUMA-Aware 优化

> **严重程度**: 🟡 中
> **影响版本**: 多插槽服务器上的 ZGC
> **修复版本**: JDK 26
> **作者**: Roman Kennke (Datadog)

---

## 1. 概述

改进 ZGC 的 NUMA 感知策略，使 ZGC 在分配和重定位对象时优先使用本地 NUMA 节点的内存，减少跨节点远程内存访问延迟。在多插槽大内存服务器上效果尤为显著。

---

## 2. 背景

### ZGC 与 NUMA

ZGC 作为低延迟垃圾收集器，其核心设计目标之一就是最小化 GC 停顿时间。然而在 NUMA 架构上，ZGC 的对象分配和移动操作可能大量访问远程内存，导致性能下降。

```
ZGC 在 NUMA 上的挑战:
┌──────────────────────────────────────────────────┐
│               ZGC 堆内存布局                     │
│  ┌──────────┬──────────┬──────────┬──────────┐  │
│  │ ZPage 0  │ ZPage 1  │ ZPage 2  │ ZPage 3  │  │
│  │ Node 0   │ Node 0   │ Node 1   │ Node 1   │  │
│  └──────────┴──────────┴──────────┴──────────┘  │
│                                                  │
│  问题: 对象分配不感知 NUMA → 随机节点分配        │
│  结果: 50% 概率访问远程内存 (+60% 延迟)          │
└──────────────────────────────────────────────────┘
```

---

## 3. 实现方案

### NUMA 感知分配策略

```
优化后的分配策略:

1. 对象分配 (Object Allocation):
   ┌─────────────────────────────────────┐
   │ 当前线程运行在 CPU 5 → NUMA Node 0 │
   │ → 从 NUMA Node 0 的 ZPage 分配     │
   │ → 本地内存访问 (~80ns)             │
   └─────────────────────────────────────┘

2. 对象重定位 (Object Relocation):
   ┌─────────────────────────────────────┐
   │ GC 线程在 Node 0 上运行            │
   │ → 将存活对象重定位到 Node 0 的 Page│
   │ → 后续访问为本地访问               │
   └─────────────────────────────────────┘

3. 大页面 (Large Pages):
   ┌─────────────────────────────────────┐
   │ 大于 50% page size 的对象           │
   │ → 在创建页面时指定 NUMA 节点       │
   └─────────────────────────────────────┘
```

### ZPage NUMA 标记

ZGC 为每个 ZPage 添加了 NUMA 节点标记：

```cpp
// ZPage 的 NUMA 感知 (简化)
class ZPage {
    const uint32_t _numa_node;  // 新增: 所属 NUMA 节点

    // 分配时优先选择当前线程所在节点的 page
    static ZPage* alloc_page(uint32_t numa_node, size_t size);
};
```

---

## 4. 性能影响

在 4 插槽 AMD EPYC 服务器（128 核 / 1TB RAM）上的测试：

| 指标 | 优化前 | 优化后 | 改善 |
|------|--------|--------|------|
| 对象分配延迟 | 120ns | 82ns | -32% |
| GC 重定位吞吐 | 1.8 GB/s | 2.5 GB/s | +39% |
| 应用吞吐量 (128GB 堆) | baseline | +8% | +8% |
| 应用吞吐量 (512GB 堆) | baseline | +18% | +18% |
| 远程内存访问比例 | ~48% | ~12% | -75% |

> **说明**: 性能提升与堆大小正相关。在单插槽或小堆（<32GB）场景下改善不明显。

## 5. 配置

```bash
# ZGC + NUMA（JDK 26 默认启用 NUMA 感知）
java -XX:+UseZGC -XX:+ZGenerational -XX:+UseNUMA -Xmx128g -jar app.jar

# 验证 NUMA 感知是否生效
java -XX:+PrintFlagsFinal -version 2>&1 | grep -i numa
#     bool UseNUMA                          = true
```
