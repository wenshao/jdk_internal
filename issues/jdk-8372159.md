## 1. 概述

优化 `java.net.http.HttpClient` 在虚拟线程环境下的内存使用和调度行为。当 `HttpClient.send()` 在虚拟线程上调用时，减少了载体线程（carrier thread）的钉住（pinning）时间，并降低了每个连接的内存开销。

---

## 2. 问题描述

### 原有问题

```java
// 在虚拟线程上使用 HttpClient
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    executor.submit(() -> {
        HttpClient client = HttpClient.newHttpClient();
        // send() 内部可能阻塞等待响应
        // → 虚拟线程被钉住在 carrier thread 上
        // → 占用 carrier thread 资源
        HttpResponse<String> resp = client.send(request,
            HttpResponse.BodyHandlers.ofString());
    });
}
```

问题：
1. **内部锁使用 synchronized**：HttpClient 内部使用 `synchronized` 保护连接池状态，导致虚拟线程 pinning
2. **Selector 线程与虚拟线程的交互**：I/O 事件通知路径存在不必要的阻塞
3. **内存开销**：每个 HTTP/2 连接分配了固定大小的 buffer，即使连接空闲也占用内存

---

## 3. 解决方案

### 3.1 替换 synchronized 为 ReentrantLock

```java
// 之前: synchronized 导致虚拟线程 pinning
synchronized (connectionPool) {
    connection = connectionPool.getConnection();
}

// 之后: ReentrantLock 允许虚拟线程正常卸载
connectionPoolLock.lock();
try {
    connection = connectionPool.getConnection();
} finally {
    connectionPoolLock.unlock();
}
```

### 3.2 异步 I/O 通知优化

```
优化前:
  VirtualThread ──send()──► HttpClient ──block──► SelectorThread
                                                          │
  VirtualThread ◄──waiting────────────────────────────────┘
  (carrier thread 被钉住)

优化后:
  VirtualThread ──send()──► HttpClient ──register──► SelectorThread
       │                                                      │
       └──unmount─────────────────────────────────────────────┘
  (carrier thread 释放)

  SelectorThread ──event ready──► VirtualThread ──remount──► continue
```

### 3.3 内存优化

| 组件 | 优化前 | 优化后 |
|------|--------|--------|
| HTTP/2 连接 buffer | 固定 16KB | 按需分配，最小 4KB |
| SSL session 缓存 | 强引用持有 | 软引用 + LRU |
| 响应体 buffer | 预分配 | 延迟分配 |

## 4. 性能影响

| 指标 | 优化前 | 优化后 | 改善 |
|------|--------|--------|------|
| 10,000 并发请求内存 | 2.4 GB | 1.6 GB | -33% |
| carrier thread 利用率 | 85% (大量 pinning) | 45% | -47% |
| 平均响应延迟 (虚拟线程) | 45ms | 38ms | -16% |
| 最大可并发连接数 (虚拟线程) | ~5,000 | ~15,000 | 3x |

## 5. 使用建议

```java
// JDK 26+: 虚拟线程 + HttpClient 推荐用法
HttpClient client = HttpClient.newBuilder()
    .version(HttpClient.Version.HTTP_2)  // HTTP/2 多路复用
    .build();

// 虚拟线程中安全使用
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    List<Future<HttpResponse<String>>> futures = urls.stream()
        .map(url -> executor.submit(() ->
            client.send(HttpRequest.newBuilder(url).build(),
                HttpResponse.BodyHandlers.ofString())))
        .toList();
}
```
