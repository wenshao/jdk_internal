# JDK-8371475: CUBIC 拥塞控制实现

> **严重程度**: 🟡 中
> **影响版本**: 无 (新功能)
> **修复版本**: JDK 26
> **作者**: Chris Hegarty (Oracle)

---

## 1. 概述

为 HTTP/3 (QUIC) 实现了 CUBIC 拥塞控制算法（RFC 9438），替代默认的 Reno/Cubic 简化实现。CUBIC 是现代互联网最广泛部署的拥塞控制算法，能更好地利用高带宽长距离链路。

---

## 2. 背景

### 为什么需要 CUBIC?

HTTP/3 基于 QUIC 协议，QUIC 需要自带的拥塞控制（不能依赖 TCP 内核实现）。JDK 的 HTTP/3 实现需要包含传输层拥塞控制。

```
Reno vs CUBIC 对比:
┌──────────────┬─────────────────────┬──────────────────────┐
│ 特性         │ Reno                │ CUBIC                │
├──────────────┼─────────────────────┼──────────────────────┤
│ 窗口增长函数 │ 线性 (AIMD)         │ 三次函数 (cubic)     │
│ 高带宽利用   │ 差                  │ 优秀                 │
│ RTT 公平性   │ RTT 不公平          │ RTT 公平             │
│ 兼容性       │ RFC 5681            │ RFC 9438             │
│ 适用场景     │ 低带宽短距离        │ 高带宽长距离 (现代)  │
└──────────────┴─────────────────────┴──────────────────────┘
```

## 3. 实现架构

### 在 QUIC 栈中的位置

```
┌───────────────────────────────────────────────────┐
│                    HTTP/3 层                      │
├───────────────────────────────────────────────────┤
│                    QUIC 层                        │
│  ┌──────────────────────────────────────────┐    │
│  │            拥塞控制 (Congestion Control)  │    │
│  │  ┌──────────┐  ┌──────────┐              │    │
│  │  │  CUBIC   │  │   Reno   │ (回退)      │    │
│  │  │ (RFC 9438)│  │(RFC 5681)│              │    │
│  │  └──────────┘  └──────────┘              │    │
│  ├──────────────────────────────────────────┤    │
│  │  丢包检测 (Loss Detection)                │    │
│  │  ECN (Explicit Congestion Notification)   │    │
│  ├──────────────────────────────────────────┤    │
│  │  Pacing (流量整形)                        │    │
│  └──────────────────────────────────────────┘    │
├───────────────────────────────────────────────────┤
│                    UDP 层                         │
└───────────────────────────────────────────────────┘
```

### CUBIC 核心公式

CUBIC 的窗口增长函数：

```
W(t) = C × (t - K)³ + W_max

其中:
- W(t)    = t 时刻的拥塞窗口大小
- C       = CUBIC 参数 (默认 0.4)
- t       = 距离上次丢包的时间
- K       = (W_max × β / C)^(1/3)
- W_max   = 上次丢包时的窗口大小
- β       = 乘法减少因子 (默认 0.7)
```

## 4. 关键代码变更

### 拥塞控制接口

```java
// 新增的拥塞控制抽象
interface QuicCongestionControl {
    void onPacketSent(long bytes);
    void onPacketAcked(long ackedBytes, long rtt);
    void onPacketLost(long lostBytes);
    long getCongestionWindow();
    long getPacingRate();
}

// CUBIC 实现
class CubicCongestionControl implements QuicCongestionControl {
    private static final double CUBIC_C = 0.4;
    private static final double CUBIC_BETA = 0.7;

    private long wMax;          // 上次丢包时的窗口
    private long k;             // K 值
    private long epochStart;    // 新 epoch 起始时间
    private long cwnd;          // 当前拥塞窗口
    // ...
}
```

## 5. 性能影响

| 场景 | Reno | CUBIC | 提升 |
|------|------|-------|------|
| 高带宽 (1Gbps, 100ms RTT) | ~200 Mbps | ~900 Mbps | 4.5x |
| 中等带宽 (100Mbps, 50ms RTT) | ~80 Mbps | ~95 Mbps | 19% |
| 低延迟 (10Mbps, 10ms RTT) | ~9 Mbps | ~9.5 Mbps | ~5% |
| 丢包率 1% | 显著下降 | 平缓下降 | 更稳定 |
