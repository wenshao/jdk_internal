## 解决方案

### 核心改动

```java
// 文件: jdk/internal/util/DecimalDigits.java

/**
 * 预计算的数字对数组 (0-99)
 * 每个 short 值按小端序打包两个 ASCII 字符
 */
@Stable
private static final short[] DIGITS;

static {
    short[] digits = new short[128];  // 2^7，配合 & 0x7f 消除边界检查
    for (int i = 0; i < 10; i++) {
        short hi = (short) (i + '0');
        for (int j = 0; j < 10; j++) {
            short lo = (short) ((j + '0') << 8);
            digits[i * 10 + j] = (short) (hi | lo);
        }
    }
    DIGITS = digits;
}

public static void appendPair(StringBuilder buf, int v) {
    int packed = DIGITS[v & 0x7f];
    buf.append(
        JLA.uncheckedNewStringWithLatin1Bytes(
            new byte[] {(byte) packed, (byte) (packed >> 8)}));
}

public static void appendQuad(StringBuilder buf, int v) {
    int packedHigh = DIGITS[(v / 100) & 0x7f];
    int packedLow  = DIGITS[(v % 100) & 0x7f];
    buf.append(
        JLA.uncheckedNewStringWithLatin1Bytes(
            new byte[] {(byte) packedHigh, (byte) (packedHigh >> 8),
                        (byte) packedLow,  (byte) (packedLow  >> 8)}));
}
```

### DateTimeHelper 重构

```java
// 修改后
public static void formatTo(StringBuilder buf, LocalDate date) {
    int year    = date.getYear();
    int absYear = Math.abs(year);

    if (absYear < 10000) {
        if (year < 0) {
            buf.append('-');
        }
        DecimalDigits.appendQuad(buf, absYear);  // 使用新方法
    } else {
        if (year > 9999) {
            buf.append('+');
        }
        buf.append(year);
    }
    buf.append('-');
    DecimalDigits.appendPair(buf, date.getMonthValue());  // 使用新方法
    buf.append('-');
    DecimalDigits.appendPair(buf, date.getDayOfMonth());   // 使用新方法
}
```

## 为什么有效

### 查找表策略

```
对于 value = 47:

原始实现:
├── 检查: 47 < 10? 否
├── 计算: 47 / 10 = 4 (idiv, ~10 周期)
├── 计算: 47 % 10 = 7 (irem, ~10 周期)
└── 转换为字符

优化后:
├── 计算: 47 & 0x7f = 47 (位运算, 1 周期)
├── 加载: DIGITS[47] = 0x3739 (L1 缓存, ~4 周期)
└── 提取字符
```

### 边界检查消除

```java
// JIT 的优化分析:
// 1. 数组长度是 128 (2^7)
// 2. 索引是 v & 0x7f
// 3. (v & 0x7f) 永远在 [0, 127] 范围内
// 4. 结论: 可以安全移除边界检查！
```

## 设计决策

### Q1: 为什么用 short[] 而不是 byte[]？

**A**: 使用 short[] 可将两个 ASCII 字符打包到一个元素：

| 方案 | 内存大小 | 访问次数 |
|------|----------|----------|
| `byte[200]` | 200 bytes | 2 次加载 |
| `short[128]` | 256 bytes | 1 次加载 |

### Q2: 为什么是 128 而不是 100？

**A**: 128 是最小的 2 的幂，能容纳 0-99：
- 允许使用 `& 0x7f` 消除边界检查
- 浪费最少 (28 个元素)
