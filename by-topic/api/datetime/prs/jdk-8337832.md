## 概述

这是 JDK-8337168 的**延续优化**，将 toString() StringBuilder 优化模式扩展到更多 java.time 类。

**关联关系**:

### 为什么 ZonedDateTime 提升最大？

```java
// 优化前 - 5 个字符串分配
public String toString() {
    return date.toString() + 'T' + time.toString() + offset.toString() + zone.toString();
    //     ^^^^^^^^^^^^^^   ^   ^^^^^^^^^^^^^^   ^^^^^^^^^^^^^^^   ^^^^^^^^^^^^
    //     String #1        |   String #2        String #3         String #4
    //                       |
    //                    (拼接产生 String #5)
}

// 优化后 - 1 个字符串分配
public String toString() {
    StringBuilder sb = new StringBuilder(32);  // 预分配
    date.toString(sb);
    sb.append('T');
    time.toString(sb);
    offset.toString(sb);
    sb.append('[');
    zone.toString(sb);
    sb.append(']');
    return sb.toString();
}
```

## 技术细节

### 新增的 API

为多个类添加了 `toString(StringBuilder)` 重载方法：

#### LocalDate.toString(StringBuilder)
```java
void toString(StringBuilder sb) {
    int yearValue = year;
    int monthValue = month;
    int dayValue = day;

    sb.append(yearValue);
    sb.append('-');
    if (monthValue < 10) {
        sb.append('0');
    }
    sb.append(monthValue);
    sb.append('-');
    if (dayValue < 10) {
        sb.append('0');
    }
    sb.append(dayValue);
}
```

#### LocalTime.toString(StringBuilder)
```java
void toString(StringBuilder sb) {
    int hourValue = hour;
    int minuteValue = minute;
    int secondValue = second;
    int nanoValue = nano;

    sb.append(hourValue);
    sb.append(':');
    if (minuteValue < 10) sb.append('0');
    sb.append(minuteValue);
    sb.append(':');
    if (secondValue < 10) sb.append('0');
    sb.append(secondValue);

    if (nanoValue > 0) {
        sb.append('.');
        // 纳秒格式化逻辑...
    }
}
```

#### ZoneOffset.toString(StringBuilder)
```java
void toString(StringBuilder sb) {
    int offsetSecs = totalSeconds;
    if (offsetSecs == 0) {
        sb.append('Z');
    } else {
        // 格式化 ±HH:MM:SS
        ...
    }
}
```

#### ZoneId.toString(StringBuilder)
```java
void toString(StringBuilder sb) {
    sb.append(id);
}
```

## ZonedDateTime 特殊处理

### ISO-8601 格式

```
2024-03-20T10:30:45+08:00[Asia/Shanghai]
│             │        │       │               │
│   date+time │ offset  │    zone ID
│             │        │
└─────────────┴────────┴─── 基础部分
```

### 优化前后对比

```java
// 优化前
@Override
public String toString() {
    String str = dateTime.toString() + offset.toString();
    if (zone != offset) {
        str += '[' + zone.toString() + ']';  // 额外拼接!
    }
    return str;
}

// 优化后
@Override
public String toString() {
    StringBuilder sb = new StringBuilder(32);
    dateTime.toString(sb);
    offset.toString(sb);
    if (zone != offset) {
        sb.append('[');
        zone.toString(sb);
        sb.append(']');
    }
    return sb.toString();
}
```

## 内存分配分析

### ZonedDateTime 示例

```
输入: ZonedDateTime.of(2024, 3, 20, 10, 30, 45, 0, ZoneId.of("Asia/Shanghai"))
输出: "2024-03-20T10:30:45+08:00[Asia/Shanghai]"

优化前:
- LocalDateTime.toString() → "2024-03-20T10:30:45" (19 chars)
- ZoneOffset.toString()    → "+08:00" (5 chars)
- ZoneId.toString()        → "Asia/Shanghai" (13 chars)
- 拼接结果                 → "2024-03-20T10:30:45+08:00[Asia/Shanghai]" (42 chars)
总分配: 5 String 对象

优化后:
- StringBuilder(32)        → 预分配 32 字符
- 最终 toString()          → 1 String 对象
总分配: 1 String 对象

减少: 80%
```

## 影响分析

### 直接影响

1. **日志框架**
   - Log4j2, Logback 使用 java.time 格式化时间戳
   - 减少日志操作的开销

2. **JSON 序列化**
   - Jackson JavaTimeModule 序列化 LocalDateTime
   - 减少序列化时的内存分配

3. **HTTP 响应**
   - REST API 返回日期时间字段
   - 减少响应生成开销

### GC 压力减轻

```
假设每秒处理 10,000 个日期时间字符串:

优化前: 10,000 * 5 = 50,000 个 String 对象/秒
优化后: 10,000 * 1 = 10,000 个 String 对象/秒

减少: 40,000 个对象/秒
Young GC 频率: ↓ 约 80%
```

## 经验总结

### 性能优化模式

**统一优化模式** (在此 PR 中确立):

```java
// Step 1: 添加私有重载方法
private void toString(StringBuilder sb) {
    // 直接写入 sb，不返回 String
}

// Step 2: 公共方法调用重载
@Override
public String toString() {
    StringBuilder sb = new StringBuilder(preallocatedSize);
    toString(sb);
    return sb.toString();
}

// Step 3: 其他组合类直接调用
public String toString() {
    StringBuilder sb = new StringBuilder(size);
    component1.toString(sb);
    sb.append(separator);
    component2.toString(sb);
    return sb.toString();
}
```
