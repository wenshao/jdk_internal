## 背景

`StringBuilder.repeat(char, int)` 是 JDK 21 引入的新方法，用于高效重复字符。在 `DateTimeFormatterBuilder` 中有多处使用循环来追加零字符的代码，可以使用 `repeat` 方法简化。

**为什么重要？**
- `DateTimeFormatterBuilder` 是日期时间格式化的核心类
- 追补零字符是常见操作（如格式化月份、日期、秒等）
- 代码更简洁，性能相当或更好

**改动位置**
```
src/java.base/share/classes/java/time/format/DateTimeFormatterBuilder.java
    └── NumberPrinterParser.format()         ← 循环补零 → repeat
    └── FractionPrinterParser.format()      ← 循环补零 → repeat (2处)
```

    buf.append(zeroDigit);

**FractionPrinterParser (补零场景2)**:
```java
// 文件: java/time/format/DateTimeFormatterBuilder.java
// 行数: ~3410

    buf.append(zero);
}
```

**FractionPrinterParser (补零场景3)**:
```java

// 行数: ~3416

for (int i = 9 - stringSize; i > 0; i--) {

```

### 性能瓶颈分析

| 问题 | 影响 |
|------|------|
| **代码冗长** | 循环语法比单次方法调用更冗长 |
| **可读性** | 循环意图不如 `repeat` 明确 |
| **性能** | `repeat` 内部优化可能更好 |

### 为什么 `StringBuilder.repeat` 更好？

```java
// StringBuilder.repeat 的简化实现
public StringBuilder repeat(char c, int count) {
    final int end = this.count + count;
    ensureCapacityInternal(end);
    // ... 使用 Arrays.fill 或类似的高效方法
    return this;
}
```

优势：
1. **意图明确**：`repeat('0', 3)` 比 `for (int i = 0; i < 3; i++) buf.append('0')` 更清晰
2. **可能更高效**：内部可以使用 `Arrays.fill` 等批量操作
3. **代码更短**：减少代码行数

## 解决方案

### 核心改动

**改动1: NumberPrinterParser**
```java
// 变更前
char zeroDigit = decimalStyle.getZeroDigit();
for (int i = 0; i < minWidth - size; i++) {
    buf.append(zeroDigit);
}

// 变更后
char zeroDigit = decimalStyle.getZeroDigit();
int zeros = minWidth - size;
if (zeros > 0) {
    buf.repeat(zeroDigit, zeros);
}
```

**改动2: FractionPrinterParser (场景1)**
```java
// 变更前
for (int i = 0; i < width; i++) {
    buf.append(zero);
}

// 变更后
buf.repeat(zero, width);
```

**改动3: FractionPrinterParser (场景2)**
```java
// 变更前
for (int i = 9 - stringSize; i > 0; i--) {
    buf.append(zero);
}

// 变更后
int zeros = 9 - stringSize;
if (zeros > 0) {
    buf.repeat(zero, zeros);
}
```

### 关键优化点

| 优化点 | 说明 |
|--------|------|
| **代码简化** | 3 行循环 → 2 行 repeat |
| **意图明确** | `repeat` 清楚表达"重复字符"的意图 |
| **零风险** | `repeat` 是标准 API，行为确定 |

## 设计决策

### Q1: 为什么需要检查 zeros > 0？

**A**:
- `StringBuilder.repeat(char, int)` 要求 count >= 0
- 如果 count = 0，`repeat` 不做任何操作，但检查可以避免方法调用开销
- 如果 count < 0，会抛出 `IllegalArgumentException`

### Q2: 为什么不把所有循环都替换？

**A**:
- 只有"重复单个字符 N 次"的循环适合替换
- 其他复杂逻辑的循环不适合
- 本 PR 只替换明确的补零场景

### Q3: 性能提升来自哪里？

**A**:
- `repeat` 内部可能使用更高效的批量复制
- 减少了循环开销（尽管 JIT 可能优化掉）
- 代码更简洁，编译器更容易优化
