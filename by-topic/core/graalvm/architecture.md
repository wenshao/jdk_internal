### Sea of Nodes IR

Graal 使用 **Sea of Nodes** 作为中间表示，这是其核心创新。

```
传统编译器 IR (线性):
┌─────┐   ┌─────┐   ┌─────┐   ┌─────┐
│ A   │ → │ B   │ → │ C   │ → │ D   │
└─────┘   └─────┘   └─────┘   └─────┘

Sea of Nodes (图结构):
     ┌─────┐
     │  A  │
     └──┬──┘
        │
   ┌────┴────┐
   │         │
   ▼         ▼
┌─────┐   ┌─────┐
│  B  │   │  C  │
└──┬──┘   └──┬──┘
   │         │
   └────┬────┘
        │
        ▼
     ┌─────┐
     │  D  │
     └─────┘

优势:
• 数据流和控制流统一表示
• 优化更精确 (基于实际依赖)
• 支持激进内联和推测优化
```

### 节点类型

| 节点类型 | 说明 | 示例 |
|----------|------|------|
| **Fixed Nodes** | 有控制流依赖 | MethodStart, Return, If |
| **Floating Nodes** | 仅数据流依赖 | Add, Load, Store |
| **Deoptimize Nodes** | 去优化信息 | Deoptimize, UncommonTrap |

### AST 节点示例

```java
// 未特化的 AST 节点
abstract class ExpressionNode extends Node {
    public abstract Object execute(VirtualFrame frame);
}

// 加法节点 (支持多类型)
public class AddNode extends ExpressionNode {
    @Child private ExpressionNode left;
    @Child private ExpressionNode right;
    
    @Specialization
    protected int add(int a, int b) {
        return a + b;
    }
    
    @Specialization
    protected double add(double a, double b) {
        return a + b;
    }
    
    @Specialization
    protected String add(String a, String b) {
        return a + b;  // 字符串连接
    }
}

// 部分求值后:
// 如果 left 和 right 总是返回 int
// → 只保留 int 特化版本
// → 消除类型检查
// → 直接生成整数加法指令
```

### 优化示例：逃逸分析

```java
// 原始代码
public int sum(int[] values) {
    int sum = 0;
    for (int v : values) {
        sum += v;
    }
    return sum;
}

// 经过逃逸分析优化后:
// 1. 循环变量未逃逸 → 寄存器分配
// 2. 数组边界检查消除 → 基于归纳变量分析
// 3. 自动向量化 → SIMD 指令

// 生成的伪代码:
mov rax, values.length
mov rcx, 0          // sum 寄存器
mov rsi, values     // 数组指针

loop:
    add rcx, [rsi + rdx*4]  // SIMD 加载
    inc rdx
    cmp rdx, rax
    jl loop

return rcx
```

## 6. 去优化机制

当优化假设失效时，Graal 支持**去优化** (Deoptimization)。

```
┌─────────────────────────────────────────────────────────────────┐
│                    去优化流程                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  优化的代码执行中...                                             │
│       │                                                         │
│       ▼                                                         │
│  检测到假设失效 (如：类型假设错误)                                │
│       │                                                         │
│       ▼                                                         │
│  触发 Uncommon Trap                                             │
│       │                                                         │
│       ▼                                                         │
│  保存当前状态 (栈帧、局部变量)                                   │
│       │                                                         │
│       ▼                                                         │
│  转换到解释器/未优化版本                                         │
│       │                                                         │
│       ▼                                                         │
│  继续执行                                                        │
│       │                                                         │
│       ▼                                                         │
│  后台重新编译 (基于新 profiling)                                 │
│       │                                                         │
│       ▼                                                         │
│  安装新版本的优化代码                                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 去优化场景

| 场景 | 触发条件 | 处理方式 |
|------|----------|----------|
| **类型假设失效** | 遇到新类型 | 重新编译，添加新特化 |
| **内联失效** | 方法被覆盖 | 去内联，虚调用 |
| **空值假设失效** | null 检查失败 | 添加 null 检查 |
| **边界检查失效** | 数组越界 | 添加边界检查 |
