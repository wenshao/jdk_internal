# x86/x64 架构

Intel/AMD x86 架构，OpenJDK 的主要开发平台。

## 概述

### 架构历史

```
1978 ─── Intel 8086 (16位)
   │
1985 ─── Intel 386 (32位, i386)
   │
2003 ─── AMD64 (64位, x86_64)
   │
JDK 1.0 ─── x86_32 支持
   │
JDK 5  ─── x86_64 支持
   │
JDK 26 ─── 移除 x86_32 (JEP 503)
```

### 特性对比

| 特性 | x86_32 | x86_64 |
|------|--------|--------|
| **寄存器** | 8个 | 16个 |
| **地址空间** | 4GB | 256TB |
| **通用寄存器宽度** | 32-bit | 64-bit |
| **SIMD** | SSE/AVX | SSE/AVX/AVX-512 |
| **JDK 支持** | ⚠️ JDK 26 移除 | ✅ 持续 |

## SIMD 演进

### SSE/AVX 支持

| 指令集 | 引入年份 | JDK 支持 | 特性 |
|--------|----------|----------|------|
| **MMX** | 1997 | JDK 1.0 | 64-bit SIMD |
| **SSE** | 1999 | JDK 1.3 | 128-bit float |
| **SSE2** | 2001 | JDK 1.4 | 128-bit int/float |
| **SSE3** | 2004 | JDK 5 | 水平指令 |
| **SSSE3** | 2006 | JDK 6 | 补充指令 |
| **SSE4.1** | 2007 | JDK 7 | 向量指令 |
| **SSE4.2** | 2008 | JDK 7 | 字符串指令 |
| **AVX** | 2011 | JDK 7 | 256-bit SIMD |
| **AVX2** | 2013 | JDK 8 | 256-bit int |
| **AVX-512** | 2016 | JDK 10+ | 512-bit SIMD |

## JVM 参数

### SIMD 控制

```bash
# AVX 支持
-XX:UseAVX=1        # AVX (256-bit)
-XX:UseAVX=2        # AVX2 (256-bit int)
-XX:UseAVX=3        # AVX-512 (512-bit)

# 向量优化
-XX:+UseVectorCmov           # 条件移动指令
-XX:+UseXMMForArrayCopy      # XMM 数组复制
-XX:+UseXMMForObjInit        # XMM 对象初始化
-XX:MaxVectorSize=32         # 最大向量大小 (字节)

# 禁用 SIMD
-XX:UseAVX=0
```

### 内存优化

```bash
# 压缩 oops
-XX:+UseCompressedOops        # 默认启用 (<32GB)
-XX:+UseCompressedClassPointers  # 压缩类指针

# 大页内存
-XX:+UseLargePages
-XX:LargePageSizeInBytes=2m

# NUMA 优化
-XX:+UseNUMA
-XX:+UseParallelGC
```

## 版本演进

### JDK 5: x86_64 支持

- AMD64 (x86_64) 首次支持
- 64位 JVM
- 更大内存支持

### JDK 7: AVX 支持

- AVX 指令集支持
- 向量 intrinsic

### JDK 10: AVX-512 支持

- 512-bit SIMD
- Skylake-X 支持

### JDK 26: 移除 x86_32

- JEP 503: 移除 32 位 x86 端口
- 专注 64 位优化

## 迁移指南

### 从 x86_32 迁移到 x86_64

```bash
# 1. 验证 64 位 JDK
java -version
# 应显示 "64-Bit"

# 2. 检查 JNI 库
file libnative.so
# 应显示 "x86-64"

# 3. 更新 JVM 参数
# -Xmx 可以设置更大值
# 无需 -d32 参数

# 4. 测试兼容性
# - 指针大小变化 (4 → 8 字节)
# - 数据结构对齐
# - JNI 签名
```
