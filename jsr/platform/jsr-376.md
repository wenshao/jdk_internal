## 2. 概述

JSR 376 定义了 Java 平台模块系统 (JPMS)，这是 Java 历史上最大的架构变革之一：

| 目标 | 说明 |
|------|------|
| **可靠的配置** | 编译时和运行时一致的依赖解析 |
| **强封装** | 模块内部的实现细节被隐藏 |
| **可组合** | 模块可以组合成更大的模块 |
| **可扩展** | 平台本身被模块化 |
| **兼容性** | 与现有代码兼容 (Class-Path) |

## 3. 模块描述符

### module-info.java

```java
module com.example.myapp {
    // 依赖其他模块
    requires java.base;           // 默认，可省略
    requires java.sql;            // 编译和运行时需要
    requires static java.annotation.processing;  // 仅编译时
    requires transitive java.logging;  // 传递给依赖此模块的模块

    // 导出包给所有模块
    exports com.example.api;
    exports com.example.internal to com.example.tests;  // 限定导出

    // 开放包给反射访问
    opens com.example.model;      // 所有模块
    opens com.example.impl to org.hibernate.core;  // 限定开放

    // 服务消费
    uses com.example.spi.MyService;

    // 服务提供
    provides com.example.spi.MyService
        with com.example.impl.MyServiceImpl;
}
```

### 关键字说明

| 关键字 | 说明 |
|--------|------|
| `requires` | 依赖模块 |
| `requires static` | 仅编译时依赖 |
| `requires transitive` | 传递依赖 |
| `exports` | 导出包 |
| `exports ... to` | 限定导出 |
| `opens` | 开放反射访问 |
| `opens ... to` | 限定开放 |
| `uses` | 声明服务消费 |
| `provides ... with` | 声明服务提供 |

## 4. 模块类型

### 系统模块 (Java SE 平台)

```bash
# 列出所有系统模块
java --list-modules

# 输出示例
java.base@17
java.sql@17
java.logging@17
java.desktop@17
jdk.httpserver@17
...
```

### 应用模块

```java
// src/com.example.myapp/module-info.java
module com.example.myapp {
    requires java.base;
    exports com.example.myapp;
}
```

### 自动模块

```java
// 将 JAR 放入 module-path，自动变成模块
// 模块名来自 JAR 文件名或 Automatic-Module-Name 清单属性

// Manifest:
Automatic-Module-Name: com.example.legacy

// 然后可以
requires com.example.legacy;
```

### 未命名模块

```java
// class-path 上的代码属于未命名模块
// 可以访问所有模块 (除了强封装的部分)
// 但不能被模块 requires
```

## 5. 模块路径 vs 类路径

```
┌─────────────────────────────────────────────────────────────────────┐
│                    模块路径 vs 类路径                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  Module Path (--module-path, -p)                                    │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐                  │
│  │  app.jar    │  │  lib1.jar   │  │  lib2.jar   │                  │
│  │  (模块)      │  │  (模块)      │  │  (模块)      │                  │
│  └─────────────┘  └─────────────┘  └─────────────┘                  │
│        │                │                │                          │
│        └────────────────┴────────────────┘                          │
│                         │                                           │
│                    模块图解析                                        │
│                                                                     │
│  Class Path (--class-path, -cp)                                     │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐                  │
│  │ legacy1.jar │  │ legacy2.jar │  │ legacy3.jar │                  │
│  │ (未命名模块) │  │ (未命名模块) │  │ (未命名模块) │                  │
│  └─────────────┘  └─────────────┘  └─────────────┘                  │
│        │                │                │                          │
│        └────────────────┴────────────────┘                          │
│                         │                                           │
│                    扁平化加载                                        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 命令行示例

```bash
# 模块化应用
java --module-path app.jar:libs/ \
     --module com.example.myapp/com.example.Main

# 混合模式 (模块 + class-path)
java --module-path app.jar \
     --class-path legacy-libs/* \
     --module com.example.myapp/com.example.Main

# 添加模块到根模块集
java --module-path app.jar \
     --add-modules java.sql,java.logging \
     --module com.example.myapp/com.example.Main
```

## 6. 服务加载

### 声明服务

```java
// 服务接口
package com.example.spi;
public interface MyService {
    String process(String input);
}

// 服务实现
package com.example.impl;
public class MyServiceImpl implements MyService {
    @Override
    public String process(String input) {
        return input.toUpperCase();
    }
}
```

### 模块描述符

```java
// 服务提供方模块
module com.example.provider {
    requires com.example.spi;
    provides com.example.spi.MyService
        with com.example.impl.MyServiceImpl;
}

// 服务消费方模块
module com.example.consumer {
    requires com.example.spi;
    uses com.example.spi.MyService;  // 声明需要此服务
}
```

### 加载服务

```java
// 使用 ServiceLoader
ServiceLoader<MyService> loader = ServiceLoader.load(MyService.class);
for (MyService service : loader) {
    String result = service.process("hello");
}
```

## 7. 反射与封装

### 强封装

```java
module com.example.internal {
    // 不导出的包对外不可见
    // 即使使用反射也无法访问 (默认)
}
```

### 开放反射

```java
module com.example.app {
    // 方式 1: opens 整个包
    opens com.example.model;

    // 方式 2: opens 给特定模块
    opens com.example.entity to org.hibernate.core;

    // 方式 3: open module (整个模块开放)
    // open module com.example.app { ... }
}
```

### 命令行绕过封装

```bash
# 添加导出
--add-exports com.example.internal/com.example.impl=ALL-UNNAMED

# 添加开放
--add-opens com.example.internal/com.example.impl=ALL-UNNAMED

# 常用于框架 (Spring, Hibernate)
--add-opens java.base/java.lang=ALL-UNNAMED
--add-opens java.base/java.util=ALL-UNNAMED
```

## 8. jlink 工具

### 创建自定义运行时

```bash
# 创建只包含所需模块的 JRE
jlink --module-path app.jar:$JAVA_HOME/jmods \
      --add-modules com.example.myapp \
      --output my-runtime \
      --compress 2 \
      --strip-debug \
      --no-header-files \
      --no-man-pages

# 使用自定义运行时
my-runtime/bin/java --module com.example.myapp/com.example.Main
```

### jlink 选项

| 选项 | 说明 |
|------|------|
| `--add-modules` | 包含的模块 |
| `--output` | 输出目录 |
| `--compress` | 压缩级别 (0-2) |
| `--strip-debug` | 移除调试信息 |
| `--no-header-files` | 不包含头文件 |
| `--no-man-pages` | 不包含手册页 |
| `--launcher` | 创建启动脚本 |
| `--bind-services` | 包含服务提供者 |

## 9. 迁移指南

### 从 class-path 迁移

#### 1. 添加 module-info.java

```java
module com.example.legacy {
    // 暂时导出所有包
    exports com.example.pkg1;
    exports com.example.pkg2;
}
```

#### 2. 处理依赖

```java
module com.example.legacy {
    // 使用自动模块
    requires spring.context;
    requires hibernate.core;

    // 或使用 class-path 兼容
}
```

#### 3. 处理反射

```java
module com.example.legacy {
    // 开放需要反射的包
    opens com.example.model;
}
```

### 常见问题

#### Split Package

```
Error: package com.example.util is in both module A and module B
```

解决：
```java
// 合并包，或使用 jar --upgrade
```

#### 循环依赖

```
Error: Cycle detected: A requires B, B requires A
```

解决：
```java
// 重构模块，提取公共部分
module com.example.common {
    exports com.example.shared;
}

module com.example.a {
    requires com.example.common;
}

module com.example.b {
    requires com.example.common;
}
```

## 10. JDK 模块化

### 关键模块

| 模块 | 说明 |
|------|------|
| `java.base` | 核心类 (自动依赖) |
| `java.sql` | JDBC API |
| `java.logging` | java.util.logging |
| `java.desktop` | AWT, Swing |
| `java.xml` | XML 处理 |
| `jdk.httpserver` | HTTP Server |
| `jdk.jfr` | Flight Recorder |

### 精简 JRE

```bash
# 只包含基础模块
jlink --add-modules java.base \
      --output minimal-jre

# 大小对比
# Full JDK:     ~300MB
# Minimal JRE:  ~40MB
```

## 11. 最佳实践

### 1. 模块命名

```java
// 推荐: 反向域名
module com.example.myapp { }
module org.apache.commons.lang3 { }

// 避免: 单词或缩写
module myapp { }
module common { }
```

### 2. API 与实现分离

```java
// API 模块
module com.example.api {
    exports com.example.api;
}

// 实现模块
module com.example.impl {
    requires com.example.api;
    provides com.example.api.Service
        with com.example.impl.ServiceImpl;
}
```

### 3. 最小化导出

```java
module com.example.app {
    // 只导出公共 API
    exports com.example.api;

    // 内部包不导出
    // exports com.example.internal;  // 不导出
}
```
