## 2. 概述

JSR 269 定义了编译时注解处理 (compile-time annotation processing) 的标准 API，取代了早期的 `apt` 工具。它允许开发者在编译阶段读取注解信息并生成新的源文件、资源文件，甚至发出编译错误和警告。

| 特性 | 说明 |
|------|------|
| 处理时机 | 编译时 (compile-time)，非运行时 |
| 核心包 | `javax.annotation.processing` |
| 类型模型 | `javax.lang.model` (element, type, util) |
| 注册方式 | `META-INF/services/javax.annotation.processing.Processor` |
| 多轮处理 | 支持多轮次 (round) 迭代处理 |
| 代码生成 | 通过 Filer API 生成 .java 源文件 |

## 3. 核心 API

### javax.annotation.processing 包

```java
// 核心接口与类
Processor                  // 处理器接口
AbstractProcessor          // 处理器基类 (常用)
RoundEnvironment           // 当前轮次的环境信息
ProcessingEnvironment      // 编译环境 (工具集合)
Filer                      // 文件创建工具
Messager                   // 编译消息 (错误/警告)

// 注解
@SupportedAnnotationTypes  // 声明处理哪些注解
@SupportedSourceVersion    // 声明支持的源码版本
@SupportedOptions          // 声明接收的编译选项
```

### ProcessingEnvironment 工具集

```java
public interface ProcessingEnvironment {
    Elements getElementUtils();    // Element 工具 (查询类信息)
    Types    getTypeUtils();       // Type 工具 (类型判断)
    Filer    getFiler();           // 文件创建 (生成源码)
    Messager getMessager();        // 编译消息输出
    Map<String, String> getOptions();  // 编译选项 -Akey=value
}
```

## 4. 编写注解处理器

### 自定义注解

```java
@Retention(RetentionPolicy.SOURCE)   // 编译时处理，不需要运行时保留
@Target(ElementType.TYPE)
public @interface GenerateBuilder {
    // 标记注解，触发 Builder 代码生成
}
```

### 处理器实现

```java
@SupportedAnnotationTypes("com.example.GenerateBuilder")
@SupportedSourceVersion(SourceVersion.RELEASE_17)
public class BuilderProcessor extends AbstractProcessor {

    @Override
    public boolean process(Set<? extends TypeElement> annotations,
                           RoundEnvironment roundEnv) {
        for (Element element : roundEnv.getElementsAnnotatedWith(GenerateBuilder.class)) {
            if (element.getKind() != ElementKind.CLASS) {
                processingEnv.getMessager().printMessage(
                    Diagnostic.Kind.ERROR,
                    "@GenerateBuilder 只能用于类", element);
                continue;
            }
            TypeElement typeElement = (TypeElement) element;
            generateBuilder(typeElement);
        }
        return true;  // 声明已处理，其他处理器不再处理
    }
}
```

### 注册处理器

```
// 文件: META-INF/services/javax.annotation.processing.Processor
com.example.BuilderProcessor
```

或使用 Google AutoService 自动生成注册文件：

```java
@AutoService(Processor.class)
public class BuilderProcessor extends AbstractProcessor { ... }
```

## 5. RoundEnvironment 与处理轮次

```java
@Override
public boolean process(Set<? extends TypeElement> annotations,
                       RoundEnvironment roundEnv) {

    // 判断是否最终轮次
    if (roundEnv.processingOver()) {
        // 最终轮次 — 清理资源
        return false;
    }

    // 获取被特定注解标记的元素
    Set<? extends Element> elements =
        roundEnv.getElementsAnnotatedWith(GenerateBuilder.class);

    // 获取本轮所有根元素 (root elements)
    Set<? extends Element> roots = roundEnv.getRootElements();

    // 检查是否有错误
    if (roundEnv.errorRaised()) {
        return false;
    }

    return true;
}
```

### 多轮处理示意

```
Round 1: 处理 UserDTO.java → 生成 UserDTOBuilder.java
Round 2: 处理 UserDTOBuilder.java (如有注解) → 无新生成
Round 3: processingOver() == true → 结束
```

## 6. 代码生成

### 使用 Filer 生成源文件

```java
private void generateBuilder(TypeElement classElement) {
    String className = classElement.getSimpleName() + "Builder";
    String packageName = processingEnv.getElementUtils()
        .getPackageOf(classElement).getQualifiedName().toString();

    try {
        JavaFileObject file = processingEnv.getFiler()
            .createSourceFile(packageName + "." + className, classElement);

        try (PrintWriter writer = new PrintWriter(file.openWriter())) {
            writer.println("package " + packageName + ";");
            writer.println();
            writer.println("public class " + className + " {");
            // ... 生成字段和方法
            writer.println("}");
        }
    } catch (IOException e) {
        processingEnv.getMessager().printMessage(
            Diagnostic.Kind.ERROR, "生成代码失败: " + e.getMessage());
    }
}
```

### 使用 JavaPoet 简化代码生成

```java
// JavaPoet (Square) — 类型安全的 Java 代码生成
TypeSpec builder = TypeSpec.classBuilder(className)
    .addModifiers(Modifier.PUBLIC)
    .addMethod(MethodSpec.methodBuilder("build")
        .addModifiers(Modifier.PUBLIC)
        .returns(ClassName.get(classElement))
        .addStatement("return new $T(this)", classElement)
        .build())
    .build();

JavaFile.builder(packageName, builder)
    .build()
    .writeTo(processingEnv.getFiler());
```

## 7. javax.lang.model 类型系统

### Element 层次结构

```
Element (元素)
├── PackageElement        包
├── TypeElement           类/接口/枚举/注解
├── VariableElement       字段/参数/枚举常量
├── ExecutableElement     方法/构造器
└── TypeParameterElement  泛型参数
```

### 常用查询

```java
// 获取类的所有字段
List<VariableElement> fields = ElementFilter.fieldsIn(
    classElement.getEnclosedElements());

// 获取类的所有方法
List<ExecutableElement> methods = ElementFilter.methodsIn(
    classElement.getEnclosedElements());

// 获取父类
TypeMirror superclass = classElement.getSuperclass();

// 判断类型关系
Types typeUtils = processingEnv.getTypeUtils();
boolean isAssignable = typeUtils.isAssignable(type1, type2);
boolean isSame = typeUtils.isSameType(type1, type2);
```

## 9. 最佳实践

### 1. 增量处理 (Incremental Processing)

```java
// Gradle 增量注解处理 — 声明类型
// 在 META-INF/gradle/incremental.annotation.processors 中注册
com.example.BuilderProcessor,aggregating
// 类型: isolating (独立), aggregating (聚合)
```

### 2. 错误处理 — 使用 Messager 而非异常

```java
// 正确: 编译器友好的错误输出
processingEnv.getMessager().printMessage(
    Diagnostic.Kind.ERROR,
    "@GenerateBuilder 不支持抽象类",
    element);  // 定位到具体元素

// 错误: 抛出异常会中断编译，信息不友好
throw new RuntimeException("不支持抽象类");
```

### 3. 处理泛型

```java
// 获取泛型参数
DeclaredType declaredType = (DeclaredType) field.asType();
List<? extends TypeMirror> typeArgs = declaredType.getTypeArguments();
// List<String> → typeArgs[0] 是 String
```
