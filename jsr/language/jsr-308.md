## 2. 概述

JSR 308 扩展了 Java 注解的使用范围。在 JDK 5 (JSR 175) 引入注解时，注解只能出现在声明位置（类、方法、字段、参数等）。JSR 308 允许注解出现在任何使用类型的位置。

| 特性 | 说明 |
|------|------|
| **TYPE_USE** | 注解可用于任何类型使用位置 |
| **TYPE_PARAMETER** | 注解可用于类型参数声明 |
| **可插拔类型系统** | 通过注解处理器实现编译期额外检查 |
| **字节码存储** | 类型注解存储在 class 文件的 `RuntimeVisibleTypeAnnotations` 属性中 |

### JDK 5 vs JDK 8 注解能力对比

```java
// JDK 5 (JSR 175): 只能注解声明
@Deprecated
public class MyClass {
    @Override
    public String toString() { ... }
}

// JDK 8 (JSR 308): 可以注解类型使用
@NonNull String name;
List<@NonNull String> names;
(@NonNull String) obj;
new @Interned MyObject();
```

## 3. 新增注解位置

JSR 308 新增了以下可以放置注解的位置：

### 类型使用位置 (TYPE_USE)

```java
// 1. 泛型类型参数
List<@NonNull String> list;
Map<@NonNull String, @Nullable Integer> map;

// 2. 类型转换
String s = (@NonNull String) obj;

// 3. instanceof 表达式
if (obj instanceof @NonNull String) { ... }

// 4. new 表达式
Object obj = new @Interned MyObject();

// 5. throws 子句
void method() throws @Critical IOException { ... }

// 6. extends / implements
class MyList extends @Readonly ArrayList<@Readonly String> { ... }

// 7. 数组类型的各个维度
@NonNull String @Nullable [] @NonNull []
// 含义: 外层数组非空，中间数组可空，元素非空

// 8. 通配符边界
List<@NonNull ? extends @Readonly Number> list;

// 9. 方法引用
@NonNull String::valueOf
```

### 类型参数声明 (TYPE_PARAMETER)

```java
// 在类型参数上使用注解
class MyClass<@Immutable T, @Covariant U> { ... }

// 方法类型参数
<@NonNull T> T getValue(List<T> list) { ... }
```

## 4. TYPE_USE 与 TYPE_PARAMETER

### ElementType 枚举新增值

```java
public enum ElementType {
    TYPE,              // 类、接口、枚举 (JDK 5)
    FIELD,             // 字段 (JDK 5)
    METHOD,            // 方法 (JDK 5)
    PARAMETER,         // 方法参数 (JDK 5)
    CONSTRUCTOR,       // 构造函数 (JDK 5)
    LOCAL_VARIABLE,    // 局部变量 (JDK 5)
    ANNOTATION_TYPE,   // 注解类型 (JDK 5)
    PACKAGE,           // 包 (JDK 5)
    TYPE_PARAMETER,    // 类型参数 (JDK 8, JSR 308)
    TYPE_USE,          // 类型使用 (JDK 8, JSR 308)
    MODULE,            // 模块 (JDK 9)
    RECORD_COMPONENT,  // Record 组件 (JDK 16)
}
```

### 定义 TYPE_USE 注解

```java
// 定义一个可用于类型使用位置的注解
@Target(ElementType.TYPE_USE)
@Retention(RetentionPolicy.RUNTIME)
public @interface NonNull { }

@Target(ElementType.TYPE_USE)
@Retention(RetentionPolicy.RUNTIME)
public @interface Nullable { }

// 同时支持声明和类型使用
@Target({ElementType.FIELD, ElementType.METHOD, ElementType.PARAMETER,
         ElementType.TYPE_USE})
@Retention(RetentionPolicy.RUNTIME)
public @interface ReadOnly { }
```

### 字节码表示

```
// TYPE_USE 注解存储在 RuntimeVisibleTypeAnnotations 属性中
// 不同于普通注解的 RuntimeVisibleAnnotations

RuntimeVisibleTypeAnnotations:
  0: #22(): LOCAL_VARIABLE, {start_pc=3, length=10, index=1}
     @NonNull
  1: #23(): METHOD_RETURN
     @Nullable
```

## 5. 可插拔类型系统

JSR 308 的核心价值在于支持**可插拔类型系统 (Pluggable Type System)**：

### 概念

```
┌─────────────────────────────────────────────────┐
│              Java 编译器 (javac)                  │
├─────────────────────────────────────────────────┤
│                                                   │
│  标准类型检查     +     注解处理器 (插件)           │
│  (Java 类型系统)       (额外类型检查)              │
│                                                   │
│  String s = 42;       @NonNull String s = null;   │
│  ↑ 编译错误            ↑ 注解处理器报错             │
│                                                   │
└─────────────────────────────────────────────────┘
```

### 工作原理

1. 开发者在代码中使用 TYPE_USE 注解标注类型
2. 注解处理器 (Annotation Processor) 在编译期扫描这些注解
3. 处理器根据注解语义执行额外的类型检查
4. 如果发现违规，报告编译警告或错误

```java
// 注解处理器可以检测以下错误:
@NonNull String name = null;           // 错误: 不能赋 null
@Positive int count = -1;              // 错误: 不能为负数
@Regex String pattern = "abc[";        // 错误: 非法正则
@Tainted String sql = userInput;       // 警告: 不安全输入
```

## 6. Checker Framework

Checker Framework 是 JSR 308 最成熟的实现，由 Michael Ernst (JSR 308 Spec Lead) 领导开发。

### 内置检查器

| 检查器 | 检查内容 |
|--------|---------|
| **Nullness Checker** | `@NonNull` / `@Nullable`，消除 NullPointerException |
| **Interning Checker** | `@Interned`，确保使用 `equals()` 而非 `==` |
| **Lock Checker** | `@GuardedBy`，确保正确的锁使用 |
| **Regex Checker** | `@Regex`，编译期验证正则表达式 |
| **Tainting Checker** | `@Tainted` / `@Untainted`，防止注入攻击 |
| **Format String Checker** | 验证 `printf` 格式字符串 |
| **Signedness Checker** | 有符号/无符号整数混用检查 |

### 使用示例

```java
// 使用 Nullness Checker
import org.checkerframework.checker.nullness.qual.NonNull;
import org.checkerframework.checker.nullness.qual.Nullable;

class UserService {
    // 返回可能为 null
    @Nullable User findUser(String id) {
        return db.query(id);
    }

    // 参数不能为 null
    void updateUser(@NonNull User user) {
        // ...
    }

    void process(String id) {
        User user = findUser(id);
        updateUser(user);  // 编译警告: user 可能为 null

        // 修复: 显式检查
        if (user != null) {
            updateUser(user);  // OK
        }
    }
}
```

### 编译集成

```xml
<!-- Maven 配置 -->
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-compiler-plugin</artifactId>
    <configuration>
        <annotationProcessorPaths>
            <path>
                <groupId>org.checkerframework</groupId>
                <artifactId>checker</artifactId>
                <version>3.42.0</version>
            </path>
        </annotationProcessorPaths>
        <compilerArgs>
            <arg>-processor</arg>
            <arg>org.checkerframework.checker.nullness.NullnessChecker</arg>
        </compilerArgs>
    </configuration>
</plugin>
```

## 7. Nullness 注解生态

JSR 308 推动了 `@NonNull` / `@Nullable` 生态的发展，但不幸的是产生了多个竞争标准：

### 主要注解库对比

| 来源 | 包名 | 说明 |
|------|------|------|
| **Checker Framework** | `org.checkerframework.checker.nullness.qual` | 最完整，支持 TYPE_USE |
| **JetBrains** | `org.jetbrains.annotations` | IntelliJ IDEA 使用 |
| **Eclipse** | `org.eclipse.jdt.annotation` | Eclipse JDT 使用 |
| **JSR 305** | `javax.annotation` | Google FindBugs，已废弃但广泛使用 |
| **Jakarta** | `jakarta.annotation` | `@Nonnull`/`@Nullable` |
| **JSpecify** | `org.jspecify.annotations` | 新标准化努力 (2023+) |

### JSpecify 项目

JSpecify 是 Google、JetBrains、Oracle 等联合推动的标准化项目，目标是统一 Nullness 注解：

```java
// JSpecify: 统一的 Nullness 注解
import org.jspecify.annotations.NullMarked;
import org.jspecify.annotations.Nullable;

@NullMarked  // 包或类级别: 默认所有类型为 @NonNull
class UserService {
    // 只需标注 @Nullable
    @Nullable User findUser(String id) { ... }

    // 未标注 = @NonNull (由 @NullMarked 推断)
    void updateUser(User user) { ... }
}
```

## 8. 与其他 JSR/JEP 的关系

### 基础依赖

| JSR/JEP | 说明 |
|---------|------|
| JSR 175 | Annotations (JDK 5) — JSR 308 的基础 |
| JSR 269 | Pluggable Annotation Processing (JDK 6) — 处理器框架 |
| JEP 104 | Annotations on Java Types — JSR 308 的 JEP 实现 |
| JEP 120 | Repeating Annotations (JDK 8) — 同版本引入 |

### 后续发展

| JEP | 说明 |
|-----|------|
| JEP 277 | Enhanced Deprecation (JDK 9) — `@Deprecated(since, forRemoval)` |
| JDK 16 | `RECORD_COMPONENT` ElementType — Record 支持 |
| JSpecify | 社区标准化 Nullness 注解 (进行中) |

### 与其他语言对比

| 语言 | Nullness 方案 |
|------|-------------|
| **Java** | 注解 (`@NonNull`) — 可选、第三方 |
| **Kotlin** | 语言级别 (`String` vs `String?`) |
| **Scala** | `Option[T]` 类型 |
| **Rust** | `Option<T>` + 编译器保证 |
| **C#** | Nullable reference types (C# 8.0) |

## 9. 最佳实践

### 1. 选择一个 Nullness 标准并全项目统一

```java
// 推荐: 使用 JSpecify (新项目) 或 Checker Framework (成熟项目)
// 不推荐: 混用多个库的注解
```

### 2. 使用 @NullMarked 减少注解噪声

```java
// 不推荐: 到处标注 @NonNull
void process(@NonNull String name, @NonNull List<@NonNull String> items) { ... }

// 推荐: 包级别 @NullMarked + 只标注 @Nullable
// package-info.java:
@NullMarked
package com.example.service;

// 类中只需标注例外情况
void process(String name, List<String> items) { ... }  // 默认非空
@Nullable String findName(int id) { ... }              // 显式标注可空
```

### 3. TYPE_USE 注解与泛型配合

```java
// 精确表达泛型中的 nullness
List<@NonNull String>          // 列表本身可空，元素非空
@NonNull List<@NonNull String> // 列表和元素都非空
@NonNull List<@Nullable String> // 列表非空，元素可空
```

### 4. 在 CI 中集成类型检查

```bash
# 在 CI 流水线中运行 Checker Framework
javac -processor org.checkerframework.checker.nullness.NullnessChecker \
      -Werror \
      src/**/*.java
```
