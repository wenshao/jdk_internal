## 2. 概述

JSR 175 为 Java 引入了注解（Annotations）机制，允许在源码中嵌入结构化元数据：

| 特性 | 说明 |
|------|------|
| **注解声明** | `@interface` 语法定义注解类型 |
| **元注解** | 注解的注解，控制注解行为 |
| **保留策略** | 控制注解在编译/运行时的可见性 |
| **内置注解** | 语言级标准注解 |
| **反射访问** | `java.lang.reflect.AnnotatedElement` |

### 设计动机

JDK 5 之前，Java 元数据只能通过以下方式表达：

```java
// 1. Javadoc 标签 (不可编程访问)
/** @author Josh */

// 2. 标记接口 (Marker Interface，功能受限)
class MyList implements Serializable { }

// 3. XML 描述符 (外部文件，易与代码脱节)
<bean id="userService" class="com.example.UserService"/>
```

注解统一了这些机制，提供了类型安全、可编程、声明式的元数据方案。

## 3. 核心语法

### 定义注解

```java
// 标记注解 (Marker Annotation) - 无元素
public @interface Immutable { }

// 单值注解 (Single-Value Annotation)
public @interface Author {
    String value();  // 使用 value() 可省略名称
}

// 多元素注解 (Multi-Value Annotation)
public @interface RequestMapping {
    String path();
    String method() default "GET";       // 默认值
    String[] headers() default {};       // 数组默认值
    boolean required() default true;
}
```

### 注解元素类型限制

注解元素只能是以下类型：

| 类型 | 示例 |
|------|------|
| 基本类型 | `int`, `long`, `boolean` 等 |
| `String` | `String value()` |
| `Class<?>` | `Class<?> type()` |
| 枚举 | `RetentionPolicy value()` |
| 注解 | `Author author()` (嵌套注解) |
| 以上类型的数组 | `String[] tags()` |

```java
// ❌ 不允许的类型
@interface Invalid {
    Object value();           // 不允许
    List<String> items();     // 不允许
    Map<String, String> map();// 不允许
}
```

### 使用注解

```java
// 标记注解
@Immutable
public class Point { }

// 单值注解 (省略 value=)
@Author("Joshua Bloch")
public class Collections { }

// 多元素注解
@RequestMapping(path = "/users", method = "POST", headers = {"Content-Type=application/json"})
public void createUser() { }
```

## 4. 元注解

### @Target — 注解应用目标

```java
@Target({ElementType.METHOD, ElementType.FIELD})
public @interface MyAnnotation { }
```

| ElementType | 说明 | 示例 |
|-------------|------|------|
| `TYPE` | 类、接口、枚举、注解 | `@Entity class User` |
| `FIELD` | 字段 | `@Column String name` |
| `METHOD` | 方法 | `@Override void run()` |
| `PARAMETER` | 方法参数 | `void setName(@NotNull String name)` |
| `CONSTRUCTOR` | 构造函数 | `@Inject public Service()` |
| `LOCAL_VARIABLE` | 局部变量 | `@SuppressWarnings("unused") int x` |
| `ANNOTATION_TYPE` | 注解类型 | `@Target @interface Meta` |
| `PACKAGE` | 包声明 | `@API package com.example` |
| `TYPE_PARAMETER` | 类型参数 (JDK 8) | `class Box<@NonNull T>` |
| `TYPE_USE` | 类型使用 (JDK 8) | `@NonNull String s` |

### @Retention — 保留策略

```java
@Retention(RetentionPolicy.RUNTIME)
public @interface MyAnnotation { }
```

| RetentionPolicy | 说明 | 用途 |
|-----------------|------|------|
| `SOURCE` | 仅存在于源码，编译后丢弃 | `@Override`, `@SuppressWarnings`, Lombok |
| `CLASS` | 保留在 .class 文件，但 JVM 不加载 (默认) | 字节码工具、静态分析 |
| `RUNTIME` | JVM 加载，反射可读取 | Spring, JPA, JAX-RS |

### @Documented

```java
@Documented
public @interface Author {
    String value();
}
// 使 @Author 出现在 Javadoc 生成的文档中
```

### @Inherited

```java
@Inherited
@Retention(RetentionPolicy.RUNTIME)
public @interface Transactional { }

@Transactional
class BaseService { }

class UserService extends BaseService { }
// UserService 自动继承 @Transactional
// 注意: 仅对类继承有效，接口实现不继承
```

## 5. 内置注解

### @Override

```java
class Parent {
    void doWork() { }
}

class Child extends Parent {
    @Override
    void doWork() { }         // ✓ 正确重写

    @Override
    void dowork() { }         // ❌ 编译错误: 拼写错误被捕获
}
```

### @Deprecated

```java
// JDK 5
@Deprecated
public class Vector<E> { }

// JDK 9 增强 (JEP 277)
@Deprecated(since = "9", forRemoval = true)
public void finalize() { }
```

### @SuppressWarnings

```java
@SuppressWarnings("unchecked")
public <T> T[] toArray(T[] a) {
    return (T[]) Arrays.copyOf(elementData, size, a.getClass());
}

@SuppressWarnings({"unchecked", "rawtypes"})
public void legacyCode() { }
```

## 6. 注解处理

### 运行时反射 API

```java
// AnnotatedElement 接口 (Class, Method, Field 等实现)
Class<?> clazz = UserService.class;

// 检查是否存在
boolean has = clazz.isAnnotationPresent(Service.class);

// 获取注解实例
Service service = clazz.getAnnotation(Service.class);
String name = service.value();

// 获取所有注解
Annotation[] all = clazz.getAnnotations();

// 方法上的注解
Method m = clazz.getMethod("findById", Long.class);
if (m.isAnnotationPresent(Transactional.class)) {
    Transactional tx = m.getAnnotation(Transactional.class);
}

// 参数上的注解
Annotation[][] paramAnnotations = m.getParameterAnnotations();
```

### 编译期注解处理（APT → JSR 269）

```java
// JSR 269 (JDK 6): Pluggable Annotation Processing API
@SupportedAnnotationTypes("com.example.AutoValue")
@SupportedSourceVersion(SourceVersion.RELEASE_21)
public class AutoValueProcessor extends AbstractProcessor {
    @Override
    public boolean process(Set<? extends TypeElement> annotations,
                          RoundEnvironment roundEnv) {
        for (Element element : roundEnv.getElementsAnnotatedWith(AutoValue.class)) {
            // 生成代码
            generateValueClass(element);
        }
        return true;
    }
}
```

## 7. 对 Java 生态的影响

JSR 175 是现代 Java 框架的基石，几乎所有主流框架都依赖注解：

| 框架/规范 | 核心注解 | 用途 |
|----------|----------|------|
| **Spring** | `@Component`, `@Autowired`, `@RequestMapping` | 依赖注入、Web MVC |
| **JPA (JSR 338)** | `@Entity`, `@Table`, `@Column`, `@Id` | ORM 映射 |
| **JAX-RS (JSR 370)** | `@Path`, `@GET`, `@POST`, `@Produces` | RESTful API |
| **Bean Validation (JSR 380)** | `@NotNull`, `@Size`, `@Email` | 数据验证 |
| **CDI (JSR 365)** | `@Inject`, `@Named`, `@Qualifier` | 依赖注入标准 |
| **JUnit** | `@Test`, `@BeforeEach`, `@DisplayName` | 单元测试 |
| **Lombok** | `@Data`, `@Builder`, `@Getter` | 代码生成 |
| **Jackson** | `@JsonProperty`, `@JsonIgnore` | JSON 序列化 |

### 从 XML 到注解的范式转变

```java
// JDK 1.4 时代: EJB 2.x XML 描述符
// ejb-jar.xml (数百行 XML)
<enterprise-beans>
    <session>
        <ejb-name>UserService</ejb-name>
        <home>com.example.UserServiceHome</home>
        <remote>com.example.UserServiceRemote</remote>
        <ejb-class>com.example.UserServiceBean</ejb-class>
    </session>
</enterprise-beans>

// JDK 5+ 时代: 注解驱动
@Stateless
public class UserService {
    @PersistenceContext
    private EntityManager em;

    @TransactionAttribute(TransactionAttributeType.REQUIRED)
    public User findById(Long id) {
        return em.find(User.class, id);
    }
}
```

## 8. 与其他 JSR/JEP 的关系

### 注解的演进

| 版本 | 增强 | 说明 |
|------|------|------|
| JDK 5 | JSR 175 | 注解基础 |
| JDK 6 | JSR 269 | 可插拔注解处理 API |
| JDK 7 | `@SafeVarargs` | 抑制泛型可变参数警告 |
| JDK 8 | JSR 308 | 类型注解 (`TYPE_USE`, `TYPE_PARAMETER`) |
| JDK 8 | `@Repeatable` | 可重复注解 |
| JDK 9 | JEP 277 | `@Deprecated(since, forRemoval)` 增强 |
| JDK 14 | Records | `@Override` 自动应用于 record 方法 |

### @Repeatable (JDK 8)

```java
// JDK 8 之前: 需要容器注解
@Schedules({
    @Schedule(dayOfWeek = "Mon"),
    @Schedule(dayOfWeek = "Fri")
})
public void biWeeklyTask() { }

// JDK 8 之后: 可重复注解
@Schedule(dayOfWeek = "Mon")
@Schedule(dayOfWeek = "Fri")
public void biWeeklyTask() { }
```

## 9. 最佳实践

### 1. 选择正确的保留策略

```java
// 编译期检查: SOURCE
@Retention(RetentionPolicy.SOURCE)
public @interface NonNull { }

// 运行时框架: RUNTIME
@Retention(RetentionPolicy.RUNTIME)
public @interface Transactional { }
```

### 2. 限定注解目标

```java
// 明确指定 @Target 避免误用
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Cacheable {
    String key() default "";
    int ttl() default 3600;
}
```

### 3. 提供合理默认值

```java
// 大多数元素应有默认值，减少使用者负担
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
public @interface Service {
    String value() default "";
    String[] profiles() default {};
    boolean lazy() default false;
}
```
