# JSR 335: Lambda Expressions

> **状态**: ✅ Final | **JDK**: 8 (2014-03) | **链接**: https://jcp.org/en/jsr/detail?id=335

---

## 概述

JSR 335 定义了 Java 语言中 Lambda 表达式和函数式接口的规范，是 Java 8 最重要的特性之一。这是 Java 历史上被请求最多的特性，经过了超过十年的社区讨论。

## 核心规范

### Lambda 表达式语法

```java
// 基本语法
(parameters) -> expression
(parameters) -> { statements; }

// 示例
(int x, int y) -> x + y
(x, y) -> x + y
x -> x * x
() -> 42
(s) -> System.out.println(s)
```

### 函数式接口

```java
// 函数式接口: 只有一个抽象方法的接口
@FunctionalInterface
public interface Predicate<T> {
    boolean test(T t);
}

// 使用
Predicate<String> isEmpty = s -> s.isEmpty();
```

### 方法引用

```java
// 静态方法引用
Function<String, Integer> parseInt = Integer::parseInt;

// 实例方法引用
Function<String, String> toUpper = String::toUpperCase;

// 构造器引用
Supplier<List<String>> listSupplier = ArrayList::new;
```

---

## 标准函数式接口

| 接口 | 参数 | 返回 | 示例 |
|------|------|------|------|
| `Runnable` | 无 | 无 | `() -> doSomething()` |
| `Supplier<T>` | 无 | T | `() -> getValue()` |
| `Consumer<T>` | T | 无 | `x -> process(x)` |
| `Function<T,R>` | T | R | `x -> transform(x)` |
| `Predicate<T>` | T | boolean | `x -> test(x)` |
| `BiFunction<T,U,R>` | T, U | R | `(x, y) -> combine(x, y)` |
| `UnaryOperator<T>` | T | T | `x -> negate(x)` |
| `BinaryOperator<T>` | T, T | T | `(x, y) -> add(x, y)` |

## Stream API

JSR 335 同时定义了 Stream API：

```java
// 流操作
List<String> names = people.stream()
    .filter(p -> p.getAge() > 18)
    .map(Person::getName)
    .sorted()
    .collect(Collectors.toList());

// 并行流
long count = data.parallelStream()
    .filter(x -> x > 0)
    .count();
```

## 实现细节

### invokedynamic

Lambda 表达式使用 `invokedynamic` 指令实现：

```java
// Java 代码
Runnable r = () -> System.out.println("Hello");

// 字节码 (简化)
invokedynamic #42:run()Ljava/lang/Runnable;
```

**优势**：
- 延迟绑定
- JIT 优化友好
- 未来实现可替换
