## 2. 概述

JSR 335 是 Java 8 最重要的语言特性，引入了函数式编程能力：

| 特性 | 说明 |
|------|------|
| **Lambda 表达式** | 匿名函数，简洁的代码块 |
| **方法引用** | 已有方法的简洁表示 |
| **默认方法** | 接口可以有方法实现 |
| **函数式接口** | 只有一个抽象方法的接口 |
| **Stream API** | 函数式数据处理 |

## 3. 核心特性

### 1. Lambda 表达式

```java
// 语法
(parameters) -> expression
(parameters) -> { statements; }

// 示例
Runnable r = () -> System.out.println("Hello");
Comparator<String> c = (a, b) -> a.compareTo(b);
Function<String, Integer> f = s -> s.length();

// 多参数
BiFunction<Integer, Integer, Integer> add = (a, b) -> a + b;

// 多语句
Predicate<String> p = s -> {
    if (s == null) return false;
    return s.length() > 0;
};
```

### 2. 方法引用

```java
// 静态方法引用
Function<String, Integer> f = Integer::parseInt;

// 实例方法引用 (特定对象)
System.out::println

// 实例方法引用 (任意对象)
String::toUpperCase

// 构造函数引用
Supplier<List<String>> s = ArrayList::new;

// 数组构造函数引用
Function<Integer, int[]> a = int[]::new;
```

### 3. 默认方法

```java
interface Collection<E> {
    // 默认方法实现
    default Stream<E> stream() {
        return StreamSupport.stream(spliterator(), false);
    }

    default Stream<E> parallelStream() {
        return StreamSupport.stream(spliterator(), true);
    }

    // 可以被重写
    default void forEach(Consumer<? super E> action) {
        for (E e : this) {
            action.accept(e);
        }
    }
}
```

### 4. 函数式接口

```java
@FunctionalInterface
public interface Function<T, R> {
    R apply(T t);
    
    // 默认方法可以组合函数
    default <V> Function<V, R> compose(Function<? super V, ? extends T> before) {
        return v -> apply(before.apply(v));
    }
    
    default <V> Function<T, V> andThen(Function<? super R, ? extends V> after) {
        return t -> after.apply(apply(t));
    }
}
```

## 4. 标准函数式接口

### java.util.function 包

| 接口 | 参数 | 返回 | 示例 |
|------|------|------|------|
| `Runnable` | 无 | 无 | `() -> doSomething()` |
| `Supplier<T>` | 无 | T | `() -> new ArrayList()` |
| `Consumer<T>` | T | 无 | `s -> System.out.println(s)` |
| `Function<T,R>` | T | R | `s -> s.length()` |
| `Predicate<T>` | T | boolean | `s -> s.isEmpty()` |
| `BiFunction<T,U,R>` | T, U | R | `(a, b) -> a + b` |
| `BinaryOperator<T>` | T, T | T | `(a, b) -> a + b` |
| `UnaryOperator<T>` | T | T | `s -> s.toUpperCase()` |

### 原始类型特化

| 接口 | 说明 |
|------|------|
| `IntFunction<R>` | int → R |
| `IntConsumer` | int → void |
| `IntPredicate` | int → boolean |
| `IntSupplier` | () → int |
| `ToIntFunction<T>` | T → int |

## 5. Stream API

### 创建 Stream

```java
// 从集合
Stream<String> s1 = list.stream();
Stream<String> s2 = set.parallelStream();

// 从数组
Stream<Integer> s3 = Arrays.stream(new Integer[]{1, 2, 3});

// 静态方法
Stream<Integer> s4 = Stream.of(1, 2, 3);
Stream<Integer> s5 = Stream.iterate(0, n -> n + 1);
Stream<Double> s6 = Stream.generate(Math::random);

// 原始类型流
IntStream ints = IntStream.range(1, 100);
LongStream longs = LongStream.rangeClosed(1, 100);
DoubleStream doubles = DoubleStream.of(1.0, 2.0, 3.0);
```

### 中间操作

```java
list.stream()
    .filter(s -> s.length() > 3)     // 过滤
    .map(String::toUpperCase)        // 映射
    .flatMap(s -> Arrays.stream(s.split("")))  // 扁平化
    .distinct()                      // 去重
    .sorted()                        // 排序
    .sorted(Comparator.reverseOrder())
    .peek(System.out::println)       // 调试
    .limit(10)                       // 限制
    .skip(5);                        // 跳过
```

### 终端操作

```java
// 收集
List<String> list = stream.collect(Collectors.toList());
Set<String> set = stream.collect(Collectors.toSet());
Map<String, Integer> map = stream.collect(Collectors.toMap(s -> s, String::length));

// 聚合
Optional<String> max = stream.max(Comparator.naturalOrder());
Optional<String> min = stream.min(Comparator.naturalOrder());
long count = stream.count();

// 归约
Optional<String> reduced = stream.reduce((a, b) -> a + b);
String reduced = stream.reduce("", (a, b) -> a + b);

// 匹配
boolean anyMatch = stream.anyMatch(s -> s.startsWith("A"));
boolean allMatch = stream.allMatch(s -> s.length() > 0);
boolean noneMatch = stream.noneMatch(Objects::isNull);

// 查找
Optional<String> first = stream.findFirst();
Optional<String> any = stream.findAny();

// 遍历
stream.forEach(System.out::println);
```

## 6. 实现原理

### invokedynamic 字节码

Lambda 表达式不生成匿名内部类，而是使用 `invokedynamic` 指令：

```java
// Java 代码
Runnable r = () -> System.out.println("Hello");

// 字节码 (javap -c -p)
0: invokedynamic #42, 0  // InvokeDynamic #0:run:()Ljava/lang/Runnable;
5: astore_1

// Bootstrap Method
BootstrapMethods:
  0: #37 invokestatic java/lang/invoke/LambdaMetafactory.metafactory:
    (Ljava/lang/invoke/MethodHandles$Lookup;
     Ljava/lang/String;
     Ljava/lang/invoke/MethodType;
     Ljava/lang/invoke/MethodType;
     Ljava/lang/invoke/MethodHandle;
     Ljava/lang/invoke/MethodType;)Ljava/lang/invoke/CallSite;
    Method arguments:
      #39 ()V
      #40 invokestatic LambdaDemo.lambda$main$0:()V
      #39 ()V
```

### LambdaMetafactory

```java
// JDK 内部实现
public final class LambdaMetafactory {
    public static CallSite metafactory(
        MethodHandles.Lookup caller,
        String invokedName,
        MethodType invokedType,
        MethodType samMethodType,
        MethodHandle implMethod,
        MethodType instantiatedMethodType) {
        // 运行时生成实现类
    }
}
```

### 优点

| 方式 | 类文件 | 内存 | 性能 |
|------|--------|------|------|
| 匿名内部类 | 每次生成新类 | 较高 | 一般 |
| invokedynamic | 无额外类 | 低 | 可优化 |

## 8. 最佳实践

### 1. 优先使用方法引用

```java
// 不推荐
list.stream().map(s -> s.toUpperCase())
list.stream().map(s -> System.out.println(s))

// 推荐
list.stream().map(String::toUpperCase)
list.stream().forEach(System.out::println)
```

### 2. 保持 Lambda 简短

```java
// 不推荐
list.stream().filter(s -> {
    if (s == null) return false;
    if (s.isEmpty()) return false;
    return s.startsWith("A");
})

// 推荐：提取方法
list.stream().filter(this::isValidString)

private boolean isValidString(String s) {
    return s != null && !s.isEmpty() && s.startsWith("A");
}
```

### 3. 使用原始类型流

```java
// 不推荐 (装箱开销)
Stream<Integer> s = list.stream();
int sum = s.mapToInt(Integer::intValue).sum();

// 推荐
IntStream s = list.stream().mapToInt(Integer::intValue);
int sum = s.sum();
```

### 4. 并行流注意事项

```java
// 小数据量不需要并行
list.parallelStream()  // 可能更慢

// 无状态操作
list.parallelStream()
    .filter(s -> s.length() > 3)  // OK: 无状态
    .map(s -> s.toUpperCase())    // OK: 无状态
    .sorted()                      // OK: 有状态但安全

// 避免共享可变状态
List<String> result = new ArrayList<>();
list.parallelStream()
    .forEach(result::add);  // 危险: 并发修改
```

## 9. 性能考量

### Lambda vs 匿名内部类

```java
// JMH 基准测试
@Benchmark
public void anonymousInnerClass() {
    Runnable r = new Runnable() {
        @Override
        public void run() {
            // do nothing
        }
    };
}

@Benchmark
public void lambda() {
    Runnable r = () -> {};
}

// 结果: Lambda 稍快 (~5-10%)
// 原因: 无额外类加载，更好的 JIT 优化
```

### Stream vs for 循环

```java
// 简单操作: for 循环更快
int sum = 0;
for (int i : list) {
    sum += i;
}

// 复杂操作: Stream 可读性更好
int sum = list.stream()
    .filter(i -> i > 0)
    .mapToInt(Integer::intValue)
    .sum();
```
