## 2. 概述

JSR 221 (JDBC 4.0) 是 JDBC API 的重要现代化版本，随 JDK 6 发布。最显著的改进是利用 `ServiceLoader` 机制实现驱动自动发现，终结了长期困扰开发者的 `Class.forName("com.mysql.jdbc.Driver")` 模式。

| 特性 | 说明 |
|------|------|
| **驱动自动发现** | 通过 `ServiceLoader` 自动加载 JDBC 驱动 |
| **SQLXML** | 原生支持 SQL/XML 数据类型 |
| **RowId** | 支持数据库原生行标识符 |
| **NClob / NString** | 国际化字符类型支持 |
| **异常链** | `SQLException` 实现 `Iterable`，支持 for-each 遍历 |
| **增强的 BLOB/CLOB** | 创建、释放方法 |

## 3. 核心改进

### JDBC 4.0 之前 vs 之后

```java
// ===== JDBC 3.0 (JDK 1.4) — 手动加载驱动 =====
Class.forName("com.mysql.jdbc.Driver");  // 必须显式加载
Connection conn = DriverManager.getConnection(url, user, pass);

// ===== JDBC 4.0 (JDK 6) — 自动发现 =====
// 无需 Class.forName()，驱动 JAR 中包含
// META-INF/services/java.sql.Driver 文件即可自动加载
Connection conn = DriverManager.getConnection(url, user, pass);
```

### try-with-resources (JDK 7 配合)

```java
// JDBC 4.0 的 Connection/Statement/ResultSet 继承 AutoCloseable
// 配合 JDK 7 try-with-resources 实现自动关闭

// 之前：冗长的 finally 块
Connection conn = null;
Statement stmt = null;
ResultSet rs = null;
try {
    conn = DriverManager.getConnection(url);
    stmt = conn.createStatement();
    rs = stmt.executeQuery("SELECT * FROM users");
} finally {
    if (rs != null) rs.close();
    if (stmt != null) stmt.close();
    if (conn != null) conn.close();
}

// 之后：简洁的 try-with-resources
try (Connection conn = DriverManager.getConnection(url);
     Statement stmt = conn.createStatement();
     ResultSet rs = stmt.executeQuery("SELECT * FROM users")) {
    while (rs.next()) {
        String name = rs.getString("name");
    }
}  // 自动关闭，逆序
```

## 4. 驱动自动发现

### ServiceLoader 机制

```
驱动 JAR 文件结构:
mysql-connector-java.jar
├── com/mysql/cj/jdbc/Driver.class
└── META-INF/
    └── services/
        └── java.sql.Driver          ← 包含: com.mysql.cj.jdbc.Driver
```

```java
// DriverManager 内部使用 ServiceLoader 加载所有驱动
// java.sql.DriverManager 源码 (简化):
ServiceLoader<Driver> loadedDrivers = ServiceLoader.load(Driver.class);
for (Driver driver : loadedDrivers) {
    // 自动注册
}
```

## 5. 新数据类型

### SQLXML

```java
// 原生 XML 类型支持
SQLXML xml = conn.createSQLXML();
xml.setString("<root><item>value</item></root>");

PreparedStatement ps = conn.prepareStatement("INSERT INTO docs(xml_col) VALUES(?)");
ps.setSQLXML(1, xml);
ps.executeUpdate();

// 读取
ResultSet rs = stmt.executeQuery("SELECT xml_col FROM docs");
if (rs.next()) {
    SQLXML result = rs.getSQLXML("xml_col");
    String xmlStr = result.getString();
    // 也可获取 StAX reader:
    // XMLStreamReader reader = result.getSource(StAXSource.class);
}
```

### RowId

```java
// 数据库原生行标识符 (如 Oracle ROWID)
ResultSet rs = stmt.executeQuery("SELECT ROWID, name FROM users");
if (rs.next()) {
    RowId rowId = rs.getRowId("ROWID");
    // 可用于快速定位行
    PreparedStatement ps = conn.prepareStatement(
        "SELECT * FROM users WHERE ROWID = ?"
    );
    ps.setRowId(1, rowId);
}
```

### NClob / National Character

```java
// 国际化字符支持
NClob nclob = conn.createNClob();
nclob.setString(1, "中文内容");

PreparedStatement ps = conn.prepareStatement("INSERT INTO t(ncol) VALUES(?)");
ps.setNClob(1, nclob);
ps.setNString(2, "日本語テキスト");
```

## 6. 异常处理增强

```java
// JDBC 4.0: SQLException 实现 Iterable<Throwable>
try {
    // 数据库操作
} catch (SQLException e) {
    // 之前：手动遍历异常链
    // SQLException next = e; while (next != null) { next = next.getNextException(); }

    // 之后：for-each 遍历 (包含 cause chain)
    for (Throwable t : e) {
        System.err.println(t.getMessage());
    }
}

// 新的异常子类层次
// SQLTransientException        — 瞬时错误，重试可能成功
//   ├── SQLTransientConnectionException
//   └── SQLTimeoutException
// SQLNonTransientException     — 非瞬时错误，重试不会成功
//   ├── SQLNonTransientConnectionException
//   ├── SQLSyntaxErrorException
//   └── SQLIntegrityConstraintViolationException
// SQLRecoverableException      — 可恢复错误
```

boolean valid = conn.isValid(5);  // 5秒超时，替代自定义心跳 SQL

## 8. JDBC 版本演进

| 版本 | JDK | 年份 | 关键特性 |
|------|-----|------|----------|
| JDBC 1.0 | 1.1 | 1997 | 基础 API: Connection, Statement, ResultSet |
| JDBC 2.0 | 1.2 | 1998 | DataSource, 可滚动 ResultSet, 批量更新 |
| JDBC 3.0 | 1.4 | 2002 | Savepoint, ParameterMetaData, 连接池标准化 |
| **JDBC 4.0** | **6** | **2006** | **自动发现, SQLXML, RowId, 异常增强** |
| JDBC 4.1 | 7 | 2011 | try-with-resources, `getObject(col, Class)` |
| JDBC 4.2 | 8 | 2014 | `java.time` 支持, `executeLargeUpdate()` |
| JDBC 4.3 | 9 | 2017 | `Connection.beginRequest/endRequest`, Sharding |

### java.time 支持 (JDBC 4.2)

```java
// JDBC 4.2 起支持 java.time 类型
PreparedStatement ps = conn.prepareStatement("INSERT INTO events(ts) VALUES(?)");
ps.setObject(1, LocalDateTime.now());

ResultSet rs = stmt.executeQuery("SELECT ts FROM events");
LocalDateTime ts = rs.getObject("ts", LocalDateTime.class);
LocalDate date = rs.getObject("date_col", LocalDate.class);
```

## 9. 最佳实践

### 1. 不再需要 Class.forName

```java
// 不推荐 (JDBC 4.0 之前的遗留写法)
Class.forName("com.mysql.cj.jdbc.Driver");

// 推荐：直接获取连接
Connection conn = DriverManager.getConnection(url, user, pass);
```

### 2. 使用 try-with-resources

```java
// 不推荐：手动 finally 关闭
// 推荐：自动资源管理
try (Connection conn = dataSource.getConnection();
     PreparedStatement ps = conn.prepareStatement(sql)) {
    ps.setString(1, value);
    try (ResultSet rs = ps.executeQuery()) {
        // 处理结果
    }
}
```

### 3. 使用 isValid 检查连接

```java
// 不推荐：执行 "SELECT 1" 做心跳检测
// 推荐：使用标准 API
if (!conn.isValid(3)) {
    conn = dataSource.getConnection();  // 重新获取
}
```
