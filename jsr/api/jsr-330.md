## 2. 概述

JSR 330 定义了一组最小化的依赖注入 (Dependency Injection) 标准注解，实现了 DI 容器之间的可移植性：

| 特点 | 说明 |
|------|------|
| 极简设计 | 仅 5 个注解 + 1 个接口，总共 6 个类型 |
| 容器无关 | 代码不绑定特定 DI 框架 (Framework-agnostic) |
| 广泛采纳 | Spring、Guice、CDI、Dagger 全部支持 |
| 由 Google 主导 | Bob Lee (Guice 作者) 担任 Spec Lead |

**Expert Group 成员**: Bob Lee (Google)、Rod Johnson (Spring)、Gavin King (Red Hat/CDI)

**包名变迁**: `javax.inject` → `jakarta.inject`（Jakarta EE 9+）

## 3. 核心注解与接口

### @Inject — 注入标记

```java
import javax.inject.Inject;

public class OrderService {
    // 构造函数注入 (Constructor Injection) — 推荐
    @Inject
    public OrderService(OrderRepository repo, PaymentGateway gateway) {
        this.repo = repo;
        this.gateway = gateway;
    }

    // 字段注入 (Field Injection)
    @Inject
    private NotificationService notifier;

    // 方法注入 (Method Injection)
    @Inject
    public void setLogger(Logger logger) {
        this.logger = logger;
    }
}
```

### @Named — 字符串限定符

```java
import javax.inject.Named;

// 按名称区分同类型的多个实现
@Inject
public PaymentService(@Named("stripe") PaymentGateway gateway) { ... }

// 在实现类上标注
@Named("stripe")
public class StripeGateway implements PaymentGateway { ... }

@Named("paypal")
public class PayPalGateway implements PaymentGateway { ... }
```

### @Singleton — 单例作用域

```java
import javax.inject.Singleton;

@Singleton
public class ConfigService {
    // 容器保证只创建一个实例 (Container Guarantees Single Instance)
}
```

## 4. 注入方式

### 注入优先级 (Injection Order)

```
┌─────────────────────────────────────────────────────────┐
│  JSR 330 注入顺序 (Injection Order)                      │
├─────────────────────────────────────────────────────────┤
│  1. 构造函数注入 (Constructor)     ← 优先推荐            │
│  2. 字段注入 (Field)               ← 简洁但难测试        │
│  3. 方法注入 (Method/Setter)       ← 可选依赖            │
└─────────────────────────────────────────────────────────┘
```

### 构造函数注入 vs 字段注入

| 方式 | 优点 | 缺点 |
|------|------|------|
| 构造函数注入 | 不可变、易测试、强制依赖 | 参数多时冗长 |
| 字段注入 | 简洁 | 难以单元测试、隐藏依赖 |
| 方法注入 | 可选依赖、可重新注入 | 使用较少 |

## 5. 限定符与作用域

### @Qualifier — 自定义限定符

```java
import javax.inject.Qualifier;
import java.lang.annotation.*;

@Qualifier
@Retention(RetentionPolicy.RUNTIME)
@Target({ElementType.FIELD, ElementType.PARAMETER, ElementType.METHOD})
public @interface Premium { }

// 使用自定义限定符
@Inject
public SubscriptionService(@Premium BillingProvider provider) { ... }

@Premium
public class PremiumBillingProvider implements BillingProvider { ... }
```

### @Scope — 自定义作用域

```java
import javax.inject.Scope;

@Scope
@Retention(RetentionPolicy.RUNTIME)
public @interface RequestScoped { }
// 各框架可自行实现 RequestScoped 的生命周期管理
```

## 6. Provider 延迟注入

```java
import javax.inject.Provider;

public class ReportService {
    private final Provider<DbConnection> connectionProvider;

    @Inject
    public ReportService(Provider<DbConnection> connectionProvider) {
        this.connectionProvider = connectionProvider;
    }

    public void generateReport() {
        // 每次调用 get() 可获取新实例 (New Instance per get())
        DbConnection conn = connectionProvider.get();
        // ... 使用连接
    }
}
```

**Provider 用途**:
- **延迟初始化** (Lazy Initialization) — 需要时才创建
- **多实例获取** (Multiple Instances) — 每次 get() 创建新对象
- **打破循环依赖** (Break Circular Dependencies)

## 7. 框架实现对比

| 框架 | JSR 330 支持 | 额外功能 |
|------|-------------|---------|
| **Spring** | 完全支持 + @Autowired 扩展 | AOP、事务、完整 IoC 容器 |
| **Google Guice** | 原生支持 (Bob Lee 设计) | Module 绑定、AOP |
| **CDI (Weld)** | JSR 330 + JSR 365 超集 | 事件、拦截器、装饰器 |
| **Dagger 2** | 编译时 DI，完全基于 JSR 330 | 零反射、Android 友好 |
| **HK2** | Jersey/GlassFish 使用 | 服务定位器模式 |

### Spring 中的 JSR 330

```java
// JSR 330 注解在 Spring 中完全可用
@Named        // 等价于 @Component
@Inject       // 等价于 @Autowired
@Singleton    // 等价于 @Scope("singleton")
```

## 8. Jakarta 迁移

```
javax.inject → jakarta.inject (Jakarta EE 9+, 2020)
```

| 版本 | 包名 | 规范 |
|------|------|------|
| JSR 330 | javax.inject | Java EE |
| Jakarta Inject 2.0 | jakarta.inject | Jakarta EE 9 |

```java
// 迁移前
import javax.inject.Inject;
import javax.inject.Named;

// 迁移后
import jakarta.inject.Inject;
import jakarta.inject.Named;
```

Maven 依赖变化：
```xml
<!-- 旧 -->
<dependency>
    <groupId>javax.inject</groupId>
    <artifactId>javax.inject</artifactId>
    <version>1</version>
</dependency>

<!-- 新 -->
<dependency>
    <groupId>jakarta.inject</groupId>
    <artifactId>jakarta.inject-api</artifactId>
    <version>2.0.1</version>
</dependency>
```

## 9. 最佳实践

1. **优先构造函数注入** — 不可变、可测试、依赖显式化
2. **使用 @Qualifier 而非 @Named** — 类型安全，编译期检查
3. **保持 JSR 330 标准注解** — 避免框架锁定 (Avoid Vendor Lock-in)
4. **Provider 用于延迟加载** — 避免不必要的初始化开销

```java
// 推荐: 构造函数注入 + 自定义限定符
@Singleton
public class OrderService {
    private final OrderRepository repo;
    private final Provider<AuditLogger> auditProvider;

    @Inject
    public OrderService(OrderRepository repo, Provider<AuditLogger> auditProvider) {
        this.repo = repo;
        this.auditProvider = auditProvider;
    }
}
```
