## 2. 概述

JSR 223 为 Java 平台定义了统一的脚本语言集成 API，允许 Java 应用程序嵌入和执行多种脚本语言，实现 Java 与动态语言的互操作：

| 特性 | 说明 |
|------|------|
| 核心包 | `javax.script` |
| 引擎发现 | SPI 机制 (`ScriptEngineFactory`) |
| 变量共享 | `Bindings` — Java 对象与脚本变量互通 |
| 编译支持 | `Compilable` — 预编译提升重复执行性能 |
| 函数调用 | `Invocable` — 从 Java 调用脚本函数 |
| 内置引擎 | Rhino (JDK 6-7), Nashorn (JDK 8-14) |

## 3. 核心 API

### javax.script 包结构

```java
// 核心接口
ScriptEngine              // 脚本执行引擎
ScriptEngineManager       // 引擎管理器 (工厂)
ScriptEngineFactory       // 引擎工厂 (SPI)
Bindings                  // 变量绑定 (key-value)
ScriptContext             // 执行上下文

// 可选接口
Compilable                // 支持预编译的引擎
Invocable                 // 支持函数调用的引擎
CompiledScript            // 已编译的脚本对象
```

## 4. ScriptEngine 基础用法

### 获取引擎

```java
ScriptEngineManager manager = new ScriptEngineManager();

// 按名称获取
ScriptEngine engine = manager.getEngineByName("js");           // JavaScript
ScriptEngine engine = manager.getEngineByName("nashorn");      // Nashorn
ScriptEngine engine = manager.getEngineByName("graal.js");     // GraalJS

// 按文件扩展名获取
ScriptEngine engine = manager.getEngineByExtension("js");

// 按 MIME 类型获取
ScriptEngine engine = manager.getEngineByMimeType("application/javascript");

// 列出所有可用引擎
for (ScriptEngineFactory factory : manager.getEngineFactories()) {
    System.out.printf("引擎: %s, 版本: %s, 语言: %s%n",
        factory.getEngineName(),
        factory.getEngineVersion(),
        factory.getLanguageName());
}
```

### 执行脚本

```java
ScriptEngine engine = manager.getEngineByName("js");

// 执行字符串脚本
Object result = engine.eval("1 + 2");   // 返回 3

// 执行脚本文件
try (Reader reader = new FileReader("script.js")) {
    Object result = engine.eval(reader);
}

// 多行脚本
engine.eval("""
    var greeting = 'Hello';
    var name = 'Java';
    greeting + ', ' + name + '!';
    """);
```

## 5. Bindings 变量绑定

### Java → 脚本

```java
ScriptEngine engine = manager.getEngineByName("js");

// 设置变量
engine.put("name", "World");
engine.put("count", 42);
engine.put("list", List.of("a", "b", "c"));

// 脚本中使用 Java 变量
engine.eval("print('Hello, ' + name)");      // Hello, World
engine.eval("print('Count: ' + count)");      // Count: 42
engine.eval("print('Size: ' + list.size())"); // Size: 3
```

### 脚本 → Java

```java
// 从脚本获取结果
engine.eval("var result = 100 * 2");
Object result = engine.get("result");   // 200
```

### Scope 作用域

```java
// ENGINE_SCOPE — 引擎级别
Bindings engineScope = engine.getBindings(ScriptContext.ENGINE_SCOPE);

// GLOBAL_SCOPE — 全局级别 (所有引擎共享)
Bindings globalScope = engine.getBindings(ScriptContext.GLOBAL_SCOPE);

// 独立的 Bindings — 隔离执行环境
Bindings isolated = engine.createBindings();
isolated.put("x", 10);
engine.eval("var y = x * 2", isolated);   // 不影响引擎默认作用域
```

## 6. Compilable 编译优化

```java
ScriptEngine engine = manager.getEngineByName("js");

if (engine instanceof Compilable compilable) {
    // 预编译脚本 — 适合重复执行
    CompiledScript compiled = compilable.compile(
        "var total = price * quantity * (1 - discount)");

    // 多次执行，避免重复解析
    Bindings bindings = engine.createBindings();

    bindings.put("price", 100.0);
    bindings.put("quantity", 5);
    bindings.put("discount", 0.1);
    compiled.eval(bindings);   // total = 450.0

    bindings.put("price", 200.0);
    bindings.put("quantity", 3);
    bindings.put("discount", 0.2);
    compiled.eval(bindings);   // total = 480.0
}
```

## 7. Invocable 函数调用

### 调用脚本函数

```java
ScriptEngine engine = manager.getEngineByName("js");

engine.eval("""
    function add(a, b) { return a + b; }
    function greet(name) { return 'Hello, ' + name; }
    """);

if (engine instanceof Invocable invocable) {
    Object sum = invocable.invokeFunction("add", 3, 4);       // 7
    Object msg = invocable.invokeFunction("greet", "Java");   // Hello, Java
}
```

### 实现 Java 接口

```java
// Java 接口
public interface Calculator {
    double add(double a, double b);
    double multiply(double a, double b);
}

// 脚本实现接口
engine.eval("""
    function add(a, b) { return a + b; }
    function multiply(a, b) { return a * b; }
    """);

Invocable invocable = (Invocable) engine;
Calculator calc = invocable.getInterface(Calculator.class);
calc.add(3.0, 4.0);       // 7.0
calc.multiply(3.0, 4.0);  // 12.0
```

## 8. 脚本引擎生态

### 引擎历史演变

| 引擎 | JDK 版本 | 状态 | 说明 |
|------|----------|------|------|
| **Rhino** | 6 - 7 | 已移除 | Mozilla JavaScript 引擎 |
| **Nashorn** | 8 - 14 | 已移除 (JEP 372) | 高性能 JS 引擎 (JEP 174 引入) |
| **GraalJS** | 外部依赖 | 活跃 | GraalVM 的 JavaScript 引擎 |

### 使用 GraalJS (现代替代方案)

```xml
<!-- Maven 依赖 -->
<dependency>
    <groupId>org.graalvm.js</groupId>
    <artifactId>js-scriptengine</artifactId>
    <version>23.0.0</version>
</dependency>
```

```java
ScriptEngine graalEngine = manager.getEngineByName("graal.js");
graalEngine.eval("print('Hello from GraalJS')");
```

## 9. 最佳实践

### 1. 线程安全

```java
// ScriptEngine 通常不是线程安全的
// 方案 1: 每线程一个引擎
ThreadLocal<ScriptEngine> engineHolder = ThreadLocal.withInitial(
    () -> new ScriptEngineManager().getEngineByName("js"));

// 方案 2: 使用独立 Bindings 隔离状态
Bindings threadBindings = engine.createBindings();
engine.eval(script, threadBindings);
```

### 2. 安全沙箱

```java
// 限制脚本访问 Java 类 (Nashorn 示例)
NashornScriptEngineFactory factory = new NashornScriptEngineFactory();
ScriptEngine engine = factory.getScriptEngine("--no-java");
// 脚本无法访问 Java.type(), Packages 等
```

### 3. 性能 — 预编译 + 复用引擎

```java
// 缓存 CompiledScript 避免重复编译
Map<String, CompiledScript> scriptCache = new ConcurrentHashMap<>();

CompiledScript getOrCompile(String key, String source) {
    return scriptCache.computeIfAbsent(key, k -> {
        try { return ((Compilable) engine).compile(source); }
        catch (ScriptException e) { throw new RuntimeException(e); }
    });
}
```
