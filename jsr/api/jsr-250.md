## 2. 概述

JSR 250 定义了一组通用注解（Common Annotations），旨在被多个 Java EE/SE 技术规范共享，避免每个规范重复定义相同语义的注解。这些注解从 JDK 6 起包含在 `javax.annotation` 包中，是 Java 生态中使用最广泛的注解之一。

| 注解 | 说明 |
|------|------|
| **@Resource** | 声明对外部资源的依赖，由容器注入 |
| **@PostConstruct** | 依赖注入完成后执行的初始化方法 |
| **@PreDestroy** | 容器销毁 Bean 前执行的清理方法 |
| **@Generated** | 标记由工具自动生成的代码 |
| **@Priority** | 指定组件的优先级（值越小优先级越高） |
| **@ManagedBean** | 轻量级容器托管 Bean（JDK 11 移除） |

### javax.annotation 包结构

```
javax.annotation (JDK 6-8) / jakarta.annotation (Jakarta EE 9+)
├── @Resource

├── @PostConstruct
├── @PreDestroy
├── @Generated
├── @Priority               (JSR 250 1.2, javax.annotation.Priority)

├── security/
│   ├── @RolesAllowed

│   ├── @DenyAll

    └── @DataSourceDefinition
```

## 4. @Resource — 资源注入

### 基本用法

```java
public class OrderService {

    // 按名称注入 DataSource
    @Resource(name = "jdbc/orderDB")
    private DataSource dataSource;

    // 按类型注入 (容器自动匹配)
    @Resource
    private DataSource ds;

    // 注入环境变量
    @Resource(name = "maxRetries")
    private int maxRetries;

    // setter 注入
    @Resource(name = "jms/orderQueue")
    public void setQueue(Queue queue) {
        this.queue = queue;
    }
}
```

### @Resource vs @Inject vs @Autowired

| 特性 | @Resource (JSR 250) | @Inject (JSR 330) | @Autowired (Spring) |
|------|---------------------|-------------------|---------------------|
| 规范 | Java EE 通用注解 | CDI 依赖注入 | Spring 专有 |
| 匹配策略 | 先按名称，再按类型 | 按类型 | 按类型 |
| name 属性 | 支持 | 不支持 (用 @Named) | 不支持 (用 @Qualifier) |
| 适用范围 | 字段、setter | 字段、构造器、setter | 字段、构造器、setter |
| JNDI 查找 | 支持 | 不支持 | 不支持 |

## 5. @PostConstruct 与 @PreDestroy — 生命周期回调

### 基本用法

```java
@Component  // Spring 或 CDI 托管
public class CacheManager {

    private Cache cache;

    @PostConstruct
    public void init() {
        // 依赖注入完成后调用
        // 适合：连接池初始化、缓存预热、资源验证
        cache = new Cache();
        cache.warmUp();
        System.out.println("CacheManager 初始化完成");
    }

    @PreDestroy
    public void cleanup() {
        // Bean 销毁前调用
        // 适合：关闭连接、释放资源、持久化状态
        cache.flush();
        cache.close();
        System.out.println("CacheManager 已清理");
    }
}
```

### 生命周期顺序

```
Bean 创建流程:
  1. 构造器调用          new CacheManager()
  2. 依赖注入            @Resource / @Inject / @Autowired
  3. @PostConstruct      init()
  4. Bean 就绪           可以被使用
  ...
  5. @PreDestroy         cleanup()
  6. Bean 销毁           GC 回收
```

### 规则约束

```java
// @PostConstruct / @PreDestroy 方法必须满足:
// - void 返回类型
// - 无参数
// - 不抛出受检异常 (checked exception)
// - 非 static
// - 每个类最多一个

@PostConstruct
public void init() { }     // 正确

@PostConstruct
public String init() { }   // 错误: 非 void

@PostConstruct
public void init(String s) { }  // 错误: 有参数
```

## 6. @Generated 与 @Priority

### @Generated — 标记自动生成的代码

```java
// 代码生成工具 (如 JAXB, Protocol Buffers, Lombok) 用此注解标记生成的代码
@Generated(
    value = "com.sun.tools.xjc.Driver",  // 生成工具
    date = "2024-01-15",                   // 生成时间
    comments = "从 schema.xsd 生成"         // 备注
)
public class CustomerDTO {
    // JAXB 自动生成的代码...
}
```

### @Priority — 定义优先级

```java
// JSR 250 1.2 引入，用于排序和选择
// 值越小优先级越高

// CDI: 当有多个实现时，选择优先级最高的
@Priority(100)
@Alternative
public class FastCache implements Cache { }

@Priority(200)
@Alternative
public class SlowCache implements Cache { }
// 容器会选择 FastCache (100 < 200)

// JAX-RS: 过滤器/拦截器执行顺序
@Priority(Priorities.AUTHENTICATION)  // 1000
public class AuthFilter implements ContainerRequestFilter { }

@Priority(Priorities.AUTHORIZATION)   // 2000
public class AuthzFilter implements ContainerRequestFilter { }
// AuthFilter 先执行 (1000 < 2000)
```

## 7. 各框架中的使用

### Spring Framework

```java
@Service
public class UserService {
    @Resource(name = "primaryDataSource")   // JSR 250 注入
    private DataSource dataSource;

    @PostConstruct                           // JSR 250 生命周期
    public void init() { validateConfig(); }

    @PreDestroy                              // JSR 250 清理
    public void destroy() { closePool(); }
}
// Spring 通过 CommonAnnotationBeanPostProcessor 处理 JSR 250 注解
```

### CDI (Jakarta Contexts and Dependency Injection)

```java
@ApplicationScoped
public class AppConfig {
    @Resource(name = "jdbc/appDB")
    private DataSource ds;

    @PostConstruct
    public void startup() { /* CDI 容器自动调用 */ }
}
```

### EJB

```java
@Stateless
public class PaymentBean {
    @Resource
    private SessionContext ctx;       // EJB 上下文注入

    @Resource(name = "jms/paymentQueue")
    private Queue paymentQueue;       // JMS 队列注入

    @PreDestroy
    public void cleanup() { /* 容器管理的 Bean 清理 */ }
}
```

## 8. javax.annotation → jakarta.annotation 迁移

### 迁移背景

随着 Java EE 向 Jakarta EE 过渡（Oracle → Eclipse Foundation），包名从 `javax.*` 迁移到 `jakarta.*`。

| 版本 | 包名 | 说明 |
|------|------|------|
| JDK 6-8 | `javax.annotation` | JDK 内置 |
| JDK 9-10 | `javax.annotation` | 标记为 deprecated for removal |
| JDK 11+ | (已移除) | JEP 320 移除，需外部依赖 |
| Jakarta EE 9+ | `jakarta.annotation` | 新包名 |

### Maven 依赖

```xml
<!-- Java EE 时代 (JDK 11+ 需要) -->
<dependency>
    <groupId>javax.annotation</groupId>
    <artifactId>javax.annotation-api</artifactId>
    <version>1.3.2</version>
</dependency>

<!-- Jakarta EE 9+ 时代 -->
<dependency>
    <groupId>jakarta.annotation</groupId>
    <artifactId>jakarta.annotation-api</artifactId>
    <version>2.1.1</version>
</dependency>
```

### 代码迁移

```java
// 之前
import javax.annotation.Resource;
import javax.annotation.PostConstruct;
import javax.annotation.PreDestroy;

// 之后
import jakarta.annotation.Resource;
import jakarta.annotation.PostConstruct;
import jakarta.annotation.PreDestroy;
```

## 9. 最佳实践

### 1. 优先使用 @PostConstruct 而非构造器初始化

```java
// 不推荐：构造器中使用注入字段 (此时字段尚未注入)
public class MyService {
    @Resource
    private DataSource ds;

    public MyService() {
        ds.getConnection();  // NullPointerException!
    }
}

// 推荐：使用 @PostConstruct
public class MyService {
    @Resource
    private DataSource ds;

    @PostConstruct
    public void init() {
        ds.getConnection();  // 注入已完成，安全
    }
}
```

### 2. @PreDestroy 中释放资源

```java
// 推荐：确保资源清理
@PreDestroy
public void cleanup() {
    if (connection != null) connection.close();
    if (executor != null) executor.shutdown();
}
```

### 3. JDK 11+ 项目添加外部依赖

```java
// JDK 11 移除了 javax.annotation，必须显式添加依赖
// 否则编译报错: package javax.annotation does not exist
```
