## 2. 概述

JSR 354 为 Java 提供标准化的货币与金额处理 API，解决 `BigDecimal` 缺乏货币语义的问题：

| 特点 | 说明 |
|------|------|
| 类型安全 | 货币与金额绑定，不同币种无法混合运算 (Type-safe) |
| 货币转换 | 内置汇率转换框架 (Exchange Rate Framework) |
| 精确计算 | 基于 BigDecimal，避免浮点精度问题 |
| 独立发布 | **未纳入 JDK SE**，作为独立依赖使用 |

**未进入 JDK 的原因**: 虽通过最终投票，Oracle 决定不纳入 JDK 9（API 设计争议和维护负担）。以 JavaMoney 社区项目独立发展。

## 3. 核心接口与操作

```java
import javax.money.*;
import org.javamoney.moneta.Money;

// CurrencyUnit — 货币单位
CurrencyUnit cny = Monetary.getCurrency("CNY");
CurrencyUnit usd = Monetary.getCurrency(Locale.US);

// MonetaryAmount — 货币金额
MonetaryAmount a = Money.of(100, "CNY");
MonetaryAmount b = Money.of(50, "CNY");

a.add(b);            // ¥150
a.subtract(b);       // ¥50
a.multiply(3);       // ¥300
a.isGreaterThan(b);  // true

// ⚠ 不同币种运算抛 MonetaryException
// Money.of(100, "CNY").add(Money.of(50, "USD")); → 异常!

// MonetaryOperator — 自定义运算 (链式调用)
MonetaryOperator discount = amt -> amt.multiply(0.8);
MonetaryOperator addTax   = amt -> amt.multiply(1.13);
Money.of(1000, "CNY").with(discount).with(addTax);  // ¥904
```

## 4. 货币转换

```java
CurrencyConversion toUSD = MonetaryConversions.getConversion("USD");
MonetaryAmount usdAmount = Money.of(1000, "CNY").with(toUSD);
```

## 5. 格式化与舍入

```java
MonetaryAmountFormat fmt = MonetaryFormats.getAmountFormat(Locale.CHINA);
fmt.format(Money.of(12345.67, "CNY"));                  // ¥12,345.67
Money.of(99.9876, "CNY").with(Monetary.getDefaultRounding());  // ¥99.99
```

## 6. Money vs FastMoney

| 实现 | 内部表示 | 精度 | 性能 |
|------|---------|------|------|
| `Money` | BigDecimal | 任意精度 | 一般 |
| `FastMoney` | long (固定5位) | 最大 5 位小数 | ~15x 快 |

## 7. 与 BigDecimal 对比

| 特性 | MonetaryAmount | BigDecimal |
|------|---------------|------------|
| 货币绑定 | 金额与货币不可分离 | 需手动关联 |
| 类型安全 | 不同币种无法混合 | 无币种概念 |
| 汇率转换 | 内置框架 | 需自行实现 |
| 标准库 | 需额外依赖 | JDK 内置 |

## 10. 最佳实践

1. **用 MonetaryAmount 替代裸 BigDecimal** — 货币语义显式化
2. **生产用 Money，性能敏感用 FastMoney** — 注意 FastMoney 最大 5 位小数
3. **通过 SPI 注册自定义汇率源** — 对接企业内部汇率系统
4. **单元测试验证舍入** — 不同货币的默认小数位不同
