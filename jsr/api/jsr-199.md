## 2. 概述

JSR 199 将 Java 编译器功能暴露为标准 API，允许程序在运行时编译 Java 源代码：

| 传统方式 | JSR 199 方案 |
|----------|-------------|
| 调用 javac 命令行 | JavaCompiler API 程序化调用 |
| 仅支持文件系统 | 支持内存编译 (In-memory Compilation) |
| 无法获取结构化错误 | DiagnosticListener 结构化诊断 |
| 无法自定义文件源 | JavaFileManager 自定义文件管理 |

**包名**: `javax.tools`（位于 `java.compiler` 模块）

## 3. 核心接口

### JavaCompiler

```java
// 获取系统编译器 (Get System Compiler)
JavaCompiler compiler = ToolProvider.getSystemJavaCompiler();

// 编译任务 (Compilation Task)
CompilationTask task = compiler.getTask(
    null,                // Writer (输出，null = System.err)
    fileManager,         // JavaFileManager
    diagnosticListener,  // DiagnosticListener
    options,             // 编译选项 List<String>
    annotationTypes,     // 注解处理器类名
    compilationUnits     // 编译单元 (源文件)
);

boolean success = task.call();  // 执行编译
```

### JavaFileManager

```java
// 标准文件管理器 (Standard File Manager)
StandardJavaFileManager sfm = compiler.getStandardFileManager(
    diagnostics, null, null
);

// 从文件路径获取编译单元
Iterable<? extends JavaFileObject> units =
    sfm.getJavaFileObjectsFromFiles(Arrays.asList(new File("Foo.java")));
```

## 4. 基本编译

```java
JavaCompiler compiler = ToolProvider.getSystemJavaCompiler();
DiagnosticCollector<JavaFileObject> diagnostics = new DiagnosticCollector<>();
StandardJavaFileManager fileManager =
    compiler.getStandardFileManager(diagnostics, null, null);

// 指定源文件
Iterable<? extends JavaFileObject> units =
    fileManager.getJavaFileObjectsFromStrings(List.of("src/Main.java"));

// 编译选项 (Compiler Options)
List<String> options = List.of("-d", "out", "-source", "17", "-target", "17");

CompilationTask task = compiler.getTask(null, fileManager, diagnostics, options, null, units);
boolean success = task.call();

// 遍历诊断信息
for (Diagnostic<? extends JavaFileObject> d : diagnostics.getDiagnostics()) {
    System.out.printf("%s:%d %s%n", d.getSource().getName(), d.getLineNumber(), d.getMessage(null));
}
fileManager.close();
```

## 5. 内存编译

### 内存源文件 (In-Memory Source)

```java
// 自定义 JavaFileObject，从字符串读取源码
public class InMemorySource extends SimpleJavaFileObject {
    private final String code;

    public InMemorySource(String className, String code) {
        super(URI.create("string:///" + className.replace('.', '/') + Kind.SOURCE.extension),
              Kind.SOURCE);
        this.code = code;
    }

    @Override
    public CharSequence getCharContent(boolean ignoreErrors) { return code; }
}
```

### 内存编译并加载 (Compile & Load)

```java
String source = "public class Hello { public String greet() { return \"Hi!\"; } }";
JavaFileObject fileObject = new InMemorySource("Hello", source);

CompilationTask task = compiler.getTask(null, fileManager, diagnostics,
    null, null, List.of(fileObject));

if (task.call()) {
    // 加载编译后的类 (Load Compiled Class)
    ClassLoader loader = fileManager.getClassLoader(StandardLocation.CLASS_OUTPUT);
    Class<?> clazz = loader.loadClass("Hello");
    Object instance = clazz.getDeclaredConstructor().newInstance();
}
```

## 6. 诊断信息

```java
DiagnosticCollector<JavaFileObject> collector = new DiagnosticCollector<>();

// Diagnostic 提供结构化信息 (Structured Diagnostic Info)
for (Diagnostic<? extends JavaFileObject> d : collector.getDiagnostics()) {
    d.getKind();          // ERROR, WARNING, NOTE 等
    d.getSource();        // 源文件 JavaFileObject
    d.getLineNumber();    // 行号
    d.getColumnNumber();  // 列号
    d.getMessage(null);   // 可读消息 (Human-readable Message)
    d.getCode();          // 编译器错误码
}
```

## 7. 实际应用场景

| 场景 | 说明 |
|------|------|
| 注解处理器 (Annotation Processors) | 编译时代码生成，如 Lombok、MapStruct |
| 构建工具 (Build Tools) | Maven Compiler Plugin、Gradle 内部 |
| IDE 支持 | IntelliJ IDEA、Eclipse 的实时编译 |
| 热部署 (Hot Reload) | JRebel、Spring DevTools 运行时重编译 |
| 模板引擎 (Template Engines) | JSP 编译、动态脚本引擎 |
| 在线评测 (Online Judge) | LeetCode 等代码评测平台 |

## 8. 最佳实践

```java
// 1. 始终关闭 FileManager (Always Close)
try (StandardJavaFileManager fm = compiler.getStandardFileManager(null, null, null)) {
    // ... 使用 fm
}

// 2. 检查编译器是否可用 (Check Availability)
JavaCompiler compiler = ToolProvider.getSystemJavaCompiler();
if (compiler == null) {
    throw new IllegalStateException("需要 JDK，JRE 不包含编译器");
}

// 3. 重用编译器实例 (Reuse Compiler Instance)
// JavaCompiler 是线程安全的，可以缓存为 static final
private static final JavaCompiler COMPILER = ToolProvider.getSystemJavaCompiler();
```
