## 2. 概述

JSR 339 是 JAX-RS 2.0 规范，Java EE 7 的 RESTful Web Services 标准 API：

| 特点 | 说明 |
|------|------|
| 注解驱动 | 通过注解声明 HTTP 资源 (Annotation-driven REST) |
| Client API | 2.0 新增标准客户端 API，取代各框架私有实现 |
| 异步支持 | 服务端异步处理 (Server-side Async Processing) |
| 过滤器/拦截器 | 请求/响应管道的可扩展过滤链 (Filter Chain) |

**Expert Group**: Santiago Pericas-Geertsen、Marek Potociar、Bill Burke (Red Hat)、Marc Hadley

## 3. 核心注解与资源类

```java
import javax.ws.rs.*;
import javax.ws.rs.core.*;

@Path("/orders")
public class OrderResource {
    @GET
    @Produces(MediaType.APPLICATION_JSON)
    public List<Order> getAll() { ... }

    @POST
    @Consumes(MediaType.APPLICATION_JSON)
    public Response create(Order order) {
        return Response.created(URI.create("/orders/" + order.getId())).build();
    }

    @PUT @Path("/{id}") @Consumes(MediaType.APPLICATION_JSON)
    public Order update(@PathParam("id") Long id, Order order) { ... }

    @DELETE @Path("/{id}")
    public void delete(@PathParam("id") Long id) { ... }
}
```

**参数绑定注解**: `@PathParam`、`@QueryParam`、`@HeaderParam`、`@FormParam`、`@BeanParam`（2.0 新增）、`@DefaultValue`

## 4. Client API

JAX-RS 2.0 最重要的新增 — 标准化客户端 API：

```java
Client client = ClientBuilder.newClient();

// 流式 API (Fluent API)
Order order = client
    .target("http://api.example.com/orders/42")
    .request(MediaType.APPLICATION_JSON)
    .get(Order.class);

// POST 请求
Response created = client.target("http://api.example.com/orders")
    .request().post(Entity.json(newOrder));
client.close();
```

## 5. 异步处理与过滤器

```java
// 服务端异步 — 释放请求线程 (Server-side Async)
@GET @Path("/report")
public void generateReport(@Suspended final AsyncResponse async) {
    executor.submit(() -> async.resume(heavyComputation()));
    async.setTimeout(30, TimeUnit.SECONDS);
}

// 客户端异步 (Client-side Async)
Future<Order> future = client.target(url).request().async().get(Order.class);
```

**过滤器** (ContainerRequestFilter/ContainerResponseFilter): 处理请求/响应元数据（日志、CORS、认证）。
**拦截器** (ReaderInterceptor/WriterInterceptor): 处理实体 Body（压缩、加密）。

## 8. 版本演进

```
JAX-RS 1.0 (JSR 311, 2008, EE 6) → 核心注解、资源类
JAX-RS 2.0 (JSR 339, 2013, EE 7) → Client API、Async、Filters
JAX-RS 2.1 (JSR 370, 2017, EE 8) → SSE、Reactive Client
JAX-RS 3.0 (Jakarta,  2020, EE 9) → jakarta.ws.rs 包名迁移
JAX-RS 3.1 (Jakarta,  2022, EE 10) → SE Bootstrap API
```

```java
// Jakarta 迁移: javax.ws.rs → jakarta.ws.rs (EE 9+)
import jakarta.ws.rs.*;
import jakarta.ws.rs.core.*;
```

## 9. 最佳实践

1. **使用 Response 构建器** — 明确状态码与头信息 (Explicit Status Codes)
2. **复用 Client 实例** — 重量级对象，避免频繁创建
3. **ExceptionMapper 统一异常** — Java 异常映射为 HTTP 响应
4. **过滤器处理横切关注点** — 日志、认证、CORS (Cross-cutting Concerns)
