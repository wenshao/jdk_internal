## 2. 概述

JSR 173 引入了 StAX（Streaming API for XML），提供拉取式（pull-parsing）XML 解析模型。在此之前，Java 有两种 XML 解析方式：DOM（将整个文档加载到内存树）和 SAX（基于回调的推送解析）。StAX 结合了两者的优点——低内存占用且由应用控制解析流程。

| 特性 | 说明 |
|------|------|
| **拉取解析 (Pull Parsing)** | 应用程序主动拉取下一个事件，而非被动接收回调 |
| **Cursor API** | `XMLStreamReader`，轻量级游标，适合高性能场景 |
| **Event API** | `XMLEventReader`，返回事件对象，可存储/转发 |
| **写入支持** | `XMLStreamWriter`，流式输出 XML |
| **低内存** | 不需要将整个文档加载到内存 |
| **包路径** | `javax.xml.stream` |

## 4. Cursor API (XMLStreamReader)

### 基本用法

```java
// 游标式解析：像数据库 ResultSet 一样遍历 XML
XMLInputFactory factory = XMLInputFactory.newInstance();
XMLStreamReader reader = factory.createXMLStreamReader(
    new FileInputStream("data.xml")
);

while (reader.hasNext()) {
    int event = reader.next();
    switch (event) {
        case XMLStreamConstants.START_ELEMENT:
            String name = reader.getLocalName();       // 元素名
            String attr = reader.getAttributeValue(null, "id");  // 属性值
            break;
        case XMLStreamConstants.CHARACTERS:
            String text = reader.getText();             // 文本内容
            break;
        case XMLStreamConstants.END_ELEMENT:
            // 元素结束
            break;
    }
}
reader.close();
```

### 跳过不关心的元素

```java
// 拉取模型的优势：可以跳过不需要的部分
while (reader.hasNext()) {
    reader.next();
    if (reader.isStartElement() && "target".equals(reader.getLocalName())) {
        String value = reader.getElementText();  // 读取元素文本并自动前进
        break;  // 找到目标后立即停止解析
    }
}
```

## 5. Event API (XMLEventReader)

```java
// 事件对象式解析：每个事件是一个不可变对象
XMLInputFactory factory = XMLInputFactory.newInstance();
XMLEventReader reader = factory.createXMLEventReader(
    new FileInputStream("data.xml")
);

while (reader.hasNext()) {
    XMLEvent event = reader.nextEvent();

    if (event.isStartElement()) {
        StartElement start = event.asStartElement();
        QName name = start.getName();
        Attribute attr = start.getAttributeByName(new QName("id"));
    } else if (event.isCharacters()) {
        Characters chars = event.asCharacters();
        String text = chars.getData();
    }
}
reader.close();
```

## 6. 写入 XML (XMLStreamWriter)

```java
XMLOutputFactory factory = XMLOutputFactory.newInstance();
XMLStreamWriter writer = factory.createXMLStreamWriter(
    new FileOutputStream("output.xml"), "UTF-8"
);

writer.writeStartDocument("UTF-8", "1.0");
writer.writeStartElement("catalog");

writer.writeStartElement("book");
writer.writeAttribute("id", "1");
writer.writeStartElement("title");
writer.writeCharacters("Java NIO");
writer.writeEndElement();  // </title>
writer.writeEndElement();  // </book>

writer.writeEndElement();  // </catalog>
writer.writeEndDocument();
writer.close();

// 输出:
// <?xml version="1.0" encoding="UTF-8"?>
// <catalog><book id="1"><title>Java NIO</title></book></catalog>
```

## 7. XML 解析模型对比

| 特性 | DOM | SAX | StAX (JSR 173) |
|------|-----|-----|-----------------|
| 解析模型 | 树 (Tree) | 推送 (Push) | 拉取 (Pull) |
| 内存占用 | 高 (全文档) | 低 | 低 |
| 控制权 | 应用 | 解析器 (回调) | 应用 |
| 随机访问 | 支持 | 不支持 | 不支持 |
| 写入支持 | 支持 | 不支持 | 支持 |
| 代码复杂度 | 低 | 高 (状态机) | 中 |
| 适用场景 | 小文档、XPath | 大文档、简单处理 | 大文档、灵活处理 |

### Push vs Pull 解析

```
SAX (Push/推送):
  解析器 ──回调──→ ContentHandler.startElement()
  解析器 ──回调──→ ContentHandler.characters()
  解析器 ──回调──→ ContentHandler.endElement()
  (解析器控制流程，应用被动接收)

StAX (Pull/拉取):
  应用 ──reader.next()──→ START_ELEMENT
  应用 ──reader.next()──→ CHARACTERS
  应用 ──reader.next()──→ END_ELEMENT
  (应用控制流程，按需拉取事件)
```

## 8. Cursor vs Event API

| 特性 | Cursor (XMLStreamReader) | Event (XMLEventReader) |
|------|--------------------------|------------------------|
| 性能 | 更快 (无对象创建) | 稍慢 (创建事件对象) |
| 内存 | 更低 | 稍高 |
| 灵活性 | 只能前进 | 事件可存储、传递 |
| API 风格 | 类似 JDBC ResultSet | 类似 Iterator |
| 适用场景 | 高性能解析 | 事件过滤、管道处理 |

## 9. 最佳实践

### 1. 大文件首选 StAX

```java
// 不推荐：DOM 加载大文件 (可能 OOM)
Document doc = DocumentBuilderFactory.newInstance()
    .newDocumentBuilder().parse(largeFile);

// 推荐：StAX 流式解析
XMLStreamReader reader = XMLInputFactory.newInstance()
    .createXMLStreamReader(new BufferedInputStream(new FileInputStream(largeFile)));
```

### 2. 确保关闭 reader/writer

```java
XMLStreamReader reader = factory.createXMLStreamReader(input);
try {
    // 解析...
} finally {
    reader.close();  // StAX reader 不实现 AutoCloseable (JDK 6 时代)
}
```

### 3. 高性能场景用 Cursor API

```java
// 每秒需要解析大量 XML 消息时，Cursor API 避免对象创建开销
XMLStreamReader reader = factory.createXMLStreamReader(input);
// 直接读取原始值，不创建中间对象
```
