## 2. 概述

JSR 380 是 Bean Validation 规范的第二个主要版本，前身为 JSR 303 (1.0) 和 JSR 349 (1.1)：

| 版本 | JSR | 年份 | 关键特性 |
|------|-----|------|---------|
| BV 1.0 | JSR 303 | 2009 | 基础约束注解、Validator API |
| BV 1.1 | JSR 349 | 2013 | 方法参数/返回值验证 |
| BV 2.0 | JSR 380 | 2017 | Java 8 支持、容器元素验证 |

**Bean Validation 2.0 新增功能**:
- 支持 `java.time` 类型 (`@Past`, `@Future` 可用于 `LocalDate` 等)
- 容器元素验证 (Container Element Validation): `List<@NotNull String>`
- 新增注解: `@NotBlank`, `@NotEmpty`, `@Email`, `@Positive`, `@Negative` 等

**包名变迁**: `javax.validation` → `jakarta.validation`（Jakarta EE 9+）

## 3. 内置约束注解

### 空值检查 (Null Checks)

| 注解 | 说明 | 示例 |
|------|------|------|
| `@NotNull` | 不为 null | 任意类型 |
| `@Null` | 必须为 null | 特定场景 |
| `@NotEmpty` | 不为 null 且不为空 | String、Collection、Map |
| `@NotBlank` | 不为 null 且含非空白字符 | String |

### 数值约束 (Numeric Constraints)

| 注解 | 说明 | 示例 |
|------|------|------|
| `@Min(value)` | 最小值 | `@Min(0)` |
| `@Max(value)` | 最大值 | `@Max(100)` |
| `@Positive` | 正数 | > 0 |
| `@PositiveOrZero` | 非负数 | >= 0 |
| `@Negative` | 负数 | < 0 |
| `@DecimalMin` | 最小小数值 | `@DecimalMin("0.01")` |
| `@Digits(integer, fraction)` | 数字精度 | `@Digits(integer=5, fraction=2)` |

### 字符串与时间 (String & Temporal)

| 注解 | 说明 |
|------|------|
| `@Size(min, max)` | 长度/大小范围 (String、Collection、Array) |
| `@Email` | 邮箱格式验证 |
| `@Pattern(regexp)` | 正则表达式匹配 |
| `@Past` | 过去的日期/时间 |
| `@PastOrPresent` | 过去或当前 |
| `@Future` | 未来的日期/时间 |
| `@FutureOrPresent` | 未来或当前 |

## 4. 验证使用

### 基本验证 (Basic Validation)

```java
public class User {
    @NotBlank(message = "用户名不能为空")
    @Size(min = 2, max = 30, message = "用户名长度 2-30")
    private String username;

    @Email(message = "邮箱格式不正确")
    private String email;

    @Past(message = "生日必须是过去的日期")
    private LocalDate birthday;

    @Min(value = 0, message = "年龄不能为负数")
    @Max(value = 150)
    private int age;
}
```

### 手动验证 (Programmatic Validation)

```java
ValidatorFactory factory = Validation.buildDefaultValidatorFactory();
Validator validator = factory.getValidator();

User user = new User();
Set<ConstraintViolation<User>> violations = validator.validate(user);

for (ConstraintViolation<User> v : violations) {
    System.out.printf("%s: %s%n", v.getPropertyPath(), v.getMessage());
}
factory.close();
```

### 级联验证 (Cascaded Validation)

```java
public class Order {
    @NotNull
    @Valid  // 级联验证 Address 内部的约束
    private Address shippingAddress;

    @NotEmpty
    private List<@Valid OrderItem> items;  // 每个元素都验证
}
```

## 5. 自定义约束

### 定义约束注解 (Custom Constraint Annotation)

```java
@Target({ElementType.FIELD, ElementType.PARAMETER})
@Retention(RetentionPolicy.RUNTIME)
@Constraint(validatedBy = PhoneNumberValidator.class)
public @interface PhoneNumber {
    String message() default "手机号格式不正确";
    Class<?>[] groups() default {};
    Class<? extends Payload>[] payload() default {};
}
```

### 实现验证逻辑 (Constraint Validator)

```java
public class PhoneNumberValidator implements ConstraintValidator<PhoneNumber, String> {
    private static final Pattern PATTERN = Pattern.compile("^1[3-9]\\d{9}$");

    @Override
    public boolean isValid(String value, ConstraintValidatorContext ctx) {
        if (value == null) return true;  // @NotNull 负责空值检查
        return PATTERN.matcher(value).matches();
    }
}

// 使用
public class Contact {
    @PhoneNumber
    private String mobile;
}
```

## 6. 分组验证

```java
// 定义验证组 (Validation Groups)
public interface OnCreate {}
public interface OnUpdate {}

public class Product {
    @Null(groups = OnCreate.class, message = "创建时不能指定 ID")
    @NotNull(groups = OnUpdate.class, message = "更新时必须指定 ID")
    private Long id;

    @NotBlank(groups = {OnCreate.class, OnUpdate.class})
    private String name;
}

// 按组验证
validator.validate(product, OnCreate.class);   // 创建验证
validator.validate(product, OnUpdate.class);   // 更新验证
```

## 7. 容器元素验证

Bean Validation 2.0 新增，支持对泛型容器内的元素进行验证：

```java
public class Team {
    // List 中每个元素不为 null
    private List<@NotNull @Size(min = 1) String> memberNames;

    // Map 的 key 和 value 分别验证
    private Map<@NotBlank String, @Positive Integer> scores;

    // Optional 内部验证
    private Optional<@Email String> contactEmail;
}
```

## 8. 与框架集成

### Spring Boot 集成

```java
// Controller 层自动验证 (Auto Validation in Controller)
@RestController
public class UserController {
    @PostMapping("/users")
    public ResponseEntity<?> create(@Valid @RequestBody User user) {
        // 验证失败自动抛出 MethodArgumentNotValidException
        return ResponseEntity.ok(userService.save(user));
    }
}

// 依赖 (Maven)

```

## 10. 最佳实践

1. **组合使用注解** — `@NotNull` + `@Size` 分离空值和长度检查
2. **自定义 message** — 提供用户友好的错误信息
3. **用 groups 区分场景** — 创建/更新使用不同验证规则
4. **@Valid 级联验证** — 嵌套对象不要遗漏
5. **自定义约束封装业务规则** — 如手机号、身份证号
