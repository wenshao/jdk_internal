## 2. 概述

JSR 203 是 JSR 51 (NIO) 的延续，为 Java 平台引入了现代化的文件 I/O API，彻底替代了 `java.io.File` 的大部分使用场景：

| java.io.File 的问题 | NIO.2 解决方案 |
|---------------------|----------------|
| 方法返回 boolean 无法区分错误 | 抛出具体异常 (IOException) |
| 不支持符号链接 (symbolic link) | 完整的符号链接支持 |
| 无文件属性 API | BasicFileAttributes, PosixFileAttributes |
| 无文件系统监控 | WatchService 事件驱动 |
| 无异步文件操作 | AsynchronousFileChannel |
| 不可扩展 | FileSystemProvider SPI 可插拔 |

## 3. Path 与 Files

### Path — 路径抽象

```java
// 创建 Path
Path path = Path.of("/home/user/docs/file.txt");       // JDK 11+
Path path = Paths.get("/home/user/docs/file.txt");      // JDK 7+
Path path = Paths.get("home", "user", "docs", "file.txt");

// 路径操作
Path fileName = path.getFileName();       // file.txt
Path parent   = path.getParent();         // /home/user/docs
Path root     = path.getRoot();           // /
int  count    = path.getNameCount();      // 4

// 路径解析与规范化
Path resolved   = dir.resolve("sub/file.txt");
Path relative   = path1.relativize(path2);
Path normalized = path.normalize();       // 消除 . 和 ..
Path real       = path.toRealPath();      // 解析符号链接
```

### Files — 文件操作工具类

```java
// 读写文件
byte[] bytes   = Files.readAllBytes(path);
String content = Files.readString(path);                // JDK 11+
List<String> lines = Files.readAllLines(path, UTF_8);

Files.write(path, bytes);
Files.writeString(path, "hello NIO.2");                 // JDK 11+

// 复制、移动、删除
Files.copy(src, dst, StandardCopyOption.REPLACE_EXISTING);
Files.move(src, dst, StandardCopyOption.ATOMIC_MOVE);
Files.delete(path);
Files.deleteIfExists(path);

// 创建文件与目录
Files.createFile(path);
Files.createDirectory(dir);
Files.createDirectories(dir);       // 递归创建
Files.createTempFile("prefix", ".tmp");

// 判断
boolean exists   = Files.exists(path);
boolean isDir    = Files.isDirectory(path);
boolean readable = Files.isReadable(path);
long    size     = Files.size(path);
```

## 4. 文件系统与 FileStore

### FileSystem

```java
// 默认文件系统
FileSystem defaultFS = FileSystems.getDefault();

// ZIP 文件系统 — 把 zip 当作目录操作
URI uri = URI.create("jar:file:/path/to/archive.zip");
try (FileSystem zipFS = FileSystems.newFileSystem(uri, Map.of("create", "true"))) {
    Path entry = zipFS.getPath("/docs/readme.txt");
    Files.writeString(entry, "inside zip");
}
```

### FileStore — 存储信息

```java
// 获取磁盘空间
FileStore store = Files.getFileStore(path);
long total     = store.getTotalSpace();
long usable    = store.getUsableSpace();
long free      = store.getUnallocatedSpace();
String fsType  = store.type();    // ext4, NTFS, APFS...
```

## 5. 目录遍历与 FileVisitor

### Files.walk / Files.list

```java
// 流式遍历 (懒加载, lazy)
try (Stream<Path> stream = Files.walk(dir, 3)) {   // maxDepth=3
    stream.filter(Files::isRegularFile)
          .filter(p -> p.toString().endsWith(".java"))
          .forEach(System.out::println);
}

// 仅列出直接子项
try (Stream<Path> stream = Files.list(dir)) {
    stream.forEach(System.out::println);
}
```

### FileVisitor — 深度优先遍历

```java
// 自定义 FileVisitor
Files.walkFileTree(startDir, new SimpleFileVisitor<Path>() {
    @Override
    public FileVisitResult visitFile(Path file, BasicFileAttributes attrs) {
        System.out.println("文件: " + file);
        return FileVisitResult.CONTINUE;
    }

    @Override
    public FileVisitResult preVisitDirectory(Path dir, BasicFileAttributes attrs) {
        System.out.println("进入目录: " + dir);
        return FileVisitResult.CONTINUE;
    }

    @Override
    public FileVisitResult postVisitDirectory(Path dir, IOException exc) {
        // 遍历完目录后的回调
        return FileVisitResult.CONTINUE;
    }
});
```

## 6. 文件属性

### BasicFileAttributes

```java
BasicFileAttributes attrs = Files.readAttributes(path, BasicFileAttributes.class);
FileTime created  = attrs.creationTime();
FileTime modified = attrs.lastModifiedTime();
boolean  isSymLink = attrs.isSymbolicLink();
long     size     = attrs.size();
```

### POSIX 文件权限

```java
// 读取权限
Set<PosixFilePermission> perms = Files.getPosixFilePermissions(path);

// 设置权限 (rwxr-xr--)
Set<PosixFilePermission> newPerms = PosixFilePermissions.fromString("rwxr-xr--");
Files.setPosixFilePermissions(path, newPerms);

// 创建文件时指定权限
FileAttribute<Set<PosixFilePermission>> attr =
    PosixFilePermissions.asFileAttribute(newPerms);
Files.createFile(path, attr);
```

## 7. WatchService 文件监控

```java
// 创建监控服务
WatchService watcher = FileSystems.getDefault().newWatchService();

// 注册目录
Path dir = Path.of("/watched/dir");
dir.register(watcher,
    StandardWatchEventKinds.ENTRY_CREATE,
    StandardWatchEventKinds.ENTRY_MODIFY,
    StandardWatchEventKinds.ENTRY_DELETE);

// 事件循环
while (true) {
    WatchKey key = watcher.take();   // 阻塞等待
    for (WatchEvent<?> event : key.pollEvents()) {
        WatchEvent.Kind<?> kind = event.kind();
        Path changed = (Path) event.context();
        System.out.printf("事件: %s -> %s%n", kind.name(), changed);
    }
    boolean valid = key.reset();
    if (!valid) break;   // 目录不可访问
}
```

## 8. 异步通道

### AsynchronousFileChannel

```java
// 异步读文件
AsynchronousFileChannel channel = AsynchronousFileChannel.open(path, READ);
ByteBuffer buffer = ByteBuffer.allocate(1024);

// Future 方式
Future<Integer> result = channel.read(buffer, 0);
int bytesRead = result.get();   // 阻塞等待结果

// CompletionHandler 回调方式
channel.read(buffer, 0, null, new CompletionHandler<Integer, Void>() {
    @Override
    public void completed(Integer bytesRead, Void attachment) {
        System.out.println("读取完成: " + bytesRead + " bytes");
    }
    @Override
    public void failed(Throwable exc, Void attachment) {
        exc.printStackTrace();
    }
});
```

### AsynchronousSocketChannel

```java
// 异步网络连接
AsynchronousSocketChannel client = AsynchronousSocketChannel.open();
Future<Void> connectFuture = client.connect(new InetSocketAddress("localhost", 8080));
connectFuture.get();

// 异步写
ByteBuffer msg = ByteBuffer.wrap("Hello".getBytes());
Future<Integer> writeFuture = client.write(msg);
```

## 9. 与 java.io.File 的对比

| 操作 | java.io.File | NIO.2 (Path + Files) |
|------|-------------|----------------------|
| 路径表示 | `new File("/a/b")` | `Path.of("/a/b")` |
| 判断存在 | `file.exists()` → boolean | `Files.exists(path)` → boolean |
| 删除 | `file.delete()` → boolean | `Files.delete(path)` → 异常 |
| 重命名 | `file.renameTo(dst)` → boolean | `Files.move(src, dst)` → 异常 |
| 读取内容 | 需要 InputStream | `Files.readAllBytes(path)` |
| 文件属性 | `file.lastModified()` | `Files.readAttributes(...)` |
| 目录遍历 | `file.listFiles()` → null on error | `Files.list(dir)` → Stream |

### 互转

```java
// File → Path
File file = new File("/path/to/file");
Path path = file.toPath();

// Path → File
Path path = Path.of("/path/to/file");
File file = path.toFile();
```

## 10. 最佳实践

### 1. 优先使用 try-with-resources

```java
// 正确: 自动关闭流
try (Stream<Path> stream = Files.walk(dir)) {
    stream.filter(Files::isRegularFile).forEach(this::process);
}

// 错误: 忘记关闭 DirectoryStream
Files.walk(dir).forEach(this::process);  // 资源泄漏
```

### 2. 原子操作

```java
// 写入临时文件后原子移动，保证数据完整性
Path tmp = Files.createTempFile(dir, "data", ".tmp");
Files.writeString(tmp, content);
Files.move(tmp, target, StandardCopyOption.ATOMIC_MOVE);
```

### 3. 使用 Files.newBufferedReader/Writer 处理大文件

```java
// 大文件逐行读取
try (BufferedReader reader = Files.newBufferedReader(path, UTF_8)) {
    String line;
    while ((line = reader.readLine()) != null) {
        process(line);
    }
}
```
