## 2. 概述

JSR 166 引入了 `java.util.concurrent` 包，彻底改变了 Java 并发编程模型。在此之前，开发者只能依赖 `synchronized`、`wait()`/`notify()` 等低级原语，极易出错且难以调试。

| 特性 | 说明 |
|------|------|
| **Executor 框架** | 线程池与任务提交分离，统一的异步执行模型 |
| **Future** | 异步计算结果的占位符 |
| **并发集合** | 线程安全的高性能集合，替代 `Collections.synchronizedXxx()` |
| **阻塞队列** | 生产者-消费者模式的基础设施 |
| **显式锁** | 比 `synchronized` 更灵活的锁机制 |
| **同步工具** | CountDownLatch、CyclicBarrier、Semaphore 等协调工具 |
| **原子变量** | 基于 CAS 的无锁并发原语 |

## 4. Executor 框架

### 核心接口

```java
// 最基础的执行器接口
public interface Executor {
    void execute(Runnable command);
}

// 扩展：支持生命周期管理和 Future
public interface ExecutorService extends Executor {
    <T> Future<T> submit(Callable<T> task);
    Future<?> submit(Runnable task);
    void shutdown();
    List<Runnable> shutdownNow();
    boolean isShutdown();
    boolean awaitTermination(long timeout, TimeUnit unit);
}
```

### ThreadPoolExecutor

```java
// 核心线程池实现
ThreadPoolExecutor executor = new ThreadPoolExecutor(
    corePoolSize,       // 核心线程数
    maximumPoolSize,    // 最大线程数
    keepAliveTime,      // 空闲线程存活时间
    TimeUnit.SECONDS,
    new LinkedBlockingQueue<>(100),  // 工作队列
    new ThreadPoolExecutor.CallerRunsPolicy()  // 拒绝策略
);
```

### Executors 工厂方法

```java
// 固定大小线程池
ExecutorService fixed = Executors.newFixedThreadPool(10);

// 缓存线程池 (按需创建，空闲回收)
ExecutorService cached = Executors.newCachedThreadPool();

// 单线程执行器 (保证任务顺序执行)
ExecutorService single = Executors.newSingleThreadExecutor();

// 定时任务线程池
ScheduledExecutorService scheduled = Executors.newScheduledThreadPool(5);
scheduled.scheduleAtFixedRate(() -> doWork(), 0, 1, TimeUnit.SECONDS);
```

### Future 与 Callable

```java
// Callable: 有返回值的任务
Callable<String> task = () -> {
    Thread.sleep(1000);
    return "结果";
};

Future<String> future = executor.submit(task);
String result = future.get();               // 阻塞等待
String result = future.get(5, TimeUnit.SECONDS);  // 超时等待
future.cancel(true);                        // 取消任务
```

## 5. 并发集合

### ConcurrentHashMap

```java
// 分段锁设计 (JDK 5-7)，CAS + synchronized (JDK 8+)
ConcurrentHashMap<String, Integer> map = new ConcurrentHashMap<>();

// 原子操作
map.putIfAbsent("key", 1);
map.computeIfAbsent("key", k -> expensiveCompute(k));
map.merge("key", 1, Integer::sum);

// JDK 8 批量操作
map.forEach(2, (k, v) -> System.out.println(k + "=" + v));  // 并行度=2
long count = map.reduceValuesToLong(2, Long::valueOf, 0L, Long::sum);
```

### CopyOnWriteArrayList

```java
// 写时复制：适用于读多写少的场景
CopyOnWriteArrayList<String> list = new CopyOnWriteArrayList<>();
list.add("item");  // 每次写操作都复制底层数组

// 迭代器是快照，不会抛出 ConcurrentModificationException
for (String s : list) {
    list.add("new");  // 安全：不影响当前迭代
}
```

### BlockingQueue

```java
// 阻塞队列：生产者-消费者模式的核心
BlockingQueue<Task> queue = new LinkedBlockingQueue<>(100);

// 生产者
queue.put(task);        // 队列满时阻塞
queue.offer(task, 5, TimeUnit.SECONDS);  // 超时放弃

// 消费者
Task t = queue.take();  // 队列空时阻塞
Task t = queue.poll(5, TimeUnit.SECONDS);  // 超时返回 null
```

## 6. 同步工具

### CountDownLatch (一次性屏障)

```java
// 等待 N 个事件完成
CountDownLatch latch = new CountDownLatch(3);

// 工作线程
executor.submit(() -> {
    doWork();
    latch.countDown();  // 计数减一
});

latch.await();  // 主线程阻塞直到计数为零
```

### CyclicBarrier (可重用屏障)

```java
// N 个线程互相等待到达同一点
CyclicBarrier barrier = new CyclicBarrier(3, () -> {
    System.out.println("所有线程到达屏障");  // 屏障动作
});

// 每个线程
barrier.await();  // 阻塞直到所有线程到达
// barrier 可重用，适用于分阶段计算
```

### Semaphore (信号量)

```java
// 控制并发访问数量
Semaphore semaphore = new Semaphore(10);  // 最多10个并发

semaphore.acquire();   // 获取许可 (阻塞)
try {
    accessResource();
} finally {
    semaphore.release();  // 释放许可
}
```

## 7. 锁机制

### ReentrantLock

```java
ReentrantLock lock = new ReentrantLock(true);  // 公平锁

lock.lock();
try {
    // 临界区
} finally {
    lock.unlock();  // 必须在 finally 中释放
}

// 尝试获取锁
if (lock.tryLock(5, TimeUnit.SECONDS)) {
    try {
        // 临界区
    } finally {
        lock.unlock();
    }
}
```

### ReentrantLock vs synchronized

| 特性 | synchronized | ReentrantLock |
|------|-------------|---------------|
| 释放方式 | 自动 | 手动 (finally) |
| 可中断获取 | 否 | `lockInterruptibly()` |
| 超时获取 | 否 | `tryLock(timeout)` |
| 公平锁 | 否 | 支持 |
| Condition | `wait/notify` | 多个 Condition |

### ReadWriteLock

```java
ReadWriteLock rwLock = new ReentrantReadWriteLock();

// 读锁：多个线程可同时持有
rwLock.readLock().lock();
try {
    return data.get(key);
} finally {
    rwLock.readLock().unlock();
}

// 写锁：独占
rwLock.writeLock().lock();
try {
    data.put(key, value);
} finally {
    rwLock.writeLock().unlock();
}
```

## 8. 原子变量

```java
// 基于 CAS (Compare-And-Swap) 的无锁并发
AtomicInteger counter = new AtomicInteger(0);

counter.incrementAndGet();       // 原子自增
counter.compareAndSet(0, 1);     // CAS 操作
counter.getAndUpdate(n -> n * 2);  // 原子更新 (JDK 8)
counter.accumulateAndGet(5, Integer::sum);  // 原子累加 (JDK 8)

// AtomicReference: 引用类型的原子操作
AtomicReference<Node> head = new AtomicReference<>();
head.compareAndSet(oldHead, newHead);

// LongAdder (JDK 8): 高并发计数器，比 AtomicLong 更快
LongAdder adder = new LongAdder();
adder.increment();
long sum = adder.sum();
```

## 9. 后续演进 (JSR 166x)

JSR 166 并非一次性完成，Doug Lea 持续推动后续更新：

### JDK 7: Fork/Join 框架

```java
// ForkJoinPool: 工作窃取 (work-stealing) 算法
ForkJoinPool pool = new ForkJoinPool();

class SumTask extends RecursiveTask<Long> {
    @Override
    protected Long compute() {
        if (数据量小) return 直接计算;
        SumTask left = new SumTask(前半部分);
        SumTask right = new SumTask(后半部分);
        left.fork();       // 异步执行
        return right.compute() + left.join();  // 合并结果
    }
}
```

### JDK 8: CompletableFuture

```java
// 声明式异步编程，支持链式组合
CompletableFuture.supplyAsync(() -> fetchData())
    .thenApply(data -> process(data))
    .thenAccept(result -> save(result))
    .exceptionally(ex -> handleError(ex));

// 组合多个异步操作
CompletableFuture<String> f1 = CompletableFuture.supplyAsync(() -> "Hello");
CompletableFuture<String> f2 = CompletableFuture.supplyAsync(() -> "World");
CompletableFuture<String> combined = f1.thenCombine(f2, (a, b) -> a + " " + b);

// 等待所有/任意完成
CompletableFuture.allOf(f1, f2).join();
CompletableFuture.anyOf(f1, f2).join();
```

### JDK 9: Flow API (Reactive Streams)

```java
// java.util.concurrent.Flow — Reactive Streams 标准接口
Flow.Publisher<String> publisher;
Flow.Subscriber<String> subscriber;
Flow.Subscription subscription;
Flow.Processor<String, String> processor;

// SubmissionPublisher: JDK 内置 Publisher 实现
SubmissionPublisher<String> pub = new SubmissionPublisher<>();
pub.subscribe(subscriber);
pub.submit("data");
```

| 版本 | 特性 | 说明 |

| JDK 5 | JSR 166 核心 | Executor, 并发集合, Lock, 原子变量 |

## 10. 最佳实践

### 1. 使用线程池而非手动创建线程

```java
// 不推荐
new Thread(() -> doWork()).start();

// 推荐
ExecutorService executor = Executors.newFixedThreadPool(
    Runtime.getRuntime().availableProcessors()
);
executor.submit(() -> doWork());
```

### 2. 优先使用并发集合

```java
// 不推荐
Collections.synchronizedMap(new HashMap<>());

// 推荐
new ConcurrentHashMap<>();
```

### 3. 使用适当的同步工具

```java
// 不推荐：用 Thread.join() 协调多个线程
// 推荐：用 CountDownLatch 或 CompletableFuture

// 不推荐：用 synchronized + wait/notify 实现生产者-消费者
// 推荐：用 BlockingQueue
```

### 4. 避免 Executors 的无界队列

```java
// 风险：可能 OOM
Executors.newFixedThreadPool(10);  // 默认用无界 LinkedBlockingQueue

// 推荐：显式配置
new ThreadPoolExecutor(10, 10, 0L, TimeUnit.MILLISECONDS,
    new LinkedBlockingQueue<>(1000),
    new ThreadPoolExecutor.CallerRunsPolicy());
```

## 11. 与其他 JSR/JEP 的关系

### 依赖关系

| JSR/JEP | 说明 |
|---------|------|
| JSR 133 | Java Memory Model (JDK 5)，JSR 166 的基础 |
| JSR 335 | Lambda Expressions (JDK 8)，让并发 API 更易用 |
| JEP 266 | More Concurrency Updates (JDK 9)，Flow API |
| JEP 425 | Virtual Threads (JDK 19 预览) |
| JEP 444 | Virtual Threads (JDK 21 正式) |
| JEP 453 | Structured Concurrency (JDK 21 预览) |
| JEP 462 | Scoped Values (JDK 23 正式) |

### 影响

JSR 166 是 Java 平台最具影响力的 JSR 之一。它：

- 奠定了 Java 所有并发编程的基础设施
- 影响了后续 CompletableFuture、Virtual Threads 等设计
- `ConcurrentHashMap` 成为 Java 最常用的线程安全集合
- Fork/Join 框架驱动了 parallel streams 的底层实现
