## 1. 迁移概览

### 为什么迁移到 JDK 11？

1. **LTS 支持**: JDK 11 是长期支持版本 (2026年9月前)
2. **性能提升**: 显著的性能改进和内存优化
3. **新特性**: 模块化、HTTP Client、ZGC 等
4. **安全增强**: 更强的安全特性和更新支持

### 迁移路径
```
JDK 8u401 (当前)
    ↓
兼容性评估 (2周)
    ↓
代码迁移 (2-4周)
    ↓
模块化改造 (可选)
    ↓
测试验证 (2周)
    ↓
性能优化 (1周)
    ↓
生产部署
```

### 关键差异
| 方面 | JDK 8 | JDK 11 | 影响 |
|------|-------|--------|------|
| **模块系统** | 无 | JPMS (Java Platform Module System) | 高 |
| **GC 选项** | G1 (可选) | ZGC, Shenandoah (实验性) | 中 |
| **HTTP 客户端** | HttpURLConnection | HttpClient (标准) | 中 |
| **API 变化** | 部分废弃 | 大量移除和新增 | 高 |

## 2. 准备工作

### 环境评估

1. **系统要求**:
```bash
# 检查操作系统兼容性
# JDK 11 最低要求:
# - Linux: glibc 2.17+, kernel 3.10+
# - Windows: Windows 7+
# - macOS: 10.13+

uname -a
ldd --version
```

2. **架构支持**:
| 架构 | JDK 8 支持 | JDK 11 支持 | 变化 |
|------|------------|-------------|------|
| x86_64 | 完全支持 | 完全支持 | 无 |
| AArch64 | 有限支持 | 完全支持 | 增强 |
| s390x | 支持 | 支持 | 无 |
| PPC64 | 支持 | 支持 | 无 |

### 依赖分析

1. **使用 jdeps 分析**:
```bash
# 分析依赖关系
jdeps --multi-release 11 --ignore-missing-deps your-app.jar

# 检查内部 API 使用
jdeps -jdkinternals your-app.jar
```

2. **常见兼容性问题**:
| 库/框架 | JDK 8 版本 | JDK 11 兼容版本 | 操作 |
|---------|------------|-----------------|------|
| Spring Framework | 4.x | 5.1+ | 升级到 5.x |
| Hibernate | 5.2 | 5.4+ | 升级 |
| Log4j | 1.x | 2.11+ | 升级到 2.x |
| Jackson | 2.9 | 2.10+ | 升级 |

### 风险评估矩阵

| **性能回归** | 低 | 中 | 性能基准测试 |

## 3. 代码迁移步骤

### 步骤 1: 编译环境更新

1. **更新构建配置**:
```xml
<!-- Maven pom.xml -->
<properties>
    <maven.compiler.source>11</maven.compiler.source>
    <maven.compiler.target>11</maven.compiler.target>
    <maven.compiler.release>11</maven.compiler.release>
</properties>

<!-- 添加模块支持 -->
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-compiler-plugin</artifactId>
    <version>3.10.0</version>
    <configuration>
        <release>11</release>
    </configuration>
</plugin>
```

```groovy
// Gradle build.gradle
plugins {
    id 'java'
}

java {
    sourceCompatibility = JavaVersion.VERSION_11
    targetCompatibility = JavaVersion.VERSION_11
    modularity.inferModulePath = true
}
```

2. **模块化决策**:
- **选项 A**: 保持类路径 (简单，兼容性好)
- **选项 B**: 逐步模块化 (推荐，长期受益)
- **选项 C**: 完全模块化 (复杂，需要重构)

### 步骤 2: 移除 API 替换

1. **使用 jdeprscan 扫描移除的 API**:
```bash
# 扫描已移除的 API
jdeprscan --release 11 --for-removal your-app.jar
```

2. **关键 API 变化**:
| 移除的 API | 替代方案 | 紧急修复 |
|------------|----------|----------|
| `java.awt.peer` 包 | 使用跨平台 API | 高 |
| `java.util.logging.LogManager` 内部类 | SLF4J/Log4j2 | 高 |
| `com.sun.image.codec.jpeg` | ImageIO | 中 |
| `sun.misc.BASE64Encoder` | `java.util.Base64` | 中 |

3. **Java EE 模块移除**:
```xml
<!-- 需要添加的依赖 -->
<dependency>
    <groupId>javax.xml.bind</groupId>
    <artifactId>jaxb-api</artifactId>
    <version>2.3.1</version>
</dependency>
<dependency>
    <groupId>com.sun.xml.bind</groupId>
    <artifactId>jaxb-impl</artifactId>
    <version>2.3.1</version>
</dependency>
```

### 步骤 3: 语言特性迁移 (可选)

1. **局部变量类型推断 (Java 10)**:
```java
// JDK 8
List<String> list = new ArrayList<>();

// JDK 11 (var 关键字)
var list = new ArrayList<String>();
```

2. **新的字符串方法 (Java 11)**:
```java
// 检查空白字符串
if (str.isBlank()) { ... }

// 去除首尾空白
String trimmed = str.strip();

// 重复字符串
String repeated = "Java ".repeat(3);  // "Java Java Java "

// 行流
str.lines().forEach(System.out::println);
```

### 步骤 4: HTTP 客户端迁移

1. **从 HttpURLConnection 迁移**:
```java
// JDK 8: HttpURLConnection
URL url = new URL("https://api.example.com");
HttpURLConnection conn = (HttpURLConnection) url.openConnection();
conn.setRequestMethod("GET");

// JDK 11: HttpClient
HttpClient client = HttpClient.newHttpClient();
HttpRequest request = HttpRequest.newBuilder()
    .uri(URI.create("https://api.example.com"))
    .GET()
    .build();
HttpResponse<String> response = client.send(request, HttpResponse.BodyHandlers.ofString());
```

2. **异步请求**:
```java
CompletableFuture<HttpResponse<String>> future = client
    .sendAsync(request, HttpResponse.BodyHandlers.ofString());
future.thenAccept(response -> System.out.println(response.statusCode()));
```

### 步骤 5: 安全性更新

1. **TLS 1.3 支持**:
```java
// 启用 TLS 1.3
SSLContext sslContext = SSLContext.getInstance("TLSv1.3");
sslContext.init(null, null, null);
```

2. **加密算法更新**:
- 默认禁用 RC4、DES、3DES
- 启用更强的密码套件
- 证书验证更严格

## 4. 模块化迁移策略

### 选项 A: 类路径模式 (简单)

1. **配置**:
```bash
# 运行应用 (不使用模块系统)
java --class-path app.jar:lib/* com.example.Main
```

2. **优点**:
- 无需代码修改
- 快速迁移
- 兼容现有库

3. **缺点**:
- 无法享受模块化好处
- 类路径地狱风险仍在
- 启动时间稍慢

### 选项 B: 自动模块 (推荐)

1. **创建 module-info.java**:
```java
// 简单的自动模块声明
module com.example.app {
    // 依赖的自动模块
    requires java.sql;
    requires com.fasterxml.jackson.databind;
    
    // 导出包
    exports com.example.api;
}
```

2. **构建配置**:
```bash
# 编译模块
javac --module-path lib -d out module-info.java $(find src -name "*.java")

# 运行模块
java --module-path out:lib -m com.example.app/com.example.Main
```

3. **自动模块命名**:
- JAR 文件名决定模块名 (去掉版本号)
- 例如: `jackson-databind-2.13.0.jar` → `jackson.databind` 模块

### 选项 C: 完全模块化 (高级)

1. **模块重构**:
- 分析包依赖关系
- 设计模块边界
- 重构包结构

2. **服务加载器迁移**:
```java
// JDK 8: META-INF/services
// JDK 11: 模块声明中的 provides/uses
module com.example.app {
    provides com.example.spi.Service
        with com.example.impl.ServiceImpl;
    uses com.example.spi.Service;
}
```

## 5. 性能优化机会

### GC 选择

1. **ZGC (低延迟)**:
```bash
# 启用 ZGC
java -XX:+UseZGC -Xmx16g -jar app.jar

# ZGC 调优参数
-XX:+UseZGC
-XX:+UnlockExperimentalVMOptions
-XX:MaxGCPauseMillis=10
-XX:ConcGCThreads=4
```

2. **Shenandoah (吞吐量和延迟平衡)**:
```bash
# 启用 Shenandoah
java -XX:+UseShenandoahGC -jar app.jar
```

3. **G1 增强**:
```bash
# JDK 11 G1 改进
-XX:+UseG1GC
-XX:G1HeapRegionSize=8m  # 支持更大区域
-XX:MaxGCPauseMillis=100
```

### 启动性能优化

1. **类数据共享 (CDS)**:
```bash
# 创建共享归档
java -Xshare:dump -XX:SharedArchiveFile=app.jsa -jar app.jar

# 使用共享归档
java -Xshare:on -XX:SharedArchiveFile=app.jsa -jar app.jar
```

2. **提前编译 (AOT)** - 实验性:
```bash
# 生成 AOT 库
jaotc --output libHelloWorld.so HelloWorld.class

# 使用 AOT 库
java -XX:AOTLibrary=./libHelloWorld.so HelloWorld
```

## 6. 配置迁移

### JVM 参数更新

1. **废弃参数替换**:
| JDK 8 参数 | JDK 11 参数 | 说明 |
|------------|-------------|------|
| `-XX:+AggressiveOpts` | 已移除 | 使用默认优化 |
| `-XX:+UseConcMarkSweepGC` | 已废弃 | 使用 G1 或 ZGC |
| `-XX:+UseParNewGC` | 已废弃 | 使用 G1 或 ZGC |

2. **新参数**:
```bash
# 支持 Unicode 10.0
-Dfile.encoding=UTF-8

# 改善 Docker 容器支持
-XX:+UseContainerSupport
-XX:MaxRAMPercentage=75.0

# 日志统一
-Xlog:gc*,safepoint:file=gc.log:time,level,tags
```

### 应用服务器配置

1. **Tomcat 9+ 配置**:
```xml
<!-- server.xml -->
<Connector port="8080" protocol="org.apache.coyote.http11.Http11NioProtocol"
           connectionTimeout="20000"
           redirectPort="8443"
           maxThreads="200"
           URIEncoding="UTF-8" />
```

2. **Spring Boot 2.1+ 配置**:
```yaml
# application.yml
spring:
  main:
    banner-mode: off
  jackson:
    default-property-inclusion: non_null

server:
  tomcat:
    max-connections: 10000
    threads:
      max: 200
      min-spare: 10
```

## 9. 常见问题解决

### 模块化问题

1. **模块未找到**:
```bash
# 错误: module not found
Error: module com.example.app not found

# 解决方案
java --module-path out:lib -m com.example.app/com.example.Main
```

2. **包未导出**:
```bash
# 错误: package is not visible
IllegalAccessError: class tried to access private method

# 解决方案: 在 module-info.java 中添加
opens com.example.internal to com.example.client;
```

### 类加载问题

1. **服务加载器失败**:
```java
// JDK 8: META-INF/services
// JDK 11: 需要模块声明
module com.example {
    provides com.example.spi.Service
        with com.example.impl.ServiceImpl;
}
```

2. **反射限制**:
```bash
# 错误: inaccessible
java.lang.reflect.InaccessibleObjectException

# 解决方案: 添加 opens 语句或使用
--add-opens java.base/java.lang=ALL-UNNAMED
```

### 性能问题

1. **启动变慢**:
```bash
# 启用 CDS
java -Xshare:dump -XX:SharedArchiveFile=app.jsa -cp app.jar
java -Xshare:on -XX:SharedArchiveFile=app.jsa -cp app.jar Main
```

2. **内存使用增加**:
```bash
# 分析内存
jmap -heap <pid>
jcmd <pid> GC.heap_info

# 调整 Metaspace
-XX:MaxMetaspaceSize=512m
```

## 10. 迁移后优化

### 代码优化

1. **使用新的 API**:
```java
// 新的集合工厂方法
List<String> list = List.of("a", "b", "c");
Set<String> set = Set.of("a", "b", "c");
Map<String, Integer> map = Map.of("a", 1, "b", 2);

// 新的 Optional 方法
optional.stream().forEach(...);
optional.ifPresentOrElse(...);
```

2. **改进的 Stream API**:
```java
// takeWhile/dropWhile
list.stream()
    .takeWhile(s -> s.length() > 0)
    .collect(Collectors.toList());
```

### 架构优化

1. **模块化设计**:
- 明确模块边界
- 减少循环依赖
- 使用服务加载器

2. **微服务优化**:
- 使用 HTTP Client 替换第三方库
- 改进的 Docker 支持
- 更好的容器资源管理

### 运维优化

1. **监控增强**:
- JFR (Java Flight Recorder) 生产使用
- 统一的 JVM 日志
- 改进的指标导出

2. **可观测性**:
```bash
# 启用 JFR
java -XX:StartFlightRecording=duration=60s,filename=recording.jfr -jar app.jar

# 分析 JFR 记录
jfr print recording.jfr
```
