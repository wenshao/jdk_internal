# 迁移到 JDK 21

> **目标版本**: JDK 21 (LTS 2023) | **建议迁移时间**: 4-8 周 | **复杂度**: 中高

---

## 迁移概览

### 为什么迁移到 JDK 21？

1. **最新 LTS**: JDK 21 是最新的长期支持版本 (支持至 2031 年)
2. **重大语言特性**: Virtual Threads, Record Patterns, Pattern Matching for switch (正式)
3. **性能飞跃**: Virtual Threads 带来的并发性能革命
4. **现代 API**: Sequenced Collections, String Templates (预览)
5. **GC 成熟**: Generational ZGC 生产就绪
6. **工具增强**: 改进的向量 API, 更好的监控

### 迁移路径
```
JDK 17 (当前)
    ↓
兼容性评估 (1-2 周)
    ↓    
核心依赖升级 (1 周)
    ↓
Virtual Threads 评估 (1-2 周)
    ↓
新特性采用 (2-3 周)
    ↓
测试验证 (2 周)
    ↓
性能优化 (1 周)
    ↓
生产部署
```

### 关键差异
| 方面 | JDK 17 | JDK 21 | 影响等级 |
|------|--------|--------|----------|
| **并发模型** | Platform Threads | Virtual Threads (JEP 444) | 高 |
| **语言特性** | Pattern Matching 预览 | Record Patterns, switch 正式 | 中 |
| **集合 API** | 传统集合 | Sequenced Collections (JEP 431) | 低 |
| **GC 特性** | ZGC 单代 | Generational ZGC (JEP 439) | 中 |
| **字符串处理** | 基本方法 | String Templates (预览, JEP 430) | 低 |

## Virtual Threads 迁移 (JEP 444)

### 理解 Virtual Threads

**传统线程 (Platform Threads)**:
- 操作系统线程 1:1 映射
- 创建成本高 (~1MB 栈)
- 上下文切换成本高
- 数量受限 (通常 <1000)

**虚拟线程 (Virtual Threads)**:
- JVM 管理的轻量级线程
- 创建成本极低 (~几百字节)
- 挂起/恢复由 JVM 管理
- 数量可达数百万

### 迁移策略

**阶段 1: 评估当前线程使用**
```java
// 分析当前线程使用模式
Thread.getAllStackTraces().keySet().stream()
    .filter(t -> !t.isDaemon())
    .count();

// 监控线程池使用
ExecutorService executor = Executors.newFixedThreadPool(100);
// 可能阻塞的操作数量
```

**阶段 2: 识别候选场景**
| 场景 | Virtual Threads 适用性 | 收益预期 |
|------|------------------------|----------|
| **阻塞 I/O 操作** | 高 | 大幅提高吞吐量 |
| **网络请求处理** | 高 | 并发连接数大幅提升 |
| **数据库查询** | 高 | 连接池需求减少 |
| **计算密集型** | 低 | 无收益，可能略差 |
| **同步阻塞操作** | 中 | 需要调整同步策略 |

**阶段 3: 逐步迁移**

```java
// JDK 17: 传统线程池
ExecutorService executor = Executors.newFixedThreadPool(100);

// JDK 21: 虚拟线程执行器
ExecutorService virtualExecutor = Executors.newVirtualThreadPerTaskExecutor();

// 或使用结构化并发 (JEP 453)
try (var scope = new StructuredTaskScope.ShutdownOnFailure()) {
    Future<String> future1 = scope.fork(() -> fetchData1());
    Future<String> future2 = scope.fork(() -> fetchData2());
    
    scope.join();
    scope.throwIfFailed();
    
    String result1 = future1.resultNow();
    String result2 = future2.resultNow();
}
```

### 常见问题解决

1. **线程局部变量 (ThreadLocal)**:
```java
// Virtual Threads 支持 ThreadLocal，但有成本
// 考虑使用 ScopedValue (JEP 446) 替代
ScopedValue<String> USER = ScopedValue.newInstance();

ScopedValue.where(USER, "alice").run(() -> {
    System.out.println(USER.get());  // "alice"
});
```

2. **同步原语调整**:
```java
// synchronized 在 Virtual Threads 上可能阻塞载体线程
// 考虑使用 ReentrantLock
private final Lock lock = new ReentrantLock();

public void process() {
    lock.lock();
    try {
        // 临界区
    } finally {
        lock.unlock();
    }
}
```

3. **线程池迁移**:
```java
// 传统线程池 → 虚拟线程执行器
// 之前
ExecutorService executor = Executors.newFixedThreadPool(200);

// 之后
ExecutorService executor = Executors.newVirtualThreadPerTaskExecutor();
// 或保持兼容性
ExecutorService executor = ThreadPoolBuilder
    .virtualThreadExecutor()
    .setThreadFactory(Thread.ofVirtual().factory())
    .build();
```

| 指标 | Platform Threads | Virtual Threads | 预期改进 |

| **并发连接数** | 1000-5000 | 10000-100000 | 10-100x |
| **内存使用** | 高 (每个线程 ~1MB) | 低 (每个线程 ~几百字节) | 90%+ 减少 |

## 语言特性迁移

### 1. Pattern Matching for switch 正式版 (JEP 441)

**状态变化**: JDK 17 预览 → JDK 21 正式

```java
// 现在可以正式使用
String formatted = switch (obj) {
    case Integer i -> String.format("int %d", i);
    case Long l    -> String.format("long %d", l);
    case Double d  -> String.format("double %f", d);
    case String s  -> String.format("String %s", s);
    case null      -> "null";
    default        -> obj.toString();
};

// 守卫表达式
static void test(Object obj) {
    switch (obj) {
        case String s when s.length() > 5 -> 
            System.out.println("Long string");
        case String s -> 
            System.out.println("Short string");
        default -> 
            System.out.println("Not a string");
    }
}
```

**迁移操作**: 移除预览标志，更新编译配置。

### 2. Record Patterns (JEP 440)

**新特性**: Record 的模式解构

```java
record Point(int x, int y) {}
record Rectangle(Point topLeft, Point bottomRight) {}

// Record 模式匹配
static void printUpperLeft(Rectangle r) {
    if (r instanceof Rectangle(Point(var x, var y), Point bottomRight)) {
        System.out.printf("Upper left corner: (%d, %d)%n", x, y);
    }
}

// 嵌套 Record 模式
static void process(Object obj) {
    switch (obj) {
        case Rectangle(Point(var x1, var y1), Point(var x2, var y2)) 
            when x1 < x2 && y1 < y2 ->
            System.out.println("Valid rectangle");
        case Rectangle r ->
            System.out.println("Invalid rectangle: " + r);
        default ->
            System.out.println("Not a rectangle");
    }
}
```

**迁移建议**: 逐步在 Record 处理代码中采用。

### 3. String Templates (预览) (JEP 430)

**新特性**: 类型安全的字符串模板

```java
// 传统方式
String name = "Alice";
int age = 30;
String message = String.format("Hello, %s. You are %d years old.", name, age);

// 字符串模板 (预览)
StringTemplate st = RAW."Hello, \{name}. You are \{age} years old.";
String message = STR.process(st);

// 或简写
String message = STR."Hello, \{name}. You are \{age} years old.";

// 自定义模板处理器
StringTemplate.Processor<String, RuntimeException> JSON = StringTemplate.Processor.of(
    (StringTemplate st) -> {
        // 生成 JSON
        return new ObjectMapper().writeValueAsString(st.values());
    }
);

String json = JSON."""
    {
        "name": "\{name}",
        "age": \{age}
    }
    """;
```

**迁移注意**: 预览特性，生产环境慎用。

### 4. Sequenced Collections (JEP 431)

**新特性**: 统一的集合序列访问 API

```java
// 新接口: SequencedCollection, SequencedSet, SequencedMap
SequencedCollection<String> collection = new ArrayList<>();

// 新增方法
collection.addFirst("first");    // 添加到开头
collection.addLast("last");      // 添加到结尾
String first = collection.getFirst();
String last = collection.getLast();
String removedFirst = collection.removeFirst();
String removedLast = collection.removeLast();

// 反向视图
SequencedCollection<String> reversed = collection.reversed();

// SequencedMap
SequencedMap<String, Integer> map = new LinkedHashMap<>();
map.putFirst("first", 1);
map.putLast("last", 100);
Map.Entry<String, Integer> firstEntry = map.firstEntry();
Map.Entry<String, Integer> lastEntry = map.lastEntry();
```

**迁移建议**: 更新使用 `LinkedList`, `LinkedHashSet`, `LinkedHashMap` 的代码。

## 性能相关迁移

### 5. Generational ZGC (JEP 439)

**改进**: ZGC 现在支持分代垃圾收集

**性能优势**:
- 减少垃圾收集停顿时间
- 降低内存占用
- 改进吞吐量

**启用配置**:
```bash
# 启用分代 ZGC
-XX:+UseZGC -XX:+ZGenerational

# 调优参数
-XX:ZAllocationSpikeTolerance=2.0
-XX:ZCollectionInterval=5
-XX:ZUncommitDelay=300

# 监控
-Xlog:gc*,gc+heap*,gc+stats*:file=gc.log:time,level,tags
```

**性能对比**:
| 场景 | 单代 ZGC | 分代 ZGC | 改进 |
|------|----------|----------|------|
| 年轻代回收 | 基准 | +50% 速度 | 显著 |
| 内存使用 | 基准 | -20% | 明显 |
| 最大停顿时间 | 基准 | -30% | 明显 |

### 6. Vector API (第六轮孵化) (JEP 448)

**状态**: 第六轮孵化

**性能优势**: 利用 SIMD 指令进行并行计算

```java
// 向量化计算示例
static final VectorSpecies<Float> SPECIES = FloatVector.SPECIES_PREFERRED;

void vectorComputation(float[] a, float[] b, float[] c) {
    for (int i = 0; i < a.length; i += SPECIES.length()) {
        var m = SPECIES.indexInRange(i, a.length);
        
        FloatVector va = FloatVector.fromArray(SPECIES, a, i, m);
        FloatVector vb = FloatVector.fromArray(SPECIES, b, i, m);
        FloatVector vc = va.add(vb).mul(va);
        
        vc.intoArray(c, i, m);
    }
}
```

**适用场景**: 数值计算，图像处理，科学计算。

## 常见问题解决

### Virtual Threads 相关问题

1. **线程被固定 (Pinned)**:
```bash
# 诊断线程固定
-Djdk.traceVirtualThreadLocals=true
-Djdk.traceVirtualThreads=true

# 检查日志中的 "Pinned" 消息
```

**解决方案**:
- 避免在 synchronized 块内进行 I/O
- 使用 `ReentrantLock` 替代 `synchronized`
- 减少线程局部变量使用

2. **内存泄漏**:
```java
// 确保正确关闭资源

    // 使用 executor
}
// 自动关闭，释放所有线程
```

```java
// 之前: 复杂的异步回调
CompletableFuture.supplyAsync(() -> fetchData())
    .thenApply(data -> transform(data))
    .thenAccept(result -> process(result))
    .exceptionally(ex -> handleError(ex));

// 之后: 同步风格的 Virtual Threads
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    Future<String> future = executor.submit(() -> {
        String data = fetchData();  // 阻塞操作
        return transform(data);
    });
    
    String result = future.get();
    process(result);
}
```
