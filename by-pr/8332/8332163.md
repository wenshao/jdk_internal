## 背景

### 原有架构问题

**JDK-8332163 之前**，SuperWord 使用 `PacksetGraph`：

```
PackSet (打包集)
    ↓
PacksetGraph (包图) - 检测循环，调度内存操作
    ↓
SuperWord::apply_vectorization - 直接生成 C2 节点
```

**问题**：
1. **紧耦合** - 分析和代码生成混在一起
2. **难以扩展** - 无法实现复杂的优化检查
3. **单一职责** - 一个类做太多事情

### 设计目标

在 [Emanuel 的博客](https://eme64.github.io/blog/2023/11/03/C2-AutoVectorizer-Improvement-Ideas.html) 中提出的改进：

- 成本模型
- if-conversion (if 转换)
- 标量到向量直接扩展
- Shuffle/pack/extract 优化
- 更多归约模式

## 核心变更

### 新架构：VTransformGraph

```
PackSet
    ↓
SuperWordVTransformBuilder
    ↓
VTransformGraph (向量化变换图)
    ↓
VTransformGraph::optimize (未来优化)
    ↓
VTransformGraph::apply_vectorization (生成 C2 节点)
```

### VTransformNode

```cpp
class VTransformNode : public ResourceObj {
public:
    // 对应原始 C2 节点
    Node* _n;

    // 向量化后的节点
    Node* _vector;

    // 变换类型
    enum TransformType {
        Load, Store, Scalar2Vec, Vec2Scalar, Reduction, ...
    } _t;

    // 成本
    int _cost;
};
```

### 变换流程

```cpp
// 1. 构建 VTransformGraph
VTransformGraph* SuperWord::build_vtransform_graph(PackSet* pset) {
    SuperWordVTransformBuilder builder;
    return builder.build(pset);
}

// 2. 优化图 (未来)
vtransform_graph->optimize();

// 3. 应用到 C2 图
vtransform_graph->apply_vectorization();
```

## 技术分析

### 与旧架构对比

| 方面 | PacksetGraph | VTransformGraph |
|------|--------------|-----------------|
| **职责** | 调度 + 代码生成 | 仅调度和优化 |
| **IR 表示** | 隐式 | 显式图结构 |
| **可扩展性** | 低 | 高 |
| **优化检查** | 在代码生成时 | 在图遍历时 |

### VTransform 的优势

#### 1. 模块化

```cpp
// 之前: 逻辑混在一起
void SuperWord::output() {
    // 分析依赖
    // 构建包
    // 调度
    // 生成节点
}

// 之后: 职责分离
VTransformGraph* graph = build_graph();
graph->schedule();      // 独立的调度
graph->optimize();      // 独立的优化
graph->apply();         // 独立的应用
```

#### 2. 可测试性

```cpp
// 可以单独测试变换图
void test_vtransform_graph() {
    VTransformGraph* graph = create_test_graph();
    ASSERT_TRUE(graph->is_valid());
    ASSERT_EQ(graph->cost(), expected_cost);
}
```

#### 3. 可扩展性

```cpp
// 未来可以轻松添加新的优化
class VTransformOptimizer {
    virtual void optimize(VTransformGraph* graph);
};

class IfConversionOptimizer : public VTransformOptimizer {
    void optimize(VTransformGraph* graph) override {
        // 实现 if-conversion
    }
};
```

## 总结

JDK-8332163 是 SuperWord 的重大架构重构：

1. **模块化** - 分离调度、优化和应用
2. **可扩展** - 为未来优化奠定基础
3. **VTransformGraph** - 显式的向量化 IR
4. **无性能回归** - 保持现有优化能力

这是 Emanuel Peter 对 SuperWord 的基础性重构，为 JDK-8340093 成本模型等后续优化铺平了道路。
