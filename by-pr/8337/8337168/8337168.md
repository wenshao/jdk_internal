## 背景

`LocalDateTime` 是 Java 8 引入的日期时间类，广泛用于表示不带时区的日期时间。原始实现的 `toString()` 方法使用字符串拼接：

```java
public String toString() {
    return date.toString() + 'T' + time.toString();
}
```

这会创建：
1. `date.toString()` 的结果 String
2. `time.toString()` 的结果 String
3. 拼接结果的 String

**为什么重要？**
- `LocalDateTime` 在日志、调试、序列化中大量使用
- 减少 GC 压力对性能敏感的应用很重要
- 这是 JDK-8337832 的前身，针对 `LocalDateTime` 的专项优化

**改动位置**
```
src/java.base/share/classes/java/time/LocalDate.java
    └── 新增 formatTo(StringBuilder) 方法
src/java.base/share/classes/java/time/LocalTime.java
    └── 新增 formatTo(StringBuilder) 方法
src/java.base/share/classes/java/time/LocalDateTime.java
    └── 重写 toString() 使用 formatTo
```

## 问题分析

### 原始实现

```java
// LocalDateTime.java
@Override
public String toString() {
    return date.toString() + 'T' + time.toString();
}
```

### 性能瓶颈分析

| 瓶颈点 | 原因 | 影响 |
|--------|------|------|
| **date.toString()** | 创建临时 String "2024-01-01" | ~40 bytes |
| **time.toString()** | 创建临时 String "12:00:00" | ~60 bytes |
| **字符串拼接** | `+` 操作创建中间 StringBuilder | ~80 bytes |
| **总分配** | 三次分配，两次拷贝 | 高 GC 压力 |

### 执行流程

```
LocalDateTime.toString()
  │
  ├─ date.toString()
  │   ├─ new StringBuilder(10)
  │   ├─ append year, month, day
  │   └─ toString()  ← 临时 String #1
  │
  ├─ time.toString()
  │   ├─ new StringBuilder(18)
  │   ├─ append hour, minute, second, nano
  │   └─ toString()  ← 临时 String #2
  │
  └─ 拼接 (String + 'T' + String)
      ├─ new StringBuilder(...)
      ├─ append date.toString()
      ├─ append 'T'
      ├─ append time.toString()
      └─ toString()  ← 最终 String
```

### 内存分配

```
优化前:
├── date.toString():  1 × StringBuilder(10) + 1 × String(10)
├── time.toString():  1 × StringBuilder(18) + 1 × String(18)
└── 拼接:             1 × StringBuilder(29) + 1 × String(29)

总计: 3 × StringBuilder + 3 × String = ~120 bytes
```

## 解决方案

### 核心改动

#### 1. 为 `LocalDate` 添加 `formatTo` 方法

```java
// LocalDate.java
@Override
public String toString() {
    var buf = new StringBuilder(10);
    formatTo(buf);
    return buf.toString();
}

/**
 * Prints the toString result to the given buf, avoiding extra string allocations.
 * Requires extra capacity of 10 to avoid StringBuilder reallocation.
 */
void formatTo(StringBuilder buf) {
    int yearValue = year;
    int monthValue = month;
    int dayValue = day;
    int absYear = Math.abs(yearValue);
    if (absYear < 1000) {
        if (yearValue < 0) {
            buf.append('-');
        }
        buf.append(String.format("%03d", absYear));
    } else {
        buf.append(yearValue);
    }
    buf.append(monthValue < 10 ? "-0" : "-")
       .append(monthValue)
       .append(dayValue < 10 ? "-0" : "-")
       .append(dayValue);
}
```

#### 2. 为 `LocalTime` 添加 `formatTo` 方法

```java
// LocalTime.java
@Override
public String toString() {
    var buf = new StringBuilder(18);
    formatTo(buf);
    return buf.toString();
}

/**
 * Prints the toString result to the given buf, avoiding extra string allocations.
 * Requires extra capacity of 18 to avoid StringBuilder reallocation.
 */
void formatTo(StringBuilder buf) {
    int hourValue = hour;
    int minuteValue = minute;
    int secondValue = second;
    int nanoValue = nano;
    buf.append(hourValue < 10 ? "0" : "")
       .append(hourValue)
       .append(minuteValue < 10 ? ":0" : ":")
       .append(minuteValue)
       .append(secondValue < 10 ? ":0" : ":")
       .append(secondValue);
    if (nanoValue > 0) {
        buf.append('.');
        int digits = 9;
        while (nanoValue % 10 == 0) {
            nanoValue /= 10;
            digits--;
        }
        buf.append(String.format("%0" + digits + "d", nanoValue));
    }
}
```

#### 3. 重构 `LocalDateTime.toString()`

```java
// LocalDateTime.java
@Override
public String toString() {
    var buf = new StringBuilder(29);
    date.formatTo(buf);  // 直接写入，不创建临时字符串
    buf.append('T');
    time.formatTo(buf);  // 直接写入
    return buf.toString();
}
```

### 关键优化点

| 优化点 | 说明 |
|--------|------|
| **单次分配** | 只分配一个 StringBuilder (29 bytes) |
| **消除临时字符串** | date 和 time 的结果不创建 String |
| **容量预知** | 10 (date) + 1 ('T') + 18 (time) = 29 |
| **可复用** | `formatTo` 方法可被其他场景使用 |

## 设计决策

### Q1: 为什么 `formatTo` 是包私有的？

> **Reviewer**: 为什么不设为 `private` 或 `public`？

**A**:
1. `private` 会导致子类无法访问
2. `public` 会使其成为 API，需要长期维护
3. 包私有是最佳平衡点

### Q2: 为什么不修改 `toString()` 的签名？

> **Reviewer**: 能否让 `toString()` 接受 `StringBuilder` 参数？

**A**:
1. `toString()` 是 `Object` 的方法，签名不能改
2. 这是公共 API，必须保持兼容
3. `formatTo` 作为内部方法提供优化路径

### Q3: 为什么不在 `LocalDate` 和 `LocalTime` 中缓存结果？

> **Reviewer**: 这些对象是不可变的，可以缓存 `toString()` 结果吗？

**A**:
1. 缓存会增加内存占用（每个实例 ~50 bytes）
2. `toString()` 不是所有场景都调用
3. 当前优化已经足够好，缓存收益有限
