## 背景

Java 8 引入的 `java.time` API（JSR-310）提供了现代化的日期时间处理能力。其中，`toString()` 方法被频繁调用来生成字符串表示。

在 `DateTimeFormatterBuilder` 的 `Instant` 打印逻辑中，存在性能问题：
1. **多次创建 StringBuilder** - 每次格式化都创建新的 StringBuilder
2. **方法过大** - `format` 方法超过 325 字节，无法被 JIT 内联
3. **代码重复** - LocalDate/Time/DateTime 的 formatTo 逻辑分散在各处

**改动位置**
```
src/java.base/share/classes/java/time/
    ├── LocalDate.java
    ├── LocalTime.java
    ├── LocalDateTime.java
    ├── OffsetDateTime.java
    ├── OffsetTime.java
    └── ZonedDateTime.java

src/java.base/share/classes/java/time/format/
    └── DateTimeFormatterBuilder.java

src/java.base/share/classes/jdk/internal/util/
    └── DateTimeHelper.java (新文件)
```

### 性能瓶颈分析

| 瓶颈点 | 原因 | 影响 |
|--------|------|------|
| **多次创建 StringBuilder** | `buf.append(ldt)` 内部调用 `ldt.toString()`，创建新的 StringBuilder | 内存分配开销 |
| **方法过大** | format 方法超过 325 字节 | JIT 无法内联 |
| **重复的格式化逻辑** | LocalDate/Time/DateTime 各自实现 formatTo | 代码重复，维护成本高 |

### JIT 内联限制

HotSpot JVM 默认的内联限制：
- **最大内联方法大小**: ~325 字节
- **频繁调用的小方法**: 更容易内联

原 `format` 方法超过这个限制，导致：
1. 无法内联到调用点
2. 每次调用都有方法开销
3. 逃逸分析无法优化 StringBuilder 创建

## 解决方案

### 核心改动

#### 1. 新增 DateTimeHelper 工具类

```java
// 新文件: jdk/internal/util/DateTimeHelper.java

package jdk.internal.util;

import java.time.LocalDate;
import java.time.LocalDateTime;
import java.time.LocalTime;

/**
 * java.time 包的内部工具类
 */
public final class DateTimeHelper {
    private DateTimeHelper() {}

    /**
     * 格式化 LocalDateTime 到指定的 StringBuilder
     * 避免 toString() 创建新的 StringBuilder
     */
    public static void formatTo(StringBuilder buf, LocalDateTime dateTime) {
        formatTo(buf, dateTime.toLocalDate());
        buf.append('T');
        formatTo(buf, dateTime.toLocalTime());
    }

    /**
     * 格式化 LocalDate 到指定的 StringBuilder
     */
    public static void formatTo(StringBuilder buf, LocalDate date) {
        int year  = date.getYear(),
            month = date.getMonthValue(),
            day   = date.getDayOfMonth();
        int absYear = Math.abs(year);
        if (absYear < 1000) {
            if (year < 0) {
                buf.append('-');
            }
            buf.repeat('0', absYear < 10 ? 3 : absYear < 100 ? 2 : 1);
            buf.append(absYear);
        } else {
            if (year > 9999) {
                buf.append('+');
            }
            buf.append(year);
        }
        buf.append(month < 10 ? "-0" : "-").append(month)
           .append(day < 10 ? "-0" : "-").append(day);
    }

    /**
     * 格式化 LocalTime 到指定的 StringBuilder
     */
    public static void formatTo(StringBuilder buf, LocalTime time) {
        int hour   = time.getHour(),
            minute = time.getMinute(),
            second = time.getSecond(),
            nano   = time.getNano();
        buf.append(hour < 10 ? "0" : "").append(hour)
           .append(minute < 10 ? ":0" : ":").append(minute);
        if ((second | nano) > 0) {
            buf.append(second < 10 ? ":0" : ":").append(second);
            if (nano > 0) {
                buf.append('.');
                int zeros = 9 - DecimalDigits.stringSize(nano);
                if (zeros > 0) {
                    buf.repeat('0', zeros);
                }
                int digits;
                if (nano % 1_000_000 == 0) {
                    digits = nano / 1_000_000;
                } else if (nano % 1000 == 0) {
                    digits = nano / 1000;
                } else {
                    digits = nano;
                }
                buf.append(digits);
            }
        }
    }
}
```

#### 3. 重构 DateTimeFormatterBuilder

```java
// 变更后: 分割 format 方法，使其可以被内联

public boolean format(DateTimePrintContext context, StringBuilder buf) {
    Instant instant = context.getValue(Instant.class);
    long inSec = instant.getEpochSecond();
    int inNano = instant.getNano();

    // 优化1: 根据小数位数选择是否在 LocalDateTime 中包含纳秒
    if (fractionalDigits == 0) {
        inNano = 0;
    }
    boolean printNanoInLocalDateTime = fractionalDigits == -2
            || (inNano == 0 && (fractionalDigits == 0 || fractionalDigits == -1));

    if (inSec >= -SECONDS_0000_TO_1970) {
        currentEra(buf, inSec, printNanoInLocalDateTime ? inNano : 0);
    } else {
        beforeCurrentEra(buf, inSec, printNanoInLocalDateTime ? inNano : 0);
    }

    // 优化2: 只在需要时打印小数秒
    if (!printNanoInLocalDateTime) {
        printNano(buf, inSec, inNano);
    }
    buf.append('Z');
    return true;
}

// 新方法: 格式化当前纪元，可被内联
private static void currentEra(StringBuilder buf, long inSec, int inNano) {
    long zeroSecs = inSec - SECONDS_PER_10000_YEARS + SECONDS_0000_TO_1970;
    long hi = Math.floorDiv(zeroSecs, SECONDS_PER_10000_YEARS) + 1;
    long lo = Math.floorMod(zeroSecs, SECONDS_PER_10000_YEARS);
    LocalDateTime ldt = LocalDateTime.ofEpochSecond(lo - SECONDS_0000_TO_1970, inNano, ZoneOffset.UTC);
    if (hi > 0) {
        buf.append('+').append(hi);
    }
    DateTimeHelper.formatTo(buf, ldt);  // 共享 StringBuilder，不创建新的
    if (ldt.getSecond() == 0 && inNano == 0) {
        buf.append(":00");
    }
}

// 新方法: 格式化公元前，可被内联
private static void beforeCurrentEra(StringBuilder buf, long inSec, int inNano) {
    // 类似逻辑，使用 DateTimeHelper.formatTo
}

// 新方法: 打印小数秒
private void printNano(StringBuilder buf, long inSec, int inNano) {
    buf.append('.');
    int div = 100_000_000;
    int fractionalDigits = this.fractionalDigits;
    for (int i = 0; ((fractionalDigits == -1 && inNano > 0) ||
            (fractionalDigits == -2 && (inNano > 0 || (i % 3) != 0)) ||
            i < fractionalDigits); i++) {
        int digit = inNano / div;
        buf.append((char) (digit + '0'));
        inNano = inNano - (digit * div);
        div = div / 10;
    }
}
```

```java
@Override
public String toString() {

// OffsetTime.java
@Override
public String toString() {
    var offsetStr = offset.toString();
    var buf = new StringBuilder(18 + offsetStr.length());
    DateTimeHelper.formatTo(buf, time);  // 使用工具类
    return buf.append(offsetStr).toString();
}

// ZonedDateTime.java
@Override
public String toString() {
    String offsetStr = offset.toString();
    String zoneStr = zone.getId();
    int length = 29 + offsetStr.length();
    if (zoneStr.length() == 1) {  // zone ID is single letter
        length++;
    } else {
        length += zoneStr.length() + 2;
    }
    var buf = new StringBuilder(length);
    DateTimeHelper.formatTo(buf, dateTime);  // 使用工具类
    buf.append(offsetStr);
    if (zoneStr.length() != 1) {
        buf.append('[').append(zoneStr).append(']');
    }
    return buf.toString();
}
```

## 设计决策

### Q1: 为什么创建 DateTimeHelper 而不是使用现有方法？

> **Reviewer**: 为什么不直接调用现有的 formatTo 方法？

**A**:
1. **跨包访问** - formatTo 是包私有的，无法跨包调用
2. **统一入口** - DateTimeHelper 提供统一的格式化入口
3. **未来扩展** - 其他内部类也可以使用 DateTimeHelper

### Q2: 为什么要拆分 format 方法？

> **Reviewer**: currentEra 和 beforeCurrentEra 为什么是 static？

**A**:
1. **可内联性** - static 方法更容易被 JIT 内联
2. **减少状态** - 不依赖实例状态，减少参数传递
3. **代码大小** - 每个方法小于 325 字节，可以内联

### Q3: printNanoInLocalDateTime 的逻辑是什么？

> **Reviewer**: 这个条件判断有点复杂

**A**:
```java
boolean printNanoInLocalDateTime = fractionalDigits == -2
        || (inNano == 0 && (fractionalDigits == 0 || fractionalDigits == -1));
```

- **fractionalDigits == -2**: 总是打印小数秒（包括末尾零）
- **fractionalDigits == -1**: 只打印有效小数（不打印末尾零）
- **fractionalDigits == 0**: 不打印小数
- **fractionalDigits > 0**: 打印指定位数

如果 `printNanoInLocalDateTime` 为 true，纳秒会在 LocalDateTime 格式化中一起打印；否则单独打印。
