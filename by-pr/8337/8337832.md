## 背景

Java 8 引入的 `java.time` 包提供了现代化的日期时间 API。原始实现中，复合日期时间类（如 `LocalDateTime`、`ZonedDateTime`）的 `toString()` 方法会拼接多个字符串。

**问题示例**：
```java
// LocalDateTime.toString() 原始实现
public String toString() {
    return date.toString() + 'T' + time.toString();
}
```

这会创建：
1. `date.toString()` 的临时 String
2. `time.toString()` 的临时 String
3. 拼接结果的 String

**为什么重要？**
- 日期时间格式化在日志、调试、序列化中广泛使用
- 减少临时对象分配可以降低 GC 压力
- 复合类的 `toString()` 调用频率很高

**改动位置**
```
src/java.base/share/classes/java/time/LocalDate.java
    └── 新增 formatTo(StringBuilder)
src/java.base/share/classes/java/time/LocalTime.java
    └── 新增 formatTo(StringBuilder)
src/java.base/share/classes/java/time/LocalDateTime.java
    └── 重写 toString()
src/java.base/share/classes/java/time/OffsetDateTime.java
    └── 重写 toString()
src/java.base/share/classes/java/time/OffsetTime.java
    └── 重写 toString()
src/java.base/share/classes/java/time/ZonedDateTime.java
    └── 重写 toString()
```

## 问题分析

### 原始实现

```java
// LocalDateTime.toString()
public String toString() {
    return date.toString() + 'T' + time.toString();
}

// ZonedDateTime.toString()
public String toString() {
    String str = dateTime.toString() + offset.toString();
    if (offset != zone) {
        str += '[' + zone.toString() + ']';
    }
    return str;
}

// OffsetDateTime.toString()
public String toString() {
    return dateTime.toString() + offset.toString();
}
```

### 性能瓶颈分析

| 瓶颈点 | 原因 | 影响 |
|--------|------|------|
| **临时字符串** | 每个组件调用 `toString()` 创建临时字符串 | 多次分配 |
| **字符串拼接** | `+` 操作符创建中间字符串 | 额外分配 |
| **重复计算** | `dateTime.toString()` 内部也创建 StringBuilder | 重复工作 |

### 对象分配流程

```
LocalDateTime.toString()
  │
  ├─ date.toString()              ← 创建 String "2024-01-01"
  │   └─ new StringBuilder(10)    ← 临时 StringBuilder
  │
  ├─ time.toString()              ← 创建 String "12:00:00"
  │   └─ new StringBuilder(18)    ← 临时 StringBuilder
  │
  └─ 拼接                          ← 创建最终 String
      └─ new StringBuilder(...)    ← 又一个 StringBuilder
```

## 解决方案

### 核心改动

#### 1. 为组件类添加 `formatTo` 方法

```java
// LocalDate.java
public String toString() {
    var buf = new StringBuilder(10);
    formatTo(buf);
    return buf.toString();
}

// 新增私有方法
void formatTo(StringBuilder buf) {
    int yearValue = year;
    int monthValue = month;
    int dayValue = day;
    int absYear = Math.abs(yearValue);
    // ... 格式化逻辑直接写入 buf ...
    buf.append(monthValue < 10 ? "-0" : "-")
       .append(monthValue)
       .append(dayValue < 10 ? "-0" : "-")
       .append(dayValue);
}
```

#### 2. 复合类重用 `formatTo`

```java
// LocalDateTime.java
public String toString() {
    var buf = new StringBuilder(29);
    date.formatTo(buf);  // 直接写入，不创建临时字符串
    buf.append('T');
    time.formatTo(buf);  // 直接写入
    return buf.toString();
}

// ZonedDateTime.java
public String toString() {
    var offsetStr = offset.toString();
    var zoneStr = (String) null;
    int length = 29 + offsetStr.length();
    if (offset != zone) {
        zoneStr = zone.toString();
        length += zoneStr.length() + 2;
    }
    var buf = new StringBuilder(length);
    dateTime.formatTo(buf);  // 复用
    buf.append(offsetStr);
    if (zoneStr != null) {
        buf.append('[').append(zoneStr).append(']');
    }
    return buf.toString();
}
```

### 关键优化点

| 优化点 | 说明 |
|--------|------|
| **消除临时字符串** | 组件不再创建中间 String |
| **单次分配** | 只分配最终的 StringBuilder |
| **容量预知** | 可以精确计算所需容量 |
| **代码复用** | `formatTo` 可被其他场景复用 |

## 设计决策

### Q1: 为什么 `formatTo` 是包私有的？

> **Reviewer**: 为什么不设为 `public`？

**A**:
1. 这是内部优化，不是公共 API
2. 包私有允许同包内的其他类复用
3. 保留未来扩展的可能性

### Q2: 为什么 `ZonedDateTime` 仍调用 `offset.toString()`？

> **Reviewer**: 为什么不给 `ZoneOffset` 也添加 `formatTo`？

**A**:
1. `ZoneOffset.toString()` 返回的是标准化格式（如 "+01:00"）
2. 这个字符串可能被缓存，直接使用更高效
3. 优化收益递减，不值得增加复杂度

### Q3: 为什么不在接口中定义 `formatTo`？

> **Reviewer**: 是否应该定义一个接口？

**A**:
1. 这不是要成为公共 API
2. 不同类型的格式化逻辑差异大
3. 保持简单，避免过度设计

### 微基准测试

```java
@State(Scope.Benchmark)
@BenchmarkMode(Mode.AverageTime)
@OutputTimeUnit(TimeUnit.NANOSECONDS)
@Warmup(iterations = 5, time = 1)
@Measurement(iterations = 15, time = 1)
@Fork(3)
public class DateTimeToStringBenchmark {

    @Benchmark
    public String localDateTimeToString() {
        return LocalDateTime.now().toString();
    }

    @Benchmark
    public String zonedDateTimeToString() {
        return ZonedDateTime.now().toString();
    }

    @Benchmark
    public String offsetDateTimeToString() {
        return OffsetDateTime.now().toString();
    }
}
```
