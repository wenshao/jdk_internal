## 背景

### FDLIBM 简介

**FDIBM** (Freely Distributable LIBM) 是：
- Sun Microsystems 的 C 数学库
- 移植到 Java (JDK-8171407, JDK-8134780)
- 实现 sin, cos, exp, log 等浮点函数

### @Stable 注解

**@Stable** 是内部 JVM 注解：

```java
package jdk.internal.vm.annotation;

@Target({ElementType.FIELD})
@Retention(RetentionPolicy.RUNTIME)
public @interface Stable {
}
```

**语义**：字段值在设置后不会改变

---

## 核心变更

### 变更前

```java
// FdLibm.java
public class FdLibm {
    // 查找表数组 - 没有 @Stable
    private static final double[] __libm_sincos_table = {
        0.0, 1.0, 2.0, ...
    };

    private static final double[] __libm_exp_table = {
        1.0, 2.71828, ...
    };
}
```

**问题**：
- JIT 不知道数组内容是否稳定
- 每次访问都需要边界检查
- 无法将数组视为常量

### 变更后

```java
// FdLibm.java
public class FdLibm {
    @Stable  // ✅ 告诉 JIT 数组内容稳定
    private static final double[] __libm_sincos_table = {
        0.0, 1.0, 2.0, ...
    };

    @Stable  // ✅ 启用优化
    private static final double[] __libm_exp_table = {
        1.0, 2.71828, ...
    };
}
```

## 技术分析

### @Stable 的优化效果

| 优化类型 | 没有 @Stable | 有 @Stable |
|----------|-------------|-----------|
| **边界检查消除** | ❌ | ✅ |
| **常量折叠** | ❌ | ✅ |
| **循环向量化** | 受限 | 启用 |
| **寄存器缓存** | 不稳定 | 稳定 |

### JIT 编译器优化

```java
// 用户代码
for (int i = 0; i < N; i++) {
    result = Math.sin(angle);  // 内部使用 FDLIBM
}
```

**没有 @Stable**:
```
每次 Math.sin 调用:
1. 加载数组引用
2. 检查边界 (每次)
3. 加载数组元素
4. 执行计算
```

**有 @Stable**:
```
首次 Math.sin 调用:
1. 加载数组引用 (缓存)
2. 检查边界 (一次)
3. 加载数组元素 (提升到寄存器)
4. 执行计算

后续调用:
- 数组元素在寄存器中
- 无边界检查
```

### 性能提升

根据相关基准测试 (JDK-8152910)：

| 场景 | 提升 |
|------|------|
| **三角函数** | +5-10% |
| **指数/对数** | +3-8% |
| **密集计算** | +10-15% |
