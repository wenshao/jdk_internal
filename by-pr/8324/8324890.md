## 背景

### VLoop 类的问题

**JDK-8324890 之前**，SuperWord 使用 `VLoop` 类：

```cpp
// VLoop 类 - 表示循环的向量化信息
class VLoop : public ResourceObj {
private:
    Arena* _arena;
    PhaseIdealLoop* _phase;
    IdealLoopTree* _lpt;
    Node* _entry;
    int _body_size;

    // 初始化和重置机制
    void init();
    void reset();

    // 循环展开分析
    bool unrolling_analysis(int vector_length);
};
```

**问题**：
1. **紧耦合** - VLoop 与 SuperWord 实现紧密耦合
2. **状态管理** - init/reset 机制复杂
3. **不可复用** - 无法被其他向量化设施使用

### 设计目标

为 **JDK-8315361** (创建共享 VLoopAnalyzer) 做准备：

```
之前: SuperWord 专属
SuperWord::VLoop → 仅 SuperWord 使用

之后: 共享设施
VLoopAnalyzer → SuperWord + 未来优化使用
```

## 核心变更

### 1. 移除 VLoop 类

**变更前**:
```cpp
// superword.hpp
class VLoop : public ResourceObj {
    Arena* _arena;
    PhaseIdealLoop* _phase;
    IdealLoopTree* _lpt;
    ...

    void init();
    void reset();
    bool unrolling_analysis(int vector_length);
};

class SuperWord : public ResourceObj {
    VLoop* _vloop;  // 成员变量
    ...
};
```

**变更后**:
```cpp
// superword.hpp
// VLoop 类完全移除

class SuperWord : public ResourceObj {
    // 不再持有 VLoop 指针
    ...
};
```

### 2. unrolling_analysis 静态化

**变更前**:
```cpp
// VLoop 的成员方法
bool VLoop::unrolling_analysis(int vector_length) {
    // 使用成员变量
    IdealLoopTree* lpt = _lpt;
    PhaseIdealLoop* phase = _phase;

    // 分析循环
    ...
}
```

**变更后**:
```cpp
// 静态方法，接受参数
static bool unrolling_analysis(PhaseIdealLoop* phase,
                               IdealLoopTree* lpt,
                               int vector_length) {
    // 使用传入的参数
    // 分析循环
    ...
}
```

### 3. 移除 init/reset 机制

**变更前**:
```cpp
// 使用 VLoop 时需要 init/reset
VLoop* vloop = new VLoop(arena, phase, lpt);
vloop->init();      // 初始化

// ... 使用 vloop ...

vloop->reset();     // 重置
// ... 再次使用 ...
```

**变更后**:
```cpp
// 直接调用静态方法，无需 init/reset
bool result = unrolling_analysis(phase, lpt, vector_length);
```

## 技术分析

### 重构模式

#### 从对象到静态方法

```
OOP 风格:
vloop->method(arg1, arg2)

函数式风格:
method(vloop_state, arg1, arg2)
```

#### 状态外部化

```cpp
// 之前: 状态在 VLoop 对象中
class VLoop {
    IdealLoopTree* _lpt;
    PhaseIdealLoop* _phase;
};

// 之后: 状态作为参数传递
static void analysis(PhaseIdealLoop* phase,
                     IdealLoopTree* lpt,
                     int vector_length);
```

### 优势分析

| 方面 | 之前 (VLoop) | 之后 (静态方法) |
|------|-------------|-----------------|
| **可复用性** | 低 - SuperWord 专属 | 高 - 任何代码可调用 |
| **测试性** | 低 - 需要 SuperWord 上下文 | 高 - 可独立测试 |
| **复杂度** | 高 - init/reset 机制 | 低 - 无状态管理 |
| **内存** | 每次 new 对象 | 无对象分配 |

## 与 JDK-8315361 的关系

### VLoopAnalyzer 共享设施

**JDK-8315361** 创建共享的 `VLoopAnalyzer`：

```cpp
// 新的共享类
class VLoopAnalyzer : public ResourceObj {
public:
    // 可被 SuperWord 和其他优化使用
    static bool analyze(PhaseIdealLoop* phase,
                       IdealLoopTree* lpt,
                       int vector_length);

    static bool is_vectorizable(LoopNode* loop);
    static int estimate_vector_length(IdealLoopTree* lpt);
};
```

### 重构步骤

```
步骤 1: JDK-8324890 (当前 PR)
├── 移除 VLoop 类
├── unrolling_analysis 静态化
└── 移除 init/reset 机制

步骤 2: JDK-8315361 (后续 PR)
├── 创建 VLoopAnalyzer 类
├── 移动分析方法到 VLoopAnalyzer
└── SuperWord 使用 VLoopAnalyzer
```
