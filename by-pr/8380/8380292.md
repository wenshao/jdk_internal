## 描述

Fixes: https://bugs.openjdk.org/browse/JDK-8380292

During AOT phase training, many indies, fields, methods, and classes are logged as "reverted" because they are being cleaned up to be able to properly process them during assembly phase to store them. The "archived/reverted" messages are therefore confusing, somehow misleading into thinking that some of those assets are not going to be stored on the AOT cache. But then they are, indeed, stored, during assembly.

On this fix I make sure those messages are not logged during training phase, and we focus on messaging during assembly.

Examples of confusing messages before this commit:

Using the code available on https://github.com/Delawen/cache-or-not/tree/main and focusing on `org.example.PlainClass` which should be stored on the cache cleanly and without issues:
