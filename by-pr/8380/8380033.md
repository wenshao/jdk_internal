## 描述

If a downstream OpenJDK build uses a different default setting for security property "jdk.includeInExceptions" with regards to jar files, the test java/util/jar/Manifest/IncludeInExceptionsTest.java will fail.

However, since the test's purpose is not to check the default value but rather the functionality, it could be made agnostic to the default.

The test now has 3 iterations, one with "-Djdk.includeInExceptions=jar", one with "-Djdk.includeInExceptions=" (empty) and one without "jdk.includeInExceptions" set, which will be the default value of the JDK distribution.
