## 描述

This change creates a separate `CopyAVX3Threshold` flag that covers special cases where it's beneficial to use 64 byte instructions for array copy and clear operations. The existing `AVX3Threshold` flag is still used by other operations. Additionally, `VM_Version::avx3_threshold()` is removed to ensure flag changes aren't made multiple times. Instead, CopyAVX3Threshold is initialized in `VM_Version::get_processor_features()`. This ensures settings can be verified in AOT mode. To validate, tier-1 tests were run on an [Intel® Xeon 6767P](https://www.intel.com/content/www/us/en/products/sku/241845/intel-xeon-6767p-processor-336m-cache-2-40-ghz/specifications.html) using [OpenJDK v27-b13](https://github.com/openjdk/jdk/releases/tag/jdk-27%2B13) as the baseline build.
