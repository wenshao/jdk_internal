## 描述

This patch prevents stack overflows in JFR DFS search. It is a simple and reliable solution, albeit maybe a bit unusual. I put this out for discussion as an alternative to https://github.com/openjdk/jdk/pull/29382, which is much more involved and harder to make airtight for some corner cases, especially in BFS+DFS mixed mode (I keep running into performance regressions with large object arrays at the boundary to DFS).

Given that we plan to abandon DFS anyway and that we all do not have much time for review, maybe this is a more pragmatic approach. As a bonus, this is really easy to downport.

I manually tested this on Linux x64 (gcc, various versions, and clang), MacOS arm64, and AIX. In all cases, it successfully prevents stack overflows even if we run with very small VMThreadStack sizes. Obviously, the smaller the stack, the fewer paths to GC roots we find, but that is not different from the static limit test we use today.

I also tested the performance impact, but os::current_stack_pointer is really fast. The new approach costs ~0.23% performance when doing pure DFS (skipping BFS) for reference-chain-heavy test case. I think that is acceptable.
