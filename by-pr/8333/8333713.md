## 背景

### 命名问题

**JDK-8333713 之前**，代码中混用两个名称：

```cpp
// vectornode.cpp/hpp 中混用
void SuperWord::some_method() { ... }           // SuperWord
void SuperWord::superword_analysis() { ... }    // superword (小写)
void superword_verify(...) { ... }              // superword (小写)
```

**问题**：
1. **命名不一致** - SuperWord vs superword vs auto_vectorization
2. **语义模糊** - "SuperWord" 是历史遗留名，不如 "auto_vectorization" 清晰
3. **代码可读性** - 新贡献者可能困惑

## 核心变更

### 1. 移除未使用方法

**变更前**:
```cpp
// vectornode.cpp
// 未使用的方法
void VectorNode::unused_method_1() { ... }
void VectorNode::unused_method_2() { ... }
int VectorNode::unused_helper() { ... }
```

**变更后**:
```cpp
// vectornode.cpp
// 方法已移除
```

### 2. 重命名 superword → auto_vectorization

**变更前**:
```cpp
// 方法名中的 superword
void SuperWord::superword_pack_pair() { ... }
bool SuperWord::superword_can_be_packed() { ... }
void superword_init() { ... }

// 注释中的 superword
// Perform superword alignment
// Check superword profitability
```

**变更后**:
```cpp
// 方法名中的 auto_vectorization
void SuperWord::auto_vectorization_pack_pair() { ... }
bool SuperWord::auto_vectorization_can_be_packed() { ... }
void auto_vectorization_init() { ... }

// 注释中的 auto_vectorization
// Perform auto-vectorization alignment
// Check auto-vectorization profitability
```

### 3. 一致性更新

| 类别 | 之前 | 之后 |
|------|------|------|
| **类名** | SuperWord | SuperWord (保留) |
| **方法前缀** | superword_ | auto_vectorization_ |
| **注释** | superword | auto_vectorization |
| **变量名** | sw | av (部分) |

## 技术分析

### 命名演进历史

```
最初: "SLP" (Superword Level Parallelism)
       ↓
JDK 7: "SuperWord" (SuperWord 作为专有名词)
       ↓
JDK 21+: "auto_vectorization" (更描述性的名称)
```

### 为什么用 auto_vectorization？

| 名称 | 优点 | 缺点 |
|------|------|------|
| **SuperWord** | 历史名称，社区熟悉 | 专有名词，不直观 |
| **superword** | 简短 | 大小写混乱 |
| **autovectorization** | 描述性强 | 无分隔符 |
| **auto_vectorization** | ✅ 清晰，标准术语 | 稍长 |

### 与其他编译器的一致性

| 编译器 | 术语 |
|--------|------|
| **GCC** | -ftree-vectorize (SLP) |
| **LLVM** | SLP Vectorizer |
| **OpenJDK (新)** | auto_vectorization |
