## 描述

This patch adds a new flag called `LoopPeeling` to control whether loop
peeling is applied.  In particular, this flag accepts numeric values in
the range 0 through 2, where 0 disables loop peeling, 1 enables it, and
2 disables it only for the iteration splitting case.  The goal of this
patch is to make it easier to reproduce problems on simpler programs
that otherwise vanish because of the loop peeling optimization.  An
additional use of this patch is in checking if loop peeling is
contributing to the problem being debugged.

To avoid disabling other transformations when loop peeling is disabled,
this patch attempts to continue without bailing out in
`create_loop_nest()` for strip-mined integer loops.  In other cases,
this patch guards calls to `do_peeling()` based on the value of the
flag.
