## 描述

Hi,

`StoreNode::Identity` tries to elide the store if the value is just loaded from the same memory location. However, for vector stores, it incorrectly checks whether `mem` is a `LoadVectorNode`, which is always false. This PR fixes that mistake and adds an IR test. Furthermore, it strengthens the code there to ensure that both the load and the store are vanilla vector accesses (not masked accesses or gathers/scatters). This is currently implied because non-vanilla vector loads return -1 for `MemNode::store_Opcode()`, but it is better to be explicit.

- [x] tier1-4,hs-precheckin-comp,hs-comp-stress
