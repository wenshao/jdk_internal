## 描述

This is a demo to be used for presentations of the Vector API.
We already have a demo for auto vectorization [JDK-8367657](https://bugs.openjdk.org/browse/JDK-8367657).

Inspiration is taken from:
https://particle-life.com/
https://www.youtube.com/@tom-mohr

It should be easy to run manually, as well as automatically through JTREG, including IR rules.

Some minor components may also be auto vectorized, and we can check for that with IR rules.

---------------------------------

If you just want to play with the demo, you can launch it stand-alone:
`java --add-modules=jdk.incubator.vector ParticleLife.java`

---------------------------------

On my AVX512 laptop, with default settings, I'm getting:
- 13 FPS with `Scalar`
- 80 FPS with `VectorAPI Inner Gather`
- 110 FPS with `VectorAPI Inner Rearranged`
- 80 FPS with `VectorAPI Outer`

This is about a 10x speedup, which is not too far off the theoretical maximum of 16x given we can put 16 floats in a 512bit vector.

(Out of curiosity, I ran it also with graalvm-jdk-25.0.2+10.1, and got: 9 FPS, 1 FPS, 68 FPS, 1 FPS. I suspect that the gather operation is not yet supported well)
