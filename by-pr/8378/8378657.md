## 描述

When an `OutputAnalyzer` instance uses a `LazyOutputBuffer` impl to capture a process' standard output, it unconditionally writes progress logs to stdout. This can easily flood the output in tests that spawn a great number of processes, each with an OutputAnalyzer attached, as part of their operations.

This change introduce an option to suppress diagnostic messages from being emitted by the analyzer and limit its overall verbosity.

(Notes on implementation)
* Introducing an "opt-in verbose mode"  would have been the more idiomatic solution, but would have implied a lot of existing tests opting into this new mode to retain their current printing behavior. 
To avoid this, an opt-in "quiet mode" was initially considered, but after reviews it was preferred to go with an opt-out verbose mode, as a compromise to avoid changes to existing tests, while retaining the more common "verbose" terminology. 
* It is implemented as an extra parameter in new constructor overloads for `OutputAnalyzer`, which really is the only way since `LazyOutputBuffer` starts logging in its constructor.
