## 描述

The new jdk/java/nio/channels/Selector/ConnectionRefusedMessage.java fails when the JDK uses enhanced exception messages of category "hostinfo", e.g. by setting 'jdk.includeInExceptions=hostInfo' in java.security.

Then, the exception message could be something like "Connection refused: localhost/127.0.0.1:47" instead of only "Connection refused".

This can be fixed by relaxing the test assertion to check for exceptions starting with "Connection refused" instead of expecting this as the full message.
