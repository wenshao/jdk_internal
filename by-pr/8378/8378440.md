## 描述

The `dlfcn,h` is needed for using `dlsym` to find the asan error callback function.  The `RTLD_DEFAULT` parameter is used to search all the loaded symbols and since the Windows equivalent os-call `GetProcAddress(HINSTANCE, char* sym_name)` does not support this search (or at least not worth to implement searching all the loaded libs); the feature is excluded in Windows builds. 
ASN build on Windows tested.
