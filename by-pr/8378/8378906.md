## 描述

Consider having an "Aggressive Deprecation" annotation processor, which adds `ACC_BRIDGE` and `ACC_SYNTHETIC` flags to the deprecated methods. Then, imagine an interface with a single deprecated method. The generated class file therefore contains the method with `ACC_PUBLIC`, `ACC_BRIDGE`, `ACC_ABSTRACT`, `ACC_SYNTHETIC` flags (a weird combination, however not against the specification).

Now, running compilation against this class file under certain circumstances, javac calls `Types.isFunctionalInterface(...)` on the interface type during overload resolution, which iterates the interface members using the `Types.DescriptorFilter`. The filter calls `Types.interfaceCandidates(...)` for the given method, which returns an empty `List` (as synthetic methods are excluded from the candidates). An attempt to directly access the first item in the returned list leads to `NullPointerException`.

The proposal here is:
Modify the `Types.DescriptorFilter.test(...)` method to simply return `false` when an empty list is returned from the `Types.interfaceCandidates(...)` call and let the compilation pass (which means that interfaces with single abstract synthetic bridge methods are not considered as functional interfaces).
