## 描述

Can I please get a review of this test-only change which introduces a workaround to address the intermittent failures in `test/hotspot/jtreg/runtime/os/TestWXHealing.java`?

As noted in https://bugs.openjdk.org/browse/JDK-8378397 this test was recently updated to have 2 `@test` definitions. That leads to concurrent launch of jshell (one in each test). This has exposed a pre-existing bug in jshell where it runs into a `NullPointerException` when trying to load jshell history from the host's filesystem. The details of that, including a trivial reproducer, are noted in https://bugs.openjdk.org/browse/JDK-8347418.

It would have been OK to just wait for that underlying but to be addressed. However, the `TestWXHealing.java` has been failing intermittently in our CI causing enough noise that a workaround in this test would be useful. The commit in this PR uses a `Map` as the backing persistence store for jshell, instead of sharing the host's filesystem by each jshell. Given the context of this test, this workaround doesn't impact the original intent of this test.

With this change, I have run this test more than 200 odd times now in our CI and it hasn't failed even once. Without this change I was able to see a failure in a test repeat of 100.
