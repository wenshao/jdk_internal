## 描述

JDK-8377389 removed `Op_URShiftI` from the `VerifyIterativeGVN` exception list after fixing the known notification gaps. This exposed a second notification gap.

`remix_address_expressions` in `loopopts.cpp` swaps `AddI` edges via `swap_edges(1, 2)` to place the loop-variant input first, but does not push the node to the IGVN worklist. This means `URShiftI` users of the `AddI` are never re-visited, missing the optimization in `URShiftINode::Ideal`.
