## 描述

The test relied on the specific formatting of `RuntimeMXBean.getInputArguments`, which has changed - flags from a settings file are now prefixed with `-XX:`.

The implementation is still correct, the `VMInfo` event explicitly gives jvmFlags as "JVM Settings File Arguments" so I think it's correct that those remain un-prefixed. 

Updated to pull the expected 'flags' and 'arguments' separately from the most convenient place I could find, `VMSupport.getAgentProperties`. If this is problematic for any reason, we could soften the check to just make sure we're including the explicit flags/args we know are given, but this would allow the implementation to add spurious arguments.
