## 描述

Can I please get a review of this test-only change which addresses the test failure on AIX?

As noted in https://bugs.openjdk.org/browse/JDK-8377015, this newly introduced test fails on AIX because the test wasn't expecting `SocketChannel.connect()` to throw a `ConnectException` when the channel was in non-blocking mode. The test has now been updated to allow for that scenario. The test continues to pass in our CI with these changes.

@MBaesken, could you please run this change against the AIX instances you have access to and see if this address the failures?
