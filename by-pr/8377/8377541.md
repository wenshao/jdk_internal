## 描述

If C2 is able to fold a memory access that requires barriers, the barriers are still emitted. This PR proposes to elide those barriers during parsing if and only if the corresponding memory access is folded to a constant.

This barrier elision during parsing will not take care of barriers for accesses that fold in later phases. I do not know of a way to locally determine if a barrier is still needed when its corresponding access was folded. To prevent walking large swaths of the memory graph, this solution seems preferable. Further, barriers are optimization blockers, so removing them as early as possible will enable more optimizations.

This issue was discovered by @minborg during his ongoing work for LazyConstants. As a consequence the best test for this is yet to be integrated with JEP 531 (#29507). This PR was tested with [`compiler/c2/irTests/stable/LazyConstantsIrTest.java`](https://github.com/minborg/jdk/blob/bedf6e2cce710d1f43177743b772b3e24e634ce6/test/hotspot/jtreg/compiler/c2/irTests/stable/LazyConstantsIrTest.java).
