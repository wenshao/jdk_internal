## 描述

Can I please get a review of this test-only change which addresses the issue noted in https://bugs.openjdk.org/browse/JDK-8377425?

The test launches a java process with `-XX:+TraceWXHealing -XX:+StressWXHealing` JVM options which are applicable only in debug VM. However the test isn't marked as requiring a debug VM and thus results in a failure when run against a product build.

The commit in this PR adds a `@requires` to mandate a debug VM. Additionally, the test code has been updated to assert that the launched java process exits normally and if it doesn't then the stdout and stderr will be printed in the test log file. Finally, a `jtreg.SkippedException` is thrown if the test identifies that `MAP_JIT` doesn't work on the system where the java process was run.

With this change I've verified that the test doesn't get executed on product builds and when executed in debug VMs, the test is marked as skipped if `MAP_JIT` isn't functional. On systems where `MAP_JIT` is functional and when run using a macos debug build, the test completely successfully and the test logs contain messages like:
