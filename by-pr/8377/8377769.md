## 描述

This PR mainly removes warnings when we can not get large pages of
sizes that has not been pre-allocated. For example, if we set up 2 MiB
pages and there are no pre-allocated 64 KiB pages, we will no longer
warn about this fact when allocating memory.

We will still warn if *no* pages has been pre-allocated, as we will still
always look at the default `_large_page_size`.

A new set of page sizes `PageSizes _pre_allocated` are kept track of.
This set contains the set `_pagesizes` filtered to contain sizes we
have pre-allocated.

When the program later allocates memory, we will choose from `_pre_allocated`
and `_large_page_size`.

I have run this through hs-tier1-3, 6 and 8.

I have also tested using the script below that will enumerate through
12 combinations on an aarch64 host with 64 KiB, 2 MiB, 32 MiB, 1 GiB
large pages. The first two runs show warnings that we want to remove,
ending in `(errno = 12)`. That is followed by two runs that show those
warnings removed after applying the code changes.

```bash
set -eu

function silent {
    "$@" &> /dev/null
}

function clear_hugepages {
    /bin/sudo find /sys/kernel/mm/hugepages -name nr_hugepages -exec bash -c "echo 0 > {}" \;
}

function set_hugepages {
    for large_page in ${1}; do
        if [[ -n ${large_page} ]]; then
            echo "Setting page: ${large_page}"
            /bin/sudo bash -c "echo 9216 > /sys/kernel/mm/hugepages/hugepages-${large_page}kB/nr_hugepages"
        fi
    done
}

declare -a build_dirs=(
    "${HOME}/workspaces/8377769_org/build/fastdebug"
    "${HOME}/workspaces/8377769/build/fastdebug"
)

declare -a large_page_lists=(
    "2048"
    ""
    "64"
)

clear_hugepages
for build_dir in "${build_dirs[@]}"; do
    echo compiling: "${build_dir}"
    silent make -C "${build_dir}"
done

for large_page_list in "${large_page_lists[@]}"; do
    echo Clear all pages
    clear_hugepages
    echo Set these pages: "${large_page_list}"
