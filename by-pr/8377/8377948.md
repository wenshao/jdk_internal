## 描述

We encountered a crash during VM exit due to the use of PerfTraceTime in ThreadWXEnable. As noted in perfData.hpp if a PerfCounter can be used concurrently with the VM shutdown safepoint then extra steps need to be taken to ensure its use is safe. ThreadWXEnable did not take those steps so I have introduced a SafePerfTraceTime class for it to use.

There remains the issue that the code is not strictly signal-handler safe - though we have a lot of such code that we typically manage to use safely, the expectation is that it would be rare to use the code when handling a signal. In the case of ThreadWXEnable however, this code will always be executed by the signal handler in non-product mode, so may still be too unsafe to keep. In which case I will just delete the PerfCounter that was added for ThreadWXEnable.

@theRealAph your review of this would be appreciated given you introduced the code.

Testing:
- all builds
- tiers 1-3 macOS-aarch64 sanity

There is no regression test.

Thanks
