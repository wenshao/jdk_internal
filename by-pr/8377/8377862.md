## 描述

There is a number of compiler tests that cannot run if a JVMTI agent is present.

The TestFramework (ir_framework) is able to detect this and throws `jtreg.SkippedException` ([related code](https://github.com/openjdk/jdk/commit/a70521c62e0841895d71cce2c872bd12f1183e33#diff-80aa3e27f1910b885230c9d2ca220cfbc5c1eaa29c46243f77dfee4a87db38feR777-R780)), however jtreg doesn't recognize this and does not mark the test as skipped, because the `jtreg.SkippedExeption` get wrapped into `compiler.lib.compile_framework.CompileFrameworkException`.

This fix unwraps and throws `jtreg.SkippedException` allowing jtreg to detect and gracefully skip tests that are unable to run due to internal checks (please see [JDK-8377862](https://bugs.openjdk.org/browse/JDK-8377862) for details).

Tested by running `jtreg:test/hotspot/jtreg/compiler` tests on macos-aarch64 with Async Profiler agent and without. Running tests with an agent produced only one expected failure (covered by another bug, [JDK-8377729](https://bugs.openjdk.org/browse/JDK-8377729)) and lot of skipped tests as expected:

```
==============================
Test summary
==============================
   TEST                                              TOTAL  PASS  FAIL ERROR  SKIP
>> jtreg:test/hotspot/jtreg/compiler                  2216  1640     1     0   575 <<
==============================
```

A run without an agent:
```
==============================
Test summary
==============================
   TEST                                              TOTAL  PASS  FAIL ERROR  SKIP
   jtreg:test/hotspot/jtreg/compiler                  2216  2028     0     0   188
==============================
```
