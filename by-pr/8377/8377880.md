## 描述

Seems we have warnings (potentially as error) for unused functions when using gcc and clang configured in the autoconf files. But for clang those warnings are disabled for hotspot.
It would be better to have them enabled for hotspot with gcc AND clang, so that developers working mostly on macOS cannot introduce those unused functions and destroy the Linux/gcc builds.
