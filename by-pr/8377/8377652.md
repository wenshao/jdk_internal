## 描述

Hi,

This PR fixes an issue with `Compile::flatten_alias_type` while dealing with loading from the array header of a constant array:

    if( ta->const_oop() ) { // MethodData* or Method*
      offset = Type::OffsetBot;   // Flatten constant access into array body
      tj = ta = ta->
              remove_speculative()->
              cast_to_ptr_type(ptr)->
              cast_to_exactness(false)->
              with_offset(offset);
    }

This sets the `offset` to `Type::OffsetBot` regardless it may actually be an offset into the array header. I found this while working on [JDK-8037084](https://bugs.openjdk.org/browse/JDK-8037084). I took the simplification in this section from Valhalla, so that this piece becomes much easier to understand. I will explain the equivalence in more details below.

Testing:

Please take a look and leave your reviews, thanks a lot.
