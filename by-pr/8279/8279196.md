## 描述

Hi all,

Test jdk/jfr/event/gc/stacktrace/TestG1OldAllocationPendingStackTrace.java was observed timed out fails intermittently. In the do-while loop in `allocAndCheck(GarbageCollectorMXBean, MemoryAllocator, String[])` method, it allocate 10KB heap memory everytime and will trigger fullgc after a long time. But sometimes the "Attempting full compaction" full gc do not trigger before test timed out. The GarbageCollectorMXBean object bean seted as 'G1 Young Generation', so only full gc will make the bead.getCollectionCount() increased to non-zero. The max heap memory 128m is too big to trigger g1 full gc.

This PR reduce the max heap memory from 128m to 64m, to make the "Attempting full compaction" full gc trigger steady.

Additional testing:

- [x] run the test 10k times on linux-x64
- [x] run the test 10k times on linux-aarch64
