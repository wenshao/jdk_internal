## 背景

`OctalDigits` 类是 JDK 内部用于八进制数字处理的工具类。该类原本为 StringTemplate 功能（JEP 430/455）设计，但在 StringTemplate 从 JDK 移除后，`OctalDigits` 类变得不再被使用。

**为什么重要？**
- 死代码增加维护负担
- 增加 JDK 体积（虽然很小）
- 可能误导未来的开发者尝试使用它

**改动位置**
```
src/java.base/share/classes/jdk/internal/util/OctalDigits.java  ← 整个文件删除
```

---

## 问题分析

### OctalDigits 类的历史

```
JEP 430: String Templates (Preview)
    └── 引入 StringTemplate 功能
        └── 需要 OctalDigits 类

JEP 455: String Templates (Second Preview)
    └── 继续 StringTemplate 工作
        └── OctalDigits 仍在使用

String Templates 功能被移除
    └── 相关代码清理
        └── OctalDigits 变成死代码

JDK-8348898: 此 PR
    └── 移除 OctalDigits 类
```

### 原始实现（被删除）

```java
// 文件: jdk/internal/util/OctalDigits.java
// 约 130 行代码

package jdk.internal.util;

/**
 * Digits class for octal digits.
 *
 * @since 21
 */
public final class OctalDigits {
    // 八进制数字查找表
    @Stable
    private static final short[] DIGITS;

    static {
        short[] digits = new short[64];
        // 初始化八进制数字对查找表
        for (int i = 0; i < 8; i++) {
            short hi = (short) (i + '0');
            for (int j = 0; j < 8; j++) {
                short lo = (short) ((j + '0') << 8);
                digits[i * 8 + j] = (short) (hi | lo);
            }
        }
        DIGITS = digits;
    }

    // 各种八进制转换方法...
}
```

### 为什么这个类不再被使用？

**搜索结果显示无引用**：

```bash
# 在 JDK 源码中搜索 OctalDigits 的引用
$ grep -r "OctalDigits" src/java.base/
# 结果：仅在 OctalDigits.java 本身中，无其他引用
```

**相关引用在 JavaDoc 中**：

```java
// 在 Integer.java 等类的 JavaDoc 中
/**
 * <i>DecimalNumeral</i>, <i>HexDigits</i>, and <i>OctalDigits</i>

```

这里的 `OctalDigits` 是指**JLS 中的语法术语**，不是指 `OctalDigits` 类。

## 设计决策

### Q1: 为什么不保留 OctalDigits 以备将来使用？

> **Reviewer**: 将来可能需要八进制转换，为什么不保留这个工具类？

**A**:

作者回复：

> "OctalDigits 类现在未被使用，移除它可以减少代码维护负担。如果将来需要，可以从 Git 历史中恢复。"

考虑因素：
- **YAGNI 原则**（You Aren't Gonna Need It）：不要保留不需要的代码
- **维护成本**：每个类都需要维护、编译、测试
- **Git 历史**：代码不会被真正删除，可以随时恢复
- **更好的替代**：如果将来需要，可以用更新的设计模式重写

### Q2: JavaDoc 中的 OctalDigits 术语是否需要修改？

> **Reviewer**: Integer.parseInt() 的 JavaDoc 中提到 OctalDigits，是否需要更新？

**A**:

不需要。JavaDoc 中的 `OctalDigits` 是指 JLS（Java Language Specification）中定义的**语法术语**，不是指 `OctalDigits` 类：

```java
/**
 * <i>DecimalNumeral</i>, <i>HexDigits</i>, and <i>OctalDigits</i>
 * are as defined in section {@jls 3.10.1} of
 * <cite>The Java Language Specification</cite>...
 *
 * <i>DecodableString:</i>
 * <dd><i>Sign<sub>opt</sub></i> {@code 0} <i>OctalDigits</i>
 */
```

这里描述的是 Java 源代码的**语法**（如 `0123` 这样的八进制字面量），不是指运行时的类。

### Q3: 为什么不使用 @Deprecated 先标记？

> **Reviewer**: 应该先标记为 @Deprecated，然后在下一个版本删除吗？

**A**:

不需要，因为：

1. **内部类**：`jdk.internal.util` 不是公开 API
2. **无使用者**：确认没有代码使用它
3. **立即生效**：删除立即减少维护负担

对于公开 API，确实应该先 `@Deprecated`，但对于内部死代码，直接删除是标准做法。
