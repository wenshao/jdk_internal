## 描述

The test fails with SIGSEGV because the freed memory is re-used by other threads and the second free in the test invalidates the memory used by them. Even without SIGSEGV case, the re-used memory is seen as valid by NMT. 
In addition, NMT does not report specifically about the "double free failure" and complains only about unexpected header/footer contents of a memory that is being freed.
So, since the "double free detection" is neither guaranteed nor reported by NMT, this test is removed.
The code coverage of the memory-corruption-detection functionality in NMT is preserved, which means that the code used in implementing the functionality is still covered by other tests even after removing this test.
 
Local macosx-aarch64 tests passed.
