## 描述

**Problem:**

C2 block-local scheduling can move a CheckCastPP with a RawPtr input below a following safepoint when -XX:TrackedInitializationLimit=0 or when the number of initialized fields exceeds the tracked initialization limit. This leaves a raw oop live at the safepoint instead of the casted oop being recorded in the OopMap, triggering the corresponding OopMap assertion. This issue has only been observed on AArch64 so far.

**Solution:**

Add a precedence edge from the following safepoint to a CheckCastPP whose input is RawPtr during local scheduling. This prevents the scheduler from moving such a CheckCastPP below the safepoint and preserves the expected ordering for OopMap construction.
