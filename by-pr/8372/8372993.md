## 描述

Hello,

`_max_eden_size` does not represent the actual upper-limit of how large eden might grow. The upper-limit was changed in [JDK-8368740](https://bugs.openjdk.org/browse/JDK-8368740), so that eden might now grow to be almost as large as the entire young generation. We run into an exception when trying to use this value through the MemoryUsage Java API. If the committed memory in eden is larger than `_max_eden_size`, we throw an exception in the Java constructor of MemoryUsage: https://github.com/openjdk/jdk/blob/master/src/java.management/share/classes/java/lang/management/MemoryUsage.java#L165-L168
