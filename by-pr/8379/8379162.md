## 描述

The issue hasd been observed when testing with fastdebug builds.

The test was not configuring the HTTP/3 request with HTTP_3_URI_ONLY, although it uses an HTTP/3 only server. As a result, the client can attempt to fallback to TCP after a reasonable (and short) timeout if the QUIC handshake is not fast enough. However since the server is HTTP/3 only there's no TCP server to fallback to.

Configuring the HTTP/3 request with HTTP_3_URI_ONLY ensures that there's no TCP fallback, and that the HTTP/3 connection attempt will make use of the full default QUIC timeout.
