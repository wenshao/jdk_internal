## 描述

Add a missing notification entry in `PhaseIterGVN::add_users_of_use_to_worklist` so that                  
  `URShiftINode::Ideal`'s `((x << z) + y) >>> z` optimization is not missed during IGVN:                        
                                                                                                                                                                              
  - When a `LShiftI`/`LShiftL` input changes, iterate over its `AddI`/`AddL` users and push 
  their `URShiftI`/`URShiftL` users to the worklist.                                                            
   
  Without this notification, `URShiftI` never gets back onto the worklist after the graph reaches the         
  optimizable shape, and the optimization is silently missed (caught by `-XX:VerifyIterativeGVN=1110`).
                                                                                                                
  This case was identified in the review thread of https://github.com/openjdk/jdk/pull/30143 (JDK-8379460).     
   
  ## Testing                                                                                                    
                  
  - Extended `MissedURShiftIAddILShiftIdeal.java` (JDK-8379460) with two new test cases `testIComputedX` /      
  `testLComputedX` covering the LShift-input-changes scenario (Case 2).
  - Ran the test with `-XX:VerifyIterativeGVN=1110` on a fastdebug build — all 8 test methods pass.
