## 描述

The test was implicitly relying on the `java.management` module but we were running it in a test task that specifically used `--limit-modules` to exclude the `java.management` module, hence the test failed with the `NoClassDefFoundError`.

The fix is to explicitly add the dependency on the `java.management` module, so that now the test is skipped when run in the --limit-modules test task.

Tested in our tier5 testing and locally.
