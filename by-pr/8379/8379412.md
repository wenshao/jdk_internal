## 描述

Hello,

When running testing in an environment that throws out of memory often, I encountered that `JfrJavaSupport::new_string` does not return early if there's a pending exception. In case of a pending exception, result is likely nullptr, which gets passed to `local_jni_handle`, which in a downstream call checks in an assert that the oop is in the heap via `Universe::heap()->is_in(...)`, which fails.

I suggest we should make this more robust by adding a CHECK to `JfrJavaSupport::new_string` so that we don't call into `local_jni_handle` if we have a pending exception.

Testing:
* open/test/jdk/:jdk_jfr locally on my Mac
* Oracle's tier 1-3
