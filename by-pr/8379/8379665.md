## 描述

Hello,

This PR obsoletes the AlwaysActAsServerClassMachine and NeverActAsServerClassMachine flags after being deprecated in JDK26. Removing the code is straightfoward in most places, and the tests where these flag(s) have been used only had them to support Windows 32-bit, which has been removed.

I've removed test/hotspot/jtreg/gc/arguments/TestSelectDefaultGC.java since it relies solely on the use of the now removed flags. Another alternative is to rewrite the test to detect what environment we're in (server/client) and see that we choose Serial or G1 correctly. With the upcoming G1 "one true default" JEP, that test is only transient, so I think we can just leave it out.

I feel strongly that the next step is to remove the code for the client emulation mode. With that gone we can simplify GC arguments-parsing code to not rely on a artificially lower memory constraint.

Testing:
