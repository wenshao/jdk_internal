## 描述

Hi,

It is generally not safe to call `transform` on the result of another call to `transform`. It is because the first call can return a `Phi`, which may not have been initialized properly yet. So, calling `transform` will result in an incorrect result. This PR fixes this issue in the C2 implementation of some intrinsics.

Testing:

- [x] tier1,tier2,tier3,tier4,hs-comp-stress

Please kindly review, thanks a lot.
