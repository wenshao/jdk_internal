## 描述

Could I have a review of a PR that prevents incorrect minimum values for the MethodTiming event? 

Due to stale reads when using a plain read, and with the initial minimum set to Long.MAX_VALUE, we must always execute the minimum check. 

This can be fixed by breaking out of the loop instead of returning. The additional cost of an if check is negligible, since the number of times a new maximum is found is likely small (<100).

Testing: test/jdk/jdk/jfr

Thanks
Erik
