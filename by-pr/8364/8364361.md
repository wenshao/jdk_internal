## 背景

### 问题：Process 资源清理复杂

**JDK-8364361 之前**，清理 Process 需要手动关闭所有流：

```java
ProcessBuilder pb = new ProcessBuilder("command", "arg");
Process p = pb.start();

try {
    // 使用进程
} finally {
    // 需要手动关闭所有流
    p.getInputStream().close();
    p.getErrorStream().close();
    p.getOutputStream().close();
    p.destroy();  // 确保进程终止
}
```

### 期望：使用 try-with-resources

```java
try (Process p = pb.start()) {
    // 自动清理所有资源
}
```

## 核心变更

### 1. Process 实现 AutoCloseable

**变更前**:
```java
public abstract class Process {
    // ...
}

// Process 不是 Closeable 或 AutoCloseable
```

**变更后**:
```java
public abstract class Process implements Comparable<Process>, Closeable {
    // ...

    /**
     * 关闭进程并清理所有流
     */
    @Override
    public void close() throws IOException {
        // 实现 Closeable 接口
    }
}
```

### 2. close() 方法实现

```java
// Process.java

@Override
public void close() throws IOException {
    // 1. 关闭所有流
    getInputStream().close();
    getErrorStream().close();
    getOutputStream().close();

    // 2. 如果进程仍在运行，销毁它
    if (isAlive()) {
        destroy();
    }
}
```

### 3. ProcessImpl 适配

```java
// ProcessImpl.java

@Override
public void close() throws IOException {
    closeDescriptors(new IOException("Process closed"));
}

private void closeDescriptors(IOException cause) throws IOException {
    // 关闭所有文件描述符
    closeInputStream(stdin_stream);
    closeOutputStream(stdout_stream);
    closeOutputStream(stderr_stream);

    // 等待进程终止
    terminateProcess(cause);
}
```

## 使用示例

### 之前的手动清理

```java
ProcessBuilder pb = new ProcessBuilder("ls", "-l");
Process p = pb.start();

BufferedReader reader = null;
try {
    reader = new BufferedReader(new InputStreamReader(p.getInputStream()));
    String line;
    while ((line = reader.readLine()) != null) {
        System.out.println(line);
    }
} finally {
    if (reader != null) {
        reader.close();
    }
    // 必须手动关闭进程的所有流
    p.getInputStream().close();
    p.getErrorStream().close();
    p.getOutputStream().close();
    if (p.isAlive()) {
        p.destroy();
    }
}
```

### 使用 try-with-resources

```java
ProcessBuilder pb = new ProcessBuilder("ls", "-l");
try (Process p = pb.start()) {
    // 自动清理所有资源
    BufferedReader reader = new BufferedReader(
        new InputStreamReader(p.getInputStream()));
    reader.lines().forEach(System.out::println);
}
// 自动调用 p.close()
```

### 管理多个进程

```java
// 并行执行多个进程
List<Process> processes = List.of(
    pb1.start(),
    pb2.start(),
    pb3.start()
);

// 使用 try-with-resources
try (var p1 = processes.get(0);
     var p2 = processes.get(1);
     var p3 = processes.get(2)) {
    // 使用进程
    p1.waitFor();
    p2.waitFor();
    p3.waitFor();
}
// 所有进程自动清理
```

## 技术分析

### Closeable vs AutoCloseable

| 接口 | 用途 |
|------|------|
| **Closeable** | 基础接口，close() 方法 |
| **AutoCloseable** | 继承 Closeable，用于 try-with-resources |

Process 实现 `Closeable` 就足够，因为：

```java
// Closeable 也支持 try-with-resources
public interface Closeable extends AutoCloseable {
    void close() throws IOException;
}
```

### 资源清理顺序

```
1. 关闭输入流 (getInputStream)
2. 关闭错误流 (getErrorStream)
3. 关闭输出流 (getOutputStream)
4. 如果进程存活，销毁进程 (destroy)
```

### Platform差异

| 平台 | 实现类 | 特殊处理 |
|------|--------|----------|
| **Unix** | UnixProcess | 关闭文件描述符 |
| **Windows** | WinProcess | 等待进程句柄 |
