## 背景

### JDK-8267332 引入的问题

**JDK-8267332** 添加了 XOR 操作的有界值优化：

```cpp
// XorINode::Value() - JDK-8267332 之后
const Type* XorINode::Value(PhaseGVN* phase) const {
    // 首先尝试有界值优化
    const Type* t1 = in(1)->Value(phase);
    const Type* t2 = in(2)->Value(phase);

    if (t1->isa_int() && t2->isa_int()) {
        // 计算有界值
        const TypeInt* i1 = t1->is_int();
        const TypeInt* i2 = t2->is_int();

        // 添加边界信息
        return TypeInt::make(i1->_lo ^ i2->_lo, i1->_hi ^ i2->_hi);
    }

    // ❌ 问题：有界值优化优先，常量折叠被跳过！
}
```

### 问题场景

```java
// 示例代码
int x = a ^ 0xFFFFFFFF;  // 应该优化为 ~a
int y = b ^ 0;           // 应该优化为 b

// 但由于有界值优化优先
// 这些常量折叠可能不会发生
```

## 核心变更

### 优化优先级调整

**变更前**:
```cpp
const Type* XorINode::Value(PhaseGVN* phase) const {
    // 1. 有界值优化 (优先执行)
    const Type* t1 = in(1)->Value(phase);
    const Type* t2 = in(2)->Value(phase);

    if (can_compute_bounded_value(t1, t2)) {
        return compute_bounded_value(t1, t2);
    }

    // 2. 常量折叠 (可能不会执行)
    return fold_constants(phase);
}
```

**变更后**:
```cpp
const Type* XorINode::Value(PhaseGVN* phase) const {
    // 1. 先尝试常量折叠
    const Type* t = fold_constants_if_possible(phase);
    if (t != nullptr) {
        return t;  // ✅ 常量折叠优先
    }

    // 2. 只有常量折叠不适用时，才用有界值优化
    return compute_bounded_value(phase);
}
```

### 具体实现

```cpp
// XorINode::Value() - 优化后
const Type* XorINode::Value(PhaseGVN* phase) const {
    const Type* t1 = in(1)->bottom_type();
    const Type* t2 = in(2)->bottom_type();

    // 1. 常量 XOR 常量 = 常量
    if (t1->is_int() && t1->is_int()->is_con() &&
        t2->is_int() && t2->is_int()->is_con()) {
        jint lo = t1->is_int()->get_con() ^ t2->is_int()->get_con();
        return TypeInt::make(lo);
    }

    // 2. XOR 0 = 恒等
    if (t2->is_int() && t2->is_int()->is_con() &&
        t2->is_int()->get_con() == 0) {
        return t1;  // a ^ 0 = a
    }

    // 3. XOR -1 = 按位取反
    if (t2->is_int() && t2->is_int()->is_con() &&
        t2->is_int()->get_con() == -1) {
        // a ^ -1 = ~a (需要类型调整)
        return t1->is_int()->xneg_id();
    }

    // 4. 有界值优化 (回退选项)
    return bounded_value_optimization(phase);
}
```

## 技术分析

### 常量折叠 vs 有界值优化

| 优化类型 | 例子 | 结果 | 精度 |
|----------|------|------|------|
| **常量折叠** | `a ^ 0xFFFFFFFF` | `~a` | 精确 |
| **有界值** | `a ^ [0, 0xFFFFFFFF]` | `[-∞, +∞]` | 保守 |

### 优化时机

```
GCC (C 编译器):
1. 常量传播
2. 常量折叠
3. 有界值分析

C2 (JDK-8267332):
1. 有界值优化 (❌ 太早)
2. 常量折叠 (可能跳过)

C2 (JDK-8347645 修复后):
1. 常量折叠 (✅ 优先)
2. 有界值优化 (回退)
```

### 性能影响

| 场景 | 修复前 | 修复后 |
|------|--------|--------|
| **常量 XOR** | 保守优化 | 完全折叠 |
| **变量 XOR** | 有界值 | 有界值 |
| **混合** | 部分优化 | 更优 |

## 总结

JDK-8347645 通过调整优化优先级：

1. **恢复常量折叠** - XOR 常量操作可以完全优化
2. **有界值作为回退** - 只在常量折叠不适用时使用
3. **更精确的类型** - 常量折叠提供更精确的结果
4. **修复 JDK-8267332 的副作用** - 平衡两种优化

这是 Johannes Graham (j3graham) 对 C2 编译器优化的重要修复。
