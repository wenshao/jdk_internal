## 描述

Running tests for JShell in the CI, `NullPointerException` is occasionally thrown from the `ConsoleIOContext.countTrailintBackslashes(...)` method.

The problem is concurrent access to the JShell history lines stored in the global Preferences. As the Preferences API does not allow for atomic reading of the keys and their values, JShell reads the keys first and then the value for each of the keys. And if some other process (like another test using JShell) overwrites the data in the global Preferences in the meantime, problem arises.

The proposal here is:
- Modify the existing `TerminalNoExecTest` that still uses the global Preferences to use an explicit test-specific persistence (so that the test is not affected by anything that is written in the global Preferences).
- While loading history on JShell startup, ignore the keys for which do not exist values anymore in the global Preference due to their concurrent modification done by another process.
