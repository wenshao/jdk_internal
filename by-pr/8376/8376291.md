## 描述

This is the next patch in the series to replace the hotspot-pid-file-based IR dumps with socket communication (see [JDK-8375271](https://bugs.openjdk.org/browse/JDK-8375271)).

This patch introduces new message classes for different `MessageTag` uses which can then directly by printed with `print()` instead of using `processOutput()`. They all implement a new `JavaMessage` interface.
- `StdoutMessages` for `MessageTag.STDOUT`.
- `ExecutedTests` for `MessageTag.TEST_LIST`.
- `MethodTimes` for `MessageTag.PRINT_TIMES`.

I included some additional small clean-ups (see GitHub code comments).
