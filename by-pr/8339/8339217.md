## 背景

`CodeBuilder.loadConstant()` 是 ClassFile API 中用于生成常量加载指令的核心方法。它根据常量类型和值选择最优的字节码指令：

```
常量值          →  字节码指令
----------------------------------------
int -1          →  iconst_m1
int 0           →  iconst_0
int 1-5         →  iconst_1/2/3/4/5
int byte 范围   →  bipush
int short 范围  →  sipush
其他            →  ldc

long 0          →  lconst_0
long 1          →  lconst_1
其他            →  ldc

float 0.0       →  fconst_0
float 1.0       →  fconst_1
float 2.0       →  fconst_2
其他            →  ldc

double 0.0      →  dconst_0
double 1.0      →  dconst_1
其他            →  ldc
```

### 改动位置

```
src/java.base/share/classes/

    │   └── CodeBuilder.java  ← 新增 4 个重载方法
    └── java/lang/invoke/
```

## 问题分析

### 原始实现

```java
// 文件: CodeBuilder.java (Before)
// 单一的大型 loadConstant 方法

default CodeBuilder loadConstant(ConstantDesc value) {
    if (value == null || value == ConstantDescs.NULL)
        return aconst_null();
    if (value instanceof Number) {
        if (value instanceof Integer iVal)
            return switch (iVal) {
                case -1 -> iconst_m1();
                case 0 -> iconst_0();
                case 1 -> iconst_1();
                case 2 -> iconst_2();
                case 3 -> iconst_3();
                case 4 -> iconst_4();
                case 5 -> iconst_5();
                default -> (iVal >= Byte.MIN_VALUE && iVal <= Byte.MAX_VALUE) ? bipush(iVal)
                        : (iVal >= Short.MIN_VALUE && iVal <= Short.MAX_VALUE) ? sipush(iVal)
                        : ldc(constantPool().intEntry(iVal));
            };
        if (value instanceof Long lVal)
            return lVal == 0L ? lconst_0()
                    : lVal == 1L ? lconst_1()
                    : ldc(constantPool().longEntry(lVal));
        if (value instanceof Float fVal)
            return Float.floatToRawIntBits(fVal) == 0 ? fconst_0()
                    : fVal == 1.0f ? fconst_1()
                    : fVal == 2.0f ? fconst_2()
                    : ldc(constantPool().floatEntry(fVal));
        if (value instanceof Double dVal)
            return Double.doubleToRawLongBits(dVal) == 0L ? dconst_0()
                    : dVal == 1.0d ? dconst_1()
                    : ldc(constantPool().doubleEntry(dVal));
    }
    return ldc(value);
}
```

### 性能瓶颈分析

| 瓶颈点 | 原因 | 影响 |
|--------|------|------|
| **方法过大** | 方法约 465 字节 | JIT 无法内联 (`callee is too large`) |
| **类型检查链** | `instanceof` + 强制类型转换 | 分支预测失败 |
| **多次 `instanceof`** | Integer/Long/Float/Double 顺序检查 | CPU 流水线停顿 |
| **开关表达式** | switch 嵌套在 instanceof 中 | 代码复杂度高 |

### JIT 编译器日志

```
java.lang.classfile.CodeBuilder::loadConstant (465 bytes)
   callee is too large
```

**问题**: 465 字节远超 HotSpot 的内联阈值 (~325 字节)

## 解决方案

### 核心改动

#### 方法重载替代类型检查

```java
// 文件: CodeBuilder.java (After)

// 原有方法保留，但简化为分发逻辑
default CodeBuilder loadConstant(ConstantDesc value) {
    if (value == null || value == ConstantDescs.NULL)
        return aconst_null();
    if (value instanceof Number) {
        // 使用方法重载自动分发
        if (value instanceof Integer) return loadConstant((int)    value);
        if (value instanceof Long   ) return loadConstant((long)   value);
        if (value instanceof Float  ) return loadConstant((float)  value);
        if (value instanceof Double ) return loadConstant((double) value);
    }
    return ldc(value);
}

// 新增: int 专用方法 (小而简单，可内联)
default CodeBuilder loadConstant(int value) {
    return switch (value) {
        case -1 -> iconst_m1();
        case  0 -> iconst_0();
        case  1 -> iconst_1();
        case  2 -> iconst_2();
        case  3 -> iconst_3();
        case  4 -> iconst_4();
        case  5 -> iconst_5();
        default -> (value >= Byte.MIN_VALUE  && value <= Byte.MAX_VALUE ) ? bipush(value)
                 : (value >= Short.MIN_VALUE && value <= Short.MAX_VALUE) ? sipush(value)
                 : ldc(constantPool().intEntry(value));
    };
}

// 新增: long 专用方法 (小而简单，可内联)
default CodeBuilder loadConstant(long value) {
    return value == 0l ? lconst_0()
         : value == 1l ? lconst_1()
         : ldc(constantPool().longEntry(value));
}

// 新增: float 专用方法 (小而简单，可内联)
default CodeBuilder loadConstant(float value) {
    return Float.floatToRawIntBits(value) == 0 ? fconst_0()
         : value == 1.0f                       ? fconst_1()
         : value == 2.0f                       ? fconst_2()
         : ldc(constantPool().floatEntry(value));
}

// 新增: double 专用方法 (小而简单，可内联)
default CodeBuilder loadConstant(double value) {
    return Double.doubleToRawLongBits(value) == 0l ? dconst_0()
         : value == 1.0d                           ? dconst_1()
         : ldc(constantPool().doubleEntry(value));
}
```

### 调用流程对比

```
原始流程:
┌─────────────────────────────────────────────────────────┐
│ loadConstant(ConstantDesc)                             │
│   │                                                     │
│   ├── instanceof Integer → switch → iconst_X           │
│   ├── instanceof Long   → compare → lconst_X/ldc       │
│   ├── instanceof Float  → compare → fconst_X/ldc       │
│   └── instanceof Double → compare → dconst_X/ldc       │
│                                                         │
│ 问题: 465 字节，无法内联                                │
└─────────────────────────────────────────────────────────┘

优化后流程:
┌─────────────────────────────────────────────────────────┐
│ loadConstant(ConstantDesc)  // ~80 字节，可内联        │
│   │                                                     │
│   ├── instanceof Integer  ─┐                           │
│   ├── instanceof Long    ──┼──> 自动分发到类型特定方法  │
│   ├── instanceof Float   ─┤   (静态分派)               │
│   └── instanceof Double ─┘                            │
│                                                         │
│ loadConstant(int)      // ~60 字节，可内联             │
│ loadConstant(long)     // ~30 字节，可内联             │
│ loadConstant(float)    // ~40 字节，可内联             │
│ loadConstant(double)   // ~30 字节，可内联             │
└─────────────────────────────────────────────────────────┘
```

### 使用示例

```java
// Before: 只能使用 ConstantDesc
CodeBuilder cb = ...;
cb.loadConstant(Integer.valueOf(42));  // 创建 Integer 对象
cb.loadConstant(Long.valueOf(100L));   // 创建 Long 对象

// After: 可以直接使用基本类型
cb.loadConstant(42);      // 直接使用 int，无需装箱
cb.loadConstant(100L);    // 直接使用 long，无需装箱
cb.loadConstant(3.14f);   // 直接使用 float，无需装箱
cb.loadConstant(2.718);   // 直接使用 double，无需装箱

// 原有用法仍然兼容
cb.loadConstant(Integer.valueOf(42));  // 仍然有效
```

### 为什么需要 CSR？

由于新增了公共 API 方法，需要通过 CSR (Compatibility and Specification Review) 审批：

- **新增方法**: `loadConstant(int/long/float/double)`
- **修改包**: `java.lang.classfile` (公共 API)
- **影响版本**: Java 24 (`@since 24`)

## 设计决策

### Q1: 为什么不直接修改原方法？

> **Reviewer**: 为什么不把原方法改小，而是新增方法？

**A**:
- **向后兼容**: 原有 `loadConstant(ConstantDesc)` 被广泛使用
- **API 设计**: 类型特定的重载是 Java 常见模式
- **性能**: 方法重载在编译时确定，无运行时开销

### Q2: 为什么不使用 `@HotSpotIntrinsicCandidate`？

> **Reviewer**: 这些方法是否可以声明为 intrinsic？

**A**:
- **不需要**: 方法已经足够简单，JIT 可以很好地优化
- **平台依赖**: intrinsic 需要为每个平台实现
- **维护成本**: intrinsic 增加维护负担

### Q3: 为什么保留 `loadConstant(ConstantDesc)`？

> **Reviewer**: 为什么不弃用原方法？

**A**:
- **通用性**: `ConstantDesc` 可以表示所有类型的常量
- **动态场景**: 运行时类型不确定时需要
- **兼容性**: 现有代码依赖此方法

## 性能对比

### Benchmark 方法

```java
@BenchmarkMode(Mode.AverageTime)
@OutputTimeUnit(TimeUnit.NANOSECONDS)
@State(Scope.Thread)
public class LoadConstantBench {

    CodeBuilder cb;

    @Setup
    public void setup() {
        ConstantPoolBuilder cp = ConstantPoolBuilder.of();
        cb = ClassFile.of().build(ClassDesc.of("Test"), cp, clb -> {
            clb.withMethod("test", MethodTypeDesc.of(CD_void),
                ClassFile.ACC_PUBLIC, mb -> {
                    mb.withCode(cob -> cob.return_());
                });
        }).get();
    }

    @Benchmark
    public void loadIntConstants() {
        for (int i = -10; i < 100; i++) {
            cb.loadConstant(i);  // 使用新的重载方法
        }
    }

    @Benchmark
    public void loadDoubleConstants() {
        for (int i = 0; i < 100; i++) {
            cb.loadConstant(i * 1.5);  // 使用新的重载方法
        }
    }
}
```

| **long 常量** | 680 | 610 | **+10.3%** | 消除装箱，内联优化 |

### 性能提升分解

```

├── 静态分派:         ~2%   (编译时确定类型)
└── 开关表达式优化:   ~1%   (更简单的 switch)

double 常量场景 (+10.0%):
├── JIT 内联:         ~6%   (小方法易内联)
├── 消除装箱:         ~3%   (无需 Double.valueOf)

```
