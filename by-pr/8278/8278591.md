## 描述

Support "--win-with-ui" boolean option. If specified, jpackage will ensure the output MSI or EXE installer will have User Interface (UI). The UI is based on the standard "WixUI_Minimal" WiX dialog set with the WelcomeEulaDlg dialog removed from the sequence.

If any option that implicitly results in the installer with UI is specified ("--win-dir-chooser", "--win-shortcut-prompt", "--license-file"), the "--win-with-ui" option is ignored.

Code changes:
 - Refactor WixUiFragmentBuilder: move code constructing a model for WiX UI source code generation to classes in a new "jdk.jpackage.internal.wixui" package with unit test coverage.
 - Refactor WixPipeline and WixVariables; add unit tests for WixVariables.
 - Refactor WinInstallerUiTest to include coverage for the "--win-with-ui" option.

Additionally, fixed an issue in WinL10nTest.java that popped up when running the tests locally with different versions of WiX toolkit.
