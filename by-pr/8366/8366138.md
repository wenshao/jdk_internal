## 描述

`Parse::jump_switch_ranges()` can stack-overflow when compiling a large switch with all-zero profiling counts. The profiling pivot degenerates to `mid == lo` every level `(0 >= 0/2)`, producing `O(N)` recursion instead of `O(log N)`.

The fix overrides the pivot to `lo + nr/2` when `total_cnt == 0`. All paths are cold so balanced is strictly better. The lower-range recursion is also converted to an iterative loop as defense-in-depth.
