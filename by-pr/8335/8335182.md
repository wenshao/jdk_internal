## 背景

### 字符串拼接优化历史

Java 字符串拼接（`+` 操作符）在编译时会被转换为 `StringConcatFactory` 的调用，运行时生成优化的字节码。

```
源代码: "Hello" + name + "!"
    ↓ javac
字节码: invokedynamic "makeConcatWithConstants"
    ↓ StringConcatFactory
生成: MethodHandle 组合树
    ↓ C2 JIT
优化: 内联、常量折叠、字符串压缩
```

### 原有实现的问题

**JDK-8335182 之前**，字符串拼接生成的 MethodHandle 组合树存在以下问题：

1. **代码路径冗余**: 同时支持有前缀和无前缀两种 prepend 变体
2. **组合模式不一致**: 有时使用 append，有时使用 prepend
3. **C2 优化困难**: 复杂的组合树结构阻碍了 JIT 深度优化

```java
// 原实现 - 多种代码路径
prepend(indexCoder, buf, value)           // 无前缀版本
prepend(indexCoder, buf, value, prefix)   // 有前缀版本
prepend(indexCoder, buf, value)           // 然后再 prepend prefix
```

## 核心变更

### 设计原则变更

| 变更前 | 变更后 |
|--------|--------|
| 支持多种 prepend 变体 | **只使用带前缀的 prepend** |
| suffix 可选处理 | **始终在 newArray 添加 suffix** |
| 多次方法调用 | **合并为单次内联友好操作** |

### 核心代码变更

#### 1. StringConcatHelper.prepend 简化

**变更前**（无前缀变体）:
```java
// 需要多次调用
static long prepend(long indexCoder, byte[] buf, boolean value) {
    // ... 写入 "true" 或 "false"
}

static long prepend(long indexCoder, byte[] buf, String value) {
    // ... 写入字符串
}

// 有前缀版本需要组合调用
static long prepend(long indexCoder, byte[] buf, boolean value, String prefix) {
    indexCoder = prepend(indexCoder, buf, value);
    indexCoder = prepend(indexCoder, buf, prefix);
    return indexCoder;
}
```

**变更后**（统一为带前缀版本）:
```java
// 直接在一个方法中完成
static long prepend(long indexCoder, byte[] buf, boolean value, String prefix) {
    int index = (int)indexCoder;
    if (indexCoder < UTF16) {
        if (value) {
            index -= 4;
            buf[index] = 't';
            buf[index + 1] = 'r';
            buf[index + 2] = 'u';
            buf[index + 3] = 'e';
        } else {
            index -= 5;
            buf[index] = 'f';
            buf[index + 1] = 'a';
            buf[index + 2] = 'l';
            buf[index + 3] = 's';
            buf[index + 4] = 'e';
        }
        index -= prefix.length();
        prefix.getBytes(buf, index, String.LATIN1);
        return index;
    } else {
        // UTF16 类似逻辑
    }
}
```

**优化效果**:
- ✅ 减少方法调用次数
- ✅ 更好的内联潜力
- ✅ 单一代码路径维护

#### 2. 移除无前缀变体

**变更前**:
```java
private static MethodHandle noPrefixPrepender(Class<?> cl) {
    int idx = classIndex(cl);
    MethodHandle prepend = NO_PREFIX_PREPENDERS[idx];
    if (prepend == null) {
        NO_PREFIX_PREPENDERS[idx] = prepend = JLA.stringConcatHelper("prepend",
                methodType(long.class, long.class, byte[].class,
                        Wrapper.asPrimitiveType(cl))).rebind();
    }
    return prepend;
}
```

**变更后**:
```java
private static MethodHandle noPrefixPrepender(Class<?> cl) {
    int idx = classIndex(cl);
    MethodHandle prepend = NO_PREFIX_PREPENDERS[idx];
    if (prepend == null) {
        // 使用 insertArguments 将 prefix 设为空字符串
        NO_PREFIX_PREPENDERS[idx] = prepend = MethodHandles.insertArguments(prepender(cl), 3, "");
    }
    return prepend;
}
```

**效果**:
- ✅ 移除了 6 个无前缀方法
- ✅ 使用 `MethodHandles.insertArguments` 实现无前缀语义
- ✅ 统一代码路径

#### 3. 始终使用 suffix

**变更前**:
```java
if (suffix != null) {
    initialLengthCoder -= suffix.length();
    newArrayCombinator = newArrayWithSuffix(suffix);
} else {
    newArrayCombinator = newArray();  // 不同的代码路径
}
```

**变更后**:
```java
// 统一使用 suffix（空字符串也行）
if (suffix == null || suffix.isEmpty()) {
    suffix = "";
}
initialLengthCoder -= suffix.length();
newArrayCombinator = newArrayWithSuffix(suffix);
```

#### 4. 移除中间层方法

**移除的方法**:
```java
// System.java - 不再需要
public long stringConcatHelperPrepend(long indexCoder, byte[] buf, String value) {
    return StringConcatHelper.prepend(indexCoder, buf, value);
}

// JavaLangAccess.java - 移除声明
// StringConcatFactory.java - 移除 NEW_ARRAY 单独方法
```

#### 5. simpleConcat 优化

**变更前**:
```java
static String simpleConcat(Object first, Object second) {
    String s1 = first.toString();
    String s2 = second.toString();
    long indexCoder = mix(initialCoder(), s1);
    indexCoder = mix(indexCoder, s2);
    byte[] buf = newArray(indexCoder);
    indexCoder = prepend(indexCoder, buf, s2);  // 两次调用
    indexCoder = prepend(indexCoder, buf, s1);
    return newString(buf, indexCoder);
}
```

**变更后**:
```java
static String simpleConcat(Object first, Object second) {
    String s1 = first.toString();
    String s2 = second.toString();
    long indexCoder = mix(initialCoder(), s1);
    indexCoder = mix(indexCoder, s2);
    byte[] buf = newArray(indexCoder);
    indexCoder = prepend(indexCoder, buf, s2, s1);  // 一次调用，s1 作为前缀
    return newString(buf, indexCoder);
}
```

## 技术分析

### 为什么这样优化有效？

#### 1. 减少方法调用开销

```java
// 变更前：2 次方法调用
indexCoder = prepend(indexCoder, buf, s2);
indexCoder = prepend(indexCoder, buf, s1);

// 变更后：1 次方法调用
indexCoder = prepend(indexCoder, buf, s2, s1);
```

#### 2. 更好的 C2 内联

| 变更前 | 变更后 |
|--------|--------|
| 多个间接调用 | 单一直接调用 |
| 需要多次内联决策 | 一次内联决策 |
| 寄存器压力大 | 寄存器压力小 |

#### 3. 简化的控制流

```
变更前:
prepend(value) → 判断类型 → 选择方法 → 调用
              ↓
prepend(prefix) → 判断类型 → 选择方法 → 调用

变更后:
prepend(value, prefix) → 判断类型 → 写入 value → 写入 prefix → 返回
```

#### 4. 方法句柄组合优化

**变更前的组合树**:
```
filterArguments
  └── foldArguments(newArray)
      └── foldArguments(prepend(s2))
          └── foldArguments(prepend(s1))
              └── ...
```

**变更后的组合树**:
```
filterArguments
  └── foldArguments(newArrayWithSuffix)
      └── foldArguments(prepend(s2, s1))  // 合并操作
          └── ...
```
