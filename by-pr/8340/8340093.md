## 核心变更

### 成本模型原理

**传统启发式方法**：
```
if (reduction && !simple && num_elements != 2) {
    vectorize();  // 仅非简单、非2元素的归约
}
```

**成本模型方法**：
```
scalar_cost = count_scalar_ops();
vector_cost = count_vector_ops();
if (vector_cost < scalar_cost) {
    vectorize();  // 基于实际成本
}
```

### 新增组件

#### 1. VTransform::is_profitable

```cpp
bool VTransform::is_profitable() {
    // 检查成本模型
    if (!cost_model_is_profitable()) {
        return false;
    }

    // 其他成本相关检查
    if (has_expensive_shuffles()) {
        return false;
    }

    return true;
}
```

#### 2. VLoopAnalyzer::cost

计算标量循环的成本：

```cpp
int VLoopAnalyzer::cost() {
    int cost = 0;
    for (auto* node : loop_nodes) {
        cost += node->cost();  // 每个节点的成本
    }
    return cost;
}
```

#### 3. VTransformGraph::cost

计算向量化循环的成本：

```cpp
int VTransformGraph::cost() {
    int cost = 0;
    for (auto* vnode : vector_nodes) {
        cost += vnode->cost();  // 向量节点的成本
    }
    return cost;
}
```

### 操作成本

| 操作类型 | 标量成本 | 向量成本 | 说明 |
|----------|----------|----------|------|
| **基本算术** | 1 | 1 | 加减乘，通常 1:1 |
| **归约 (add/mul)** | 1 | 2 | 需要额外步骤 |
| **Shuffle** | 0 | 2-4 | 重新排列向量元素 |
| **扩展** | 1 | 1 | 标量到向量 |

## 技术分析

### 为什么需要成本模型？

**问题 1**: 向量操作不一定更快

```java
// 标量版本 (4次操作)
sum = a[0] + a[1] + a[2] + a[3];

// 向量版本
vsum = load(vector) + reduce(vsum);  // 1次load + 1次reduce
```

如果 `reduce` 操作昂贵，向量化可能更慢。

**问题 2**: Shuffle 代价高

```java
// 需要重新排列数据
shuffled = shuffle(vector);  // 可能需要 2-4 个周期
```

### 成本计算

```
标量循环成本 = Σ(每个标量节点的成本)
向量化循环成本 = Σ(每个向量节点的成本) + 额外开销

如果 向量化成本 < 标量成本:
    允许向量化
```

### 移除的限制

**之前的限制**：
```cpp
// 禁用"简单"归约
if (is_simple_reduction && num_elements == 1) {
    return false;  // 不向量化
}
```

**现在**：
```cpp
// 使用成本模型判断
if (is_profitable()) {
    return true;  // 向量化
}
```
