## 描述

`VectorMaskToLongNode::Ideal_MaskAll` looks through `VectorStoreMask` to fold constant masks, but when a `VectorStoreMask` input changes, its `VectorMaskToLong` users aren't added to the IGVN worklist, so the folding never fires. This causes an assertion failure with `-XX:VerifyIterativeGVN=1110 -XX:+StressIncrementalInlining` in the tests mentioned in the bug.

The fix adds `VectorStoreMask` users to the worklist so downstream nodes get a chance to re-run `Ideal`.
