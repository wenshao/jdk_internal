## 背景

### SuperWord 的设计假设

**SuperWord** 优化假设：

```
高迭代次数循环:
├── 超展开主循环: 占 90% 时间
├── 预热/清理循环: 占 10% 时间
└── 向量化收益: 显著

小迭代次数循环:
├── 主循环: 可能只有 1-2 次迭代
├── 向量化开销: 可能超过收益
└── 向量化收益: 不确定
```

### 问题场景

```java
// 示例: 小循环
for (int i = 0; i < 8; i++) {  // 只有 8 次迭代
    result[i] = data[i] * 2;
}

// 向量化后 (假设 4 路):
for (int i = 0; i < 8; i += 4) {
    // 2 次向量操作
    vec_result = vec_data * 2;
}

// 问题:
// - 向量设置开销
// - 可能不需要的标量-向量转换
// - 实际可能比标量更慢
```

## 核心变更

### 成本模型改进

**变更前**:
```cpp
// superword.cpp
int SuperWord::estimate_profitability() {
    // 简单估计: 只考虑循环体大小
    int vector_cost = loop_body_size();
    int scalar_cost = loop_body_size() * iteration_count;

    return scalar_cost > vector_cost;  // 简化判断
}
```

**变更后**:
```cpp
// superword.cpp
int SuperWord::estimate_profitability() {
    int iteration_count = loop()->trip_count();

    // 小循环特殊处理
    if (iteration_count < UNROLLING_THRESHOLD) {
        return estimate_small_loop_profitability();
    }

    // 大循环使用原有逻辑
    return estimate_large_loop_profitability();
}

int SuperWord::estimate_small_loop_profitability() {
    // 考虑:
    // 1. 向量设置开销
    // 2. 标量-向量转换开销
    // 3. 实际计算量

    int vector_setup_cost = 10;  // 假设 10 个周期
    int scalar_cost = iteration_count * loop_body_size();
    int vector_cost = vector_setup_cost +
                     (iteration_count / vector_width()) * loop_body_size();

    return scalar_cost > vector_cost;
}
```

### 迭代次数阈值

```cpp
// 新增阈值定义
enum LoopSize {
    TINY = 4,      // 极小循环: 4 次迭代
    SMALL = 16,    // 小循环: 16 次迭代
    MEDIUM = 64,   // 中等循环: 64 次迭代
    LARGE = 256    // 大循环: 256 次迭代
};

// 决策逻辑
if (iteration_count < LoopSize::TINY) {
    return NEVER_VECTORIZE;
} else if (iteration_count < LoopSize::SMALL) {
    return CONSERVATIVE_VECTORIZE;
} else {
    return AGGRESSIVE_VECTORIZE;
}
```

## 技术分析

### 向量化开销分解

| 开销类型 | 周期数 | 说明 |
|----------|--------|------|
| **向量设置** | ~10 | 加载向量常量 |
| **标量转向量** | ~5 | 打包标量数据 |
| **向量转标量** | ~5 | 解包向量数据 |
| **向量计算** | 1/N | N 是向量宽度 |

### 小循环分析

```
8 次迭代的循环:

标量版本:
8 次操作 × 1 周期 = 8 周期

4 路向量化:
设置: 10 周期
打包: 5 周期
计算: 2 次向量操作 × 1 周期 = 2 周期
解包: 5 周期
总计: 22 周期

结论: 标量更快!
```

### 大循环分析

```
1000 次迭代的循环:

标量版本:
1000 次操作 × 1 周期 = 1000 周期

4 路向量化:
设置: 10 周期
打包: 5 周期
计算: 250 次向量操作 × 1 周期 = 250 周期
解包: 5 周期
总计: 270 周期

结论: 向量化快 3.7 倍!
```
