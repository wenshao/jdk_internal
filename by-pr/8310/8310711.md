## 描述

This clean-up PR removes the handling of the `<!-- safepoint while printing -->` message in the IR framework. It is no longer required since we dump the output of `PrintIdeal` to the hotspot_pid file differently since [JDK-8306922](https://bugs.openjdk.org/browse/JDK-8306922). There is no interrupting `<!-- safepoint while printing -->` message anymore. I removed the corresponding now unneeded code together with the previously added test case for it.

Testing: tier1-4
