## 描述

`RShiftNode::IdentityIL` recognizes `RShift(LShift(x, C), C) -> x` when `x` fits in the sign-extension mask. It compared shift amounts by node pointer (`in(1)->in(2) == in(2)`).

During IGVN, `mask_and_replace_shift_amount` normalizes shift counts (e.g., -1 -> 31), but LShift and RShift may be processed at different times, leaving them with different constant nodes for the same effective count. The pointer comparison then fails.

Here fix is to use `const_shift_count` to extract and compare masked shift values instead of node pointers.

Tested with:

```bash
make CONF=linux-x86_64-server-fastdebug test TEST="test/hotspot/jtreg/compiler/c2" JTREG="JAVA_OPTIONS=-XX:+StressIGVN -XX:+StressCCP -XX:VerifyIterativeGVN=1110"
```
