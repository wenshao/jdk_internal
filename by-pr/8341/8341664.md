## 背景

### ClassDesc 与 internalName

`ClassDesc` 是 Java 中描述类类型的常量描述符。`internalName` 是类在 JVM 内部使用的名称格式（如 `java/lang/String`）。

**为什么重要？**
- ClassFile API 频繁需要将 `ClassDesc` 转换为内部名称
- 原始实现每次都通过 `descriptorString()` + substring 转换
- 大量重复的字符串创建和复制操作

### 原始实现

public final class ReferenceClassDescImpl implements ClassDesc {
    private final String descriptor;  // 如 "Ljava/lang/String;"

    public String descriptorString() {
        return descriptor;
    }

    // 每次 resolveConstantDesc 都要转换
    @Override
    public Class<?> resolveConstantDesc(MethodHandles.Lookup lookup)

        // 每次调用都需要计算 internalName
        return lookup.findClass(internalToBinary(

    }

// 文件: Util.java (优化前)
public static String toInternalName(ClassDesc cd) {
    var desc = cd.descriptorString();
    if (desc.charAt(0) == 'L')
        return desc.substring(1, desc.length() - 1);  // 每次创建新字符串
    throw new IllegalArgumentException(desc);

    var utf = tryFindUtf8OfRegion(..., desc, 1, desc.length() - 1);

### 性能瓶颈分析

| 瓶颈点 | 问题 | 影响 |
|--------|------|------|
| **重复 substring** | 每次调用都创建新字符串 | 内存分配压力 |
| **重复边界检查** | `charAt(0)` + `length()` 检查 | CPU 周期浪费 |
| **复杂的 hash 计算** | `internalNameHash` 需要特殊算法 | 计算开销 |
| **equalsRegion 调用** | 比较时需要遍历字符 | 性能下降 |

### 为什么频繁调用？

```
每次写入类引用时:
1. ClassDesc → internalName 转换
2. internalName → Utf8Entry 查找/创建
3. 写入常量池

一个类的生成可能涉及数百次这样的转换
```

## 解决方案

### 核心改动

#### 1. 在 ReferenceClassDescImpl 中添加缓存

```java
// 文件: ReferenceClassDescImpl.java (优化后)

public final class ReferenceClassDescImpl implements ClassDesc {
    private final String descriptor;  // 如 "Ljava/lang/String;"
    private transient String internalName;  // 缓存的内部名称

    // 懒加载缓存
    public String internalName() {
        var internalName = this.internalName;
        if (internalName == null) {
            var desc = this.descriptor;
            this.internalName = internalName =
                desc.charAt(0) == 'L' ? dropFirstAndLastChar(desc) : desc;
        }
        return internalName;
    }

    @Override
    public Class<?> resolveConstantDesc(MethodHandles.Lookup lookup)
            throws ReflectiveOperationException {
        // 使用缓存的 internalName
        return lookup.findClass(internalToBinary(internalName()));
    }
}
```

#### 2. 简化 Util.toInternalName

```java
// 文件: Util.java (优化后)

public static String toInternalName(ClassDesc cd) {
    if (cd instanceof ReferenceClassDescImpl rcd) {
        // 快速路径：直接使用缓存
        return rcd.internalName();
    }
    // 回退：处理其他 ClassDesc 实现
    return cd.descriptorString();
}
```

#### 3. 简化 SplitConstantPool

```java
// 文件: SplitConstantPool.java (优化后)

private AbstractPoolEntry.ClassEntryImpl classEntryForClassOrInterface(ClassDesc cd) {
    // 直接使用 internalName，不需要复杂的 hash 计算
    String internalName = Util.toInternalName(cd);
    var utfHash = internalName.hashCode();  // 标准 hash，足够了
    var utf = tryFindUtf8(AbstractPoolEntry.hashString(utfHash), internalName);
    if (utf == null)
        utf = internalAdd(new AbstractPoolEntry.Utf8EntryImpl(this, size, internalName, utfHash));

    return internalAdd(new AbstractPoolEntry.ClassEntryImpl(this, size, utf, hash, cd));
}
```

#### 4. 删除冗余代码

```java
// 文件: AbstractPoolEntry.java (删除)
// 删除 equalsRegion 方法 - 不再需要特殊的区域比较

// 文件: Util.java (删除)
// 删除 internalNameHash 方法 - 不再需要特殊的 hash 计算
```

### 关键优化点

| 优化点 | 说明 | 效果 |
|--------|------|------|
| **懒加载缓存** | 首次访问时计算并缓存 | 后续访问 O(1) |
| **transient 字段** | 序列化不影响 | 缓存只在运行时有用 |
| **instanceof 检查** | 快速类型判断 | 避免 descriptorString 调用 |
| **标准 hashCode** | 不再需要特殊 hash | 简化代码 |

## 设计决策

### Q1: 为什么使用懒加载而不是急加载？

> **Reviewer**: 为什么不在构造函数中直接计算 internalName？

**A**:
1. **节省初始化时间**：不是所有 ClassDesc 都需要 internalName
2. **节省内存**：不使用的 ClassDesc 不占用缓存内存
3. **序列化友好**：transient 字段不影响序列化大小

### Q2: 为什么要删除 internalNameHash 方法？

> **Reviewer**: 特殊的 hash 不是更快吗？

**A**:
1. **复杂度不值得**：特殊 hash 只是为了避免 substring，现在有缓存了
2. **标准 hash 足够好**：String.hashCode() 已经高度优化
3. **代码简化**：删除 30+ 行复杂代码

### Q3: instanceof 检查的性能影响？

> **Reviewer**: `instanceof ReferenceClassDescImpl` 会很慢吗？

**A**:
1. **JIT 优化**：JIT 会优化常见类型的 instanceof 检查
2. **内联后等价**：热点代码会被内联，消除虚方法调用
3. **快速失败**：不符合类型时快速返回

## 风险评估

| 风险类型 | 等级 | 缓解措施 |
|----------|------|----------|
| **内存泄漏** | 🟢 无 | String 是不可变的，会被 GC |

| **兼容性** | 🟢 无影响 | 内部实现，API 不变 |
