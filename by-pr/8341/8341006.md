## 背景

`StackMapGenerator` 是 ClassFile API 中生成 StackMapTable 属性的核心类。`detectFrames()` 方法负责检测哪些字节码位置需要生成栈映射帧。

**为什么重要？**
- StackMapTable 是 JVM 类型验证的基础
- 帧检测是字节码生成的必经步骤
- 原始实现使用 ArrayList 和 BitSet，有额外开销

**改动位置**
```
src/java.base/share/classes/jdk/internal/classfile/impl/StackMapGenerator.java
    ├── generate()              ← 重构
    ├── generateHandlers()      ← 新增
    ├── deadCodePatching()      ← 新增
    ├── detectFrames()          ← 重构
    └── addFrame()              ← 新增
```

## 问题分析

### 原始实现

```java

private BitSet detectFrameOffsets() {

        @Override
        public void set(int i) {
            Preconditions.checkIndex(i, bytecode.length(), RawBytecodeHelper.IAE_FORMATTER);
            super.set(i);
        }
    };

    // 扫描字节码，设置帧偏移
    // ...

    // 处理异常处理器
    for (var exhandler : rawHandlers) {
        offsets.set(exhandler.handler());
    }
    return offsets;
}

// 在 generate() 中使用 BitSet 创建 Frame 列表
BitSet frameOffsets = detectFrameOffsets();
int framesCount = frameOffsets.cardinality();
frames = new ArrayList<>(framesCount);
int offset = -1;
for (int i = 0; i < framesCount; i++) {
    offset = frameOffsets.nextSetBit(offset + 1);
    frames.add(new Frame(offset, classHierarchy));  // ← 两次遍历
}
```

### 性能瓶颈分析

| 瓶颈点 | 原因 | 影响 |
|--------|------|------|
| **BitSet 开销** | BitSet 需要位操作和扩容 | 增加内存分配 |
| **两次遍历** | 一次收集偏移，一次创建 Frame | 增加循环开销 |
| **ArrayList 扩容** | 需要预分配容量 | 可能的内存浪费 |
| **迭代器开销** | `frames.get(i)` 使用 ArrayList 索引访问 | 增加方法调用 |

## 解决方案

### 核心改动

#### 1. 使用数组直接存储 Frame

```java
// 优化前
private List<Frame> frames;

// 优化后
private static final Frame[] EMPTY_FRAME_ARRAY = new Frame[0];
private Frame[] frames = EMPTY_FRAME_ARRAY;
private int framesCount = 0;
```

#### 2. 重构 detectFrames

```java
// 优化后：直接创建 Frame 对象
private void detectFrames() {
    var bcs = bytecode.start();
    boolean no_control_flow = false;
    int opcode, bci = 0;

    try {
        while (bcs.next()) {
            opcode = bcs.opcode();
            bci = bcs.bci();
            if (no_control_flow) {
                addFrame(bci);  // ← 直接添加 Frame
            }
            no_control_flow = switch (opcode) {
                case GOTO -> {
                    addFrame(bcs.dest());  // ← 直接添加
                    yield true;
                }
                case GOTO_W -> {
                    addFrame(bcs.destW());  // ← 直接添加
                    yield true;
                }
                // ... 其他 case
            };
        }
    } catch (IllegalArgumentException iae) {
        throw generatorError("Detected branch target out of bytecode range", bci);
    }

    for (int i = 0; i < rawHandlers.size(); i++) {
        addFrame(rawHandlers.get(i).handler());  // ← 直接添加
    }
}
```

#### 3. 新增 addFrame 方法

```java
private void addFrame(int offset) {
    Preconditions.checkIndex(offset, bytecode.length(), RawBytecodeHelper.IAE_FORMATTER);
    var frames = this.frames;
    int i = 0, framesCount = this.framesCount;

    // 查找插入位置（保持有序）
    for (; i < framesCount; i++) {
        var frameOffset = frames[i].offset;
        if (frameOffset == offset) {
            return;  // 已存在，直接返回
        }
        if (frameOffset > offset) {
            break;
        }
    }

    // 扩容
    if (framesCount >= frames.length) {
        int newCapacity = framesCount + 8;
        this.frames = frames = framesCount == 0
            ? new Frame[newCapacity]
            : Arrays.copyOf(frames, newCapacity);
    }

    // 插入新 Frame
    if (i != framesCount) {
        System.arraycopy(frames, i, frames, i + 1, framesCount - i);
    }
    frames[i] = new Frame(offset, classHierarchy);
    this.framesCount = framesCount + 1;
}
```

#### 4. 提取异常处理方法

```java
// 优化前：在 generate() 中处理异常
for (var exhandler : handlers) {
    int start_pc = labelContext.labelToBci(exhandler.tryStart());
    // ...
}

// 优化后：提取到单独方法
private void generateHandlers() {
    var labelContext = this.labelContext;
    for (int i = 0; i < handlers.size(); i++) {
        var exhandler = handlers.get(i);
        int start_pc = labelContext.labelToBci(exhandler.tryStart());
        // ...
    }
}
```

### 关键优化点

| 优化点 | 说明 |
|--------|------|
| **消除 BitSet** | 直接创建 Frame 对象，避免中间存储 |
| **一次遍历** | 边检测边创建，无需两次遍历 |
| **空集合优化** | `EMPTY_FRAME_ARRAY` 避免空方法分配 |
| **数组索引访问** | 比 ArrayList.get() 更快 |
| **有序插入** | 保持数组有序，便于二分查找 |

## 设计决策

### Q1: 为什么要保持数组有序？

> **Reviewer**: 直接添加到末尾不是更快吗？

**A**:
1. **二分查找**: `getFrame(int offset)` 使用二分查找，需要有序数组
2. **后续处理**: StackMap 生成需要按偏移量顺序处理
3. **插入开销小**: 帧数量通常不多，插入开销可接受

### Q2: 为什么用 EMPTY_FRAME_ARRAY 而不是 null？

> **Reviewer**: null 检查不是更简单吗？

**A**:
1. **避免空指针**: 无需每次检查 null
2. **更清晰**: 空数组和 null 的语义更明确
3. **性能**: `framesCount == 0` 检查比 null 检查更快

### Q3: 为什么扩容是 +8 而不是 2 倍？

> **Reviewer**: ArrayList 用 1.5 倍扩容，为什么这里 +8？

**A**:
1. **帧数量少**: 大多数方法的帧数量在 10 以内
2. **避免浪费**: 2 倍扩容对小数组浪费内存
3. **经验值**: +8 对大多数方法足够

                    var code = mb.codeBuilder();
                    // 生成包含分支的代码

                    code.iload(0).ifne(l1);

                    code.ireturn();
