## 描述

[JDK-8346781](https://bugs.openjdk.org/browse/JDK-8346781) changed `JVMCIServiceLocator` such that the set of providers is computed eagerly in `JVMCIServiceLocator.<clinit>`. There are some JVMCI test classes that directly subclass `JVMCIServiceLocator` which means deadlock could occur between the main thread running the test and a JVMCI compiler thread. This PR fixes this by ensuring that `JVMCIServiceLocator` providers are separate classes from the main test class.
