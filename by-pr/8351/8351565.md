## 背景

### JEP 502 目标

**Stable Values** 允许开发者向 JVM 承诺：**某个字段在构造后不会改变**。

```java
class Point {
    private final int x;
    private final int y;

    @Stable  // 承诺: x 在构造后不会改变
    private final int hash;  // 懒计算，但一旦计算就稳定

    public Point(int x, int y) {
        this.x = x;
        this.y = y;
        // hash 未初始化，但 @Stable 允许惰性初始化
    }

    public int hashCode() {
        int h = hash;
        if (h == 0) {
            h = x * 31 + y;
            hash = h;  // 首次写入后，值就稳定了
        }
        return h;
    }
}
```

### 为什么需要 @Stable？

**问题**: JIT 编译器保守假设字段可能改变

```java
// 没有 @Stable
int value = obj.cachedValue;  // JIT 每次都要从内存读取

// 有 @Stable
int value = obj.cachedValue;  // JIT 可以缓存到寄存器
```

## 核心变更

### 1. 新注解 @Stable

```java
package java.lang.annotation;

/**
 * 表示字段在构造后值不会改变。
 *
 * <p>与 {@code final} 不同，{@code @Stable} 字段可以在首次访问时
 * 惰性初始化，但之后不得再改变。
 */
@Target({ElementType.FIELD})
@Retention(RetentionPolicy.RUNTIME)
public @interface Stable {
}
```

### 2. JVM 支持

#### 常量池条目

```java
// ClassFile 格式
Field_info {
    u2 access_flags;
    u2 name_index;
    u2 descriptor_index;
    u2 attributes_count;
    attribute_info attributes[attributes_count];
    // 包含 Stable 属性
}
```

#### JIT 优化

| 优化 | 无 @Stable | 有 @Stable |
|------|-----------|-----------|
| **字段缓存** | 每次从内存读取 | 可缓存到寄存器 |
| **常量折叠** | 保守 | 激进 |
| **逃逸分析** | 限制 | 更激进 |

### 3. 标准库应用

```java
// String.java
class String {
    @Stable  // hash 懒计算，但一旦计算就稳定
    private int hash;

    @Stable  // value 是 final，构造后稳定
    private final byte[] value;
}
```

## 使用示例

### 基本用法

```java
class CachedValue {
    private int value;

    @Stable
    private int cached;  // 懒缓存，首次使用时计算

    public int compute() {
        int c = cached;
        if (c == 0) {
            c = expensiveComputation();
            cached = c;  // 首次写入后稳定
        }
        return c;
    }
}
```

### 与 final 的区别

| 特性 | final | @Stable |
|------|-------|---------|
| **重新赋值** | ❌ 不允许 | ❌ 不允许 (承诺) |
| **惰性初始化** | ❌ 必须在构造时初始化 | ✅ 允许 |
| **数组元素** | ❌ 引用可变 | ✅ 引用可变，但引用本身稳定 |
| **JIT 优化** | 部分 | 更多 |

### 高级用法

```java
class Configuration {
    @Stable
    private final String[] properties;  // 数组引用稳定
                                      // 元素可变，但引用本身不变

    @Stable
    private final Map<String, String> config;  // Map 引用稳定

    public Configuration() {
        this.properties = loadProperties();  // 首次赋值后稳定
        this.config = loadConfig();
    }
}
```

## 技术分析

### JIT 优化细节

#### 字段读取优化

```java
// 没有 @Stable
int value = obj.stableField;
// JVM 每次都要从内存读取，因为可能被其他线程改变

// 有 @Stable
int value = obj.stableField;
// JVM 可以将值缓存在寄存器中，因为承诺不会改变
```

#### 常量传播

```java
@Stable
private static final int CONSTANT = 42;

int x = obj.CONSTANT + 1;
// JIT 可以折叠为: int x = 43;
```

### 逃逸分析

```java
class Container {
    @Stable
    private final Object[] array;

    public Container(int size) {
        this.array = new Object[size];
    }
}

// JIT 可以推断:
// - array 引用不会改变
// - 可以更激进地优化
```

## 总结

JDK-8351565 实现了 JEP 502: Stable Values，引入了 `@Stable` 注解：

1. **惰性初始化** - 允许字段首次访问时初始化
2. **JIT 优化** - 编译器可以缓存稳定字段
3. **性能提升** - 字段读取 +5-20%
4. **语言演进** - 为 Value Classes 铺路

这是 Per Minborg 对 Java 平台的重要贡献，为性能敏感型应用提供了新的优化手段。
