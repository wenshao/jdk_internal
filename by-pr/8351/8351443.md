## 背景

### StringBuilder 非线程安全

**StringBuilder** 不是线程安全的：

```java
StringBuilder sb = new StringBuilder();
// 多线程同时调用:
// 线程 1: sb.append("hello");
// 线程 2: sb.append("world");
// 结果: 未定义行为，可能崩溃
```

### 内部状态不一致问题

**StringBuilder 维护三个关键状态**：

```java
abstract class AbstractStringBuilder {
    byte[] value;      // 字符数据
    byte coder;        // 编码: 0=Latin1, 1=UTF16
    int count;         // 当前长度
    int capacity;      // 容量
}
```

**问题场景**：

```java
// 某些异常操作可能导致状态不一致
// 例如: count > capacity
// 或者: coder 与实际数据不匹配
```

## 核心变更

### 状态一致性检查

**变更前**:
```java
// StringBuilder.java
public StringBuilder append(char c) {
    try {
        ensureCapacityInternal(count + 1);
        // ... 添加字符
    } catch (OutOfMemoryError e) {
        throw e;
    }
}
```

**变更后**:
```java
// StringBuilder.java
public StringBuilder append(char c) {
    // 新增: 状态一致性检查
    if (count > value.length) {
        throw new IllegalStateException("count > length");
    }
    if (count > capacity) {
        throw new IllegalStateException("count > capacity");
    }

    ensureCapacityInternal(count + 1);
    // ... 添加字符
}
```

### coder 一致性检查

```java
// 检查 coder 与实际内容一致
private void checkCoder() {
    if (coder == LATIN1) {
        // 检查所有字节都符合 Latin1
        for (int i = 0; i < count; i++) {
            if (value[i] < 0) {  // UTF-16 字节在 byte[] 中是负数
                throw new IllegalStateException("Inconsistent coder");
            }
        }
    }
}
```

## 技术分析

### 状态关系

| 状态 | 约束 |
|------|------|
| **count ≤ capacity** | 当前长度不能超过容量 |
| **count ≤ value.length** | 当前长度不能超过数组长度 |
| **coder 匹配内容** | coder 必须与实际数据编码一致 |

### 并发问题检测

```java
// 并发修改可能的状态:
// 线程 1: count = 10
// 线程 2: capacity = 8
// 结果: count > capacity ❌

// 新增检查会抛出异常而不是静默错误
if (count > capacity) {
    throw new IllegalStateException("count > capacity");
}
```
