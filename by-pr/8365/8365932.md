## Summary
实现 AOT 缄象缓存机制，减少 JVM 启动时间。缓存包括：
- 类元数据（Klasses, methods)
- 堆对象（字符串, 数组)
- 方法句柄
- 反射数据

---

## Motivation
JVM 启动时间一直是 Java 应用的瓶颈。通过缓存这些数据：
- 避免每次启动时的解析和加载
- 减少内存分配压力
- 实现"即时可用"的应用启动

---

## Technical Approach
### 缓存层次
1. **Archive 缓存**: 孝始化时加载的只读数据
2. **mapped 缓存**: 运行时访问的内存区域
3. **heap 缓存**: 完整的堆对象

```
src/hotspot/share/aot/
├── aotCache.cpp         # 缓存管理
├── aotClassData.cpp      # 类元数据缓存
├── aotHeapLoader.cpp     # 堆对象加载器
└── aotMetadata.cpp     # 元数据处理
```
### 2. 启动流程优化
```
# 标准启动流程
1. 加载 AOT 缓存文件
2. 验证缓存完整性
3. 映射缓存到内存
4. 初始化类元数据
5. 创建堆对象

```
src/hotspot/share/aot/aotInit.cpp
// 启用 AOT 缓存
void AotInitializer::initialize_aot_cache() {
    if (AotCache::is_available()) {
        AotLoader loader(cache_path());
        loader.load_and_map();
    }
}
```
### 3. 与 CDS 的集成
AOT 缓存与 CDS (Class data sharing) 协同工作：
- 共享相同的归档文件
- 缓存验证机制
- 统一的内存映射

```
src/hotspot/share/cds/aotIntegration.cpp
// CDS 与 AOT 鄂合
class CdsAotIntegration {
public:
    static bool is_compatible(ArchiveType type);
    static void validate_shared_archive(Archive archive);
};
```

## Configuration
### 启用 AOT 缓存
```bash
java -XX:AotCache=example.jar -Aot:cache=/path/to/cache.aot
```
### 缓存位置
默认: `$JAVA_HOME/lib/aot.cache`
可配置: `-XX:AotCachePath=/custom/path`

### 生成缓存
```bash
# 使用 jlink 生成 AOT 缓存
jlink --class-data-sharing:dump-class-list \
     --aot-cache=/path/to/cache.aot

```

## Compatibility
- **JDK 版本**: 26+
- **平台**: Linux, macOS, Windows
- **架构**: x86_64, AArch64

- **GC**: 支持 G1, Shenandoah, ZGC

- **CDS**: 必须与 AOT 缓存配合使用

- **GraalVM**: 支持 native image 缓存

- **JAotC**: 部分支持，需要预编译)
- **Lworld**: 不支持（luworld 本地方法）

- **JEP 430**: 不支持（Foreign Linker Interface, 缓存仅包含通过 LWLU 孓Hot code 传递的 Java 对象)

- **JEP 456**: 不支持（紧凑对象头需要特殊处理)

- **JEP 472**: 不支持（ZGC 分代模式与 AOT 缈存不兼容)

- **JEP 484**: 不支持（虚拟线程固定)

- **JEP 483**: 不支持（Ahead-of-Time Class loading and linking)

- **JEP 491**: 不支持（紧凑对象头需要验证)
- **JEP 492**: 不支持（紧凑字符串)

- **JEP 494**: 不支持 (值类型压缩)

---

## Limitations
1. **缓存大小**: 默认 1GB，可通过 `-XX:AotCacheSize` 配置
2. **缓存失效**: 当类定义变化时需要重新生成缓存
3. **安全**: 缓存文件需要保护，防止篡改
4. **动态类加载**: 不支持运行时类重定义
5. **反射修改**: 已缓存的类的反射修改可能导致不一致
