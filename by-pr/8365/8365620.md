## 背景

`MethodHandleDesc` 是 Java 12 引入的常量描述符接口，位于 `java.lang.constant` 包中，用于描述 `MethodHandle` 的常量池形式。

**代码不一致问题**
在 JDK 21+ 引入增强 switch 表达式后，JDK 代码库逐步迁移到新语法。`MethodHandleDesc` 类中存在不一致：
- `ofField()` 方法使用**增强 switch**（新语法）
- `of()` 和 `ofMethod()` 方法使用**传统 switch**（旧语法）

**为什么重要？**
- 代码一致性是 JDK 代码质量的重要指标
- 增强 switch 更简洁、更易读
- 统一代码风格降低维护成本

**改动位置**
```
src/java.base/share/classes/java/lang/constant/MethodHandleDesc.java
    ├── of()           ← 修改为增强 switch
    ├── ofMethod()     ← 修改为增强 switch
    └── ofField()      ← 已使用增强 switch（无需修改）
```

## 问题分析

### 原始实现

```java
// 文件: java/lang/constant/MethodHandleDesc.java
// 旧版本

public sealed interface MethodHandleDesc extends ConstantDesc
        permits AsTypeMethodHandleDesc, DirectMethodHandleDesc {

    // of() - 使用传统 switch
    static DirectMethodHandleDesc of(DirectMethodHandleDesc.Kind kind,
                                     ClassDesc owner,
                                     String name,
                                     String lookupDescriptor) {
        switch (kind) {
        case GETTER:
        case SETTER:
        case STATIC_GETTER:
        case STATIC_SETTER:
            return ofField(kind, owner, name, ClassDesc.ofDescriptor(lookupDescriptor));
        default:
            return new DirectMethodHandleDescImpl(kind, owner, name,
                MethodTypeDesc.ofDescriptor(lookupDescriptor));
        }
    }

    // ofMethod() - 使用传统 switch
    static DirectMethodHandleDesc ofMethod(DirectMethodHandleDesc.Kind kind,
                                           ClassDesc owner,
                                           String name,
                                           MethodTypeDesc lookupMethodType) {
        switch (kind) {
        case GETTER:
        case SETTER:
        case STATIC_GETTER:
        case STATIC_SETTER:
            throw new IllegalArgumentException(kind.toString());
        case VIRTUAL:
        case SPECIAL:
        case INTERFACE_VIRTUAL:
        case INTERFACE_SPECIAL:
        case INTERFACE_STATIC:
        case STATIC:
        case CONSTRUCTOR:
            return new DirectMethodHandleDescImpl(kind, owner, name, lookupMethodType);
        default:
            throw new IllegalArgumentException(kind.toString());
        }
    }

    // ofField() - 已使用增强 switch
    static DirectMethodHandleDesc ofField(DirectMethodHandleDesc.Kind kind,
                                          ClassDesc owner,
                                          String fieldName,
                                          ClassDesc fieldType) {
        MethodTypeDesc mtr = switch (kind) {
            case GETTER        -> MethodTypeDesc.of(fieldType, owner);
            case SETTER        -> MethodTypeDesc.of(CD_void, owner, fieldType);
            case STATIC_GETTER -> MethodTypeDesc.of(fieldType);
            case STATIC_SETTER -> MethodTypeDesc.of(CD_void, fieldType);
            default -> throw new IllegalArgumentException(kind.toString());
        };
        return new DirectMethodHandleDescImpl(kind, owner, fieldName, mtr);
    }
}
```

### 代码不一致分析

| 方法 | switch 类型 | case 数量 | 代码行数 |
|------|-------------|----------|----------|
| `of()` | 传统 | 5 | 9 行 |
| `ofMethod()` | 传统 | 12 | 17 行 |
| `ofField()` | 增强 | 5 | 7 行 |

**问题点**：
1. 同一个类中使用两种不同的 switch 风格
2. 传统 switch 需要显式 `break` 或 `return`
3. 增强 switch 使用 `->` 和箭头语法更简洁
4. case 标签可以合并（逗号分隔）

---

## 解决方案

### 核心改动

**1. 修改 `of()` 方法**

```java
// 变更后: 使用增强 switch

static DirectMethodHandleDesc of(DirectMethodHandleDesc.Kind kind,
                                 ClassDesc owner,
                                 String name,
                                 String lookupDescriptor) {
    return switch (kind) {
        case GETTER, SETTER, STATIC_GETTER, STATIC_SETTER
                -> ofField(kind, owner, name, ClassDesc.ofDescriptor(lookupDescriptor));
        default -> new DirectMethodHandleDescImpl(kind, owner, name,
                MethodTypeDesc.ofDescriptor(lookupDescriptor));
    };
}
```

**2. 修改 `ofMethod()` 方法**

```java
// 变更后: 使用增强 switch

static DirectMethodHandleDesc ofMethod(DirectMethodHandleDesc.Kind kind,
                                       ClassDesc owner,
                                       String name,
                                       MethodTypeDesc lookupMethodType) {
    return switch (kind) {
        case GETTER, SETTER, STATIC_GETTER, STATIC_SETTER
                -> throw new IllegalArgumentException(kind.toString());
        case VIRTUAL, SPECIAL, INTERFACE_VIRTUAL, INTERFACE_SPECIAL,
             INTERFACE_STATIC, STATIC, CONSTRUCTOR
                -> new DirectMethodHandleDescImpl(kind, owner, name, lookupMethodType);
        default -> throw new IllegalArgumentException(kind.toString());
    };
}
```

### 关键优化点

| 优化点 | 说明 |
|--------|------|
| **语法统一** | 所有方法使用增强 switch |
| **代码简化** | 使用 `->` 代替 `:` + `break/return` |
| **case 合并** | 逗号分隔多个 case 值 |
| **直接返回** | switch 作为表达式，直接 `return` |

### 代码对比

```
传统 switch:
┌─────────────────────────────────────┐
│ switch (kind) {                     │
│     case A:                          │
│     case B:                          │
│         return foo();                │  ← 需要 break 或 return
│     default:                         │
│         return bar();                │
│ }                                    │
└─────────────────────────────────────┘

增强 switch:
┌─────────────────────────────────────┐
│ return switch (kind) {              │
│     case A, B -> foo();             │  ← 箭头语法，自动返回
│     default   -> bar();             │
│ };                                   │
└─────────────────────────────────────┘
```

## 设计决策

### Q1: 为什么不保持向后兼容的旧语法？

> **Reviewer**: 传统 switch 也能工作，为什么一定要改？

**A**:
1. **代码一致性** - 同一个类应该使用统一的代码风格
2. **增强 switch 是未来** - JDK 21+ 推荐使用新语法
3. **更简洁** - 增强 switch 代码行数更少
4. **降低维护成本** - 统一风格更容易理解和维护

### Q2: 增强 switch 有性能差异吗？

> **Reviewer**: 两种语法编译后的字节码是否相同？

**A**:
- 两者编译后的字节码基本相同
- JIT 编译器对两者优化效果一致
- 性能差异可忽略不计

### Q3: 为什么 `ofField()` 已经使用增强 switch？

> **Reviewer**: 为什么只有 `ofField()` 用了新语法？

**A**:
- `ofField()` 可能在后续 PR 中单独更新过
- 这次 PR 统一了整个类的风格

## 性能对比

### 编译结果对比

```java
// 传统 switch
switch (kind) {
    case GETTER:
    case SETTER:
        return foo();
    default:
        return bar();
}

// 增强 switch
return switch (kind) {
    case GETTER, SETTER -> foo();
    default -> bar();
};
```

两者编译后的字节码几乎相同，性能无明显差异。

| 指标 | 传统 switch | 增强 switch |
|------|-------------|-------------|
| 字节码大小 | 相同 | 相同 |
| JIT 优化 | 相同 | 相同 |
| 执行效率 | 相同 | 相同 |
| 代码可读性 | 较低 | 较高 |
