## 背景

`DecimalDigits` 类是 JDK 内部用于数字到字符串转换的核心工具类，位于 `jdk.internal.util` 包。该类使用 `Unsafe` 操作直接操作内存以获得最佳性能。

**为什么重要？**
- `Integer.toString()`、`Long.toString()` 等核心方法依赖此类
- `StringBuilder` 数字追加功能使用此类
- 日期时间格式化（`ZoneOffset`）使用此类

**前置条件的必要性**

在 JDK-8353197 中，引入了使用 Unsafe 操作的优化方法。这些方法为了性能**不进行边界检查**，因此调用者必须确保满足前置条件。

**改动位置**
```
src/java.base/share/classes/jdk/internal/util/DecimalDigits.java
    ├── uncheckedGetCharsLatin1(int, int, byte[])
    ├── uncheckedGetCharsLatin1(long, int, byte[])
    ├── uncheckedGetCharsUTF16(int, int, byte[])
    ├── uncheckedGetCharsUTF16(long, int, byte[])
    ├── uncheckedPutPairLatin1(byte[], int, int)
    └── uncheckedPutPairUTF16(byte[], int, int)
```

## 问题分析

### 原始实现（缺少文档）

```java
// 文件: jdk/internal/util/DecimalDigits.java
// 修改前: 没有明确的前置条件文档

/**
 * Places characters representing the integer i into the
 * character array buf. The characters are placed into
 * the buffer backwards starting with the least significant
 * digit at the specified index (exclusive), and working
 * backwards from there.
 *
 * @param i     value to convert
 * @param index next index, after the least significant digit
 * @param buf   target buffer, Latin1-encoded
 * @return index of the most significant digit or minus sign, if present
 */
public static int uncheckedGetCharsLatin1(int i, int index, byte[] buf) {
    // ... 使用 Unsafe.putByte 直接写入内存
    // 没有边界检查！
}
```

### 潜在风险

| 风险类型 | 描述 | 后果 |
|----------|------|------|
| **数组越界** | 调用者传入的 `buf` 容量不足 | 内存损坏 |
| **索引错误** | `index` 参数超出范围 | 静默写入错误位置 |
| **编码不匹配** | Latin1 方法用于 UTF16 数据 | 数据损坏 |

### 为什么不能用普通边界检查？

这些方法的设计目标是**极致性能**：

```java
// 带边界检查的版本（慢）
public static void putCharLatin1(byte[] buf, int pos, int c) {
    if (pos < 0 || pos >= buf.length) {
        throw new IndexOutOfBoundsException();
    }
    buf[pos] = (byte) c;  // 字节码有边界检查指令
}

// Unsafe 版本（快）
private static void uncheckedPutCharLatin1(byte[] buf, int charPos, int c) {
    assert charPos >= 0 && charPos < buf.length;  // 仅在 debug 模式
    UNSAFE.putByte(buf, ARRAY_BYTE_BASE_OFFSET + charPos, (byte) c);  // 直接写内存
}
```

性能差异：
- 边界检查版本：每次调用 ~2-5ns
- Unsafe 版本：每次调用 ~0.5-1ns（**4-5倍快**）

public static int uncheckedGetCharsLatin1(int i, int index, byte[] buf) {
    // ...
}

## 设计决策

### Q1: 为什么使用 WARNING 而不是 @throws？

> **Reviewer**: 为什么不声明 `@throws IndexOutOfBoundsException`？

**A**: 作者回复：

> "这些方法使用 Unsafe 操作，在错误条件下不会抛出异常。使用 WARNING 更准确地反映了实际行为。"

关键区别：
- `@throws IndexOutOfBoundsException`：方法会检查并抛出异常
- `WARNING`：方法不会检查，错误条件导致未定义行为

### Q2: 为什么不添加 assert 检查？

> **Reviewer**: 可以添加 `assert` 语句在调试模式下检查吗？

**A**:

实际上代码中已有 `assert`：

```java
private static void uncheckedPutCharLatin1(byte[] buf, int charPos, int c) {
    assert charPos >= 0 && charPos < buf.length;  // 已存在
    UNSAFE.putByte(buf, ARRAY_BYTE_BASE_OFFSET + charPos, (byte) c);
}
```

但这些 `assert` 仅在调试模式下生效。JavaDoc 的 WARNING 是为了提醒**生产环境**的使用者。

### Q3: 为什么只文档化 unchecked 方法？

> **Reviewer**: `appendPair` 和 `appendQuad` 方法也使用 Unsafe，为什么不文档化？

**A**:

`appendPair` 和 `appendQuad` 是公开方法，它们的实现**不直接暴露 Unsafe 调用**：

```java
// 公开方法：有明确的参数范围文档
public static void appendPair(StringBuilder buf, int v) {
    // The integer v is formatted as two decimal digits.
    // Values from 0 to 9 are formatted with a leading zero...
    // If the value is outside the range 0-99, the behavior is unspecified.
    // ...
}

// 私有方法：直接使用 Unsafe
private static void uncheckedPutCharLatin1(byte[] buf, int charPos, int c) {
    // WARNING: This method does not perform any bound checks.
    UNSAFE.putByte(buf, ARRAY_BYTE_BASE_OFFSET + charPos, (byte) c);
}
```

区别：
- 公开方法：使用 StringBuilder，相对安全
- 私有 unchecked 方法：直接操作字节数组，需要 WARNING

### 无性能影响

这是一个**纯文档变更**：

| 指标 | 值 |
|------|-----|

| **删除代码** | 0 行 |

| **性能影响** | 无 |

### 为什么文档化很重要？

虽然不影响性能，但影响**正确性**：

```java
// 错误使用示例（现在文档明确指出这是不安全的）
byte[] buf = new byte[5];
int index = 2;
// 错误！可能只有 2 字节空间，但方法会写入更多字节
DecimalDigits.uncheckedGetCharsLatin1(12345, index, buf);  // 💥 潜在越界
```

## 实际影响场景

### 受影响的调用链

```
Integer.toString(int)
    └── Integer.getChars(int, int, byte[])
        └── DecimalDigits.uncheckedGetCharsLatin1()  ← 文档化的方法

Long.toString(long)
    └── Long.getChars(long, int, byte[])
        └── DecimalDigits.uncheckedGetCharsLatin1()  ← 文档化的方法

StringBuilder.append(int)
    └── Integer.getChars()
        └── DecimalDigits.uncheckedGetCharsLatin1()  ← 文档化的方法

ZoneOffset.buildId()
    └── DecimalDigits.appendPair()
        └── DecimalDigits.uncheckedPutPairLatin1()  ← 文档化的方法
```

### 安全使用模式

所有 JDK 内部调用者都遵循安全模式：

```java
// 安全使用示例（来自 Integer.getChars）
public static int getChars(int i, int index, byte[] buf) {
    // 1. 确保容量足够
    // 已知：int 最多 10 位 + 负号 = 11 字节
    // 调用者确保 buf[index - 11 ... index] 有效

    // 2. 调用 unchecked 方法
    return DecimalDigits.uncheckedGetCharsLatin1(i, index, buf);
}
```
