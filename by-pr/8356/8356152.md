## 背景

### 问题发现

**JDK 24** 中，`String.concat()` 开始抛出 `StringIndexOutOfBoundsException`：

```java
// JDK 24
byte[] huge = new byte[100_000_000];
String s = new String(huge, 0);  // 大数组
String result = "prefix".concat(s);  // ❌ 抛出异常！

// 期望: 抛出 OutOfMemoryError
// 实际: 抛出 StringIndexOutOfBoundsException
```

### 根本原因

**JDK 24** 引入了新的字符串拼接实现，但边界检查有错误：

```java
// String.java (JDK 24)
byte[] result = new byte[len + other.value.length];
//                          ^^^^^^^^^^^^^^^^^^^^
// 如果 len + other.value.length > Integer.MAX_VALUE,
// 这里会溢出，变成负数，但数组创建不会失败
// 后续访问 result 时会越界
```

---

## 核心变更

### 问题代码

```java
// 变更前 (JDK 24)
static String concat(String str, String other) {
    byte[] result = new byte[str.length() + other.length()];
    // ...
}

// 问题: str.length() + other.length() 可能溢出
// 例如: Integer.MAX_VALUE + 1 → Integer.MIN_VALUE (负数!)
// new byte[Integer.MIN_VALUE] 会创建一个小数组
// 后续访问时抛出 StringIndexOutOfBoundsException
```

### 修复代码

```java
// 变更后
static String concat(String str, String other) {
    int len = str.value.length;
    int otherLen = other.value.length;

    // 检查溢出
    if (len > Integer.MAX_VALUE - otherLen) {
        throwOutOfMemoryError("String is too large");
    }

    byte[] result = new byte[len + otherLen];
    // ...
}
```

### 异常类型变更

| 场景 | JDK ≤ 23 | JDK 24 | 修复后 |
|------|---------|--------|--------|
| **正常拼接** | 正常返回 | 正常返回 | 正常返回 |
| **超大拼接** | OutOfMemoryError | StringIndexOutOfBoundsException ❌ | OutOfMemoryError ✅ |

## 技术分析

### 整数溢出

```java
// 最大可能的字符串长度
int MAX_LENGTH = Integer.MAX_VALUE;  // 约 2GB

// 拼接两个最大字符串
int len1 = Integer.MAX_VALUE;
int len2 = 1;
int sum = len1 + len2;  // 溢出 → Integer.MIN_VALUE

// 错误处理
new byte[sum];  // 创建了很小的数组
// 访问时越界
```

### 为什么是 OutOfMemoryError？

**语义正确性**：

```java
// 尝试创建超大字符串
String result = huge1.concat(huge2);

// 实际含义是: "我想创建一个超大的字符串"
// 正确异常应该是 OutOfMemoryError，而不是数组越界
```
