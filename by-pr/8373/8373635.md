## 描述

Prior to this patch, in the optimization that avoids adding the zero
offset to the pointer, the code used `intcon(0)` for comparison, which
fails on 64-bit systems, since in that case, `offset` is a `ConL` node
and not a `ConI` node. This patch fixes the problem by using `MakeConX`,
which, based on `_LP64`, correctly uses either `intcon` or `longcon`.

This patch also adds a debug assertion to prevent such regressions in
the future.  We add this assertion in place of a test since the
`basic_plus_adr()` function is called from many internal code paths
during compilation, making it difficult to construct a specific Java
test that exercises this change.
