# JDK-8349400: Improve Startup Speed via Eliminating Nested Classes

> **Issue**: [JDK-8349400](https://bugs.openjdk.org/browse/JDK-8349400)
> **Author**: Shaojin Wen
> **Date**: 2025-08-25
> **Impact**: ⭐⭐⭐⭐ (+5% startup)

---

## Summary

通过消除不必要的嵌套类，减少类加载时间和内存占用，提升 JVM 启动速度约 5%。

---

## Problem Analysis

### 原始结构

许多 JDK 类使用内部类来组织代码，但有些内部类是不必要的:

```java
// 原始实现 - 使用不必要的内部类
public class StringFormatter {
    private static class Helper {
        static String format(String pattern, Object... args) {
            // 格式化逻辑
        }
    }
    
    public String format(String pattern, Object... args) {
        return Helper.format(pattern, args);
    }
}
```

**问题**:
1. 每个 inner class 都需要单独加载和验证
2. 增加了类的元数据大小
3. 额外的间接调用开销

---

## Solution

### 优化后实现

```java
// 优化后 - 消除不必要的内部类
public class StringFormatter {
    public String format(String pattern, Object... args) {
        // 直接实现，无需内部类
        return formatInternal(pattern, args);
    }
    
    private String formatInternal(String pattern, Object... args) {
        // 格式化逻辑
    }
}
```

---

## Key Changes

### 消除的内部类

| 类 | 消除的内部类 | 原因 |
|----|-------------|------|
| java.lang.String | StringHelper | 仅包含静态方法 |
| java.util.UUID | UUIDParser | 可以内联 |
| java.lang.Integer | DigitExtractor | 单一职责可合并 |
| java.time.format.DateTimeFormatter | FormatHelper | 调用频率高 |

### 优化策略

1. **内联静态方法**: 将仅包含静态方法的内部类内联到外部类
2. **合并单一职责**: 将只被调用一次的内部类合并
3. **延迟初始化**: 使用 `@Stable` 注解替代延迟初始化内部类

## Technical Details

### 为什么能提升启动速度?

1. **减少类加载**: 每个类都需要从 JAR 文件读取、解析、验证
2. **减少元数据**: 每个类占用 Metaspace
3. **减少初始化**: 静态初始化块需要执行

### 安全性考虑

- 保持 API 兼容性
- 内部类如果被反射访问，需要保留
- 确保 serialVersionUID 不变
