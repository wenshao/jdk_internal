# JDK-8355177: Speed up StringBuilder::append(char[])

> **Issue**: [JDK-8355177](https://bugs.openjdk.org/browse/JDK-8355177)
> **Author**: Shaojin Wen
> **Date**: 2025-06-17
> **Impact**: ⭐⭐⭐⭐ (+15% performance)

---

## Summary

通过使用 `Unsafe::copyMemory` 优化 `StringBuilder::append(char[])` 方法，性能提升约 15%。

---

## Problem

原有实现使用 `System.arraycopy` 复制字符数组:

```java
// 原有实现
public AbstractStringBuilder append(char[] str) {
    int len = str.length;
    ensureCapacityInternal(count + len);
    System.arraycopy(str, 0, value, count, len);
    count += len;
    return this;
}
```

**问题**: `System.arraycopy` 在小数组场景下有额外的边界检查和方法调用开销。

---

## Solution

使用 `Unsafe::copyMemory` 直接内存复制:

```java
// 优化后实现
public AbstractStringBuilder append(char[] str) {
    int len = str.length;
    ensureCapacityInternal(count + len);
    Unsafe.getUnsafe().copyMemory(
        str, 
        Unsafe.ARRAY_CHAR_BASE_OFFSET,
        value, 
        Unsafe.ARRAY_CHAR_BASE_OFFSET + (count << 1),
        len << 1  // char = 2 bytes
    );
    count += len;
    return this;
}
```

---

## Technical Details

### 为什么更快?

| 因素 | System.arraycopy | Unsafe.copyMemory |
|------|------------------|-------------------|
| 边界检查 | 是 | 否 |
| 方法调用 | 2次 | 1次 |
| 类型检查 | 是 | 否 |
| 适用场景 | 通用 | 受控环境 |

### 安全性考虑

- StringBuilder 内部已确保容量
- 输入数组长度已验证
- 不会发生数组越界
