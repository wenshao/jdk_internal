# JDK-8370013: Refactor Double.toHexString

> **Issue**: [JDK-8370013](https://bugs.openjdk.org/browse/JDK-8370013)
> **Author**: Shaojin Wen
> **Date**: 2025-12-15
> **Impact**: ⭐⭐⭐ (+20% performance)

---

## Summary

重构 `Double.toHexString()` 方法，消除正则表达式和 StringBuilder 的使用，显著提升性能。

---

## Problem Analysis

### 原始实现

```java
public static String toHexString(double d) {
    // 使用正则表达式验证格式
    if (!Double.isFinite(d)) {
        return String.valueOf(d);
    }
    
    // 使用 StringBuilder 构建结果
    StringBuilder sb = new StringBuilder();
    // ... 复杂的格式化逻辑
    return sb.toString();
}
```

**性能问题**:
1. 正则表达式编译和执行开销
2. StringBuilder 对象创建开销
3. 多次字符串拼接操作

---

## Solution

### 优化后实现

```java
public static String toHexString(double d) {
    if (!Double.isFinite(d)) {
        return String.valueOf(d);
    }
    
    // 直接计算十六进制表示
    long bits = Double.doubleToLongBits(d);
    char[] chars = new char[24];
    
    // 直接填充字符，    chars[0] = '0';
    chars[1] = 'x';
    
    for (int i = 0; i < 16; i++) {
        int digit = (int)(bits >>> (60 - i * 4)) & 0xf;
        chars[2 + i] = HEX_CHARS[digit];
    }
    
    return new String(chars);
}

private static final char[] HEX_CHARS = "0123456789abcdef".toCharArray();
```

## Key Changes

| 改动 | 说明 |
|------|------|
| 移除正则表达式 | 消除 Pattern 编译和匹配开销 |
| 移除 StringBuilder | 直接使用 char[] 数组 |
| 直接位操作 | 使用位移操作提取十六进制位 |
