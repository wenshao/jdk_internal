# Optimize Java implementation of ML-DSA

> JDK-8347606 | Ben Perez (blperez01) | JDK 25

---

## 概述

本 PR 优化了 **ML-DSA (Module-Lattice-Based Digital Signature Algorithm)** 的 Java 实现，通过改进多维数组分配策略提升约 10% 的性能。

**注意**: 此 issue 是关于 ML-DSA (数字签名)，不是 ML-KEM (密钥封装)。

## 背景

### 问题分析

ML-DSA 的 Java 实现中使用了大量多维数组分配。标准的 `int[][] a = new int[rows][cols]` 初始化方式比在循环中逐列分配更慢。由于 ML-DSA 需要执行许多大型多维数组分配，优化这些分配方式可显著提升性能。

---

## 技术实现

### 核心优化

```java
// 优化前: 标准多维数组分配
int[][] matrix = new int[rows][cols];

// 优化后: 逐列分配
int[][] matrix = new int[rows][];
for (int i = 0; i < rows; i++) {
    matrix[i] = new int[cols];
}
```

此优化还包括对布尔矩阵分配的类似改进。

### 文件变更

主要修改 ML-DSA 相关的安全提供者实现代码，优化 `matrixAllocate` 方法。

ML-DSA 操作性能提升约 10%（基于 PR 描述）。
