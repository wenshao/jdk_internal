# JEP 491: Synchronize Virtual Threads without Pinning

> **状态**: ✅ 正式版 (JDK 24) | **JEP 链接**: https://openjdk.org/jeps/491

---

## 概述

改进虚拟线程在 synchronized 块中的行为，避免固定 (pinning) 导致的性能问题。

---

## 问题

之前，虚拟线程在 synchronized 块或方法中被"固定"到平台线程：

```java
synchronized (lock) {
    // 虚拟线程被固定，无法被卸载
    // 阻塞操作会阻塞底层平台线程
    Thread.sleep(1000);  // 阻塞平台线程
}
```

---

## 解决方案

JDK 24 改进了 synchronized 语义：
- 虚拟线程在 synchronized 块中不再被固定
- 固定仅在必要时发生（如本地代码调用）

---

## 使用示例

### 无需修改代码

```java
// 之前：可能导致性能问题
synchronized (lock) {
    blockingIO();  // 固定虚拟线程
}

// JDK 24+：自动优化
synchronized (lock) {
    blockingIO();  // 虚拟线程继续正常运行
}
```
