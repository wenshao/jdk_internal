## 1. 背景与动机

### HTTP 协议演进

```
HTTP/1.1 (1997)     - 文本协议，队头阻塞
    ↓
HTTP/2 (2015)       - 二进制分帧，多路复用，仍基于 TCP
    ↓
HTTP/3 (2022)       - 基于 QUIC，无队头阻塞，UDP 传输
```

### 问题

HTTP/2 虽然解决了应用层队头阻塞，但 TCP 层仍存在：
- 丢包导致所有流阻塞
- TCP 握手延迟
- 中间设备兼容性问题

### 解决方案

HTTP/3 基于 QUIC 协议：
- **UDP 基础**: 避免 TCP 队头阻塞
- **内置 TLS 1.3**: 安全性内置
- **快速连接**: 0-RTT 连接恢复
- **连接迁移**: 网络切换不断开

## 2. 使用示例

### 基本用法

```java
import java.net.URI;
import java.net.http.HttpClient;
import java.net.http.HttpRequest;
import java.net.http.HttpResponse;

// 创建支持 HTTP/3 的客户端
HttpClient client = HttpClient.newBuilder()
    .version(HttpClient.Version.HTTP_3)  // 指定 HTTP/3
    .build();

HttpRequest request = HttpRequest.newBuilder()
    .uri(URI.create("https://cloudflare.com/"))
    .build();

HttpResponse<String> response = client.send(
    request, 
    HttpResponse.BodyHandlers.ofString()
);

System.out.println("Status: " + response.statusCode());
System.out.println("Version: " + response.version());  // HTTP_3
System.out.println("Body: " + response.body());
```

### 自动协商

```java
// 自动选择最佳版本 (HTTP/3 > HTTP/2 > HTTP/1.1)
HttpClient client = HttpClient.newBuilder()
    .version(HttpClient.Version.HTTP_3_AUTO)  // 自动协商
    .build();
```

### 配置选项

```java
HttpClient client = HttpClient.newBuilder()
    .version(HttpClient.Version.HTTP_3)
    .connectTimeout(Duration.ofSeconds(10))
    .followRedirects(HttpClient.Redirect.NORMAL)
    .build();
```

### 异步请求

```java
client.sendAsync(request, HttpResponse.BodyHandlers.ofString())
    .thenApply(HttpResponse::body)
    .thenAccept(System.out::println)
    .join();
```

## 4. 开发者影响

### 兼容性

| 方面 | 说明 |
|------|------|
| API 兼容 | 完全兼容，只需更改 Version |
| 服务器支持 | 需服务器支持 HTTP/3 |
| 网络环境 | UDP 可能被防火墙阻止 |
| 回退机制 | 自动回退到 HTTP/2 或 HTTP/1.1 |

### 配置要求

```java
// 安全配置 (src/java.base/share/conf/security/java.security)
// 新增 HTTP/3 相关配置项
jdk.httpclient.http3.enabled=true
jdk.httpclient.http3.maxStreams=100
```

### 最佳实践

```java
// ✅ 推荐：使用自动协商
HttpClient client = HttpClient.newBuilder()
    .version(HttpClient.Version.HTTP_3_AUTO)
    .build();

// ✅ 推荐：设置超时
HttpRequest request = HttpRequest.newBuilder()
    .uri(uri)
    .timeout(Duration.ofSeconds(30))
    .build();

// ✅ 推荐：处理降级
HttpResponse<String> response = client.send(request, handler);
if (response.version() != HttpClient.Version.HTTP_3) {
    log.info("HTTP/3 不可用，使用 " + response.version());
}

// ⚠️ 注意：UDP 可能被阻止
// 建议实现降级逻辑
```

## 5. 性能对比

### 连接建立时间

```
场景: 首次连接到已支持 HTTP/3 的服务器

HTTP/1.1:  TCP(1-RTT) + TLS(2-RTT) = 3-RTT
HTTP/2:    TCP(1-RTT) + TLS(2-RTT) = 3-RTT
HTTP/3:    QUIC(1-RTT 包含 TLS)   = 1-RTT

场景: 恢复连接 (0-RTT)

HTTP/1.1:  TCP(1-RTT) + TLS(1-RTT) = 2-RTT
HTTP/2:    TCP(1-RTT) + TLS(1-RTT) = 2-RTT
HTTP/3:    0-RTT                   = 0-RTT
```

### 丢包恢复

```
场景: 1% 丢包率，10 个并发请求

HTTP/2:  所有请求受影响（TCP 队头阻塞）
HTTP/3:  仅丢包的流受影响（独立流）
```
