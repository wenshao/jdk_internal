## 1. 背景与动机

### 历史背景

32位 x86 架构（i386/i686）曾是主流平台，但随着技术发展：

| 时间 | 事件 |
|------|------|
| 1985 | Intel 386 发布，32位时代开始 |
| 2003 | AMD64 发布，64位时代开始 |
| 2010s | 64位成为主流 |
| 2025 | JDK 26 移除 32位 x86 支持 |

### 移除原因

1. **使用率极低**: 大多数用户已迁移到 64位
2. **维护成本高**: 需要维护大量平台特定代码
3. **测试负担**: 每个特性都需要 32位测试
4. **功能限制**: 无法使用现代 CPU 特性

## 3. 开发者影响

### 受影响的用户

| 用户类型 | 影响 |
|----------|------|
| 64位用户 | 无影响 |
| 32位应用 | 需迁移到 64位 JDK |
| 嵌入式设备 | 需使用其他 JDK 发行版或 ARM |

### 迁移指南

```bash
# 检查当前 JVM 架构
java -version
# 64-Bit Server VM (build ...)  ← 无影响
# 32-Bit Server VM (build ...)  ← 需要迁移

# 检查操作系统架构
uname -m
# x86_64  ← 可使用 64位 JDK
# i686    ← 需要升级系统或使用其他 JDK
```

### 替代方案

| 场景 | 建议 |
|------|------|
| 服务器 | 升级到 64位系统 |
| 桌面应用 | 升级到 64位系统 |
| 嵌入式 | 使用 ARM 或其他架构 |
| 遗留系统 | 使用 JDK 25 或更早版本 |

## 4. 保留的平台

JDK 26 仍支持以下平台：

| 平台 | 架构 | 状态 |
|------|------|------|
| Linux x86_64 | 64位 | ✅ 支持 |
| Linux aarch64 | ARM 64位 | ✅ 支持 |
| Linux riscv64 | RISC-V 64位 | ✅ 支持 |
| macOS x86_64 | 64位 | ✅ 支持 |
| macOS aarch64 | ARM 64位 | ✅ 支持 |
| Windows x86_64 | 64位 | ✅ 支持 |
| Windows aarch64 | ARM 64位 | ✅ 支持 |
