## 1. 背景与动机

### 问题

JIT 编译器需要运行时分析来决定编译哪些方法：
- 预热时间长
- 初始性能差
- 生产环境不稳定

### 解决方案

JEP 515 允许在 AOT 阶段收集方法分析数据：
- 提前确定热点方法
- 加速 JIT 预热
- 提升启动性能

## 3. 实现分析

### 编译策略

```cpp
// src/hotspot/share/compiler/compilationPolicy.cpp

// 基于分析数据决定编译优先级
class AOTCompilationPolicy : public CompilationPolicy {
    // 使用预收集的分析数据
    void method_invocation(methodHandle method);
    void method_back_branch_event(methodHandle method);
};
```

### 数据存储

```
profile.prof 包含:
- 方法调用计数
- 分支预测信息
- 类型反馈
- 内联决策
```
