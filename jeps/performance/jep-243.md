# JEP 243: Java Level JVM Compiler Interface (JVMCI)

> **Status**: Permanent | **Type**: Feature | **Component**: HotSpot / Compiler
>
> **Lead**: Doug Simon (Oracle Labs)
>
> **JDK Version**: JDK 9 (2017-09)
>
> **Organization**: [Oracle Labs](../../contributors/orgs/oracle.md)

---

## 概览

引入 JVM 编译器接口（JVMCI），允许基于 Java 的编译器（如 Graal）集成到 HotSpot JVM 中，替代传统的 C++ 编译器。

---

## 目标

- 提供 Java 级别的编译器接口
- 支持 Graal 作为 JIT 编译器
- 允许动态替换编译器
- 简化编译器开发和集成

---

## 动机

### 传统 JVM 编译器架构

```
┌─────────────────────────────────────┐
│         Java Application            │
└─────────────────────────────────────┘
                 ↓
┌─────────────────────────────────────┐
│         HotSpot JVM (C++)           │
│  ┌───────────────────────────────┐  │
│  │    C1/C2 Compiler (C++)       │  │
│  │    - 硬编码到 JVM             │  │
│  │    - 难以扩展和定制           │  │
│  └───────────────────────────────┘  │
└─────────────────────────────────────┘
```

### JVMCI 架构

```
┌─────────────────────────────────────┐
│         Java Application            │
└─────────────────────────────────────┘
                 ↓
┌─────────────────────────────────────┐
│         HotSpot JVM (C++)           │
│  ┌───────────────────────────────┐  │
│  │      JVMCI Interface          │  │
│  │    - 标准化接口               │  │
│  │    - 支持多种编译器           │  │
│  └───────────────────────────────┘  │
│                 ↓                   │
│  ┌───────────────────────────────┐  │
│  │   Graal Compiler (Java)       │  │
│  │    - 易于开发和扩展           │  │
│  │    - 支持高级优化             │  │
│  └───────────────────────────────┘  │
└─────────────────────────────────────┘
```

## 关键特性

### 1. 编译器接口

```java
// JVMCI 核心接口
public interface JVMCICompiler {
    CompiledMethod compileMethod(
        HotSpotResolvedJavaMethod method,
        int entryBci,
        boolean installAsNonVMEntryPoint,
        CompilationRequest request
    );
}
```

### 2. 编译器注册

```java
// 注册自定义编译器
public class GraalCompiler implements JVMCICompiler {
    @Override
    public CompiledMethod compileMethod(...) {
        // Graal 编译逻辑
    }
}
```

### 3. 运行时选择

```bash
# 使用 Graal 作为 JIT 编译器
-XX:+UnlockExperimentalVMOptions \
-XX:+EnableJVMCI \
-XX:+UseJVMCICompiler

# 指定编译器
-XX:JVMCICompiler=org.graalvm.compiler.hotspot.GraalCompiler
```

## 使用方式

### 启用 JVMCI

```bash
# JDK 9+
java -XX:+UnlockExperimentalVMOptions \
     -XX:+EnableJVMCI \
     -XX:+UseJVMCICompiler \
     -jar myapp.jar
```

### 验证 JVMCI 状态

```bash
# 检查 JVMCI 是否启用
java -XX:+UnlockDiagnosticVMOptions \
     -XX:+PrintFlagsFinal \
     | grep JVMCI

# 输出:
# EnableJVMCI = true
# UseJVMCICompiler = true
```

### Graal 集成

```xml
<!-- Maven 依赖 -->
<dependency>
    <groupId>org.graalvm.compiler</groupId>
    <artifactId>compiler</artifactId>
    <version>23.0.0</version>
</dependency>
```

## 性能对比

### Graal vs C2

| 基准测试 | C2 | Graal | 差异 |
|----------|-----|-------|------|
| SPECjbb2015 | 100% | 95-105% | ±5% |
| DaCapo | 100% | 90-110% | ±10% |
| 启动时间 | 100% | 85-95% | -5~15% |
| 峰值性能 | 100% | 95-105% | ±5% |

### JVMCI 开销

| 操作 | 原生 C2 | JVMCI | 开销 |
|------|---------|-------|------|
| 编译请求 | 直接调用 | JNI 调用 | ~5% |
| 元数据访问 | 直接 | 通过接口 | ~3% |
| 代码安装 | 相同 | 相同 | 无 |

## 应用场景

### 推荐

- ✅ 研究和实验新优化技术
- ✅ 领域特定编译器
- ✅ 动态优化策略
- ✅ 教学和学习编译器开发

### 不推荐

- ❌ 生产环境（除非充分测试）
- ❌ 对启动时间敏感的应用
- ❌ 资源受限环境
