## 1. 背景与动机

### 问题

JFR 提供采样分析，但缺少方法级精确计时：
- 无法知道方法执行时间
- 无法追踪方法调用链
- 需要额外的 profiler 工具

### 解决方案

JEP 520 提供方法级计时和追踪：
- 精确的方法执行时间
- 调用链追踪
- 低开销

---

## 2. 使用示例

### 启用方法追踪

```bash
# 启用方法追踪
java -XX:StartFlightRecording=method-tracing=true,filename=recording.jfr MyApp
```

### 配置追踪

```java
// JFR 配置文件
<event name="jdk.MethodTiming">
  <setting name="enabled">true</setting>
  <setting name="filter">com.example.*</setting>
</event>
```

### 查看结果

```bash
# 打印方法计时
jfr print --events jdk.MethodTiming recording.jfr
```

## 3. 实现分析

### 架构

```
方法追踪框架:
├── jfrMethodTracer.cpp      # 追踪核心
├── jfrFilter.cpp            # 过滤器
├── jfrFilterManager.cpp     # 过滤器管理
├── jfrMethodProcessor.cpp   # 方法处理
└── jfrClassFilterClosure.cpp# 类过滤
```

### 字节码插桩

```java
// 方法入口/出口插桩
public void myMethod() {
    // 插桩: 记录开始时间
    long start = JFR.methodEnter("myMethod");
    try {
        // 原始代码
        ...
    } finally {
        // 插桩: 记录结束时间
        JFR.methodExit("myMethod", start);
    }
}
```

## 4. 开发者影响

### 开销

```
方法追踪开销:
- 全部方法: 10-20%
- 过滤后:   2-5%
```

### 最佳实践

```bash
# 使用过滤器减少开销
java -XX:StartFlightRecording=\
  method-tracing=true,\
  method-filter="com.example.*"\
  MyApp
```
