## 1. 概述

JEP 393 是 Foreign-Memory Access API 的第三轮孵化（`jdk.incubator.foreign`）。本轮在 JEP 383（JDK 16）基础上继续演进，同时与并行开发的 Foreign Linker API（JEP 389）在设计上开始对齐，为后续合并为统一的 Foreign Function & Memory API 铺平道路。

---

## 2. 与前一轮的变更

| 变更项 | JEP 383 (JDK 16) | JEP 393 (JDK 16) |
|--------|-------------------|-------------------|
| **MemorySegment 统一** | MemorySegment 与 MemoryAddress 分离 | MemoryAddress 实现 MemorySegment 接口（零长度段） |
| **共享段支持** | 段仅限创建线程访问 | 支持跨线程共享段，通过 `share()` 方法 |
| **性能优化** | 基础 VarHandle 访问 | 结合 JIT 优化，内存访问性能显著提升 |
| **与 Foreign Linker 对齐** | 独立发展 | API 设计开始与 JEP 389 Foreign Linker 协调 |

---

## 3. 核心改进

- **共享内存段** — 允许多个线程安全地共享同一个 MemorySegment，解决了并发场景下的实用性限制
- **性能提升** — 通过 JIT 编译器优化 VarHandle 调用链，内存访问的开销接近 `Unsafe` 水平
- **API 简化** — 减少了构建内存访问 VarHandle 所需的样板代码

---

## 4. 代码示例

```java
// 跨线程共享内存段
MemorySegment shared;
try (MemorySegment segment = MemorySegment.allocateNative(1024)) {
    shared = segment.share(); // 创建可跨线程共享的段
    executor.submit(() -> {
        VarHandle intHandle = MemoryHandles.varHandle(int.class, ByteOrder.nativeOrder());
        intHandle.set(shared.baseAddress(), 99);
    });
}
```
