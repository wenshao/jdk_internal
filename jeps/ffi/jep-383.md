## 1. 概述

JEP 383 是 Foreign-Memory Access API 的第二轮孵化（`jdk.incubator.foreign`），首次孵化为 JEP 370（JDK 15）。该 API 允许 Java 程序安全、高效地访问 Java 堆外的外部内存，目标是最终替代 `sun.misc.Unsafe` 和 `ByteBuffer` 在外部内存访问场景中的使用。

---

## 2. 与前一轮的变更

基于 JEP 370（JDK 15）第一轮孵化期间收集的反馈，JEP 383 引入以下关键变更：

| 变更项 | JEP 370 (JDK 15) | JEP 383 (JDK 15) |
|--------|-------------------|-------------------|
| **MemorySegment / MemoryAddress 分离** | MemoryAddress 是 MemorySegment 的内部概念 | 明确分离为两个独立抽象，MemoryAddress 可独立存在 |
| **MemoryLayout 改进** | 基础布局描述 | 增加更丰富的布局组合能力与路径导航 |
| **Spliterator 支持** | 无 | MemorySegment 支持 Spliterator，便于并行处理 |
| **映射段增强** | 基础文件映射 | 支持 MappedMemorySegment 的 `force()`、`isLoaded()` 等操作 |

---

## 3. 核心 API 概览

API 位于 `jdk.incubator.foreign` 模块，核心类型包括：

- **MemorySegment** — 表示一段连续的外部内存区域，与生命周期绑定
- **MemoryAddress** — 表示一个内存地址（可附着于段，也可脱离）
- **MemoryLayout** — 以编程方式描述内存布局（结构体、联合体、序列等）
- **VarHandle** — 通过 MemoryLayout 生成的访问句柄，用于读写内存

---

## 4. 代码示例

```java
try (MemorySegment segment = MemorySegment.allocateNative(100)) {
    MemoryAddress base = segment.baseAddress();
    VarHandle intHandle = MemoryHandles.varHandle(int.class, ByteOrder.nativeOrder());
    intHandle.set(base, 42);
    int value = (int) intHandle.get(base); // 42
}
```
