## 1. 概述

JEP 442 是 Foreign Function & Memory API 的第三轮预览（`java.lang.foreign`），距最终版仅一步之遥。API 已从孵化模块（`jdk.incubator.foreign`）迁移至 `java.base` 模块，表明其稳定性已接近正式标准。该 API 使 Java 程序能够与 Java 运行时之外的代码和数据进行互操作——调用外部函数、安全地访问外部内存——而无需 JNI 的脆弱性和危险性。

---

## 2. 与前一轮的变更

基于 JEP 434（JDK 20，第二轮预览）的反馈，JEP 442 引入以下关键变更：

| 变更项 | JEP 434 (JDK 20) | JEP 442 (JDK 21) |
|--------|-------------------|-------------------|
| **Arena 替代 ResourceScope** | 使用 SegmentScope（原 ResourceScope） | 引入 Arena 作为内存分配和生命周期管理的统一入口 |
| **Linker API 精化** | 基础的 downcall/upcall | Linker 选项增强（如 `captureCallState` 捕获 errno） |
| **MemorySegment 改进** | 段与地址已统一 | 零长度段的处理更加安全和一致 |
| **布局常量命名** | C 风格命名（`C_INT`） | 采用 Java 风格命名（`ValueLayout.JAVA_INT`），同时保留平台相关常量 |

---

## 3. 核心 API 概览

- **Arena** — 管理 MemorySegment 生命周期的统一入口，支持 confined、shared、auto 和 global 四种模式
- **MemorySegment** — 表示一段连续的内存区域（堆内或堆外）
- **Linker** — 连接 Java 代码与本地代码，支持 downcall 和 upcall
- **SymbolLookup** — 查找本地库中的函数地址
- **FunctionDescriptor** — 描述本地函数的参数和返回值布局

---

## 4. 代码示例

```java
// 使用 Arena 调用 C 标准库的 strlen
Linker linker = Linker.nativeLinker();
MethodHandle strlen = linker.downcallHandle(
    linker.defaultLookup().find("strlen").orElseThrow(),
    FunctionDescriptor.of(ValueLayout.JAVA_LONG, ValueLayout.ADDRESS)
);
try (Arena arena = Arena.ofConfined()) {
    MemorySegment str = arena.allocateFrom("Hello");
    long len = (long) strlen.invoke(str); // 5
}
```
