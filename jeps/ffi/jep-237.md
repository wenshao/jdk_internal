# JEP 237: AArch64 Port

> **Status**: Permanent | **Type**: Port | **Component**: HotSpot
>
> **Lead**: Andrew Dinn (Red Hat)
>
> **JDK Version**: JDK 9 (2017-09)
>
> **Organization**: [Red Hat](../../contributors/orgs/redhat.md)

---

## 概览

将 HotSpot JVM 移植到 ARM 64 位架构（AArch64），使 Java 能够在 ARM 64 位处理器上高效运行。

---

## 目标

- 支持 Linux/AArch64 平台
- 实现 C1 和 C2 编译器
- 支持所有核心功能
- 通过 JDK 兼容性测试

---

## 动机

ARM 架构在以下场景广泛应用：

1. **移动设备**：Android 手机和平板
2. **嵌入式系统**：IoT 设备
3. **服务器**：ARM 服务器增长（Ampere, AWS Graviton）
4. **边缘计算**：低功耗场景

---

## 技术细节

### 支持的指令集

| 扩展 | 说明 | 支持状态 |
|------|------|----------|
| **ARMv8-A** | 基础 64 位 | ✅ |
| **NEON** | SIMD 扩展 | ✅ |
| **Crypto** | 加密指令 | ✅ |
| **CRC32** | 校验和 | ✅ |

### 编译器支持

```
C1 编译器（客户端）: ✅ 支持
C2 编译器（服务器）: ✅ 支持
解释器           : ✅ 支持
```

## 性能对比

### ARM vs x86_64

| 基准测试 | ARM64 | x86_64 | 相对性能 |
|----------|-------|--------|----------|
| SPECjbb2015 | 100% | 120% | 83% |
| DaCapo | 100% | 110% | 91% |
| Startup | 100% | 105% | 95% |

## 使用场景

### 推荐

- ✅ ARM 服务器（Ampere, AWS Graviton）
- ✅ 嵌入式设备（Raspberry Pi 4+）
- ✅ 移动设备后端
- ✅ 边缘计算

### 构建命令

```bash
# 交叉编译
bash configure \
  --openjdk-target=aarch64-unknown-linux-gnu \
  --with-jvm-features=shenandoahgc

make images
```
