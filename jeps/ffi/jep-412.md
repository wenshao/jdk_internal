## 1. 概述

JEP 412 是 Project Panama 的里程碑——首次将 Foreign-Memory Access API（JEP 370/383/393）和 Foreign Linker API（JEP 389）合并为统一的 **Foreign Function & Memory API**，以 `jdk.incubator.foreign` 模块孵化。该合并 API 允许 Java 程序直接调用本地函数并安全操作外部内存，无需编写 JNI 代码。

---

## 2. 合并背景

JDK 16 中，Foreign-Memory Access API（JEP 393）和 Foreign Linker API（JEP 389）作为两个独立 JEP 并行孵化。实践表明两者高度关联——调用本地函数需要传递和接收外部内存。JDK 17 将二者合并为单一 API，统一了内存管理与函数调用的生命周期模型。

| 来源 | 贡献 |
|------|------|
| Foreign-Memory Access (JEP 393) | MemorySegment、MemoryAddress、MemoryLayout |
| Foreign Linker (JEP 389) | CLinker、FunctionDescriptor、本地函数调用 |
| 新增 | ResourceScope — 统一的资源生命周期管理 |

---

## 3. 核心 API 概览

- **CLinker** — 提供 Java 与 C 函数之间的互操作，支持 downcall（Java 调用 C）和 upcall（C 回调 Java）
- **MemorySegment** — 表示一段外部内存，绑定到 ResourceScope
- **ResourceScope** — 管理内存段和其他本地资源的生命周期，支持确定性释放
- **FunctionDescriptor** — 描述本地函数的签名（参数和返回值的内存布局）
- **SymbolLookup** — 查找本地库中的函数符号

---

## 4. 代码示例

```java
// 调用 C 标准库的 strlen 函数
CLinker linker = CLinker.getInstance();
MethodHandle strlen = linker.downcallHandle(
    CLinker.systemLookup().lookup("strlen").get(),
    MethodType.methodType(long.class, MemoryAddress.class),
    FunctionDescriptor.of(CLinker.C_LONG, CLinker.C_POINTER)
);
try (ResourceScope scope = ResourceScope.newConfinedScope()) {
    MemorySegment str = CLinker.toCString("Hello", scope);
    long len = (long) strlen.invoke(str.address()); // 5
}
```
