# JEP 422: Linux/RISC-V Port

> **Status**: Permanent | **Type**: Port | **Component**: HotSpot
>
> **Lead**: [Fei Yang](../../by-contributor/profiles/fei-yang.md) (Huawei)
>
> **JDK Version**: JDK 19 (2022-09)
>
> **Organization**: [Huawei](../../contributors/orgs/huawei.md)

---

## 概览

将 RISC-V 架构移植到 Linux 平台，使 Java 能够在 RISC-V 处理器上运行。这是 RISC-V 架构的官方支持开始。

---

## 目标

- 支持 Linux/RISC-V 平台
- 实现客户端和服务器编译器
- 支持所有核心功能
- 通过 JDK 兼容性测试（JCK）

---

## 动机

RISC-V 是一个开放的指令集架构（ISA），具有以下优势：

1. **开放标准**：免版税，开放授权
2. **模块化设计**：基础 ISA + 可选扩展
3. **生态系统增长**：硬件和软件生态快速发展
4. **中国自主可控**：减少对外部架构依赖

---

## 技术细节

### 支持的 RISC-V 扩展

| 扩展 | 说明 | 支持状态 |
|------|------|----------|
| **RV64G** | 基础 64 位 + 通用扩展 | ✅ |
| **Zicsr** | 控制状态寄存器 | ✅ |
| **Zifencei** | 指令获取屏障 | ✅ |
| **Zba** | 位操作加速 | ✅ |
| **Zbb** | 基础位操作 | ✅ |
| **Zbc** | 进位操作 | ✅ |
| **Vector (V)** | 向量扩展 | JDK 23+ |

### 编译器支持

```
C1 编译器（客户端）: ✅ 支持
C2 编译器（服务器）: ✅ 支持
解释器           : ✅ 支持
```

## 构建指南

### 前置条件

```bash
# 安装 RISC-V 工具链
sudo apt install gcc-riscv64-linux-gnu g++-riscv64-linux-gnu

# 或使用交叉编译
export CROSS_COMPILE=riscv64-linux-gnu-
```

### 构建命令

```bash
# 配置
bash configure \
  --openjdk-target=riscv64-unknown-linux-gnu \
  --with-jvm-features=shenandoahgc \
  --disable-warnings-as-errors

# 构建
make images

# 验证
build/linux-riscv64-server-release/jdk/bin/java -version
```

## 性能对比

### 基准测试（RISC-V vs x86_64）

|----------|--------|--------|----------|

## 使用场景

### 推荐

- ✅ RISC-V 开发板（如 HiFive）
- ✅ RISC-V 服务器
- ✅ 嵌入式系统
- ✅ 学术研究和教学

### 限制

- ⚠️ 性能优化仍在进行中
- ⚠️ 部分 native 库可能不兼容
- ⚠️ 工具链生态不如 x86 成熟
