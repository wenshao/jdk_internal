## 1. 概述

JEP 330 增强了 `java` 启动器（launcher），使其能够直接运行 Java 源文件，无需显式调用 `javac` 编译：

```bash
java HelloWorld.java
```

启动器在内存中自动编译源文件并立即执行，整个过程对用户透明。这一改进大幅降低了运行简单 Java 程序的仪式感（ceremony），使 Java 在脚本化、教学和快速原型验证场景中更加实用。

---

## 2. 背景

Java 传统流程要求先编译再运行（`javac HelloWorld.java` → `java HelloWorld`），对于简单程序带来不必要的摩擦：初学者必须理解 `javac`/`java` 两个命令的区别；相比 `python script.py` 的一步执行多了编译步骤；即使只测试一小段逻辑也会留下 `.class` 文件。Python、Ruby、Go、Kotlin 等语言均支持直接运行源文件，Java 缺乏这种便利性是一个显著不足。

JDK 9 的 JShell（[JEP 222](jep-222.md)）解决了交互式探索的问题，但不适合编写完整的、可重复执行的脚本。JEP 330 填补了这一空白——面向单文件的完整 Java 程序而非交互式片段。同时，Unix/Linux 的 shebang（`#!`）机制也需要 `java` 命令能接受源文件作为输入。

---

## 3. 工作原理

### 3.1 内部编译流程

当 `java` 启动器检测到参数是 `.java` 文件时，切换到 "source-file mode"：

```
java HelloWorld.java arg1 arg2
     ↓ 检测到 .java 后缀
     ↓ 调用 javac 内部 API（内存编译，不写 .class 文件）
     ↓ 加载字节码到专用类加载器
     ↓ 查找第一个类的 main() 方法并执行
```

也可用 `--source` 显式指定语言版本（此时文件名不必以 `.java` 结尾）：

```bash
java --source 11 my-script arg1 arg2
```

### 3.2 内存编译

关键设计：**不在磁盘上写入 `.class` 文件**。编译在内存中完成，字节码直接加载执行，进程结束后一切消失。每次运行都重新编译，但对单文件程序的启动延迟通常可忽略。

### 3.3 入口类与命令行参数

源文件中**第一个类**必须包含 `main()` 方法。文件中可定义多个辅助类，但都必须在同一文件中。

在 source-file mode 下，`.java` 文件替代传统模式中的类名位置：`java -cp lib MyApp.java arg1 arg2`。可通过 `--enable-preview` 等选项启用预览功能：`java --source 21 --enable-preview MyApp.java`

## 4. Shebang 支持

JEP 330 为 Java 源文件引入了 Unix shebang 支持：

```java
#!/usr/bin/java --source 11

public class HelloScript {
    public static void main(String[] args) {
        System.out.println("Hello from a Java script!");
    }
}
```

```bash
chmod +x hello-script
./hello-script
```

**规则**：shebang 必须在首行；使用 shebang 时文件名不应以 `.java` 结尾；`--source` 在 shebang 中是必需的；编译器自动忽略 shebang 行。

为提高可移植性，可使用 `env`：`#!/usr/bin/env -S java --source 17`

## 5. 使用场景

- **脚本与自动化**：无需构建工具，直接用 Java 编写 shell 脚本式的工具，利用 NIO、HTTP Client 等标准库
- **学习与教学**：学生只需 `java HelloWorld.java` 一条命令，教师可在教学早期跳过编译概念
- **快速原型**：开发者无需创建 Maven/Gradle 项目即可验证想法
- **DevOps**：运维人员可利用 Java 丰富的标准库编写跨平台可移植的运维脚本

---

## 6. 限制与约束

| 限制 | 说明 |
|------|------|
| **单文件** | 所有类必须在同一个文件中，不能引用其他源文件（直到 JEP 458 解除） |
| **无依赖管理** | 可通过 `-cp` 添加 JAR，但无自动下载机制 |
| **每次重编译** | 不缓存编译结果，每次运行都重新编译 |
| **不适合生产** | 设计用于开发/学习/脚本，非性能敏感场景 |

## 7. 演进路线

JEP 330 是 Java 降低入门门槛系列改进的起点：

| JEP | JDK | 标题 | 改进 |
|-----|-----|------|------|
| **JEP 330** | **JDK 11** | Launch Single-File Source-Code Programs | 单文件直接运行，shebang 支持 |
| [JEP 458](jep-458.md) | JDK 22 | Launch Multi-File Source-Code Programs | 突破单文件限制，自动编译依赖的源文件 |
| [JEP 512](jep-512.md) | JDK 25 | Compact Source Files and Instance Main Methods | 简化 main 声明，隐式类 |

**JEP 458**（JDK 22）解除了单文件限制——当主源文件引用其他源文件中的类时，启动器自动查找并编译。

**JEP 512**（JDK 25）进一步减少样板代码，最简单的程序只需：

```java
void main() {
    println("Hello, World!");
}
```

结合 source-file mode，`java HelloWorld.java` 即可运行。

## 8. 与脚本语言的比较

| 特性 | Java (JEP 330) | Python | Bash | Go (`go run`) |
|------|----------------|--------|------|---------------|
| 一步运行 | `java X.java` | `python x.py` | `./x.sh` | `go run x.go` |
| Shebang | 是 | 是 | 是 | 否 |
| 类型安全 | 编译时 | 运行时 | 无 | 编译时 |
| 启动速度 | 较慢（JVM+编译） | 较快 | 快 | 中等 |
| 标准库 | 非常丰富 | 丰富 | 有限 | 丰富 |

**优势**：编译期类型检查、JIT 执行性能、庞大的标准库与生态、Java 开发者无需学新语言、跨平台可移植性。

**劣势**：JVM 启动 + 编译带来几百毫秒到一两秒的延迟；Java 语法比 Python/Bash 更冗长；缺乏内置依赖自动下载。
