## 1. 概述

JEP 458 扩展了 `java` 启动器的 source-file mode，使其能够运行由**多个 `.java` 源文件**组成的程序，无需显式调用 `javac` 或使用构建工具：

```bash
java Main.java
```

当 `Main.java` 引用了其他 `.java` 文件中定义的类时，启动器会**自动发现并编译**这些依赖文件，所有编译在内存中完成，不产生 `.class` 文件。这一改进将 JEP 330 的单文件便利性推广到多文件程序，使 Java 在中小规模脚本、教学项目和快速原型场景中更加实用。

---

## 2. 从 JEP 330 到 JEP 458

[JEP 330](jep-330.md)（JDK 11）首次让 `java` 命令能直接运行源文件，但有一个关键限制：**所有代码必须在同一个文件中**。对于超过几十行的程序，开发者必须将所有类塞入一个文件，或者回退到传统的 `javac` + `java` 流程。

这一限制在实际使用中带来显著摩擦：将多个类塞入一个文件违反 Java 编码习惯；教师无法演示类分离和包组织；脚本随功能增长自然需要拆分为多文件。

JEP 458 移除了单文件限制，让 source-file mode 适用于由多个源文件组成的程序，同时保持了零配置、一条命令运行的便利性。

## 3. 工作原理

### 3.1 自动依赖解析

当启动器以 source-file mode 运行 `Main.java` 时，遇到未解析的类引用会**自动搜索**对应的 `.java` 文件：

```
java Main.java
     ↓ 编译 Main.java
     ↓ 发现引用 Helper → 搜索 Helper.java（递归解析）
     ↓ 内存中加载所有字节码 → 执行 main()
```

搜索遵循标准目录布局约定（全限定名映射到路径）。默认以主源文件所在目录为根搜索，无包的源文件放在同一目录即可。也可通过 `--source-path` 指定搜索路径：`java --source-path src src/Main.java`

### 3.2 内存编译与入口点

与 JEP 330 一致，所有编译在内存中完成——不产生 `.class` 文件，不缓存编译结果，每次运行都重新编译。入口点规则也与 JEP 330 相同：命令行指定的 `.java` 文件中的**第一个类**必须包含 `main()` 方法。

## 4. 使用示例

### 4.1 基本多文件程序

```java
// Main.java
class Main {
    public static void main(String[] args) {
        var greeter = new Greeter("World");
        greeter.greet();
    }
}
```

```java
// Greeter.java
class Greeter {
    private final String name;
    Greeter(String name) { this.name = name; }
    void greet() { System.out.println("Hello, " + name + "!"); }
}
```

```bash
java Main.java
# 输出: Hello, World!
```

无需任何额外配置，启动器自动发现并编译同目录下的 `Greeter.java`。

### 4.2 带包结构的程序

```
src/
  app/
    Main.java        # package app;
    model/
      User.java      # package app.model;
    util/
      Formatter.java # package app.util;
```

```bash
java --source-path src src/app/Main.java
```

启动器根据 `import app.model.User` 等声明自动定位并编译相关源文件。多文件模式同样支持 `--enable-preview` 等编译选项，所有被编译的源文件共享相同的编译设置。

## 5. 限制与约束

| 限制 | 说明 |
|------|------|
| **无外部依赖/JAR** | 不能自动下载或管理第三方 JAR；可手动通过 `-cp` 添加 |
| **每次重编译** | 不缓存编译结果，每次运行都编译所有源文件 |
| **无模块支持** | 不支持 `module-info.java`，仅适用于非模块化程序 |
| **源文件布局** | 有包的类必须遵循标准目录布局（包名映射到目录路径） |
| **不适合生产** | 设计用于开发、学习、脚本场景，非性能敏感的生产环境 |

对于需要外部依赖的项目，仍需使用 Maven、Gradle 等构建工具，或手动管理 classpath。

## 6. 演进路线

JEP 458 是 Java 持续降低入门门槛和减少仪式感（ceremony）系列改进中的关键一环：

| JEP | JDK | 标题 | 改进 |
|-----|-----|------|------|
| [JEP 330](jep-330.md) | JDK 11 | Launch Single-File Source-Code Programs | 单文件直接运行，shebang 支持 |
| **JEP 458** | **JDK 22** | **Launch Multi-File Source-Code Programs** | **突破单文件限制，自动编译依赖源文件** |
| [JEP 512](../language/jep-512.md) | JDK 25 | Compact Source Files and Instance Main Methods | 紧凑源文件，简化 main 声明 |

**JEP 512**（JDK 22）在 JEP 458 的基础上进一步演进，引入紧凑源文件（Compact Source Files）和实例 main 方法，并允许通过声明式语法引入模块依赖，部分解决了外部依赖的问题。结合隐式声明的类，最简单的程序只需：

```java
void main() {
    println("Hello, World!");
}
```

从 JEP 330 到 JEP 458 再到 JEP 512，Java 在保持类型安全和编译期检查的同时，逐步逼近脚本语言的易用性。
