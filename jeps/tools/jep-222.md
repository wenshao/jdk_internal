## 1. 概述

JEP 222 为 JDK 引入了 `jshell`——Java 平台的官方 REPL（Read-Eval-Print Loop）工具。用户可以在交互式环境中逐行输入 Java 代码片段（snippet），立即看到执行结果，无需编写完整的类、`main` 方法或编译步骤。该工具同时提供了 `jdk.jshell` 模块，允许 IDE 和其他工具通过 API 编程式地集成 REPL 功能。

---

## 2. 背景

### 2.1 Java 缺乏 REPL 的历史

在 JDK 9 之前，Java 是少数没有官方 REPL 的主流编程语言。Python、Ruby、Scala、Groovy 等语言都内置了交互式 shell，而 Java 开发者即使只想测试一个简单表达式，也必须：

1. 创建一个 `.java` 文件
2. 编写 `public class` + `public static void main`
3. 用 `javac` 编译
4. 用 `java` 运行

这一流程对初学者尤其不友好，大幅提高了 Java 的学习门槛。

### 2.2 第三方尝试

社区曾出现过 BeanShell、DrJava 等工具提供类似 REPL 的功能，但它们要么不完全兼容 Java 语法，要么缺乏与 JDK 的深度集成，无法成为标准方案。

### 2.3 Project Kulla

JEP 222 源自 **Project Kulla**（2014 年启动），由 Robert Field 主导。项目名称 "Kulla" 来自美索不达米亚神话中的砖匠之神——寓意一砖一瓦地交互式构建程序。Kulla 的目标是为 Java 提供一个一流的、与语言规范紧密对齐的 REPL 体验。

## 3. 关键特性

### 3.1 表达式求值与即时反馈

`jshell` 允许直接输入表达式、语句和声明，无需包装在类或方法中：

```
jshell> 2 + 3
$1 ==> 5

jshell> "Hello".toUpperCase()
$2 ==> "HELLO"

jshell> Math.PI * 10 * 10
$3 ==> 314.1592653589793
```

表达式的结果会自动赋值给临时变量（`$1`、`$2` 等），可以在后续代码中引用。不需要 `System.out.println()`，结果自动打印。

### 3.2 方法与类定义

用户可以在 REPL 中定义方法、类、接口和 record，并支持 forward reference（前向引用）：

```
jshell> int add(int a, int b) { return a + b; }
|  created method add(int,int)

jshell> add(3, 4)
$2 ==> 7

jshell> class Point { int x, y; Point(int x, int y) { this.x = x; this.y = y; } }
|  created class Point

jshell> new Point(1, 2)
$4 ==> Point@6d06d69c
```

方法和类可以随时重新定义，新定义会覆盖旧的版本，适合迭代式开发。

### 3.3 自动补全与 /commands

**Tab 补全**：按 Tab 键可以自动补全类名、方法名、变量名，还能显示方法签名和 Javadoc 摘要。

**内置命令**（以 `/` 开头）：

| 命令 | 功能 |
|------|------|
| `/list` | 列出所有已输入的 snippet |
| `/edit` | 在外部编辑器中编辑 snippet |
| `/drop` | 删除指定 snippet |
| `/save` | 保存 snippet 到文件 |
| `/open` | 从文件加载 snippet |
| `/vars` | 列出所有变量 |
| `/methods` | 列出所有方法 |
| `/types` | 列出所有类型定义 |
| `/imports` | 列出当前导入 |
| `/reset` | 重置 JShell 状态 |
| `/exit` | 退出 JShell |

### 3.4 Snippet 编辑与历史

- **内联编辑**：支持通过上下方向键浏览历史记录，Ctrl+R 反向搜索
- **外部编辑器**：`/edit` 命令可启动外部编辑器（通过 `/set editor` 配置），在编辑器中修改多行代码后保存即可执行
- **Snippet ID**：每个 snippet 都有唯一编号，可以用 `/list 5` 或 `/drop 5` 精确操作

### 3.5 Feedback Modes

`jshell` 提供四种反馈模式，控制输出的详细程度：

| 模式 | 说明 |
|------|------|
| `verbose` | 显示完整信息（值、类型、名称）及附加提示 |
| `normal` | 默认模式，显示必要的反馈 |
| `concise` | 简洁模式，减少输出 |
| `silent` | 静默模式，仅显示显式输出 |

通过 `/set feedback <mode>` 切换。也可以用 `/set mode` 自定义反馈模式。

## 4. 架构

### 4.1 JShell API (jdk.jshell 模块)

`jshell` 工具基于 `jdk.jshell` 模块构建，该模块提供了完整的编程式 API。核心类：

| 类 | 职责 |
|------|------|
| `JShell` | 入口类，代表一个 JShell 评估状态 |
| `Snippet` | 表示一个代码片段（变量、方法、类、表达式等） |
| `SnippetEvent` | 描述 snippet 状态变化的事件 |
| `SourceCodeAnalysis` | 提供代码补全、文档查询等分析功能 |
| `Diag` | 编译诊断信息（错误、警告） |

模块结构：

```
jdk.jshell
├── jdk.jshell          // 核心 API（JShell, Snippet, SnippetEvent 等）
├── jdk.jshell.spi      // SPI：ExecutionControl, ExecutionEnv
└── jdk.jshell.tool     // 命令行工具实现
```

### 4.2 编译管道

`jshell` 的编译管道将用户输入的代码片段转换为可执行代码：

1. **解析（Parse）**：分析输入文本，判断 snippet 类型（表达式、声明、语句等）
2. **包装（Wrap）**：将 snippet 包装进合成类中——例如表达式 `2 + 3` 被包装为合成类中某个方法的 return 语句
3. **编译（Compile）**：使用 javac 编译包装后的代码，支持 forward reference（引用尚未定义的方法/类）
4. **加载（Load）**：将编译后的字节码发送到执行引擎

```
用户输入: int x = 42;
    ↓ Parse
snippet 类型: VarSnippet
    ↓ Wrap
class REPL$$Group1 { static int x = 42; }
    ↓ Compile (javac)
字节码
    ↓ Load
发送到 Remote JVM 执行
```

### 4.3 执行引擎 (Remote JVM)

默认情况下，`jshell` 在独立的远程 JVM 进程中执行用户代码。这种设计有重要的安全和稳定性考量：

| 设计决策 | 原因 |
|----------|------|
| **进程隔离** | 用户代码中的 `System.exit()` 或无限循环不会终止 JShell 工具本身 |
| **类加载隔离** | 用户代码的类路径与 JShell 工具的类路径分离 |
| **可恢复性** | 远程 JVM 崩溃后可以自动重启 |

执行引擎通过 SPI（`jdk.jshell.spi.ExecutionControl`）可替换，支持自定义执行环境（如本地执行、远程服务器执行等）。

## 5. 使用场景

### 5.1 学习与教学

`jshell` 降低了 Java 的入门门槛：学生可以直接输入表达式和语句，立即观察结果，无需理解类、包、编译等概念：

```
jshell> var list = List.of("Java", "Kotlin", "Scala")
list ==> [Java, Kotlin, Scala]

jshell> list.stream().filter(s -> s.startsWith("J")).toList()
$2 ==> [Java]
```

### 5.2 快速原型验证

开发者可以在 JShell 中快速测试 API 行为、验证正则表达式、尝试不同的数据结构，而无需创建临时项目。

### 5.3 调试与探索

通过 `/classpath` 添加项目依赖后，可以在 JShell 中交互式调用项目中的类和方法，探索第三方库的行为。

### 5.4 脚本式使用

`jshell` 支持从文件执行脚本（`.jsh` 文件），适合自动化任务：

```bash
jshell script.jsh
```

也可以通过管道传入代码：`echo '2+3' | jshell`

## 6. API：编程式访问

`jdk.jshell` 模块的 API 允许 IDE 和工具集成 JShell 功能：

```java
import jdk.jshell.*;

// 创建 JShell 实例
try (JShell jshell = JShell.create()) {
    // 执行代码片段
    List<SnippetEvent> events = jshell.eval("int x = 42;");

    for (SnippetEvent event : events) {
        if (event.status() == Snippet.Status.VALID) {
            System.out.println("Value: " + event.value());
        }
    }

    // 代码补全
    SourceCodeAnalysis analysis = jshell.sourceCodeAnalysis();
    List<SourceCodeAnalysis.Suggestion> suggestions =
        analysis.completionSuggestions("System.out.print", 17, new int[1]);
}
```

这使得 IntelliJ IDEA、Eclipse、NetBeans 等 IDE 可以嵌入 JShell 作为交互式控制台，提供比传统 "Evaluate Expression" 更强大的功能。
