## 1. 概述

JEP 414 是 Vector API 的第二轮孵化（`jdk.incubator.vector`），在 JEP 338（JDK 17）基础上继续演进。该 API 提供了一种在 Java 中表达向量计算的方式，能够在支持的 CPU 架构上可靠地编译为 SIMD 指令（如 x64 的 SSE/AVX、AArch64 的 NEON/SVE），从而显著提升数据并行计算的性能。

---

## 2. 与前一轮的变更

| 变更项 | JEP 338 (JDK 17) | JEP 414 (JDK 17) |
|--------|-------------------|-------------------|
| **性能改进** | 初始 JIT 编译支持 | 增强了超字操作（superword）优化，改善 SIMD 代码生成 |
| **新操作** | 基础算术与逻辑操作 | 新增 bitwise 逐位操作（`LEADING_ZEROS_COUNT` 等）和跨道操作 |
| **字符转换** | 无 | 支持 byte 向量与 boolean 数组之间的转换 |
| **掩码改进** | 基础掩码支持 | 改善了掩码操作的性能和可用性 |

---

## 3. 核心 API 概览

- **VectorSpecies** — 描述向量的形状（位宽）和元素类型
- **Vector<E>** — 抽象基类，表示一个固定大小的 SIMD 向量
- **IntVector / FloatVector / ...** — 具体元素类型的向量类
- **VectorMask** — 用于条件操作的掩码
- **VectorShuffle** — 用于元素重排的 shuffle 描述符

---

## 4. 代码示例

```java
static final VectorSpecies<Float> SPECIES = FloatVector.SPECIES_256;

void vectorAdd(float[] a, float[] b, float[] c) {
    int i = 0;
    for (; i < SPECIES.loopBound(a.length); i += SPECIES.length()) {
        FloatVector va = FloatVector.fromArray(SPECIES, a, i);
        FloatVector vb = FloatVector.fromArray(SPECIES, b, i);
        va.add(vb).intoArray(c, i);
    }
    for (; i < a.length; i++) c[i] = a[i] + b[i]; // tail
}
```

## 5. 长期孵化原因

Vector API 的长期孵化（截至 JDK 26 仍在孵化）主要因为其最终形态依赖 Project Valhalla 引入的 **值类型（value types）**。向量值应当是值类型以避免堆分配开销，但 Valhalla 尚未交付。在此之前，API 保持孵化状态以避免锁定次优设计。
