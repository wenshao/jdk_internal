> **Status**: Permanent | **Type**: Feature | **Component**: JDK Tools
>
> **Lead**: Daniel Fuchs (Oracle)

> **JDK Version**: JDK 18 (2022-03)
>

## 概览

提供一个简单的 Web 服务器作为 JDK 工具，用于快速测试和原型开发，无需外部依赖。

---

## 目标

- 提供零配置的 Web 服务器
- 用于快速测试和演示
- 无需外部依赖
- 简单易用

---

## 使用方式

### 命令行

```bash
# 启动服务器（默认端口 8000）
jwebserver

# 指定端口
jwebserver -p 8080

# 指定目录
jwebserver -d /path/to/webroot

# 绑定特定地址
jwebserver -b 127.0.0.1

# 显示帮助
jwebserver -h
```

### 输出示例

```
$ jwebserver -p 8080
Listening on http://127.0.0.1:8080/

127.0.0.1 - - [21/Mar/2026:10:00:00 +0800] "GET / HTTP/1.1" 200 1234
127.0.0.1 - - [21/Mar/2026:10:00:01 +0800] "GET /style.css HTTP/1.1" 200 567
```

## 功能特性

### 支持的功能

| 功能 | 说明 |
|------|------|
| **静态文件服务** | HTML, CSS, JS, 图片等 |
| **目录列表** | 自动列出目录内容 |
| **MIME 类型** | 自动检测常见类型 |
| **HTTP 日志** | 类似 Apache 的访问日志 |
| **HTTPS** | ❌ 不支持 |
| **虚拟主机** | ❌ 不支持 |
| **URL 重写** | ❌ 不支持 |

### 默认 MIME 类型

```properties
.html=text/html
.css=text/css
.js=application/javascript
.png=image/png
.jpg=image/jpeg
.gif=image/gif
.json=application/json
.xml=application/xml
```

## 使用场景

### 推荐

```bash
# 1. 快速测试静态页面
cd my-website
jwebserver

# 2. 前端开发
cd frontend/dist
jwebserver -p 3000

# 3. 演示原型
cd demo
jwebserver

# 4. 测试 API 响应
echo '{"status":"ok"}' > api.json
jwebserver
curl http://localhost:8000/api.json
```

### 不推荐

```bash
# ❌ 生产环境
# ❌ 需要 HTTPS 的场景
# ❌ 需要认证的场景
# ❌ 高并发场景
```

## 安全考虑

### 默认配置

- 仅绑定 `127.0.0.1`（本地回环）
- 不暴露到外部网络
- 无目录遍历保护

### 安全建议

```bash
# ✅ 仅本地测试
jwebserver -b 127.0.0.1

# ⚠️ 如需外部访问，明确指定
jwebserver -b 0.0.0.0  # 谨慎使用

# ❌ 不要暴露敏感目录
jwebserver -d /etc  # 危险！
```

## 与其他工具对比

| 工具 | 安装 | 配置 | 功能 | 适用场景 |

## 示例

### 服务静态网站

```bash
# 目录结构
my-site/
├── index.html
├── style.css
└── images/
    └── logo.png

# 启动服务器
cd my-site
jwebserver -p 8080

# 访问
# http://localhost:8080/
# http://localhost:8080/style.css
# http://localhost:8080/images/logo.png
```

### 目录列表

```bash
# 访问没有 index.html 的目录
# 自动显示目录列表：
# Index of /images/
# - logo.png
# - banner.jpg
```
