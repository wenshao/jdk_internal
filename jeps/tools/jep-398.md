## 1. 概述

JEP 398 在 JDK 17 中将 Applet API 标记为 `@Deprecated(forRemoval=true)`，这是
Applet 退役进程中的关键一步。Applet API 最初在 JDK 9 中被普通废弃（JEP 289），
JEP 398 将其升级为"计划移除"状态。

### 1.1 核心变更

- **`java.applet` 包**：整个包标注 `@Deprecated(since="9", forRemoval=true)`
- **`javax.swing.JApplet`**：Swing 版 Applet 基类同样标记为 forRemoval
- **`java.beans.AppletInitializer`**：关联的 Bean 初始化接口一并废弃
- **编译警告升级**：使用上述 API 的代码在编译时触发 `removal` 级别警告

## 2. 背景：Applet 的兴衰

### 2.1 从辉煌到消亡

Java Applet 是 Java 平台最早的杀手级应用（1995），依赖浏览器 NPAPI 插件架构运行。
随着安全问题频发和 Web 技术进步，主流浏览器逐步淘汰了 NPAPI：

| 浏览器 | NPAPI 移除时间 |
|--------|--------------|
| Google Chrome | 2015 年 9 月（Chrome 45） |
| Mozilla Firefox | 2017 年（Firefox 52 ESR 结束） |
| Microsoft Edge / Safari | 从未支持 / 2017 年默认禁用 |

与 Applet 密切相关的 Java Web Start（JNLP）也同步退场——JDK 9 废弃，JDK 11 移除。

## 3. JDK 17 中的变更

### 3.1 废弃的 API 清单

```
java.applet.Applet                    # Applet 基类
java.applet.AppletContext             # Applet 运行环境上下文
java.applet.AppletStub                # Applet 与浏览器的通信接口
java.applet.AudioClip                 # Applet 音频播放接口
javax.swing.JApplet                   # Swing Applet 基类
java.beans.AppletInitializer          # Bean 在 Applet 中的初始化接口
```

### 3.2 废弃级别变化

| 版本 | 废弃状态 | JEP |
|------|---------|-----|
| JDK 9 | `@Deprecated(since="9")` | JEP 289 |
| **JDK 17** | **`@Deprecated(since="9", forRemoval=true)`** | **JEP 398** |

---

## 4. 为何标记 forRemoval

Applet 所依赖的浏览器插件架构已被全部主流浏览器移除，API 毫无实际用途。Applet
API 与 AWT/Swing 事件模型、Security Manager、类加载机制深度耦合，保留废弃代码
增加了客户端库复杂性。Applet 沙箱高度依赖 Security Manager，而 JEP 411 同时在
JDK 17 中废弃了 Security Manager，两者协同推进。

---

## 5. 废弃与移除时间线

| 时间 | 版本 | 事件 |
|------|------|------|
| 1995 | JDK 1.0 | Applet API 引入 |
| 2015-2017 | — | 主流浏览器移除 NPAPI 插件支持 |
| 2017 | JDK 9 | JEP 289：Applet API 标记为 Deprecated |
| 2018 | JDK 11 | Java Plugin 和 Web Start 从 JDK 中移除 |
| **2021** | **JDK 17** | **JEP 398：升级为 Deprecated(forRemoval=true)** |

---

## 6. 迁移建议

| 当前用法 | 迁移方案 |
|---------|---------|
| 企业内网 Applet | 重写为 Web 应用（HTML5 + JavaScript/WebAssembly） |
| 数据可视化 Applet | 使用 JavaFX 桌面应用或 Web 前端框架 |
| 继承 `JApplet` | 重构为 `JFrame` 或 `JPanel` 桌面应用 |
| Java Web Start 部署 | 使用 jpackage（JDK 14+）打包独立应用 |
