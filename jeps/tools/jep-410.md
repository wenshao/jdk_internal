## 1. 概述

JEP 410 在 JDK 17 中移除了基于 Graal 编译器的实验性提前编译（AOT）功能和实验性
即时编译（JIT）功能。这包括 `jaotc` 工具、`jdk.aot` 模块以及 `jdk.internal.vm.compiler`
模块。

### 1.1 核心变更

- **移除 `jaotc` 工具**：不再提供 Java 提前编译命令行工具
- **移除 `jdk.aot` 模块**：包含 AOT 编译基础设施的模块被删除
- **移除 `jdk.internal.vm.compiler` 模块**：Graal JIT 编译器从 JDK 中删除
- **移除 `jdk.internal.vm.compiler.management` 模块**：Graal 管理接口一并删除

## 2. 背景：Graal 在 JDK 中的实验之路

### 2.1 JEP 295：AOT 编译（JDK 9, 2017）

JDK 9 通过 JEP 295 引入了实验性的 `jaotc` 工具，允许开发者将 Java 类预编译为
本地共享库（.so 文件），减少启动时间。该功能基于 Graal 编译器，仅在 Linux/x64
平台上可用，且标记为实验性。

### 2.2 JEP 317：Graal JIT 编译器（JDK 10, 2018）

JDK 10 通过 JEP 317 将 Graal 作为实验性 JIT 编译器集成到 HotSpot 中。使用者可通过
`-XX:+UnlockExperimentalVMOptions -XX:+UseJVMCICompiler` 启用 Graal 替代 C2
作为顶层 JIT 编译器。然而 Graal 编译器在 JDK 中以源代码形式维护，需要与上游
GraalVM 项目保持同步，每次 JDK 内部 API 变更都需额外更新。

## 3. JDK 17 中的变更

### 3.1 移除的模块与工具

```
jdk.aot                              # jaotc 工具及 AOT 编译支持
jdk.internal.vm.compiler             # Graal JIT 编译器实现
jdk.internal.vm.compiler.management  # Graal MBean 管理接口
jaotc                                # AOT 编译命令行工具
```

### 3.2 保留的组件

| 组件 | 状态 |
|------|------|
| JVMCI（JVM 编译器接口） | **保留** — 仍可加载外部 JIT 编译器 |
| C1 / C2 编译器 | **保留** — HotSpot 默认 JIT 编译器不受影响 |
| GraalVM（独立发行版） | **不受影响** — 作为独立项目继续发展 |

JVMCI 的保留意味着用户仍然可以通过 GraalVM CE/EE 使用 Graal 编译器，只是不再
随 JDK 发行版一起分发。

---

## 4. 为何移除

### 4.1 采用率极低

AOT 编译（`jaotc`）和实验性 Graal JIT 自引入以来实际使用率非常低。标记为实验性
功能，主流应用和框架几乎没有依赖这些组件。

### 4.2 维护成本与收益不成比例

Graal 编译器代码量庞大（数十万行 Java 代码），与 JDK 内部 API 深度耦合。每次
JDK 发布周期中同步上游 Graal 变更消耗了大量工程资源，而收益极为有限。

### 4.3 GraalVM 作为独立项目更适合

Oracle 持续投资 GraalVM 作为独立产品（CE 和 EE 版本），提供了包括 Native Image、
多语言支持等比 JDK 内嵌 Graal 更完整的体验。移除不影响其独立发展。

### 4.4 释放工程资源

移除后 JDK 团队可将精力集中在 C2 优化、虚拟线程（Loom）、值类型（Valhalla）等
更高优先级的工作上。

---

## 5. 迁移建议

| 当前用法 | 迁移方案 |
|---------|---------|
| 使用 `jaotc` 减少启动时间 | 使用 CDS / AppCDS 或 GraalVM Native Image |
| 使用 Graal JIT 替代 C2 | 下载 GraalVM CE/EE，通过 JVMCI 加载 |
| 依赖 `jdk.aot` 模块 | 迁移至 GraalVM Native Image 工具链 |
| 峰值吞吐优化 | C2 编译器（JDK 默认）或 GraalVM EE JIT |
| 提前编译 | GraalVM Native Image |
