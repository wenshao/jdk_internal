## 1. 背景与动机

### 问题

在 Java 模块化系统中，使用模块的类需要逐个导入包：

```java
// 传统方式：需要多个 import 语句
import java.util.List;
import java.util.ArrayList;
import java.util.Map;
import java.util.HashMap;
import java.io.IOException;
import java.nio.file.Path;
// ... 可能需要几十个 import
```

对于 `java.base` 这样的大型模块，开发者经常需要导入大量常用类，导致：
- 代码冗长，import 列表占据大量空间
- 维护成本高，新增类使用需要添加 import
- 初学者门槛高，需要记住各类所在的包

### 解决方案

JEP 511 引入 **模块导入声明**，允许一次性导入模块的所有导出包：

```java
import module java.base;  // 导入 java.base 模块的所有导出包
```

## 2. 使用示例

### 基本用法

```java
// 以前需要多个 import
import java.util.List;
import java.util.ArrayList;
import java.util.Map;

// 现在只需一行
import module java.base;

public class Example {
    public static void main(String... args) {
        List<String> list = new ArrayList<>();  // 直接使用
        Map<String, Integer> map = Map.of();    // 直接使用
    }
}
```

### 多模块导入

```java
import module java.base;
import module java.logging;

public class LoggerExample {
    private static final Logger logger = Logger.getLogger("MyApp");
    // Logger 来自 java.logging 模块
}
```

### 与普通 import 混用

```java
import module java.base;
import java.sql.Date;  // 明确指定解决歧义

public class DateExample {
    Date sqlDate;      // java.sql.Date
    java.util.Date utilDate;  // java.util.Date (来自模块导入)
}
```

### JShell 中的使用

JShell 默认启动脚本已更新：

```java
// 旧版 DEFAULT.jsh
import java.io.*;
import java.math.*;
import java.net.*;
import java.nio.file.*;
import java.util.*;
import java.util.concurrent.*;
import java.util.function.*;
import java.util.prefs.*;
import java.util.regex.*;
import java.util.stream.*;

// 新版 DEFAULT.jsh (JDK 26)
import module java.base;
```

### 隐式类中使用

```java
// Test.java - 隐式类
import module java.base;

void main() {
    println("Hello, World!");
}
```

## 3. 实现分析

### 编译器处理流程

```
源码解析
    ↓
遇到 "import module X"
    ↓
解析模块 X 的模块描述符
    ↓
获取所有导出包列表
    ↓
将这些包加入导入作用域
    ↓
后续类型解析时搜索这些包
```

### 关键源码变更

#### 1. ImportTree API 扩展

```java
// src/jdk.compiler/share/classes/com/sun/source/tree/ImportTree.java
public interface ImportTree extends Tree {
    boolean isModule();  // 新增：判断是否为模块导入
    Tree getQualifiedIdentifier();
    String getModuleImportName();  // 新增：获取模块名
}
```

#### 2. 模块信息解析

```java
// src/java.base/share/classes/jdk/internal/module/ModuleInfo.java
// 简化了 requires transitive java.base 的验证逻辑
// 允许模块导入时正确处理依赖关系
```

#### 3. JShell 集成

```java
// src/jdk.jshell/share/classes/jdk/jshell/SnippetMaps.java
// 新增模块导入的映射处理
// 支持在 JShell 中动态添加模块导入
```

### 作用域规则

1. **模块必须可读**: 当前模块必须 `requires` 目标模块
2. **只导入导出包**: 仅导入 `exports` 的包，不导入内部包
3. **传递依赖**: 通过 `requires transitive` 的模块也会被导入
4. **歧义处理**: 多个模块导入导致类名冲突时，需显式 import 指定

## 4. 开发者影响

### 迁移指南

| 场景 | 建议 |
|------|------|
| 新项目 | 推荐使用 `import module java.base` |
| 教学场景 | 大幅简化入门代码 |
| 库开发 | 谨慎使用，考虑向后兼容性 |
| 已有项目 | 可逐步替换，无需强制迁移 |

### 兼容性

- **二进制兼容**: 完全兼容，模块导入编译后与普通 import 相同
- **源码兼容**: 新语法，旧版本编译器不支持
- **IDE 支持**: 需要更新 IDE 以支持新语法

### 最佳实践

```java
// ✅ 推荐：应用入口类
import module java.base;

public class Application {
    public static void main(String[] args) {
        // 简洁的代码
    }
}

// ✅ 推荐：JShell 脚本
import module java.base;
import module java.sql;

// ⚠️ 谨慎：库代码
// 考虑用户可能使用旧版本 JDK
// 建议保留传统 import

// ❌ 避免：导入过多模块
import module java.base;
import module java.sql;
import module java.logging;
import module java.management;
// 可能导致命名冲突
```

## 5. 性能影响

### 编译时

| 指标 | 传统 import | 模块 import |
|------|-------------|-------------|
| 解析时间 | O(n) n=import数 | O(m) m=导出包数 |
| 内存占用 | 较低 | 略高（需加载模块信息） |
| 增量编译 | 无影响 | 无影响 |

### 运行时

- **无影响**: 编译后字节码与传统 import 完全相同
- **类加载**: 无额外开销
