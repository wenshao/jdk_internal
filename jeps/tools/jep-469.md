## 1. 概述

JEP 469 是 Vector API 的第八轮孵化。该 API 提供了一种在 Java 中表达向量计算的方式，使其在运行时可靠地编译为支持的 CPU 架构上的最优向量指令（如 SSE、AVX），从而获得相较于标量计算的显著性能提升。

---

## 2. 变更内容

Vector API 通过 `jdk.incubator.vector` 模块提供，包括 `Vector<E>`、`VectorSpecies` 等核心抽象，允许开发者编写与平台无关的向量化代码，由 HotSpot JIT 编译器映射到硬件 SIMD 指令。

---

## 3. 与前一孵化版的差异

相较于 JEP 460（JDK 22 第七轮孵化），本次孵化的变更较小：

- **细微 API 更新**：根据持续反馈进行了小幅 bug 修复和性能改进。
- **继续等待 Valhalla**：Vector API 的最终定稿依赖于 Project Valhalla 的 value types 特性。value types 能消除装箱开销，使 `Vector<E>` 成为真正的值类型，这对 API 的性能和最终形态至关重要。在 Valhalla 就绪之前，Vector API 将继续以孵化状态迭代。

---

## 4. 核心 API

| 类/接口 | 说明 |
|---------|------|
| `Vector<E>` | 向量的抽象基类 |
| `VectorSpecies<E>` | 描述向量的元素类型和形状 |
| `IntVector`, `FloatVector` 等 | 特定类型的向量实现 |
| `VectorOperators` | 定义向量运算操作 |
| `VectorMask<E>` | 向量掩码，支持条件操作 |

---

## 5. 代码示例

```java
static final VectorSpecies<Float> SPECIES = FloatVector.SPECIES_256;

void vectorAdd(float[] a, float[] b, float[] c) {
    for (int i = 0; i < a.length; i += SPECIES.length()) {
        var va = FloatVector.fromArray(SPECIES, a, i);
        var vb = FloatVector.fromArray(SPECIES, b, i);
        va.add(vb).intoArray(c, i);
    }
}
```
