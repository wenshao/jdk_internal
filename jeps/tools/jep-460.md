## 1. 概述

JEP 460 是 Vector API 的第七轮孵化（`jdk.incubator.vector`），在 JDK 22 中继续提供对 SIMD 向量计算的 Java 级抽象。该 API 允许开发者编写与硬件无关的向量化代码，由 HotSpot JIT 编译器在运行时映射到底层 CPU 的 SIMD 指令（SSE、AVX-512、NEON、SVE 等）。API 本身在本轮中变化不大，重点在于运行时优化和 bug 修复。

本轮的主要目的是保持 API 的活跃状态，同时等待 Project Valhalla 提供的值类型支持。

---

## 3. 为何仍在孵化

Vector API 持续孵化的核心原因是对 **Project Valhalla 值类型** 的依赖：

- **性能关键**：向量值（如 `FloatVector`）应当是值类型（value types），以实现栈分配并消除堆分配开销
- **API 影响**：值类型的引入可能影响 Vector API 的类型层次结构和使用模式
- **时间线**：Valhalla 的值类型（JEP 401）仍在开发中，预计需要更多 JDK 版本才能交付
- **策略**：保持孵化状态可避免将次优的装箱语义锁定为正式 API

在 Valhalla 交付之前，Vector API 将在每个 JDK 版本中以 re-incubation 的形式继续演进。

## 4. 代码示例

```java
// 使用 Vector API 进行 FMA（fused multiply-add）计算
static final VectorSpecies<Float> SPECIES = FloatVector.SPECIES_PREFERRED;

void fma(float[] a, float[] b, float[] c, float[] result) {
    int i = 0;
    for (; i < SPECIES.loopBound(a.length); i += SPECIES.length()) {
        var va = FloatVector.fromArray(SPECIES, a, i);
        var vb = FloatVector.fromArray(SPECIES, b, i);
        var vc = FloatVector.fromArray(SPECIES, c, i);
        va.fma(vb, vc).intoArray(result, i); // a*b + c
    }
    for (; i < a.length; i++) result[i] = Math.fma(a[i], b[i], c[i]);
}
```
