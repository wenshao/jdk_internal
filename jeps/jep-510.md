## 1. 背景与动机

### 问题

密钥派生函数 (KDF) 是密码学中的基础操作，用于：
- 从密码生成密钥 (PBKDF2)
- 从共享密钥派生会话密钥 (HKDF)
- 密钥协商后的密钥派生

之前 Java 没有标准 KDF API，开发者需要：
- 使用第三方库 (Bouncy Castle)
- 自己实现 (容易出错)
- 使用特定算法的专有 API

### 解决方案

JEP 510 提供标准化的 KDF API：
- 统一的 `javax.crypto.KDF` 接口
- 支持 HKDF 等现代算法
- 可扩展的 Provider 机制

## 2. 使用示例

### 基本用法

```java
import javax.crypto.KDF;
import javax.crypto.SecretKey;
import javax.crypto.spec.HKDFParameterSpec;

// 获取 HKDF 实例
KDF kdf = KDF.getInstance("HKDF");

// 准备参数
byte[] salt = generateRandomSalt();
byte[] ikm = getSharedSecret();  // 输入密钥材料

// 派生密钥
HKDFParameterSpec params = HKDFParameterSpec.ofExtract()
    .addSalt(salt)
    .extract(ikm)
    .expand("aes-256-gcm-key".getBytes(), 32);

SecretKey key = kdf.deriveKey("AES", params);
```

### HKDF 完整示例

```java
// HKDF-Extract + HKDF-Expand
public class HKDFExample {
    
    public static SecretKey deriveKey(byte[] ikm, byte[] salt, byte[] info, int length) 
            throws NoSuchAlgorithmException, InvalidAlgorithmParameterException {
        
        KDF kdf = KDF.getInstance("HKDF-SHA256");
        
        HKDFParameterSpec params = HKDFParameterSpec.ofExtract()
            .addSalt(salt)
            .extract(ikm)
            .expand(info, length);
        
        return kdf.deriveKey("AES", params);
    }
    
    public static void main(String[] args) throws Exception {
        byte[] sharedSecret = getECDHSharedSecret();
        byte[] salt = new byte[32];
        new SecureRandom().nextBytes(salt);
        
        // 派生 AES-256 密钥
        SecretKey aesKey = deriveKey(sharedSecret, salt, "encryption".getBytes(), 32);
        
        // 派生 HMAC 密钥
        SecretKey hmacKey = deriveKey(sharedSecret, salt, "mac".getBytes(), 32);
    }
}
```

### 仅 Extract

```java
// 仅执行 Extract 步骤，获取伪随机密钥 (PRK)
KDF kdf = KDF.getInstance("HKDF-SHA256");

HKDFParameterSpec extractOnly = HKDFParameterSpec.ofExtract()
    .addSalt(salt)
    .extract(ikm);

SecretKey prk = kdf.deriveKey("Generic", extractOnly);
```

### 仅 Expand

```java
// 从已有的 PRK 派生多个密钥
HKDFParameterSpec expandOnly = HKDFParameterSpec.expandOnly()
    .prk(prk)
    .expand(info, length);

SecretKey derivedKey = kdf.deriveKey("AES", expandOnly);
```

## 3. 实现分析

### API 结构

```
javax.crypto.KDF                    # 主接口
├── getInstance(String algorithm)   # 获取实例
├── getInstance(String, Provider)   # 指定 Provider
├── deriveKey(String alg, KDFParameters params)  # 派生密钥
├── deriveData(KDFParameters params)             # 派生原始数据
└── getAlgorithm()                   # 获取算法名

javax.crypto.KDFParameters          # 参数基类
javax.crypto.KDFSpi                 # SPI 实现基类

javax.crypto.spec.HKDFParameterSpec # HKDF 参数规范
├── ofExtract()                     # 开始 Extract
├── addSalt(byte[])                 # 添加盐值
├── extract(byte[])                 # 执行 Extract
├── expand(byte[] info, int len)    # 执行 Expand
└── expandOnly()                    # 仅 Expand
```

### 支持的算法

| 算法 | 描述 |
|------|------|
| HKDF-SHA256 | HMAC-SHA256 based KDF |
| HKDF-SHA384 | HMAC-SHA384 based KDF |
| HKDF-SHA512 | HMAC-SHA512 based KDF |

### Provider 实现

```java
// SunJCE Provider 注册

put("KDF.HKDF-SHA384", "com.sun.crypto.provider.HKDF");
put("KDF.HKDF-SHA512", "com.sun.crypto.provider.HKDF");
```

## 4. 开发者影响

### 迁移指南

```java
// 旧方式: 手动实现 HKDF
public static byte[] hkdfSha256(byte[] ikm, byte[] salt, byte[] info, int length) {
    // Extract
    Mac mac = Mac.getInstance("HmacSHA256");
    mac.init(new SecretKeySpec(salt, "HmacSHA256"));
    byte[] prk = mac.doFinal(ikm);
    
    // Expand
    byte[] result = new byte[length];
    byte[] t = new byte[0];
    int offset = 0;
    int hashLen = 32;
    
    for (int i = 1; offset < length; i++) {
        mac.init(new SecretKeySpec(prk, "HmacSHA256"));
        mac.update(t);
        mac.update(info);
        mac.update((byte) i);
        t = mac.doFinal();
        
        int copyLen = Math.min(hashLen, length - offset);
        System.arraycopy(t, 0, result, offset, copyLen);
        offset += copyLen;
    }
    return result;
}

// 新方式: 使用标准 API
KDF kdf = KDF.getInstance("HKDF-SHA256");
HKDFParameterSpec params = HKDFParameterSpec.ofExtract()
    .addSalt(salt).extract(ikm).expand(info, length);
SecretKey key = kdf.deriveKey("AES", params);
```

### 最佳实践

```java
// ✅ 推荐: 使用强盐值
byte[] salt = new byte[32];
SecureRandom.getInstanceStrong().nextBytes(salt);

// ✅ 推荐: 使用不同的 info 区分密钥用途
SecretKey encKey = deriveKey(ikm, salt, "encryption".getBytes(), 32);
SecretKey macKey = deriveKey(ikm, salt, "mac".getBytes(), 32);

// ✅ 推荐: 使用标准算法名
KDF kdf = KDF.getInstance("HKDF-SHA256");

// ⚠️ 注意: 盐值可以为空，但不推荐
// 空盐值会使用全零盐值
```

## 5. 安全考虑

### HKDF 安全性

- **Extract**: 将输入密钥材料压缩为固定长度的伪随机密钥
- **Expand**: 从 PRK 派生多个密钥
- **Salt**: 增强安全性，防止彩虹表攻击

### 推荐配置

```java
// ECDH 密钥协商后的密钥派生
public class SecureKeyDerivation {
    
    public static KeyPair generateKeyPair() throws Exception {
        KeyPairGenerator kpg = KeyPairGenerator.getInstance("X25519");
        return kpg.generateKeyPair();
    }
    
    public static SecretKey deriveSessionKey(PrivateKey myPrivate, PublicKey theirPublic) 
            throws Exception {
        // 1. ECDH 密钥协商
        KeyAgreement ka = KeyAgreement.getInstance("X25519");
        ka.init(myPrivate);
        ka.doPhase(theirPublic, true);
        byte[] sharedSecret = ka.generateSecret();
        
        // 2. HKDF 派生会话密钥
        KDF kdf = KDF.getInstance("HKDF-SHA256");
        
        // 使用对方的公钥作为盐值（简化示例）
        byte[] salt = theirPublic.getEncoded();
        
        HKDFParameterSpec params = HKDFParameterSpec.ofExtract()
            .addSalt(salt)
            .extract(sharedSecret)
            .expand("session-key".getBytes(), 32);
        
        return kdf.deriveKey("AES", params);
    }
}
```
