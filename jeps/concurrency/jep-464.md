## 1. 概述

JEP 464 是 Scoped Values 的第二轮预览（首次预览为 JEP 446，JDK 21）。Scoped Values 提供了一种在线程内及子线程间安全、高效地共享不可变数据的机制，是 `ThreadLocal` 在虚拟线程时代的替代方案。

---

## 2. 变更内容

`ScopedValue` 允许方法将值绑定到当前线程的作用域中，在该作用域内被调用的方法可以读取该值，但不能修改它。与 `ThreadLocal` 不同，Scoped Values 具有明确的生命周期，绑定在作用域结束时自动移除，避免内存泄漏。

```java
private static final ScopedValue<User> CURRENT_USER = ScopedValue.newInstance();

ScopedValue.where(CURRENT_USER, user).run(() -> {
    // 在此作用域内，任何代码均可读取 CURRENT_USER
    handle(request);
});
```

---

## 3. 与前一预览版的差异

相较于 JEP 446（JDK 21 首次预览），本次预览的主要变更：

- **API 细化**：对 `ScopedValue.where()` 的绑定 API 进行了细微调整，提升了易用性。
- **结构化并发继承**：在与结构化并发（`StructuredTaskScope`）配合使用时，子线程自动继承父线程的 scoped value 绑定，无需额外配置。

---

## 4. 核心 API

| 类/方法 | 说明 |
|---------|------|
| `ScopedValue<T>` | 核心类型，表示一个作用域内绑定的值 |
| `ScopedValue.newInstance()` | 创建新的 ScopedValue 实例 |
| `ScopedValue.where(key, value)` | 创建绑定，返回 Carrier |
| `Carrier.run(Runnable)` | 在绑定作用域内执行任务 |
| `Carrier.call(Callable)` | 在绑定作用域内执行并返回结果 |
| `ScopedValue.get()` | 读取当前作用域中的绑定值 |
