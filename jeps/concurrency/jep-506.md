## 1. 背景与动机

### 问题

线程局部变量 (ThreadLocal) 存在多个问题：

```java
// ThreadLocal 的问题
private static final ThreadLocal<User> currentUser = new ThreadLocal<>();

// 1. 无界生命周期：忘记 remove() 导致内存泄漏
currentUser.set(user);
// ... 忘记 remove()
// 内存泄漏！

// 2. 可变性：任何代码都可以修改
currentUser.set(otherUser);  // 意外修改

// 3. 继承复杂性：InheritableThreadLocal 行为复杂
// 4. 虚拟线程不友好：每个虚拟线程都有自己的副本
```

### 解决方案

Scoped Values 提供：
- **有界生命周期**: 自动清理
- **不可变**: 设置后不可修改
- **虚拟线程友好**: 高效共享
- **结构化**: 与作用域绑定

## 2. 使用示例

### 基本用法

```java
import java.lang.ScopedValue;

public class ScopedValueExample {
    // 声明 ScopedValue
    private static final ScopedValue<User> CURRENT_USER = ScopedValue.create();
    
    public static void main(String[] args) {
        User user = new User("Alice");
        
        // 在作用域内绑定值
        ScopedValue.where(CURRENT_USER, user).run(() -> {
            // 在此作用域内，CURRENT_USER.get() 返回 user
            processRequest();
        });
        
        // 作用域外，CURRENT_USER 未绑定
        // CURRENT_USER.get() 抛出 NoSuchElementException
    }
    
    static void processRequest() {
        User user = CURRENT_USER.get();  // 获取当前用户
        System.out.println("Processing for: " + user.name());
    }
}
```

### 嵌套作用域

```java
ScopedValue.where(CURRENT_USER, user)
    .where(REQUEST_ID, "req-123")
    .run(() -> {
        // CURRENT_USER 和 REQUEST_ID 都可用
        
        ScopedValue.where(LOG_LEVEL, Level.DEBUG).run(() -> {
            // 三个值都可用
            // LOG_LEVEL 在此内层作用域有效
        });
        
        // LOG_LEVEL 不再可用
    });
```

### 与虚拟线程配合

```java
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    ScopedValue.where(CURRENT_USER, user).run(() -> {
        // 所有在此作用域内创建的虚拟线程都共享此值
        executor.submit(() -> {
            User u = CURRENT_USER.get();  // 可访问
            processRequest();
        });
        
        executor.submit(() -> {
            User u = CURRENT_USER.get();  // 可访问
            processRequest();
        });
    });
}
```

### 与结构化并发配合

```java
try (var scope = new StructuredTaskScope.ShutdownOnFailure()) {
    ScopedValue.where(CURRENT_USER, user).run(() -> {
        // 子任务继承 ScopedValue
        Future<String> task1 = scope.fork(() -> fetchUserData());
        Future<String> task2 = scope.fork(() -> fetchUserPermissions());
        
        scope.join();
        scope.throwIfFailed();
        
        return new Result(task1.resultNow(), task2.resultNow());
    });
}
```

## 3. 实现分析

### API 结构

```java
public final class ScopedValue<T> {
    // 创建 ScopedValue
    public static <T> ScopedValue<T> create();
    public static <T> ScopedValue<T> forType(Class<T> type);
    
    // 绑定值
    public ScopedValue.CallableOp<T, R> call(R defaultValue);
    public ScopedValue.RunOp<T> run();
    
    // 获取值
    public T get();
    public T orElse(T other);
    public T orElseThrow();
    public boolean isBound();
}

// 绑定操作
public static <T> ScopedValue.Where<T> where(ScopedValue<T> key, T value);

public final class ScopedValue.Where<T> {
    public ScopedValue.Where<T> where(ScopedValue<T> key, T value);
    public void run(Runnable op);
    public <R> R call(Callable<R> op);
}
```

### 内部实现

```java
// 简化示意
class ScopedValue<T> {
    // 使用线程局部存储，但生命周期受控
    private static final ThreadLocal<Bindings> BINDINGS = new ThreadLocal<>();
    
    static class Bindings {
        Map<ScopedValue<?>, Object> values;
        Bindings parent;  // 支持嵌套
    }
    
    public T get() {
        Bindings bindings = BINDINGS.get();
        while (bindings != null) {
            T value = (T) bindings.values.get(this);
            if (value != null) return value;
            bindings = bindings.parent;
        }
        throw new NoSuchElementException();
    }
}
```

## 4. 开发者影响

### 与 ThreadLocal 对比

| 特性 | ThreadLocal | ScopedValue |
|------|-------------|-------------|
| 生命周期 | 无界 | 有界 |
| 可变性 | 可变 | 不可变 |
| 内存泄漏 | 风险高 | 无风险 |
| 虚拟线程 | 每线程一份 | 高效共享 |
| 继承 | 复杂 | 简单 |

### 迁移指南

```java
// 旧代码：ThreadLocal
private static final ThreadLocal<Context> CONTEXT = new ThreadLocal<>();

public void process() {
    CONTEXT.set(context);
    try {
        doWork();
    } finally {
        CONTEXT.remove();  // 必须清理
    }
}

// 新代码：ScopedValue
private static final ScopedValue<Context> CONTEXT = ScopedValue.create();

public void process() {
    ScopedValue.where(CONTEXT, context).run(() -> {
        doWork();  // 自动清理
    });
}
```

### 最佳实践

```java
// ✅ 推荐：不可变值
record User(String name, String id) {}
private static final ScopedValue<User> CURRENT_USER = ScopedValue.create();

// ✅ 推荐：结构化作用域
ScopedValue.where(CURRENT_USER, user).run(() -> {
    // 清晰的作用域边界
    processRequest();
});

// ❌ 避免：在作用域外访问
try {
    CURRENT_USER.get();  // NoSuchElementException
} catch (NoSuchElementException e) {
    // 处理未绑定情况
}

// ✅ 推荐：使用 orElse 处理未绑定
User user = CURRENT_USER.orElse(User.ANONYMOUS);
```

## 5. 性能优势

### 内存效率

```
ThreadLocal:
- 每个线程一个 Map
- 虚拟线程数量可能很大
- 内存占用高

ScopedValue:
- 绑定在作用域栈上
- 虚拟线程共享父线程的绑定
- 内存占用低
```

### 访问速度

```
ThreadLocal.get():  ~10-20 ns
ScopedValue.get():  ~5-10 ns (更快)
```
