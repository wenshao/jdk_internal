## 1. 背景与动机

### 问题

传统并发编程容易出错：

```java
// 传统方式：手动管理线程
ExecutorService executor = Executors.newCachedThreadPool();
Future<String> userFuture = executor.submit(() -> fetchUser());
Future<Integer> orderFuture = executor.submit(() -> fetchOrder());

try {
    String user = userFuture.get();
    Integer order = orderFuture.get();
    return new Response(user, order);
} catch (InterruptedException | ExecutionException e) {
    // 如果 fetchUser 失败，fetchOrder 仍在运行
    // 需要手动取消
    userFuture.cancel(true);
    orderFuture.cancel(true);
}
```

问题：
- 子任务生命周期不受控
- 异常处理复杂
- 资源泄漏风险
- 取消逻辑繁琐

### 解决方案

结构化并发确保：
- 子任务生命周期与父任务绑定
- 自动取消和清理
- 简化的错误处理

## 2. 使用示例

### 基本用法

```java
import java.util.concurrent.StructuredTaskScope;
import java.util.concurrent.Future;

// 并发执行多个任务
try (var scope = new StructuredTaskScope.ShutdownOnFailure()) {
    Future<String> user = scope.fork(() -> fetchUser());
    Future<Integer> order = scope.fork(() -> fetchOrder());
    
    scope.join();           // 等待所有任务完成
    scope.throwIfFailed();  // 如果有任务失败，抛出异常
    
    return new Response(user.resultNow(), order.resultNow());
}
// 自动关闭，取消未完成的任务
```

### 策略类型

```java
// 1. ShutdownOnFailure: 任一任务失败则取消其他任务
try (var scope = new StructuredTaskScope.ShutdownOnFailure()) {
    Future<String> task1 = scope.fork(() -> fetchFromServer1());
    Future<String> task2 = scope.fork(() -> fetchFromServer2());
    
    scope.join();
    scope.throwIfFailed();
    
    return task1.resultNow();  // 或 task2
}

// 2. ShutdownOnSuccess: 任一任务成功则取消其他任务
try (var scope = new StructuredTaskScope.ShutdownOnSuccess<String>()) {
    scope.fork(() -> fetchFromServer1());
    scope.fork(() -> fetchFromServer2());
    scope.fork(() -> fetchFromServer3());
    
    scope.join();
    return scope.result();  // 返回第一个成功的结果
}

// 3. 自定义策略
try (var scope = new StructuredTaskScope<String>()) {
    scope.fork(() -> task1());
    scope.fork(() -> task2());
    
    scope.join();
    
    // 处理所有结果
    scope.forEach((subtask) -> {
        switch (subtask.state()) {
            case SUCCESS -> handleSuccess(subtask.get());
            case FAILED -> handleFailure(subtask.exception());
        }
    });
}
```

### 与虚拟线程配合

```java
// 使用虚拟线程执行结构化并发
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    try (var scope = new StructuredTaskScope.ShutdownOnFailure()) {
        // 每个子任务在独立的虚拟线程中运行
        Future<String> user = scope.fork(() -> fetchUser());
        Future<List<Order>> orders = scope.fork(() -> fetchOrders());
        
        scope.join();
        scope.throwIfFailed();
        
        return new Response(user.resultNow(), orders.resultNow());
    }
}
```

### 嵌套结构化并发

```java
try (var outerScope = new StructuredTaskScope.ShutdownOnFailure()) {
    Future<String> user = outerScope.fork(() -> fetchUser());
    
    Future<List<Order>> orders = outerScope.fork(() -> {
        // 嵌套的结构化并发
        try (var innerScope = new StructuredTaskScope.ShutdownOnSuccess<Order>()) {
            innerScope.fork(() -> fetchOrderFromDB());
            innerScope.fork(() -> fetchOrderFromCache());
            innerScope.join();
            return List.of(innerScope.result());
        }
    });
    
    outerScope.join();
    outerScope.throwIfFailed();
    
    return new Response(user.resultNow(), orders.resultNow());
}
```

## 3. 实现分析

### API 结构

```java
// 主类
public class StructuredTaskScope<T> implements AutoCloseable {
    // 创建子任务
    public Subtask<T> fork(Callable<? extends T> task);
    
    // 等待所有任务完成
    public StructuredTaskScope<T> join() throws InterruptedException;
    
    // 等待直到指定时间
    public StructuredTaskScope<T> joinUntil(Instant deadline) 
        throws InterruptedException;
    
    // 关闭作用域
    public void close();
    
    // 处理结果
    public void forEach(Consumer<Subtask<? extends T>> handler);
}

// 子任务状态
public enum Subtask.State {
    AVAILABLE,   // 可用
    SUCCESS,     // 成功
    FAILED,      // 失败
}

// 预定义策略
public static final class ShutdownOnFailure extends StructuredTaskScope<Object> {
    public void throwIfFailed() throws ExecutionException;
}

public static final class ShutdownOnSuccess<T> extends StructuredTaskScope<T> {
    public T result();
}
```

### Joiners 工具类

```java
// JDK 26 新增/改进
public class Joiners {
    // 等待所有任务完成
    public static <T> void awaitAll(StructuredTaskScope<T> scope);
    
    // 等待任一任务成功
    public static <T> T awaitAny(StructuredTaskScope<T> scope);
}
```

## 4. 开发者影响

### 与 CompletableFuture 对比

```java
// CompletableFuture
CompletableFuture<String> userFuture = CompletableFuture.supplyAsync(() -> fetchUser());
CompletableFuture<Integer> orderFuture = CompletableFuture.supplyAsync(() -> fetchOrder());

CompletableFuture<Response> result = userFuture.thenCombine(orderFuture, Response::new);

// 问题：如果 userFuture 失败，orderFuture 不会自动取消

// StructuredTaskScope
try (var scope = new StructuredTaskScope.ShutdownOnFailure()) {
    Future<String> user = scope.fork(() -> fetchUser());
    Future<Integer> order = scope.fork(() -> fetchOrder());
    
    scope.join();
    scope.throwIfFailed();
    
    return new Response(user.resultNow(), order.resultNow());
}
// 任一失败，其他自动取消
```

### 最佳实践

```java
// ✅ 推荐：使用 try-with-resources
try (var scope = new StructuredTaskScope.ShutdownOnFailure()) {
    // ...
}  // 自动关闭

// ✅ 推荐：配合虚拟线程
try (var scope = new StructuredTaskScope.ShutdownOnFailure()) {
    // 每个子任务在虚拟线程中运行
    scope.fork(() -> blockingIO());
    scope.fork(() -> blockingIO());
    scope.join();
}

// ❌ 避免：忘记 join()
try (var scope = new StructuredTaskScope.ShutdownOnFailure()) {
    scope.fork(() -> task());
    // 忘记 scope.join()
}  // 子任务可能未完成就被取消
```
