## 1. 概述

JEP 481 是 Scoped Values 的第三轮预览（前两轮为 JEP 446 和 JEP 464）。Scoped Values 提供了一种在线程内及子线程间安全、高效地共享不可变数据的机制，旨在取代虚拟线程场景下的 `ThreadLocal`。

---

## 2. 变更内容

`ScopedValue` 允许将值绑定到当前线程的作用域中，作用域内的所有方法调用均可读取该值。绑定在作用域结束时自动移除，确保无泄漏。与 `ThreadLocal` 相比，Scoped Values 具有不可变性、有界生命周期和更低的内存开销。

```java
private static final ScopedValue<Principal> PRINCIPAL = ScopedValue.newInstance();

ScopedValue.where(PRINCIPAL, currentUser).run(() -> {
    serve(request); // PRINCIPAL.get() 在此作用域内可用
});
```

## 3. 与前一预览版的差异

相较于 JEP 464（JDK 22 第二次预览），本次预览的变更非常小：

- **无重大语义变更**：核心设计和行为保持不变，继续收集实际使用经验以为最终定稿做准备。

## 4. 核心 API

| 类/方法 | 说明 |
|---------|------|
| `ScopedValue<T>` | 核心类型，表示一个作用域内绑定的值 |
| `ScopedValue.newInstance()` | 创建新的 ScopedValue 实例 |
| `ScopedValue.where(key, value)` | 创建绑定，返回 Carrier |
| `Carrier.run(Runnable)` | 在绑定作用域内执行任务 |
| `Carrier.call(Callable)` | 在绑定作用域内执行并返回结果 |
| `ScopedValue.get()` | 读取当前作用域中的绑定值 |
