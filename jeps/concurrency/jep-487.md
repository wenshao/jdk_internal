## 1. 概述

JEP 487 是 Scoped Values 的第四轮预览，在 JDK 24 中发布。Scoped Values 提供了一种在方法之间以及与子线程之间共享不可变数据的机制，作为 `ThreadLocal` 的轻量级、不可变替代方案。本轮预览在 JEP 481（第三轮预览，JDK 23）的基础上仅做了少量改进。

---

## 2. 背景

`ThreadLocal` 长期以来是 Java 中线程内数据传递的主要手段，但存在可变性、内存泄漏和性能开销等问题。Scoped Values 通过限定作用域和不可变性来解决这些问题，尤其适合与虚拟线程和结构化并发配合使用。

```java
private static final ScopedValue<String> USER = ScopedValue.newInstance();

ScopedValue.runWhere(USER, "Alice", () -> {
    System.out.println(USER.get()); // "Alice"
    handleRequest();
});
```

## 3. 与上一轮预览的变更

JEP 487 相对于 JEP 481（JDK 23）的变更非常小：

| 变更领域 | 说明 |
|----------|------|
| **API 微调** | 根据反馈对方法签名进行了细微优化 |
| **规范文本** | 更新了 Javadoc 说明以提高清晰度 |
| **无重大语义变更** | 核心行为与第三轮预览保持一致 |

---

## 4. 核心 API 与 ThreadLocal 对比

| 特性 | ThreadLocal | ScopedValue |
|------|-------------|-------------|
| **可变性** | 可变，随时可 set | 不可变，绑定后不可更改 |
| **生命周期** | 需手动 remove | 作用域结束自动释放 |
| **继承** | InheritableThreadLocal 复制数据 | 与 StructuredTaskScope 自动共享 |
| **性能** | 每线程一个哈希表 | 优化的栈帧绑定 |

核心方法：`newInstance()`、`runWhere()`、`callWhere()`、`get()`、`isBound()`。值的绑定具有严格的词法作用域，返回后自动失效。
