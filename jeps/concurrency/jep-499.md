---

## 1. 概述

JEP 499 是 Structured Concurrency 的第四轮预览，在 JDK 24 中发布。结构化并发通过 `StructuredTaskScope` API 将多个并发任务视为一个工作单元，确保子任务的生命周期被严格限定在父任务的作用域内，简化错误处理和取消逻辑。

---

## 2. 背景与变更

传统的 `ExecutorService` 中子任务的生命周期独立于父任务，容易导致线程泄漏和错误丢失。结构化并发借鉴了结构化编程的思想。JEP 499 相对于 JEP 480（JDK 24）的变更：

| 变更领域 | 说明 |
|----------|------|
| **StructuredTaskScope 改进** | 对 API 进行了微调以提高易用性 |
| **Joiner 接口** | 优化了自定义 Joiner 的扩展点设计 |
| **异常处理** | 改进了子任务失败时的异常传播机制 |

---

## 3. 代码示例

```java
// 并发获取用户和订单，任一失败则取消全部
try (var scope = StructuredTaskScope.open(
        Joiner.allSuccessfulOrThrow())) {
    Subtask<User> userTask = scope.fork(() -> findUser(userId));
    Subtask<Order> orderTask = scope.fork(() -> fetchOrder(orderId));
    scope.join();
    return new Response(userTask.get(), orderTask.get());
}

// ShutdownOnSuccess：首个成功即返回
try (var scope = StructuredTaskScope.open(
        Joiner.anySuccessfulResultOrThrow())) {
    scope.fork(() -> queryPrimary(key));
    scope.fork(() -> queryReplica(key));
    return scope.join();
}
```

---

## 4. 错误处理策略

| 策略 | 说明 |
|------|------|
| `allSuccessfulOrThrow` | 所有子任务成功才继续，任一失败则取消其余 |
| `anySuccessfulResultOrThrow` | 首个成功的子任务结果即可，取消其余 |
| **自定义 Joiner** | 实现 Joiner 接口以定义自定义汇聚逻辑 |
