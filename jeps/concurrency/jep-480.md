## 1. 概述

JEP 480 是结构化并发的第三轮预览（前两轮为 JEP 453 和 JEP 462）。结构化并发通过 `StructuredTaskScope` API 将在不同线程中运行的相关任务视为一个工作单元，简化并发编程中的错误处理和取消逻辑。

---

## 2. 变更内容

`StructuredTaskScope` 将并发任务的生命周期限定在一个明确的词法作用域内。父任务创建 scope，在其中 fork 子任务，然后 join 等待所有子任务完成。如果任何子任务失败，其余子任务会被自动取消。

---

## 3. 与前一预览版的差异

相较于 JEP 462（JDK 22 第二次预览），本次预览未引入重大 API 变更：

- **持续收集反馈**：核心 API 保持稳定，本轮预览旨在让更多开发者在实际项目中验证 API 的可用性。
- **与 Scoped Values 的协同**：进一步明确了 scoped values 在结构化并发中自动继承的行为。

---

## 4. 核心 API

| 类/方法 | 说明 |
|---------|------|
| `StructuredTaskScope` | 核心类，定义结构化并发作用域 |
| `scope.fork(Callable)` | 在作用域内启动子任务 |
| `scope.join()` | 等待所有子任务完成 |
| `ShutdownOnFailure` | 策略：任一子任务失败时关闭作用域 |
| `ShutdownOnSuccess` | 策略：任一子任务成功时关闭作用域 |

---

## 5. 代码示例

```java
try (var scope = new StructuredTaskScope.ShutdownOnFailure()) {
    Subtask<User> user = scope.fork(() -> findUser(id));
    Subtask<Order> order = scope.fork(() -> fetchOrder(id));
    scope.join().throwIfFailed();
    return new Response(user.get(), order.get());
}
```
