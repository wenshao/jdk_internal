# JEP 425: Virtual Threads (Preview)

> **Status**: Preview | **Type**: Feature | **Component**: Java SE / Concurrency
>
> **Lead**: Ron Pressler (Oracle)
>
> **JDK Version**: JDK 19 (2022-09)
>
> **Organization**: [Oracle](../../contributors/orgs/oracle.md)

---

## 概览

引入虚拟线程作为预览功能。虚拟线程是轻量级线程，由 JVM 管理而非操作系统，大幅降低并发编程的复杂度。

---

## 目标

- 提供轻量级线程实现
- 简化高并发编程
- 保持现有 Thread API 兼容
- 收集社区反馈

---

## 动机

### 平台线程的问题

```java
// 传统平台线程（1 线程 = 1 OS 线程）
for (int i = 0; i < 10000; i++) {
    new Thread(() -> {
        // 每个线程占用 ~1MB 栈内存
    }).start();  // OOM: 无法创建这么多线程
}
```

### 虚拟线程的优势

```java
// 虚拟线程（1 线程 = 轻量级对象）
for (int i = 0; i < 10000; i++) {
    Thread.startVirtualThread(() -> {
        // 每个虚拟线程仅占用 ~1KB 内存
    });  // 轻松创建百万级线程
}
```

---

## 使用方式

### 创建虚拟线程

```java
// 方式 1：直接启动
Thread.startVirtualThread(() -> {
    System.out.println("Running in virtual thread");
});

// 方式 2：使用 Thread.Builder
Thread factory = Thread.builder().virtual().factory();
Runnable task = () -> System.out.println("Hello");
Thread thread = factory.newThread(task);
thread.start();

// 方式 3：使用 ExecutorService
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    executor.submit(() -> doWork());
}
```

### 虚拟线程 vs 平台线程

```java
// 虚拟线程
Thread vt = Thread.startVirtualThread(() -> task());

// 平台线程（传统）
Thread pt = new Thread(() -> task());
pt.start();

// 检查类型
System.out.println(vt.isVirtual());  // true
System.out.println(pt.isVirtual());  // false
```

---

## 关键特性

### 1. 平台线程兼容性

```java
// 虚拟线程可以使用所有 Thread API
Thread vt = Thread.startVirtualThread(() -> {
    Thread.sleep(1000);
    Thread.interrupted();
    // 所有传统 API 都可用
});
vt.join();  // 等待完成
```

### 2. 自动挂载/解挂

```java
// 虚拟线程在阻塞操作时自动解挂
Thread.startVirtualThread(() -> {
    socket.read();    // 解挂，不占用载体线程
    Thread.sleep(1);  // 解挂
    lock.lock();      // 解挂
    // 恢复执行时重新挂载
});
```

### 3. 线程池优化

```java
// 传统线程池（需要调优）
ExecutorService pool = Executors.newFixedThreadPool(100);

// 虚拟线程池（无需调优）
ExecutorService pool = Executors.newVirtualThreadPerTaskExecutor();
```

## 性能对比

### 内存占用

| 线程类型 | 栈大小 | 10000 线程内存 |
|----------|--------|---------------|
| **平台线程** | ~1MB | ~10GB |
| **虚拟线程** | ~1KB | ~10MB |

### 创建时间

| 操作 | 平台线程 | 虚拟线程 | 改进 |
|------|----------|----------|------|
| 创建 | 100μs | 1μs | 100x |
| 切换 | 10μs | 0.1μs | 100x |

### 并发能力

| 场景 | 平台线程 | 虚拟线程 |
|------|----------|----------|
| 最大线程数 | ~10,000 | ~1,000,000 |
| 适用场景 | CPU 密集 | I/O 密集 |

## 使用场景

### 推荐

```java
// ✅ 高并发 I/O 操作
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    for (URL url : urls) {
        executor.submit(() -> download(url));
    }
}

// ✅ 每个请求一个线程
class WebServer {
    private final ExecutorService executor = 
        Executors.newVirtualThreadPerTaskExecutor();
    
    void handleRequest(Socket client) {
        executor.submit(() -> process(client));
    }
}

// ✅ 定时任务
ScheduledExecutorService scheduler = 
    Executors.newScheduledThreadPool(10);  // 仍用平台线程
```

### 不推荐

```java
// ❌ CPU 密集型任务（使用平台线程池）
ExecutorService cpuPool = Executors.newFixedThreadPool(
    Runtime.getRuntime().availableProcessors()
);

// ❌ 长时间运行的计算任务
// ❌ 需要线程局部变量大量存储的场景
```

---

## 限制（预览阶段）

1. **不支持**：`Thread.stop()` 等废弃 API
2. **不支持**：线程栈跟踪的完整信息
3. **需要**：`--enable-preview` 标志
4. **不推荐**：用于生产环境
