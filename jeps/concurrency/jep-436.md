# JEP 436: Virtual Threads (Second Preview)

> **Status**: Incubator | **Type**: Feature | **Component**: Java SE / Concurrency
>
> **Lead**: Ron Pressler (Oracle)
>
> **JDK Version**: JDK 20 (2023-03)
>
> **Organization**: [Oracle](../../contributors/orgs/oracle.md)

---

## 概览

引入 Scoped Value 作为孵化器功能。Scoped Value 允许在线程内和跨线程安全高效地共享数据，特别适用于虚拟线程。

---

## 目标

- 提供线程安全的数据共享机制
- 替代 ThreadLocal
- 支持虚拟线程
- 收集社区反馈

---

## 动机

### ThreadLocal 的问题

```java
// ThreadLocal 的问题
private static final ThreadLocal<User> userContext = new ThreadLocal<>();

// 1. 内存泄漏风险
userContext.set(user);
// 忘记 remove() 会导致内存泄漏

// 2. 继承问题
Thread child = new Thread(() -> {
    // 子线程无法访问父线程的 ThreadLocal
    User user = userContext.get();  // null
});

// 3. 虚拟线程不友好
// ThreadLocal 在虚拟线程中效率低
```

### Scoped Value 的优势

```java
// Scoped Value
private static final ScopedValue<User> USER = ScopedValue.newInstance();

ScopedValue.where(USER, user).run(() -> {
    // 安全访问
    User u = USER.get();
    
    // 自动清理，无内存泄漏
});
```

## 使用方式

### 基本用法

```java
// 定义 ScopedValue
static final ScopedValue<User> CURRENT_USER = ScopedValue.newInstance();

// 绑定和使用
User user = authenticate();
ScopedValue.where(CURRENT_USER, user).run(() -> {
    // 访问绑定的值
    processRequest(CURRENT_USER.get());
});
// 运行结束后自动解绑
```

### 跨线程共享

```java
// 在线程间共享
ScopedValue<User> USER = ScopedValue.newInstance();
User user = authenticate();

ScopedValue.where(USER, user).run(() -> {
    // 主线程可访问
    Thread.startVirtualThread(() -> {
        // 子线程也可访问（继承）
        System.out.println(USER.get());
    }).join();
});
```

### 嵌套作用域

```java
ScopedValue.where(USER, admin).run(() -> {
    // 第一层：admin
    System.out.println(USER.get());  // admin
    
    ScopedValue.where(USER, guest).run(() -> {
        // 第二层：guest（遮蔽外层）
        System.out.println(USER.get());  // guest
    });
    
    // 恢复外层
    System.out.println(USER.get());  // admin
});
```

## 与 ThreadLocal 对比

| 特性 | ThreadLocal | ScopedValue |
|------|-------------|-------------|
| **内存泄漏** | 可能 | ✅ 自动清理 |
| **继承性** | 需 InheritableThreadLocal | ✅ 自动继承 |
| **虚拟线程** | ❌ 效率低 | ✅ 优化 |
| **可变性** | ✅ 可变 | ❌ 不可变 |
| **作用域** | 线程生命周期 | 代码块范围 |
| **性能** | 中等 | 高 |

---

## 使用场景

### 推荐

```java
// ✅ 请求上下文
static final ScopedValue<Request> REQUEST = ScopedValue.newInstance();

void handleRequest(Request req) {
    ScopedValue.where(REQUEST, req).run(() -> {
        process();
        log();
        cleanup();
    });
}

// ✅ 事务上下文
static final ScopedValue<Connection> DB_CONNECTION = ScopedValue.newInstance();

void doInTransaction(Connection conn) {
    ScopedValue.where(DB_CONNECTION, conn).run(() -> {
        // 所有嵌套调用都可访问
        dao.save();
        service.process();
    });
}

// ✅ 安全上下文
static final ScopedValue<Authentication> AUTH = ScopedValue.newInstance();

void withAuth(Authentication auth, Runnable task) {
    ScopedValue.where(AUTH, auth).run(task);
}
```

### 不推荐

```java
// ❌ 需要可变状态
ScopedValue<int[]> counter = ScopedValue.newInstance();
// 不能修改数组内容

// ❌ 长期存储
// ScopedValue 是短期的，不适合缓存
```
