## 1. 背景与动机

### 问题

传统 Java 对象头占用大量内存：

```
64位 JVM 对象头结构:
┌─────────────────────────────────────────────────────┐
│ Mark Word (8 bytes)                                 │
│ - 锁状态、GC 年龄、hashCode 等                        │
├─────────────────────────────────────────────────────┤
│ Klass Pointer (8 bytes, 压缩后 4 bytes)             │
│ - 指向类元数据                                        │
└─────────────────────────────────────────────────────┘

总计: 12-16 字节/对象
```

对于小对象，对象头可能占用 50%+ 的内存：

```java
class Point { int x, y; }  // 8 字节数据 + 12-16 字节头 = 20-24 字节
```

### 解决方案

JEP 519 引入 **紧凑对象头**：
- 减少对象头大小
- 提高内存效率
- 改善缓存命中率

---

## 2. 技术细节

### 对象头对比

| 配置 | Mark Word | Klass Pointer | 总计 |
|------|-----------|---------------|------|
| 传统 64位 | 8 bytes | 8 bytes | 16 bytes |
| 传统 + 压缩指针 | 8 bytes | 4 bytes | 12 bytes |
| **紧凑对象头** | **优化** | **优化** | **更小** |

### 实现原理

紧凑对象头通过以下方式减少内存：

1. **位压缩**: 将部分信息压缩存储
2. **延迟加载**: 部分信息延迟到需要时计算
3. **共享存储**: 某些状态存储在类元数据中

## 3. 使用示例

### 启用方式

```bash
# JDK 26 默认启用
java -XX:+UseCompactObjectHeaders MyApp

# 禁用（如遇兼容性问题）
java -XX:-UseCompactObjectHeaders MyApp
```

### 内存节省示例

```java
// 测试对象
class SmallObject {
    int value;  // 4 bytes
}

// 传统对象头
// 12 bytes 头 + 4 bytes 数据 + 4 bytes 填充 = 20 bytes

// 紧凑对象头
// 更小的头 + 4 bytes 数据 = 更少内存
```

### 批量对象内存对比

```java
// 创建 100 万个对象
SmallObject[] objects = new SmallObject[1_000_000];
for (int i = 0; i < objects.length; i++) {
    objects[i] = new SmallObject();
}

// 传统: ~20 MB
// 紧凑: ~12-16 MB (节省 20-40%)
```

## 5. 开发者影响

### 兼容性

| 方面 | 影响 |
|------|------|
| 应用代码 | 无需修改 |
| JNI 代码 | 可能需要检查对象头访问 |
| JVM 工具 | 可能需要更新 |
| CDS 归档 | 需要重新生成 |

### 配置选项

```bash
# 相关 JVM 选项
-XX:+UseCompactObjectHeaders    # 启用紧凑对象头
-XX:+UseCompressedClassPointers # 压缩类指针（配合使用）
-XX:+UseCompressedOops          # 压缩普通对象指针
```

### 最佳实践

```bash
# 推荐配置
-XX:+UseCompactObjectHeaders    # 默认启用
-XX:+UseCompressedClassPointers # 配合使用
-XX:+UseCompressedOops          # 配合使用

# 大堆场景
-Xmx32g -XX:+UseCompactObjectHeaders
```

### 内存占用

```
对象类型          传统      紧凑      节省
-----------------------------------------
空对象            16 bytes  8-12 bytes  25-50%
Point (2 int)     24 bytes  16 bytes    33%

```
