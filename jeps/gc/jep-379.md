# JEP 379: Shenandoah GC (Standard)

> **Status**: Permanent | **Type**: Feature | **Component**: HotSpot / GC
>
> **Lead**: [Aleksey Shipilev](../../by-contributor/profiles/aleksey-shipilev.md) (Red Hat，当时；现 Amazon)
>
> **JDK Version**: JDK 15 (2020-09)
>
> **Organization**: [Red Hat](../../contributors/orgs/redhat.md)（当时）

---

## 概览

将 Shenandoah 垃圾收集器从孵化器提升为标准功能。Shenandoah 提供超低暂停时间的垃圾收集，暂停时间与堆大小无关。

---

## 目标

- 将 Shenandoah GC 提升为标准功能
- 移除 `-XX:+UnlockExperimentalVMOptions` 要求
- 推荐用于生产环境的低延迟场景

---

## 动机

经过 JDK 12-14 三个版本的孵化器阶段，Shenandoah GC 已经：

1. **稳定性验证**：在生产环境中得到广泛测试
2. **性能成熟**：吞吐量和暂停时间达到预期
3. **功能完整**：支持 Class Unloading、String Deduplication 等
4. **社区认可**：收到积极的用户反馈

---

## 技术特性

### 核心算法

```
+-------------------+     +-------------------+
| Concurrent Mark   | --> | STW Update Refs   |
+-------------------+     +-------------------+
        ↓                         ↓
+-------------------+     +-------------------+
| STW Init Mark     |     | Concurrent Update |
+-------------------+     +-------------------+
```

### 读屏障 vs 写屏障

| 特性 | Shenandoah (读屏障) | ZGC (读写屏障) |
|------|-------------------|---------------|
| 暂停时间 | 亚毫秒级 | 亚毫秒级 |
| 吞吐量影响 | 中等 | 较低 |
| 实现复杂度 | 中等 | 高 |

## 使用方式

### JDK 15+ (标准功能)

```bash
# 无需解锁实验选项
-XX:+UseShenandoahGC

# 与 G1 切换
-XX:+UseG1GC  # 默认
-XX:+UseShenandoahGC  # 低延迟
```

### 推荐配置

```bash
# 低延迟应用
-XX:+UseShenandoahGC -XX:ShenandoahTargetPauseTime=10ms

# 大堆应用 (>32GB)
-XX:+UseShenandoahGC -XX:ShenandoahGCHeuristics=static

# 容器环境
-XX:+UseShenandoahGC -XX:+ShenandoahUncommitDelay
```

## 性能数据

### 暂停时间对比（JDK 15）

| 堆大小 | G1 GC | Shenandoah | 改进 |
|--------|-------|------------|------|
| 4GB | 15ms | 1.5ms | 10x |
| 16GB | 45ms | 2ms | 22x |
| 64GB | 150ms | 3ms | 50x |
| 256GB | 500ms+ | 5ms | 100x+ |

### 吞吐量对比

| 基准测试 | G1 GC | Shenandoah | 差异 |
|----------|-------|------------|------|
| SPECjbb2015 | 100% | 92% | -8% |
| DaCapo | 100% | 95% | -5% |
| 实际应用平均 | 100% | 90-95% | -5~10% |

## 适用场景

### 推荐使用

- ✅ 低延迟要求的交易系统
- ✅ 实时数据处理
- ✅ 大堆应用（>32GB）
- ✅ 对暂停时间敏感的服务

### 不推荐

- ❌ CPU 密集型批处理（选择 Parallel GC）
- ❌ 小内存应用（<4GB，选择 Serial GC）
- ❌ 吞吐量优先场景（选择 G1 GC）
