# JEP 333: ZGC

> **Status**: Permanent | **Type**: Feature | **Component**: HotSpot / GC
>
> **Lead**: Per Lidén (Oracle)
>
> **JDK Version**: JDK 11 (2018-09)
>
> **Organization**: [Oracle](../../contributors/orgs/oracle.md)

---

## 概览

引入 ZGC（Z Garbage Collector），一个可扩展的低延迟垃圾收集器，暂停时间不超过 10ms，不影响应用吞吐量。

---

## 目标

- 暂停时间 < 10ms
- 暂停时间不随堆大小增加
- 支持堆大小从几百 MB 到几 TB
- 吞吐量开销 < 15%

---

## 技术细节

### 并发标记

```
+------------------+
| STW Initial Mark |
+------------------+
        ↓
+------------------+
| Concurrent Mark  |
+------------------+
        ↓
+------------------+
| STW Final Mark   |
+------------------+
```

### 并发整理

ZGC 使用**读屏障**和**染色指针**技术：

```
染色指针格式（64 位）：
┌──────┬─────┬─────┬────────────────┐
│ 未用 │ Mark│ Mark│ 物理地址 (42 位) │
│ 4 位  │ 1   │ 0   │                │
└──────┴─────┴─────┴────────────────┘
       ^     ^
       |     └─ Mark 0: 对象是否存活
       └─ Mark 1: 对象是否被移动
```

### 读屏障

```java
// 伪代码：ZGC 读屏障
Object loadField(Object obj, long offset) {
    Object value = UNSAFE.getObject(obj, offset);
    if (isMarked(value)) {
        value = remap(value);  // 读取新位置
    }
    return value;
}
```

## 使用方式

### JDK 11-14 (实验性)

```bash
-XX:+UnlockExperimentalVMOptions \
-XX:+UseZGC
```

### JDK 15+ (生产可用)

```bash
-XX:+UseZGC
```

### 典型配置

```bash
# 堆大小
-Xms4g -Xmx4g

# ZGC 特定参数
-XX:ConcGCThreads=2        # 并发 GC 线程数
-XX:ZCollectionInterval=0  # 无间隔限制
-XX:ZFragmentationLimit=25 # 碎片化限制
```

## 性能对比

### 暂停时间

| GC | 平均暂停 | 最大暂停 | 99 百分位 |
|----|----------|----------|-----------|
| **G1 GC** | 5-50ms | 200ms+ | 100ms |
| **ZGC** | 1-2ms | 10ms | 5ms |

### 吞吐量

| 基准测试 | G1 GC | ZGC | 差异 |
|----------|-------|-----|------|
| SPECjbb2015 | 100% | 90-95% | -5~10% |
| DaCapo | 100% | 85-95% | -5~15% |
| 实际应用 | 100% | 90-98% | -2~10% |

### 堆大小支持

| GC | 最小堆 | 最大堆 | 推荐堆 |
|----|--------|--------|--------|
| **G1 GC** | 1GB | 32TB | 4-128GB |
| **ZGC** | 1GB | 16TB | 8-512GB |

## 适用场景

### 推荐使用

- ✅ 低延迟要求的应用
- ✅ 大堆应用（>8GB）
- ✅ 延迟敏感的服务
- ✅ 云原生应用

### 不推荐

- ❌ 小堆应用（<4GB，使用 G1）
- ❌ CPU 密集型批处理
- ❌ 吞吐量优先场景
