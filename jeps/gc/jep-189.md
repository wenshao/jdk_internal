# JEP 189: Shenandoah GC (Incubator)

> **Status**: Incubator | **Type**: Feature | **Component**: HotSpot / GC
>
> **Lead**: [Aleksey Shipilev](../../by-contributor/profiles/aleksey-shipilev.md) (Red Hat)
>
> **JDK Version**: JDK 12 (2019-03)
>
> **Organization**: [Red Hat](../../contributors/orgs/redhat.md)

---

## 概览

将 Shenandoah 垃圾收集器作为孵化器功能引入 OpenJDK。Shenandoah 是一个超低暂停时间的 GC，适用于需要可预测暂停时间的应用。

---

## 目标

- 提供亚毫秒级暂停时间
- 暂停时间与堆大小无关
- 与 G1 GC 互补，提供低延迟选项
- 通过孵化器机制收集社区反馈

---

## 动机

在 JDK 12 之前，Java 开发者在低延迟 GC 方面选择有限：

| GC | 暂停时间 | 吞吐量 | 适用场景 |
|----|----------|--------|----------|
| **G1 GC** | 中等（毫秒级） | 高 | 通用场景 |
| **Parallel GC** | 高 | 最高 | 批处理 |
| **Serial GC** | 高 | 低 | 小应用 |
| **CMS** (已废弃) | 低 | 中等 | 低延迟 |

**Shenandoah 的优势**：
- 暂停时间与堆大小无关
- 即使在 TB 级堆上也能保持亚毫秒级暂停
- 适合对延迟敏感的应用

## 技术细节

### 并发标记

```
+------------------+
|  STW Initial Mark |
+------------------+
        ↓
+------------------+
| Concurrent Mark   |
+------------------+
        ↓
+------------------+
|  STW Final Mark   |
+------------------+
```

### 并发整理

Shenandoah 的核心创新是**并发整理**（Concurrent Compaction）：

```
传统 GC:
[对象 A] → [对象 B] → [对象 C]
  ↓ 整理时 STW
[对象 A][对象 B][对象 C] ← 紧凑排列

Shenandoah:
[对象 A] → [对象 B] → [对象 C]
  ↓ 并发整理
[转发指针] → 新位置
```

### 读屏障

```java
// 伪代码：Shenandoah 读屏障
Object loadField(Object obj, long offset) {
    Object value = UNSAFE.getObject(obj, offset);
    if (value != null && isForwarded(value)) {
        value = readForward(value);  // 读取新位置
    }
    return value;
}
```

## 使用方式

### 启用 Shenandoah

```bash
# JDK 12+ (孵化器)
-XX:+UnlockExperimentalVMOptions -XX:+UseShenandoahGC

# JDK 15+ (标准功能)
-XX:+UseShenandoahGC
```

### 典型配置

```bash
# 低延迟模式
-XX:+UseShenandoahGC -XX:ShenandoahGCHeuristics=compact

# 吞吐量优先
-XX:+UseShenandoahGC -XX:ShenandoahGCHeuristics=static

# 目标暂停时间
-XX:ShenandoahTargetPauseTime=10ms
```

## 性能对比

### 暂停时间对比

| GC | 平均暂停 | 最大暂停 | 与堆大小关系 |
|----|----------|----------|--------------|
| **G1 GC** | 5-50ms | 200ms+ | 相关 |
| **Shenandoah** | 0.5-2ms | 10ms | 无关 |

### 吞吐量对比

| 场景 | G1 GC | Shenandoah | 差异 |
|------|-------|------------|------|
| CPU 密集 | 100% | 85-95% | -5~15% |
| 内存密集 | 100% | 80-90% | -10~20% |
| 混合负载 | 100% | 90-95% | -5~10% |

## 限制（孵化器阶段）

1. **不支持**：Class Unloading
2. **不支持**：String Deduplication
3. **需要**：`-XX:+UnlockExperimentalVMOptions`
4. **不推荐**：用于生产环境
