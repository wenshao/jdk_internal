## 1. 背景与动机

### 问题

JVM 启动需要处理大量命令行选项：
- GC 选择、内存设置、编译器选项
- 手动调优复杂
- 不同场景需要不同配置

### 解决方案

JEP 514 改进 AOT (Ahead-of-Time) 缓存的命令行处理：
- 自动选择最佳配置
- 简化启动参数
- 与 CDS (Class Data Sharing) 集成

---

## 2. 使用示例

### 创建 AOT 缓存

```bash
# 训练运行，收集配置
java -XX:ArchiveClassesAtExit=app.aot -cp myapp.jar MyApp

# 使用 AOT 缓存启动
java -XX:SharedArchiveFile=app.aot -cp myapp.jar MyApp
```

### 自动配置

```bash
# JDK 26 自动选择最佳 GC
java -XX:SharedArchiveFile=app.aot MyApp

# 自动设置堆大小
java -XX:SharedArchiveFile=app.aot -Xms:auto MyApp
```
