# JEP 504: Remove the Applet API

> **版本**: JDK 26  
> **状态**: 正式移除  
> **Commit**: `5cf672e7784`  
> **作者**: Phil Race  
> **Reviewer**: aivanov, kizune, kcr, achung, serb

---

## 1. 背景与动机

### Applet 的历史

```
1995  JDK 1.0 发布，Applet 作为核心特性
      - 在浏览器中运行 Java 程序
      - "Write Once, Run Anywhere" 的代表
      
2000s 浏览器开始限制 Applet
      - 安全问题
      - Flash/JavaScript 竞争
      
2010s 浏览器移除 NPAPI 支持
      - Chrome 2015 移除 NPAPI
      - Firefox 2017 移除 NPAPI
      
2017  JDK 9 标记 Applet API 为废弃
2021  JDK 17 标记为待移除
2025  JDK 26 正式移除
```

### 移除原因

1. **浏览器不支持**: 所有主流浏览器已移除 Applet 支持
2. **安全风险**: Applet 存在大量安全漏洞
3. **技术过时**: 现代替代方案更安全、更高效
4. **维护成本**: 维护无用代码浪费资源

## 3. 开发者影响

### 受影响的代码

```java
// ❌ 不再编译
import java.applet.Applet;

public class MyApplet extends Applet {
    public void init() {
        // ...
    }
}

// ❌ 不再可用
import javax.swing.JApplet;

public class MyJApplet extends JApplet {
    // ...
}
```

### 迁移方案

| 原方案 | 替代方案 |
|--------|----------|
| Applet | Java Web Start (也已移除) |
| Applet | 独立 Java 应用 |
| Applet | JavaScript/WebAssembly |
| Applet | 桌面应用 (JavaFX/Swing) |

### 迁移示例

```java
// 旧代码: Applet
public class MyApplet extends Applet {
    public void init() {
        add(new Label("Hello"));
    }
}

// 新代码: 独立应用
public class MyApplication extends JFrame {
    public MyApplication() {
        add(new JLabel("Hello"));
        setDefaultCloseOperation(EXIT_ON_CLOSE);
        pack();
        setVisible(true);
    }
    
    public static void main(String[] args) {
        SwingUtilities.invokeLater(MyApplication::new);
    }
}
```

### 检查依赖

```bash
# 检查代码是否使用 Applet API
grep -r "java.applet" src/
grep -r "JApplet" src/
grep -r "AppletContext" src/

# 检查编译错误
javac -source 26 -target 26 MyCode.java
```

```
        - 正式移除
```

---

## 6. 现代替代方案

### Web 应用

```
Applet → JavaScript/TypeScript
       → WebAssembly (可编译 Java)
       → Web Components
```

### 桌面应用

```
Applet → JavaFX
       → Swing 独立应用
       → SWT
```

### 企业应用

```
Applet → Web 应用 (Spring Boot, Jakarta EE)
       → 微服务架构
```
