# JEP 120: Repeating Annotations

> **Status**: Final | **Type**: Feature | **Component**: specification/language
>
> **Lead**: Joe Darcy (Oracle)
>
> **JDK Version**: JDK 8 (2014-03)
>
> **Organization**: [Oracle](../../contributors/orgs/oracle.md)

---

## 1. 概述

JEP 120 允许同一个注解在同一个程序元素（类、方法、字段等）上多次使用。JDK 8 之前，开发者必须手动使用容器注解包装同类型的多个注解。JEP 120 通过 `@Repeatable` 元注解消除了这种样板代码。

---

## 2. 背景

在 JDK 7 及更早版本中，Java 语言规范明确禁止同一注解在同一元素上出现多次。如果框架需要为某个元素关联多个同类型的配置，开发者必须手动定义并使用容器注解：

```java
// JDK 7: 必须手动使用容器注解包装
@Schedules({@Schedule(dayOfMonth = "last"), @Schedule(dayOfWeek = "Fri", hour = "23")})
public void doPeriodicCleanup() { ... }
```

这种模式语法冗余、可读性差，且每个可重复注解都需要配套定义容器注解类型。

---

## 3. @Repeatable 元注解

JDK 8 引入了 `java.lang.annotation.Repeatable` 元注解，用于标记一个注解类型可以在同一元素上重复使用。`@Repeatable` 的唯一属性 `value` 指定该注解对应的容器注解类型：

```java
@Repeatable(Schedules.class)
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.METHOD)
public @interface Schedule {
    String dayOfMonth() default "first";
    String dayOfWeek() default "Mon";
    int hour() default 12;
}
```

---

## 4. 容器注解模式

即使有了 `@Repeatable`，容器注解仍然需要定义——但开发者不再需要直接使用它。容器注解必须包含一个 `value()` 方法返回可重复注解数组，且其 `@Retention` 和 `@Target` 范围必须兼容可重复注解：

```java
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.METHOD)
public @interface Schedules {
    Schedule[] value();
}
```

---

## 5. 编译器如何包装重复注解

当编译器遇到同一元素上的多个重复注解时，会自动将它们包装进容器注解。这是纯编译期转换——源码中写两个 `@Schedule`，字节码中存储为一个 `@Schedules` 容器。如果只出现一次 `@Schedule`，编译器不会包装；两次及以上时才合并为容器注解。

---

## 6. 反射 API 支持

JDK 8 在 `AnnotatedElement` 接口上新增了 `getAnnotationsByType()` 方法，透明获取重复注解：

```java
Schedule[] schedules = method.getAnnotationsByType(Schedule.class);
```

| 方法 | 行为 |
|------|------|
| `getAnnotation(Schedule.class)` | 仅当只有一个 `@Schedule` 时返回，多个时返回 `null` |
| `getAnnotationsByType(Schedule.class)` | 无论一个还是多个，都以数组返回所有实例 |
| `getDeclaredAnnotationsByType(Schedule.class)` | 同上，但不包含继承的注解 |

---

## 7. 典型用例

同一类型的元数据以不同参数值多次应用于同一程序元素是常见需求：

```java
@Schedule(dayOfMonth = "1")              // 多时间调度
@Schedule(dayOfWeek = "Fri", hour = 23)
public void generateReport() { ... }

@Role("admin")                            // 多角色授权
@Role("manager")
public void approveExpense() { ... }

@Filter(pattern = "*.json")               // 多过滤条件
@Filter(pattern = "*.xml")
public void processConfigFiles(Path dir) { ... }
```

---

## 8. 限制与约束

- 容器注解类型仍必须由开发者定义，`@Repeatable` 仅免去手动使用它
- `getAnnotation()` 与 `getAnnotationsByType()` 返回结果不同，迁移时需注意行为差异
