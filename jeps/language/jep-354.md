## 1. 概述

JEP 354 是 switch 表达式的第二轮预览，在 JDK 12 中发布。它继承了 JEP 325（JDK 12，第一轮预览）引入的核心能力——switch 作为表达式使用、箭头标签（`case L ->`）、多值 case 标签——并对从 switch 表达式中返回值的语法进行了重要调整。

---

## 2. 与 JEP 325 的变更

JEP 325（JDK 13）使用 `break value` 语法从 switch 表达式中返回值。社区反馈认为复用 `break` 关键字语义不清晰，容易与循环中的 `break` 混淆。JEP 354 的核心变更是引入专用关键字 `yield` 替代 `break value`。

| 方面 | JEP 325（JDK 13） | JEP 354（JDK 13） |
|------|-------------------|-------------------|
| **表达式返回值** | `break value;` | `yield value;` |
| **箭头标签** | `case L -> expr;` | 不变 |
| **多值 case** | `case L1, L2:` | 不变 |
| **break 语义** | 既可退出 switch 又可返回值 | 仅用于退出 switch 语句 |

---

## 3. yield 关键字

`yield` 是上下文关键字（context-sensitive keyword），仅在 switch 表达式内部作为返回值的语句使用。它不是保留关键字，不影响现有代码中名为 `yield` 的变量或方法。

在箭头标签（`case L ->`）中，如果右侧是单个表达式，则隐式 yield；如果右侧是代码块，则必须使用 `yield` 显式返回值。在冒号标签（`case L:`）中，始终使用 `yield` 返回值。

---

## 4. 代码示例

```java
// 箭头标签 — 单表达式隐式 yield
int numLetters = switch (day) {
    case MONDAY, FRIDAY, SUNDAY -> 6;
    case TUESDAY                -> 7;
    case THURSDAY, SATURDAY     -> 8;
    case WEDNESDAY              -> 9;
};

// 冒号标签 — 使用 yield 返回值
int numLetters = switch (day) {
    case MONDAY, FRIDAY, SUNDAY:
        yield 6;
    case TUESDAY:
        yield 7;
    default:
        yield 0;
};
```

## 5. 演进路径

| JEP | 版本 | 阶段 | 关键变化 |
|-----|------|------|----------|
| JEP 325 | JDK 12 | 第一轮预览 | 引入 switch 表达式，使用 `break value` |
| **JEP 354** | **JDK 13** | **第二轮预览** | **`break value` → `yield value`** |
| JEP 361 | JDK 14 | 正式发布 | switch 表达式成为永久特性 |
