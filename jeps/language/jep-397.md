## 1. 概述

JEP 397 是密封类（Sealed Classes）的第二轮预览，在 JDK 16 中发布。密封类允许类或接口的作者明确控制哪些类可以扩展或实现它，通过 `sealed` 修饰符和 `permits` 子句限定允许的子类集合。这一特性是代数数据类型和模式匹配的基础设施。

---

## 2. 与 JEP 360 的变更

JEP 360（JDK 15，第一轮预览）引入了密封类的核心设计。第二轮预览根据社区反馈进行了以下调整：

| 方面 | JEP 360（JDK 16） | JEP 397（JDK 16） |
|------|-------------------|-------------------|
| **permits 子句语法** | `permits` 关键字 | 不变，但 JLS 语法规则更精确 |
| **局部类和匿名类** | 未明确限制 | 明确禁止局部类和匿名类扩展密封类 |
| **编译器推断 permits** | 支持同文件推断 | 不变 |
| **abstract sealed 组合** | 允许 | 不变 |

局部类和匿名类本质上是实现细节，不应参与密封层次结构的类型约束，因此被明确排除。

---

## 3. 密封类核心语法

密封类的子类必须使用以下三个修饰符之一：

| 修饰符 | 含义 |
|--------|------|
| `final` | 不允许进一步扩展 |
| `sealed` | 自身也是密封的，必须声明 `permits` |
| `non-sealed` | 开放扩展，退出密封层次 |

---

## 4. 代码示例

```java
public sealed interface Shape
    permits Circle, Rectangle, Triangle {
}

public final class Circle implements Shape {
    private final double radius;
    public Circle(double radius) { this.radius = radius; }
}

public sealed class Rectangle implements Shape
    permits Square { /* ... */ }

public final class Square extends Rectangle { /* ... */ }

public non-sealed class Triangle implements Shape { /* open */ }
```

## 5. 演进路径

| JEP | 版本 | 阶段 | 关键变化 |
|-----|------|------|----------|
| JEP 360 | JDK 15 | 第一轮预览 | 引入密封类核心设计 |
| **JEP 397** | **JDK 16** | **第二轮预览** | **语法精化，禁止局部/匿名类扩展密封类** |
| JEP 409 | JDK 17 | 正式发布 | 密封类成为永久特性 |
