## 1. 概述

JEP 323 允许在隐式类型 lambda 表达式的形式参数声明中使用 `var`，使 lambda 参数的声明语法与 JDK 10 中 JEP 286 引入的局部变量类型推断保持一致。开发者可以写 `(var x, var y) -> x + y`，而不仅仅是 `(x, y) -> x + y`。

---

## 2. 背景

JDK 10 通过 JEP 286 引入了局部变量类型推断（`var list = new ArrayList<String>()`），但明确将 lambda 参数排除在外。在 JDK 10 中，lambda 参数只有两种声明方式：

| 方式 | 示例 | 说明 |
|------|------|------|
| **显式类型** | `(String x, String y) -> x + y` | 每个参数标注完整类型 |
| **隐式类型** | `(x, y) -> x + y` | 省略类型，由编译器推断 |

这种不一致性意味着 `var` 可以用于局部变量却不能用于 lambda 参数，语法上存在不必要的缺口。

---

## 3. 变更内容

JDK 11 起，`var` 可以用于 lambda 表达式的形式参数声明，成为第三种合法的声明方式：

```java
// 隐式类型（JDK 8+）
(x, y) -> x + y

// 显式类型（JDK 8+）
(int x, int y) -> x + y

// var 声明（JDK 11+）
(var x, var y) -> x + y
```

编译器仍然根据目标类型推断参数的实际类型，行为与隐式类型 lambda 完全相同。`var` 不会改变推断结果，只提供一种新的语法形式。

---

## 4. 为什么需要这个特性

核心动机是允许为 lambda 参数添加注解而无需写出完整类型。在 JDK 10 中，隐式类型的 lambda 参数不能附加注解，必须退回显式类型声明：

```java
// JDK 10: 必须写出完整类型才能添加注解
(@Nonnull String x, @Nullable String y) -> x + y

// JDK 11: 使用 var 即可添加注解，无需写出完整类型
(@Nonnull var x, @Nullable var y) -> x + y
```

这在使用 Checker Framework、NullAway 等静态分析工具时尤为有用，避免了为添加注解而写出冗长的参数类型。

---

## 5. 限制与约束

在同一个 lambda 表达式中，所有参数必须使用相同的声明方式，不允许混合：

| 语法形式 | 合法性 | 说明 |
|----------|--------|------|
| `(var x, var y) -> ...` | 合法 | 所有参数均使用 var |
| `(@Nonnull var x) -> ...` | 合法 | var 的核心用途之一 |
| `(var x, int y) -> ...` | 非法 | 不能混合 var 与显式类型 |
| `(var x, y) -> ...` | 非法 | 不能混合 var 与隐式类型 |
| `var x -> ...` | 非法 | var 必须在括号内使用 |

单参数 lambda 省略括号时不能使用 `var`，必须写成 `(var x) -> x + 1`。

---

## 6. 与 JEP 286 的一致性

JEP 323 是 JEP 286 的自然延伸，两者共同建立了 `var` 在 Java 中的统一语义：

| 方面 | JEP 286（JDK 10） | JEP 323（JDK 11） |
|------|-------------------|-------------------|
| **适用范围** | 局部变量声明 | Lambda 形式参数 |
| **类型推断** | 从初始化表达式推断 | 从目标类型（函数式接口）推断 |
| **注解支持** | `@Nonnull var x = ...` | `(@Nonnull var x) -> ...` |
| **var 的语义** | 保留类型名称（非关键字） | 同 JEP 286，保留类型名称 |

`var` 在两种上下文中的核心含义一致：声明一个由编译器推断类型的变量。

---

## 7. 对开发者的影响

JEP 323 是一个纯粹的增量特性——不改变任何现有语法的行为，只增加了一种新的合法写法。现有代码无需修改。在不需要注解的场景下，`(var x, var y) -> ...` 与 `(x, y) -> ...` 在语义上完全等价，后者更简洁。`var` 的主要价值体现在需要为 lambda 参数添加注解的场景。
