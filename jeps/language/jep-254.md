# JEP 254: Compact Strings

> **状态**: Released
> **JDK**: JDK 9
> **作者**: [Claes Redestad](/by-contributor/profiles/claes-redestad.md)
> **链接**: [OpenJDK JEP 254](https://openjdk.org/jeps/254)

---

## 概述

将 Java String 的内部表示从 UTF-16 `char[]` 数组改为更紧凑的 `byte[]` 数组，加上编码标识字段。

---

## 动机

### 内存问题

**JDK 9 之前**：所有 String 都使用 UTF-16 (`char[]`) 存储

```java
class String {
    final char[] value;    // 每个 char 2 字节
    final int offset;      // 已废弃 (JDK 7u6)
    final int count;       // 已废弃 (JDK 7u6)
    int hash;              // 缓存哈希
}
```

**问题**：
- ASCII 字符串浪费 50% 空间（每个字符 2 字节）
- 大量 Latin1 字符（英文、数字、HTML、JSON）没有优化空间

### 影响范围

| 应用类型 | Latin1 占比 | 潜在节省 |
|----------|------------|----------|
| 典型 Web 应用 | 60-80% | 30-40% 堆内存 |
| JSON 处理 | 90%+ | 45% 堆内存 |
| 日志系统 | 70-90% | 35-45% 堆内存 |

---

## 解决方案

### 新的 String 内部结构

```java
class String {
    final byte[] value;      // 字节数组 (紧凑)
    final byte coder;        // 编码标识: 0=Latin1, 1=UTF16
    // private final int hash;  // 移到字段末尾
}
```

### 编码模式

| Coder | 名称 | 范围 | 每字符大小 |
|-------|------|------|-----------|
| 0 | LATIN1 | U+0000 - U+00FF | 1 字节 |
| 1 | UTF16 | U+0000 - U+FFFF+ | 2 字节 |

### 自动转换

```java
// 创建时自动选择编码
String s1 = "Hello";        // Latin1 (coder=0)
String s2 = "你好";         // UTF16 (coder=1)

// 拼接时自动升级
String result = s1 + s2;    // UTF16 (coder=1)
```

## 实现

### StringLatin1 和 StringUTF16

```java
// 包级私有类
package java.lang;

final class StringLatin1 {
    static int indexOf(byte[] value, int ch) {
        // 单字节字符搜索
    }
    static void inflate(byte[] src, char[] dst) {
        // Latin1 → UTF16 扩展
    }
}

final class StringUTF16 {
    static int indexOf(byte[] value, int ch) {
        // 双字节字符搜索
    }
    static byte[] toBytes(char[] value) {
        // UTF16 → 压缩
    }
}
```

### 关键方法实现

```java
// String.charAt()
public char charAt(int index) {
    if (isLatin1()) {
        return StringLatin1.charAt(value, index);
    } else {
        return StringUTF16.charAt(value, index);
    }
}

private boolean isLatin1() {
    return coder == 0;
}
```

## VM 参数

```bash
# 启用 Compact Strings (默认)
-XX:+CompactStrings

# 禁用 Compact Strings
-XX:-CompactStrings

# 检查状态
-XX:+PrintCompactStringsInfo
```

**禁用场景**：
- 调试编码相关 bug
- 性能基准测试对比
- 特殊字符处理需求
