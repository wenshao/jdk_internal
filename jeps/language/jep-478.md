## 1. 概述

JEP 478 是密钥派生函数（Key Derivation Function, KDF）API 的第一轮预览，在 JDK 24 中引入。它在 `javax.crypto` 包中新增 `KDF` 类，为 HKDF（HMAC-based Extract-and-Expand Key Derivation Function）等密钥派生算法提供标准化的 Java API。此前 JDK 内部实现了 HKDF 用于 TLS 1.3，但未向开发者公开。

---

## 2. 核心 API

| 类/接口 | 说明 |
|---------|------|
| `javax.crypto.KDF` | 密钥派生函数的引擎类，类似 `Cipher`、`Mac` |
| `javax.crypto.KDFParameters` | KDF 参数的标记接口 |
| `javax.crypto.spec.HKDFParameterSpec` | HKDF 专用参数规范，支持 extract、expand、extractExpand |

`KDF` 遵循 JCA（Java Cryptography Architecture）的标准工厂模式，使用 `KDF.getInstance(algorithm)` 获取实例。

---

## 3. HKDF 支持

JEP 478 内置支持以下 HKDF 变体，符合 RFC 5869 和 NIST SP 800-56C Rev. 2 规范：

| 算法名称 | 说明 |
|----------|------|
| `HKDF-SHA256` | 使用 HMAC-SHA-256 的 HKDF |
| `HKDF-SHA384` | 使用 HMAC-SHA-384 的 HKDF |
| `HKDF-SHA512` | 使用 HMAC-SHA-512 的 HKDF |

每种算法支持三种操作模式：仅提取（extract）、仅扩展（expand）、提取后扩展（extract-then-expand）。

---

## 4. 代码示例

```java
// HKDF extract-then-expand
KDF kdf = KDF.getInstance("HKDF-SHA256");

SecretKey inputKey = /* 输入密钥材料 */;
byte[] salt = /* 盐值 */;
byte[] info = /* 上下文信息 */;

HKDFParameterSpec params = HKDFParameterSpec
    .extractThenExpand(inputKey, salt, info, 32);

SecretKey derivedKey = kdf.deriveKey("AES", params);
```

```

主要应用场景包括 TLS 1.3 密钥调度、后量子密码学（KEM 输出需通过 KDF 派生会话密钥）、NIST SP 800-56C 合规要求等。
