## 1. 概述

JEP 368 是文本块（Text Blocks）的第二轮预览，在 JDK 13 中发布。它保留了 JEP 355（JDK 13，第一轮预览）的所有核心特性——三引号定界符、自动缩进管理、行终止符规范化——并新增了两个转义序列以提供对空白和换行的精细控制。

---

## 2. 与 JEP 355 的变更

第二轮预览的变更范围较小且聚焦，核心是新增两个转义序列：

| 方面 | JEP 355（JDK 14） | JEP 368（JDK 14） |
|------|-------------------|-------------------|
| **尾部空白** | 默认去除，无法保留 | 新增 `\s` 可显式保留空白 |
| **行续符** | 不支持 | 新增 `\`（行末反斜杠）抑制换行 |
| **其他特性** | 三引号、缩进管理、转义 | 不变 |

---

## 3. 新增转义序列

### `\s` — 空格转义

`\s` 转换为单个空格字符（U+0020）。它的主要用途是防止编译器去除行尾空白。放在行末时，`\s` 及其之前的空白字符都会被保留。

### `\` — 行续符（行末反斜杠）

行末的 `\` 抑制该行的换行符，使当前行与下一行在结果字符串中合并为一行。这在文本块中需要写很长的单行文本时非常有用。

行末的 `\` 使当前行与下一行合并为一行，适用于在文本块中书写很长的单行文本。

---

## 4. 代码示例

```java
// \s 保留尾部空白（常见于对齐表格数据）
String aligned = """
    red  \s
    green\s
    blue \s
    """;

// \ 行续符 — 生成单行 SQL
String query = """
    SELECT id, name, email \
    FROM users \
    WHERE active = true\
    """;
```

这两个转义序列同样适用于普通字符串字面量（`"..."`），但其主要价值体现在文本块中。
