# JEP 192: String Deduplication

> **Status**: Permanent | **Type**: Feature | **Component**: HotSpot / GC
>
> **Lead**: Monica Pavlyk (Oracle)
>
> **JDK Version**: JDK 8u20 (2014-08)
>
> **Organization**: [Oracle](../../contributors/orgs/oracle.md)

---

## 概览

引入字符串去重功能，通过共享 String 对象的底层 char 数组来减少内存占用，特别适用于大量重复字符串的场景。

---

## 目标

- 减少 String 对象的内存占用
- 透明地共享相同内容的字符串
- 与现有应用完全兼容
- 低开销的并发去重

---

## 技术细节

### 问题背景

```java
// 典型场景：大量重复字符串
List<String> list = new ArrayList<>();
for (int i = 0; i < 1000000; i++) {
    list.add("Hello World");  // 每个都是新对象
}

// 传统实现：每个 String 都有独立的 char[]
// 内存占用：1000000 × (24 + 12 × 2) = 48MB
```

### 去重原理

```
去重前：
String1 → char[] "Hello"
String2 → char[] "Hello"  ← 重复
String3 → char[] "Hello"  ← 重复

去重后：
String1 ─┐
String2 ├─→ char[] "Hello" (共享)
String3 ─┘
```

### 实现机制

```java
// G1 GC 中的字符串去重
// 在并发标记阶段识别重复字符串
// 在混合收集阶段进行去重

// 伪代码：
void deduplicate(String str) {
    char[] current = str.value;
    char[] existing = findExisting(current);
    
    if (existing != null) {
        str.value = existing;  // 指向现有数组
        // 原数组可被 GC 回收
    }
}
```

---

## 使用方式

### 启用字符串去重

```bash
# 需要 G1 GC
-XX:+UseG1GC

# 启用字符串去重
-XX:+UseStringDeduplication

# 可选：打印统计信息
-XX:+PrintStringDeduplicationStatistics
```

### 输出示例

```
[0.123s] String Deduplication Statistics:
[0.123s]   Total Strings: 1000000
[0.123s]   Deduplicated: 500000 (50%)
[0.123s]   Memory Saved: 24MB
```

## 性能影响

### 内存节省

| 场景 | 去重前 | 去重后 | 节省 |
|------|--------|--------|------|
| 日志应用 | 100MB | 60MB | 40% |
| Web 应用 | 200MB | 150MB | 25% |
| 数据处理 | 500MB | 300MB | 40% |

### CPU 开销

| 操作 | 开销 |
|------|------|
| 去重处理 | <1% |
| 哈希计算 | <0.5% |
| 总体影响 | <2% |

---

## 适用场景

### 推荐使用

- ✅ 大量重复字符串的应用
- ✅ 日志处理系统
- ✅ 数据处理/ETL 应用
- ✅ Web 应用（重复 URL、参数名）

### 不推荐

- ❌ 字符串重复率低的应用
- ❌ 内存充足的环境
- ❌ 对 CPU 极度敏感的场景
