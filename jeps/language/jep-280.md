## 1. 概述

将字符串拼接操作从 `StringBuilder` 模式改为使用 `invokedynamic` 和 `java.lang.invoke.StringConcatFactory`，为未来的优化提供更好的基础。

---

## 2. 动机

### JDK 9 之前的问题

**编译方式**：
```java
String result = "Hello " + name + "!";

// 编译为:
new StringBuilder().append("Hello ").append(name).append("!").toString();
```

**问题**：
1. **每次都创建 StringBuilder 对象**
2. **StringBuilder 方法无法内联**（太大）
3. **无法针对特定场景优化**
4. **字节码膨胀**

### 性能问题

| 场景 | JDK 8 | 问题 |
|------|-------|------|
| 简单拼接 | `new SB().append().append()` | 对象创建开销 |
| 循环拼接 | 多次 SB 创建 | 无法优化 |
| 启动阶段 | 大量 SB 对象 | GC 压力 |

---

## 3. 解决方案

### invokedynamic 调用

**JDK 9+ 编译方式**：
```java
String result = "Hello " + name + "!";

// 编译为:
invokedynamic makeConcatWithConstants(Ljava/lang/String;)Ljava/lang/String;
```

### StringConcatFactory

```java
// java.lang.invoke.StringConcatFactory
public final class StringConcatFactory {

    // 生成拼接方法
    public static CallSite makeConcatWithConstants(
            MethodHandles.Lookup lookup,
            String name,
            MethodType type,
            Object... constants) throws StringConcatException {
        // 返回优化的 MethodHandle
    }

    // 生成带常量的拼接方法
    public static CallSite makeConcatWithConstants(
            MethodHandles.Lookup lookup,
            String name,
            MethodType type,
            String recipe,
            Object... constants) throws StringConcatException {
        // recipe: "1\u0001\u0002" (参数位置)
        // constants: {"Hello ", "!"}
    }
}
```

## 4. 优化策略

### 策略选择

| 策略 | 名称 | 说明 | 适用场景 |
|------|------|------|----------|
| BC | Bytecode | 生成字节码 (SB 风格) | 兜底 |
| HB | HandleInline | 内联句柄 | 少量参数 |
| HM | HandleMixin | 混合句柄 | 中等参数 |
| MC | MethodHandleInlineClass | 隐藏类 | 大量参数 |

### 配置参数

```bash
# 查看策略
-Djava.lang.invoke.StringConcat.DEBUG=true
-Djava.lang.invoke.StringConcat.cacheThreshold=256
-Djava.lang.invoke.StringConcat.factory=BC|HB|HM|MC
```

## 6. 字节码对比

### JDK 8

```
 0: new           #2  // class java/lang/StringBuilder
 3: dup
 4: invokespecial #3  // Method StringBuilder."<init>":()V
 7: ldc           #4  // String Hello
 9: invokevirtual #5  // Method StringBuilder.append:(Ljava/lang/String;)Ljava/lang/StringBuilder;
12: aload_1
13: invokevirtual #5  // Method StringBuilder.append:(Ljava/lang/String;)Ljava/lang/StringBuilder;
16: ldc           #6  // String !
18: invokevirtual #5  // Method StringBuilder.append:(Ljava/lang/String;)Ljava/lang/StringBuilder;
21: invokevirtual #7  // Method StringBuilder.toString:()Ljava/lang/String;
```

### JDK 9

```
 0: aload_1
 1: invokedynamic #2,  0  // InvokeDynamic #0:makeConcatWithConstants:(Ljava/lang/String;)Ljava/lang/String;
 6: areturn
```

**字节码减少**: 21 → 6 字节 (71% 减少)

## 8. 实现细节

### MethodHandle 组合树

```java
// "a" + b + "c" + d 的 MH 组合
filterArgument(d, String::valueOf)
  .foldArguments(newArray())
  .foldArguments(prepend(d, "c"))
  .foldArguments(prepend(b, "a"))
```

### 隐藏类策略 (JDK 24+)

```java
// 为每个"形状"生成隐藏类
// "String" + "String" → 共享同一个类
// "int" + "String"    → 共享另一个类

// 性能优势:
// - 元空间共享 (-60%)
// - 启动性能 (+40%)
```
