# JEP 492: Flexible Constructor Bodies (Third Preview)

> **Status**: Preview (3rd) | **Type**: Feature | **Component**: specification/language
>
> **Lead**: Archie Cobbs (Oracle)
>
> **JDK Version**: JDK 24 (2025-03)
>
> **Organization**: [Oracle](../../contributors/orgs/oracle.md)

---

## 1. 概述

JEP 492 是 Flexible Constructor Bodies 的第三轮预览，在 JDK 24 中发布。该特性允许在构造器中在调用 `super(...)` 或 `this(...)` 之前执行语句，突破了 Java 长期以来要求构造器第一条语句必须是父类调用的限制。

---

## 2. 背景与变更

自 Java 1.0 以来，构造器的第一条语句必须是 `super(...)` 或 `this(...)` 调用，迫使开发者将参数验证等逻辑放入辅助静态方法中。JEP 447（JDK 24）首次预览了该特性。JEP 492 相对于 JEP 482（JDK 23）的变更：

| 变更领域 | 说明 |
|----------|------|
| **Prologue 语句** | 进一步明确了在 super() 之前允许的语句类型 |
| **字段访问规则** | 细化了 prologue 中访问未初始化字段的限制 |
| **错误消息** | 改进了编译器对违规使用的诊断信息 |

构造器主体分为 **Prologue**（super() 之前，不可访问 this）和 **Epilogue**（super() 之后，传统行为）两个阶段。

---

## 3. 代码示例

```java
// JDK 24 之前：需要辅助静态方法
class PositiveInteger {
    PositiveInteger(int value) {
        super(validate(value));
    }
    private static int validate(int v) {
        if (v <= 0) throw new IllegalArgumentException();
        return v;
    }
}

// JDK 24：直接在 super() 之前验证
class PositiveInteger {
    PositiveInteger(int value) {
        if (value <= 0)
            throw new IllegalArgumentException("Must be positive");
        super(value);
    }
}
```

---

## 4. 限制与约束

| 限制 | 说明 |
|------|------|
| **不可访问 this** | Prologue 中不能读写实例字段或调用实例方法 |
| **不可访问 super** | Prologue 中不能调用父类方法 |
| **局部变量可用** | 可声明和使用局部变量、调用静态方法 |
| **预览特性** | 需 `--enable-preview` 编译和运行 |
