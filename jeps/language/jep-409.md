# JEP 409: Sealed Classes

> **Status**: Permanent | **Type**: Feature | **Component**: Java SE / Language
>
> **Lead**: Brian Goetz (Oracle)
>
> **JDK Version**: JDK 17 (2021-09)
>
> **Organization**: [Oracle](../../contributors/orgs/oracle.md)

---

## 概览

引入密封类和密封接口，限制哪些类可以继承或实现它们，为模式匹配提供基础。

---

## 目标

- 限制类层次结构的使用
- 提供更强的封装性
- 支持模式匹配的穷尽性检查
- 保持向后兼容

---

## 语法

### 基本用法

```java
// 密封类
public sealed class Shape 
    permits Circle, Rectangle, Triangle {
    // ...
}

// 子类必须明确声明
public final class Circle extends Shape { }
public final class Rectangle extends Shape { }
public final class Triangle extends Shape { }
```

### 修饰符组合

```java
// sealed: 允许指定子类
public sealed class A permits B, C { }

// non-sealed: 允许继续扩展
public non-sealed class B extends A { }

// final: 不允许继承
public final class C extends A { }
```

### 密封接口

```java
public sealed interface Expr 
    permits Lit, Add, Mul {
    // ...
}

public final class Lit implements Expr { }
public sealed class Add implements Expr 
    permits Add2 { }
public final class Add2 extends Add { }
```

## 使用场景

### 代数数据类型

```java
// 表达式树
public sealed interface Expr {
    int eval();
}

public record Lit(int value) implements Expr {
    public int eval() { return value; }
}

public record Add(Expr left, Expr right) implements Expr {
    public int eval() { 
        return left.eval() + right.eval(); 
    }
}

public record Mul(Expr left, Expr right) implements Expr {
    public int eval() { 
        return left.eval() * right.eval(); 
    }
}

// 模式匹配保证穷尽性
int compile(Expr e) {
    return switch (e) {
        case Lit(int v) -> 1;
        case Add(Expr l, Expr r) -> 2;
        case Mul(Expr l, Expr r) -> 3;
        // 编译器知道这是穷尽的
    };
}
```

### 领域模型

```java
// 支付系统
public sealed class Payment 
    permits CreditCardPayment, PayPalPayment, CryptoPayment {
    // ...
}

// 只有指定的子类可以扩展
public final class CreditCardPayment extends Payment { }
public final class PayPalPayment extends Payment { }
public final class CryptoPayment extends Payment { }
```

### API 设计

```java
// 限制 API 扩展
public sealed class Result<T> 
    permits Success, Error {
    // ...
}

public final class Success<T> extends Result<T> { }
public final class Error<T> extends Result<T> { }
```

## 编译时检查

### 允许的类型

```java
// ✅ 同一文件中
public sealed class A permits B { }
class B extends A { }

// ✅ 不同文件中（需要 permits）
// A.java
public sealed class A permits B { }

// B.java
public final class B extends A { }

// ❌ 未授权子类
class C extends A { }  // 编译错误
```

### 子类要求

```java
// 子类必须是 final、sealed 或 non-sealed
public sealed class A permits B { }

// ✅ 正确
public final class B extends A { }
public sealed class C extends A permits D { }
public non-sealed class E extends A { }

// ❌ 错误
class F extends A { }  // 缺少修饰符
```

## 与模式匹配集成

### Switch 表达式

```java
String format(Shape s) {
    return switch (s) {
        case Circle c -> "Circle with radius " + c.radius();
        case Rectangle r -> "Rectangle " + r.width() + "x" + r.height();
        case Triangle t -> "Triangle";
        // 编译器验证穷尽性
    };
}
```

### Instanceof 模式

```java
void process(Expr e) {
    if (e instanceof Lit(int value)) {
        System.out.println("Literal: " + value);
    } else if (e instanceof Add(Expr l, Expr r)) {
        System.out.println("Addition");
    }
    // 不需要 else：密封类保证穷尽
}
```
