## 1. 概述

JEP 488 是 Primitive Types in Patterns, instanceof, and switch 的第二轮预览，在 JDK 24 中发布。该特性将模式匹配扩展到所有原始类型（`int`、`long`、`float`、`double`、`boolean` 等），允许在 `instanceof`、`switch` 和模式匹配上下文中使用原始类型模式。

---

## 2. 背景与变更

Java 的模式匹配（自 JDK 23 起逐步引入）最初仅支持引用类型。JEP 455（JDK 23）首次预览了原始类型模式的支持。JEP 488 的主要变更：

| 变更领域 | 说明 |
|----------|------|
| **类型转换安全性** | 改进了窄化转换时的精确匹配语义 |
| **与 record 模式的集成** | 优化了原始类型在 record 解构中的行为 |
| **规范更新** | 根据社区反馈细化了 JLS 中的匹配规则 |

---

## 3. 核心特性

| 上下文 | 说明 |
|--------|------|
| **instanceof** | `x instanceof int i` 可测试值是否可安全窄化为 int |
| **switch** | `switch (obj)` 中可使用原始类型 case 标签 |
| **模式匹配** | record 模式中可直接解构出原始类型分量 |

关键语义：原始类型模式仅在值可以无损转换时匹配成功（精确匹配），避免数据丢失。

---

## 4. 代码示例

```java
// instanceof 中使用原始类型模式
long value = 42L;
if (value instanceof int i) {
    System.out.println("Fits in int: " + i);
}

// switch 中使用原始类型
switch (statusCode) {
    case 0 -> System.out.println("Success");
    case 1 -> System.out.println("Warning");
    case int i when i > 1 -> System.out.println("Error: " + i);
}
```
