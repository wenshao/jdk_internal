# JEP 286: Local-Variable Type Inference

> **Status**: Permanent | **Type**: Feature | **Component**: Java SE / Language
>
> **Lead**: Brian Goetz (Oracle)
>
> **JDK Version**: JDK 10 (2018-03)
>
> **Organization**: [Oracle](../../contributors/orgs/oracle.md)

---

## 概览

引入 `var` 关键字用于局部变量类型推断，允许声明局部变量时不显式指定类型，由编译器根据初始化表达式推断类型。

---

## 目标

- 简化代码编写
- 保持静态类型安全
- 不改变运行时行为
- 与现有代码完全兼容

---

## 动机

Java 作为静态类型语言，变量声明往往冗长：

```java
// JDK 9 及之前
HashMap<String, List<Integer>> hashMap = new HashMap<String, List<Integer>>();

// JDK 10+
var hashMap = new HashMap<String, List<Integer>>();
```

---

## 语法

### 基本用法

```java
// 基本类型
var i = 42;           // int
var d = 3.14;         // double
var b = true;         // boolean

// 引用类型
var s = "Hello";      // String
var list = new ArrayList<String>();  // ArrayList<String>

// 匿名类
var obj = new Object() {
    void method() {}
};
```

### 限制

```java
// ❌ 不允许的情况
var x;          // 错误：需要初始化
var x = null;   // 错误：无法推断类型
var x = {1,2};  // 错误：集合字面量需要显式类型

// ❌ 不能用于
var field;      // 错误：不能用于字段
var method() {} // 错误：不能用于方法返回类型
```

---

## 最佳实践

### 推荐

```java
// 类型明显时
var stream = list.stream();
var map = new HashMap<String, Integer>();

// 泛型复杂时
var comparator = (Comparator<String>) (a, b) -> a.length() - b.length();
```

### 不推荐

```java
// 类型不明显
var result = compute();  // ❌ 读者不知道 compute() 返回什么

// 基本类型
var i = 0;  // ❌ int i = 0; 更清晰
```

---

## 编译后

```java
// 源代码
var list = new ArrayList<String>();
list.add("Hello");

// 字节码（反编译后）
ArrayList<String> list = new ArrayList<String>();
list.add("Hello");
```

**类型推断发生在编译期**，运行时类型不变。

## 影响

### 代码简洁性

| 场景 | JDK 9 | JDK 10+ | 减少 |
|------|-------|---------|------|
| 集合声明 | 45 字符 | 25 字符 | 44% |
| 匿名类 | 60 字符 | 35 字符 | 42% |
| 泛型嵌套 | 80 字符 | 40 字符 | 50% |

### 可读性

- ✅ 减少样板代码
- ⚠️ 过度使用可能降低可读性
- ✅ 配合 IDE 类型提示效果最佳
