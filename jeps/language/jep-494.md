## 1. 概述

JEP 494 是 Module Import Declarations 的第二轮预览，在 JDK 24 中发布。该特性引入了 `import module` 声明，允许一次性导入整个模块导出的所有包中的公共类型，简化了 import 语句的编写。

---

## 2. 背景与变更

Java 开发者经常需要编写大量 import 语句。模块系统（JDK 24）已定义了包的逻辑分组，`import module` 利用了这一分组结构。JEP 494 相对于 JEP 476（JDK 23）的变更：

| 变更领域 | 说明 |
|----------|------|
| **歧义解析规则** | 改进了当多个模块导出同名类型时的解析策略 |
| **与传统导入的交互** | 明确了 `import module` 与 `import` 语句的优先级规则 |
| **编译器诊断** | 增强了歧义类型的错误提示信息 |

---

## 3. 核心语法与示例

```java
import module java.base;

// 无需单独 import List、Map、Stream 等
public class Example {
    public static void main(String[] args) {
        List<String> names = List.of("Alice", "Bob");
        Map<String, Integer> scores = Map.of("Alice", 95);
        String result = names.stream()
            .collect(Collectors.joining(", "));
    }
}
```

当多个模块导出相同简单名称的类型时，使用传统导入消除歧义：

```java
import module java.base;
import module java.sql;
import java.sql.Date;  // 明确选择 java.sql.Date
```

传统的单类型导入始终优先于模块导入中的同名类型。
