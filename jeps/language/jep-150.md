## 1. 概述

JEP 150 实现了 JSR 310 (Date and Time API)，为 Java 平台引入了全新的日期与时间库 `java.time`。该 API 取代了长期被诟病的 `java.util.Date` 和 `java.util.Calendar`，以 Joda-Time 库为灵感来源，由 Joda-Time 作者 Stephen Colebourne 主导设计。

核心目标：
- 提供不可变 (immutable)、线程安全的日期时间类型
- 清晰区分人类可读时间与机器时间
- 以 ISO-8601 为默认日历系统
- 提供流畅 (fluent) 的 API 风格

## 2. 背景：旧 API 的问题

### 2.1 java.util.Date 的缺陷

```java
// 问题 1: 可变性 (Mutable) — 线程不安全
Date date = new Date();
date.setTime(0L); // 可以被任意修改

// 问题 2: 命名误导 — Date 实际上包含时间信息
Date now = new Date(); // 包含日期 + 时间 + 时区偏移

// 问题 3: 月份从 0 开始 (0 = January)
Date d = new Date(2014, 2, 18); // 实际是 3914 年 3 月 18 日
// year = 1900 + 2014, month = 2 表示 March

// 问题 4: 大量方法已废弃
// getYear(), getMonth(), getDate() 等均标记 @Deprecated
```

**关键问题总结**：

| 问题 | 说明 |
|------|------|
| Mutable | 无法安全地在多线程间共享 |
| 线程不安全 | `SimpleDateFormat` 也不是线程安全的 |
| API 设计糟糕 | 年份偏移 1900，月份从 0 开始 |
| 职责不清 | `Date` 既表示日期又表示时间戳 |

### 2.2 java.util.Calendar 的缺陷

```java
// 问题 1: 月份仍然从 0 开始
Calendar cal = Calendar.getInstance();
cal.set(2014, 2, 18); // 2 表示 March，不是 February

// 问题 2: 同样是 Mutable
cal.add(Calendar.DAY_OF_MONTH, 1); // 直接修改原对象

// 问题 3: API 冗长且容易出错
int year = cal.get(Calendar.YEAR);
int month = cal.get(Calendar.MONTH) + 1; // 必须 +1 才能得到实际月份

// 问题 4: 类型不安全 — 字段使用 int 常量
cal.set(Calendar.DAY_OF_WEEK, 42); // 编译通过，运行时行为未定义
```

### 2.3 Joda-Time 的启示

Joda-Time 是 Stephen Colebourne 在 2002 年创建的第三方日期时间库，成为 Java 社区的事实标准。JSR 310 吸收了 Joda-Time 的核心设计理念，但进行了重要改进：

| 方面 | Joda-Time | JSR 310 (java.time) |
|------|-----------|---------------------|
| 包名 | `org.joda.time` | `java.time` |
| Null 处理 | 接受 null（用默认值） | 拒绝 null（抛出 NullPointerException） |
| 内部实现 | 基于毫秒时间戳 | 各类型独立存储字段 |
| Chronology | 可插拔日历系统 | 简化为 `java.time.chrono` |
| 与旧 API 互操作 | 提供转换方法 | 内建转换方法 |

## 3. 核心 API 设计

### 3.1 无时区类型（人类本地时间）

```java
// LocalDate — 仅日期，无时间、无时区
LocalDate date = LocalDate.of(2014, 3, 18);       // 2014-03-18
LocalDate today = LocalDate.now();                  // 当前日期
LocalDate parsed = LocalDate.parse("2014-03-18");   // 从字符串解析

// LocalTime — 仅时间，无日期、无时区
LocalTime time = LocalTime.of(14, 30, 0);           // 14:30:00
LocalTime noon = LocalTime.NOON;                     // 12:00

// LocalDateTime — 日期 + 时间，无时区
LocalDateTime dt = LocalDateTime.of(2014, 3, 18, 14, 30);
LocalDateTime combined = LocalDateTime.of(date, time);
```

**核心特征**：这些类型不包含时区信息，表示"本地"概念。适用于生日、会议时间等不依赖时区的场景。

### 3.2 带时区类型

```java
// ZonedDateTime — 完整的日期时间 + 时区
ZonedDateTime zdt = ZonedDateTime.of(
    LocalDateTime.of(2014, 3, 18, 14, 30),
    ZoneId.of("Asia/Shanghai")
);
// 2014-03-18T14:30+08:00[Asia/Shanghai]

// OffsetDateTime — 日期时间 + UTC 偏移量（无时区规则）
OffsetDateTime odt = OffsetDateTime.of(
    LocalDateTime.of(2014, 3, 18, 14, 30),
    ZoneOffset.ofHours(8)
);
// 2014-03-18T14:30+08:00

// ZoneId vs ZoneOffset
ZoneId shanghai = ZoneId.of("Asia/Shanghai");     // 包含夏令时规则
ZoneOffset offset = ZoneOffset.ofHours(8);         // 固定偏移，无夏令时
```

**ZonedDateTime vs OffsetDateTime 的选择**：

| 场景 | 推荐类型 |
|------|---------|
| 用户界面展示 | `ZonedDateTime` |
| 数据库存储 | `OffsetDateTime` |
| 网络传输/序列化 | `OffsetDateTime` |
| 跨时区计算 | `ZonedDateTime` |

### 3.3 机器时间

```java
// Instant — UTC 时间轴上的一个点（纳秒精度）
Instant now = Instant.now();                         // 当前时刻
Instant epoch = Instant.EPOCH;                       // 1970-01-01T00:00:00Z
Instant specific = Instant.ofEpochSecond(1395100200);

// 与旧 API 互转
Instant fromDate = new Date().toInstant();
Date fromInstant = Date.from(instant);
```

**Instant 的定位**：纯粹的机器时间戳，不关联任何时区或日历。适用于日志时间戳、性能测量、事件排序。

### 3.4 时间量

```java
// Duration — 基于时间的量（秒 + 纳秒）
Duration fiveMinutes = Duration.ofMinutes(5);
Duration between = Duration.between(time1, time2);
long seconds = fiveMinutes.getSeconds(); // 300

// Period — 基于日期的量（年、月、日）
Period tenDays = Period.ofDays(10);
Period threeMonths = Period.ofMonths(3);
Period between = Period.between(date1, date2);

// 关键区别
Duration d = Duration.ofDays(1);   // 精确的 86400 秒
Period p = Period.ofDays(1);        // 概念上的"一天"（可能因夏令时为 23 或 25 小时）
```

| 类型 | 存储 | 适用场景 |
|------|------|---------|
| `Duration` | 秒 + 纳秒 | 计时、超时、精确时间差 |
| `Period` | 年 + 月 + 日 | 人类概念的时间段（年龄、合同期限） |

## 4. 设计原则

### 4.1 不可变性 (Immutability)

所有核心类型都是 immutable 且线程安全的。"修改"操作返回新实例：

```java
LocalDate date = LocalDate.of(2014, 3, 18);
LocalDate nextDay = date.plusDays(1);    // 返回新对象，date 不变
LocalDate nextMonth = date.plusMonths(1); // 同上
```

### 4.2 Fluent API

链式调用风格，代码可读性强：

```java
LocalDateTime meeting = LocalDate.now()
    .atTime(14, 30)
    .plusHours(2)
    .with(TemporalAdjusters.nextOrSame(DayOfWeek.MONDAY));
```

### 4.3 人类时间与机器时间分离

API 明确区分两种时间概念：

| 概念 | 类型 | 含义 |
|------|------|------|
| 人类时间 | `LocalDate`, `LocalTime`, `ZonedDateTime` | 日历/时钟上的显示 |
| 机器时间 | `Instant`, `Duration` | 时间轴上的精确点/段 |

### 4.4 ISO-8601 作为默认标准

所有 `toString()` 和 `parse()` 方法默认使用 ISO-8601 格式：

```java
LocalDate.of(2014, 3, 18).toString();  // "2014-03-18"
LocalTime.of(14, 30).toString();        // "14:30"
Instant.now().toString();               // "2014-03-18T06:30:00Z"
```

## 5. 实现细节

### 5.1 java.time 包结构

| 包 | 用途 |
|----|------|
| `java.time` | 核心类：`LocalDate`, `Instant`, `Duration` 等 |
| `java.time.chrono` | 非 ISO 日历系统（日本历、泰国历等） |
| `java.time.format` | `DateTimeFormatter` 格式化与解析 |
| `java.time.temporal` | 底层框架：`TemporalField`, `TemporalUnit`, `TemporalAdjuster` |
| `java.time.zone` | 时区规则：`ZoneRules`, `ZoneOffsetTransition` |

### 5.2 ChronoField 与 ChronoUnit

`ChronoField` 和 `ChronoUnit` 是 `TemporalField` 和 `TemporalUnit` 的标准实现：

```java
// ChronoField — 日期时间字段
int dayOfYear = date.get(ChronoField.DAY_OF_YEAR);      // 第几天
int hourOfDay = time.get(ChronoField.HOUR_OF_DAY);       // 小时

// ChronoUnit — 时间单位
long daysBetween = ChronoUnit.DAYS.between(date1, date2);
long hoursBetween = ChronoUnit.HOURS.between(time1, time2);
boolean supported = date.isSupported(ChronoUnit.HOURS);   // false — LocalDate 不支持小时
```

### 5.3 TemporalAdjusters

`TemporalAdjuster` 提供常见日期调整的工具方法：

```java
import static java.time.temporal.TemporalAdjusters.*;

LocalDate date = LocalDate.of(2014, 3, 18);

date.with(firstDayOfMonth());               // 2014-03-01
date.with(lastDayOfMonth());                // 2014-03-31
date.with(firstDayOfNextMonth());           // 2014-04-01
date.with(next(DayOfWeek.FRIDAY));          // 下一个周五
date.with(previousOrSame(DayOfWeek.MONDAY)); // 上一个（或当天）周一

// 自定义 TemporalAdjuster
TemporalAdjuster nextWorkingDay = t -> {
    LocalDate d = LocalDate.from(t);
    DayOfWeek dow = d.getDayOfWeek();
    if (dow == DayOfWeek.FRIDAY) return d.plusDays(3);
    if (dow == DayOfWeek.SATURDAY) return d.plusDays(2);
    return d.plusDays(1);
};
date.with(nextWorkingDay); // 跳过周末的下一个工作日
```

## 6. DateTimeFormatter

### 6.1 预定义格式

```java
LocalDate date = LocalDate.of(2014, 3, 18);

date.format(DateTimeFormatter.ISO_LOCAL_DATE);      // "2014-03-18"
date.format(DateTimeFormatter.BASIC_ISO_DATE);      // "20140318"

ZonedDateTime zdt = ZonedDateTime.now();
zdt.format(DateTimeFormatter.ISO_ZONED_DATE_TIME);  // 完整 ISO 格式
zdt.format(DateTimeFormatter.RFC_1123_DATE_TIME);   // HTTP 日期格式
```

### 6.2 自定义格式

```java
DateTimeFormatter formatter = DateTimeFormatter.ofPattern("yyyy/MM/dd HH:mm");
String text = LocalDateTime.now().format(formatter);     // "2014/03/18 14:30"
LocalDateTime parsed = LocalDateTime.parse(text, formatter);

// 本地化格式
DateTimeFormatter chinese = DateTimeFormatter
    .ofLocalizedDate(FormatStyle.FULL)
    .withLocale(Locale.CHINA);
date.format(chinese); // "2014年3月18日 星期二"
```

### 6.3 与 SimpleDateFormat 对比

| 特性 | `SimpleDateFormat` | `DateTimeFormatter` |
|------|-------------------|---------------------|
| 线程安全 | 否 | 是（immutable） |
| 模式语法 | 类似 | 类似但更严格 |
| 解析结果 | `Date` | `TemporalAccessor` |
| 推荐使用 | JDK 8 之前 | JDK 8+ |

## 7. 从旧 API 迁移

### 7.1 互转方法

JDK 8 为旧类型添加了桥接方法：

```java
// Date <-> Instant
Date date = new Date();
Instant instant = date.toInstant();
Date backToDate = Date.from(instant);

// Date <-> LocalDateTime (需要时区)
LocalDateTime ldt = date.toInstant()
    .atZone(ZoneId.systemDefault())
    .toLocalDateTime();

// Calendar <-> ZonedDateTime
Calendar cal = Calendar.getInstance();
Instant calInstant = cal.toInstant();
ZonedDateTime zdt = cal.toInstant()
    .atZone(cal.getTimeZone().toZoneId());

// TimeZone <-> ZoneId
TimeZone tz = TimeZone.getDefault();
ZoneId zoneId = tz.toZoneId();

// java.sql.Date <-> LocalDate
java.sql.Date sqlDate = java.sql.Date.valueOf(LocalDate.now());
LocalDate localDate = sqlDate.toLocalDate();

// java.sql.Timestamp <-> LocalDateTime
Timestamp ts = Timestamp.valueOf(LocalDateTime.now());
LocalDateTime fromTs = ts.toLocalDateTime();
```

### 7.2 迁移策略

| 旧类型 | 新类型 | 说明 |
|--------|--------|------|
| `java.util.Date` | `Instant` | 时间戳场景 |
| `java.util.Date` | `LocalDateTime` | 本地日期时间场景 |
| `java.util.Calendar` | `ZonedDateTime` | 带时区的日期时间 |
| `java.util.TimeZone` | `ZoneId` / `ZoneOffset` | 时区信息 |
| `java.text.SimpleDateFormat` | `DateTimeFormatter` | 格式化/解析 |
| `java.sql.Date` | `LocalDate` | 仅日期 |
| `java.sql.Time` | `LocalTime` | 仅时间 |
| `java.sql.Timestamp` | `LocalDateTime` | 日期 + 时间 |

## 8. 常见用例

### 8.1 计算年龄

```java
LocalDate birthday = LocalDate.of(1990, 5, 15);
long age = ChronoUnit.YEARS.between(birthday, LocalDate.now());
// 或
Period period = Period.between(birthday, LocalDate.now());
int years = period.getYears();
```

### 8.2 计算工作日

```java
LocalDate start = LocalDate.of(2014, 3, 1);
LocalDate end = LocalDate.of(2014, 3, 31);
long workDays = start.datesUntil(end) // JDK 9+
    .filter(d -> d.getDayOfWeek().getValue() <= 5)
    .count();
```

### 8.3 时区转换

```java
ZonedDateTime shanghai = ZonedDateTime.now(ZoneId.of("Asia/Shanghai"));
ZonedDateTime newYork = shanghai.withZoneSameInstant(ZoneId.of("America/New_York"));
// 同一时刻，不同时区的显示
```

### 8.4 定时任务中的时间计算

```java
LocalDateTime now = LocalDateTime.now();
LocalDateTime nextRun = now.with(TemporalAdjusters.next(DayOfWeek.MONDAY))
    .withHour(9).withMinute(0).withSecond(0);
Duration delay = Duration.between(now, nextRun);
```
