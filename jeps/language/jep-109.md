# JEP 109: Enhance Core Libraries with Lambda

> **Status**: Final | **Type**: Feature | **Component**: core-libs
>
> **Lead**: Brian Goetz (Oracle)
>
> **JDK Version**: JDK 8 (2014-03)
>
> **Organization**: [Oracle](../../contributors/orgs/oracle.md)

---

## 1. 概述

JEP 109 是一个总括性 JEP，利用 Lambda 表达式和默认方法（JEP 126）对核心库进行增强。在 `Iterable`、`Collection`、`Map`、`Comparator` 等关键接口上添加 lambda 友好的 API，使集合操作更加简洁和声明式。

---

## 2. 背景

JDK 7 及之前的集合操作以外部迭代为主，开发者必须编写显式的 `for` 循环来处理集合元素。Lambda 和默认方法的引入为在现有接口上添加新方法提供了基础，无需破坏二进制兼容性。

---

## 3. Iterable.forEach()

`Iterable` 接口新增 `forEach(Consumer)` 默认方法，替代传统的 `for-each` 循环：

```java
// JDK 8: 内部迭代替代 for-each 循环
list.forEach(System.out::println);
```

---

## 4. Collection.removeIf() 与 Collection.stream()

`Collection` 接口新增两个关键默认方法：

- `removeIf(Predicate)` — 移除所有满足条件的元素
- `stream()` / `parallelStream()` — 返回流对象，开启声明式聚合操作

```java
names.removeIf(String::isEmpty);
List<String> upper = names.stream()
    .filter(s -> s.length() > 3)
    .map(String::toUpperCase)
    .collect(Collectors.toList());
```

---

## 5. Map 接口增强

`Map` 接口新增多个默认方法，消除 null 检查和条件更新的样板代码：

| 方法 | 用途 |
|------|------|
| `forEach(BiConsumer)` | 对每个键值对执行操作 |
| `computeIfAbsent(key, Function)` | 键不存在时计算并插入值 |
| `computeIfPresent(key, BiFunction)` | 键存在时重新计算值 |
| `merge(key, value, BiFunction)` | 合并新旧值 |
| `getOrDefault(key, defaultValue)` | 键不存在时返回默认值 |
| `putIfAbsent(key, value)` | 键不存在时插入 |

---

## 6. Comparator 工厂与组合方法

`Comparator` 接口通过静态工厂方法和默认方法实现声明式比较器构建：

```java
people.sort(Comparator.comparing(Person::getLastName)
                       .thenComparing(Person::getFirstName)
                       .reversed());
```

另有 `naturalOrder()`、`reverseOrder()`、`nullsFirst/Last()` 等辅助方法。

---

## 7. 默认方法的关键作用

JEP 109 的实现完全依赖于接口默认方法。没有默认方法，在 `Iterable`、`Collection`、`Map` 等已有大量第三方实现的接口上添加新方法将破坏兼容性。默认方法允许接口提供默认实现，现有实现类无需修改。

---

## 8. 设计原则

- **内部迭代优于外部迭代** — `forEach` 将控制权交给库，便于并行化
- **行为参数化** — 通过 lambda 传递行为而非数据，减少样板代码
- **惰性求值** — 流管道延迟执行，仅在终端操作时触发计算
