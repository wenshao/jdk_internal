## 1. 概述

JEP 473 是 Stream Gatherers 的第二轮预览（首次预览为 JEP 461，JDK 22）。该特性通过引入 `Gatherer` 接口，允许开发者定义自定义的中间流操作，弥补了 Stream API 中间操作不可扩展的长期缺陷。

---

## 2. 变更内容

`java.util.stream.Gatherer` 接口是 Stream 中间操作的用户自定义扩展点。通过 `Stream.gather(Gatherer)` 方法，开发者可以插入任意有状态或无状态的中间变换逻辑，不再受限于 `map`、`filter`、`flatMap` 等内置操作。

---

## 3. 与前一预览版的差异

相较于 JEP 461（JDK 22 首次预览），本次预览的变更较小：

- **API 细化**：根据社区反馈对 `Gatherer` 接口及内置 gatherer 工厂方法进行了细微调整。
- **无重大语义变更**：核心设计保持不变，本轮预览旨在收集更多实际使用反馈。

---

## 4. 核心 API

| 类/方法 | 说明 |
|---------|------|
| `Gatherer<T,A,R>` | 核心接口，定义自定义中间操作 |
| `Stream.gather(Gatherer)` | 将 Gatherer 应用为中间操作 |
| `Gatherers.fold(...)` | 内置：折叠操作 |
| `Gatherers.scan(...)` | 内置：前缀扫描 |
| `Gatherers.windowFixed(n)` | 内置：固定大小滑动窗口 |
| `Gatherers.windowSliding(n)` | 内置：滑动窗口 |
| `Gatherers.mapConcurrent(n, fn)` | 内置：并发映射（限制并发度） |

---

## 5. 代码示例

```java
// 使用 windowFixed 将流分组为固定大小的批次
List<List<Integer>> batches = Stream.of(1, 2, 3, 4, 5)
    .gather(Gatherers.windowFixed(2))
    .toList();
// [[1, 2], [3, 4], [5]]
```
