## 1. 概述

JEP 355 是文本块（Text Blocks）的第一轮预览，在 JDK 13 中引入。文本块是多行字符串字面量，使用三引号 `"""` 作为定界符，避免了传统字符串中大量的转义序列和字符串拼接，使 HTML、SQL、JSON 等多行文本的嵌入更加清晰易读。

---

## 2. 核心特性

| 特性 | 说明 |
|------|------|
| **三引号定界符** | 以 `"""` 开头（后必须换行）和 `"""` 结尾 |
| **自动缩进管理** | 编译器根据闭合 `"""` 的位置自动去除公共前导空白 |
| **行终止符规范化** | 所有行终止符统一为 `\n`，无论源文件使用何种换行符 |
| **尾部空白去除** | 每行末尾的空白字符默认被去除 |
| **传统转义支持** | 支持 `\n`、`\t`、`\"` 等标准转义序列 |

---

## 3. 缩进管理与代码示例

编译器根据闭合 `"""` 的位置自动去除公共前导空白（对应 `String::stripIndent` 的逻辑）。将闭合定界符左移可保留更多缩进。

```java
String query = """
    SELECT id, name
    FROM users
    WHERE active = true
    """;

String json = """
    {
        "name": "Java",
        "version": 13
    }
    """;
```

文本块的类型是 `java.lang.String`，与普通字符串字面量完全互操作。

---

## 4. 限制与设计决策

- 开头的 `"""` 后不能有内容，必须换行
- 文本块不支持字符串插值（模板功能由后续 JEP 探索）
- 第一轮预览尚未引入 `\s` 和 `\`（行续符）转义序列，这些在第二轮预览中添加
