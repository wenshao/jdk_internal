## 1. 概述

JEP 406 是 switch 模式匹配的第一轮预览，在 JDK 17 中引入。它扩展了 switch 语句和 switch 表达式，允许在 case 标签中使用模式（pattern），而不仅限于常量。这使得 switch 可以对对象的类型和结构进行匹配和解构，大幅简化了复杂的条件分支逻辑。

---

## 2. 核心特性

| 特性 | 说明 |
|------|------|
| **类型模式** | `case Integer i` — 匹配类型并绑定变量 |
| **守卫模式** | `case String s && s.length() > 5` — 模式附加布尔条件 |
| **null case** | `case null` — 显式处理 null，不再自动抛 NullPointerException |
| **密封类穷举** | 配合密封类实现编译期穷举检查，可省略 default |
| **模式变量作用域** | 模式变量仅在对应的 case 分支中可见 |

---

## 3. 类型模式与守卫模式

类型模式（type pattern）将 `instanceof` 的模式匹配能力引入 switch。守卫模式（guarded pattern）使用 `&&` 在模式之后附加条件：

```java
static String describe(Object obj) {
    return switch (obj) {
        case Integer i && i > 0 -> "positive integer: " + i;
        case Integer i          -> "other integer: " + i;
        case String s           -> "string of length " + s.length();
        default                 -> "something else";
    };
}
```

注意：后续预览版本将守卫模式的语法从 `&&` 改为 `when` 关键字。

JEP 406 还引入了 `case null` 显式处理 null 值（不再自动抛 NullPointerException），可与 `default` 合并为 `case null, default`。

---

## 4. 代码示例

```java
// 配合密封类实现穷举匹配
sealed interface Shape permits Circle, Rectangle {}
record Circle(double radius) implements Shape {}
record Rectangle(double w, double h) implements Shape {}

double area(Shape shape) {
    return switch (shape) {
        case Circle c    -> Math.PI * c.radius() * c.radius();
        case Rectangle r -> r.w() * r.h();
        // 无需 default — 密封类保证穷举
    };
}
```

## 5. 演进路径

| JEP | 版本 | 阶段 | 关键变化 |
|-----|------|------|----------|
| **JEP 406** | **JDK 17** | **第一轮预览** | **引入 switch 模式匹配，守卫用 `&&`** |
| JEP 420 | JDK 18 | 第二轮预览 | 守卫模式语法调整 |
| JEP 427 | JDK 19 | 第三轮预览 | `&&` 替换为 `when` 关键字 |
| JEP 433 | JDK 20 | 第四轮预览 | 进一步细化 |
