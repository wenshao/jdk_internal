# JEP 160: Lambda-Form Representation for Method Handles

> **Status**: Permanent | **Type**: Feature | **Component**: Java SE / API
>
> **Lead**: John Rose (Oracle)
>
> **JDK Version**: JDK 8 (2014-03)
>
> **Organization**: [Oracle](../../contributors/orgs/oracle.md)

---

## 概览

引入 Lambda-Form 作为 Method Handles 的中间表示，提供更高效的执行和编译机制。

---

## 目标

- 创建名为 "lambda form" 的中间表示
- 支持直接执行
- 支持直接编译到字节码
- 支持缓存和重用
- 用 JIT 技术替换汇编语言路径

---

## 技术细节

### Lambda-Form 结构

```java
// Lambda-Form 是 method handles 的中间表示
// 比 JVM 字节码更抽象，比 Java 源代码更底层

// 示例：方法组合
MethodHandle mh1 = ...;
MethodHandle mh2 = ...;
MethodHandle combined = MethodHandles.compose(mh1, mh2);

// 内部表示为 lambda form
// 可以被 JIT 编译器直接优化
```

### 执行路径

```
MethodHandle
     ↓
Lambda-Form (中间表示)
     ↓
JIT 编译 → 本地代码
```

### 优势

| 特性 | 传统方式 | Lambda-Form |
|------|----------|-------------|
| **执行速度** | 慢 | 快 |
| **编译开销** | 高 | 低 |
| **缓存支持** | 有限 | 完整 |
| **重用性** | 低 | 高 |

---

## 使用场景

### Lambda 表达式实现

```java
// Lambda 表达式底层使用 lambda-form
List<String> list = Arrays.asList("a", "b", "c");
list.forEach(s -> System.out.println(s));

// 编译器生成 lambda-form
// 而不是匿名内部类
```

### Method Handle 组合

```java
// MethodHandle 组合使用 lambda-form
MethodHandle add = MethodHandles.lookup()
    .findVirtual(java.lang.String.class, "concat", 
                 MethodType.methodType(String.class, String.class));

MethodHandle prefix = MethodHandles.constant(String.class, "Hello ");
MethodHandle result = MethodHandles.foldArguments(add, prefix);

// 内部表示为优化的 lambda-form
```
