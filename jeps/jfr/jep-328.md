# JEP 328: Flight Recorder

> **Status**: Permanent | **Type**: Feature | **Component**: HotSpot / JFR
>
> **Lead**: Marcus Hirt (Oracle)
>
> **JDK Version**: JDK 11 (2018-09)
>
> **Organization**: [Oracle](../../contributors/orgs/oracle.md)

---

## 概览

将 JDK Flight Recorder (JFR) 开源并集成到 OpenJDK 中，提供低开销的性能诊断工具。

---

## 目标

- 提供低开销的事件记录
- 支持生产环境诊断
- 与 JDK 深度集成
- 提供标准化事件 API

---

## 使用方式

### 启用 JFR

```bash
# 基本使用
-XX:StartFlightRecording

# 记录到文件
-XX:StartFlightRecording=filename=recording.jfr

# 指定时长和大小
-XX:StartFlightRecording=duration=60s,maxsize=100M

# 使用预设配置
-XX:StartFlightRecording=profile
```

### JFR 命令行

```bash
# 启动记录
jcmd <pid> JFR.start

# 停止记录
jcmd <pid> JFR.stop

# 转储记录
jcmd <pid> JFR.dump filename=recording.jfr

# 查看配置
jcmd <pid> JFR.check
```

## 事件类型

### JVM 事件

| 事件 | 说明 | 开销 |
|------|------|------|
| **GC 事件** | GC 暂停、堆使用 | 低 |
| **线程事件** | 线程创建、阻塞 | 低 |
| **类加载** | 类加载、卸载 | 低 |
| **编译事件** | JIT 编译 | 低 |

### 应用事件

```java
// 自定义事件
@Name("com.example.MyEvent")
@Label("My Application Event")
public class MyEvent extends Event {
    @Label("Message")
    public String message;
    
    @Label("Duration")
    public long duration;
    
    public void commit() {
        super.commit();
    }
}

// 使用
try (MyEvent e = new MyEvent()) {
    e.message = "Processing";
    e.begin();
    // ... 业务逻辑
    e.duration = System.currentTimeMillis() - start;
    e.commit();
}
```

## 性能开销

### CPU 开销

| 配置 | 开销 | 适用场景 |
|------|------|----------|
| **关闭** | 0% | - |
| **默认** | <1% | 生产环境 |
| **详细** | 1-3% | 测试环境 |
| **调试** | 5-10% | 开发环境 |

### 内存开销

| 配置 | 内存占用 |
|------|----------|
| **默认** | ~10MB |
| **最大缓冲区** | 可配置 (默认 10MB) |
