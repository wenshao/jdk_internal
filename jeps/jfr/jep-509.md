> **Reviewer**: mgronlun

## 1. 背景与动机

### 问题

传统 JFR 执行采样 (Execution Sample) 存在局限：

```
Execution Sample:
- 采样线程执行状态
- 无法区分 CPU 时间 vs 等待时间
- I/O 密集型应用分析不准确
```

示例问题：

```java
// 线程大部分时间在等待
synchronized (lock) {
    // 等待锁 - 不消耗 CPU
    // 但 Execution Sample 会记录为"运行中"
}
```

### 解决方案

JEP 509 引入 **CPU-Time Profiling**：
- 仅采样实际使用 CPU 的时间
- 更准确的性能分析
- 区分 CPU 时间和等待时间

## 2. 使用示例

### 启用 CPU-Time 采样

```bash
# 使用 JFR 启动
java -XX:StartFlightRecording=settings=profile,filename=recording.jfr MyApp

# 或通过 jcmd
jcmd <pid> JFR.start settings=profile
```

### 新增事件

```
jdk.CPUTimeSample           # CPU 时间采样事件
jdk.CPUTimeThreadAllocation # CPU 时间线程分配
```

### JMC 中查看

在 JDK Mission Control 中：
1. 打开 JFR 录制文件
2. 查看 "CPU Time" 视图
3. 分析实际 CPU 热点

### 代码示例

```java
// 应用代码
public class CPUBoundApp {
    public static void main(String[] args) {
        // CPU 密集型操作
        long sum = 0;
        for (long i = 0; i < 1_000_000_000L; i++) {
            sum += i;
        }
        System.out.println("Sum: " + sum);
        
        // I/O 密集型操作
        try {
            Thread.sleep(1000);  // 不消耗 CPU
        } catch (InterruptedException e) {}
    }
}
```

```
Execution Sample 结果:
- main 线程 100% 运行中（不准确）

CPU-Time Sample 结果:
- main 线程 50% CPU（循环计算）
- main 线程 0% CPU（sleep 等待）
```


## 4. 开发者影响

### 配置选项

```bash
# JFR 配置文件 (default.jfc)
<event name="jdk.CPUTimeSample">
  <setting name="enabled">true</setting>
  <setting name="period">10 ms</setting>
</event>

<event name="jdk.CPUTimeThreadAllocation">
  <setting name="enabled">true</setting>
</event>
```

### 与 Execution Sample 对比

| 特性 | Execution Sample | CPU-Time Sample |
|------|------------------|-----------------|
| 采样依据 | 线程状态 | 实际 CPU 使用 |
| I/O 等待 | 记为运行 | 不记录 |
| 准确性 | 中等 | 高 |
| 开销 | 低 | 中等 |

### 最佳实践

```bash
# 生产环境推荐配置
java -XX:StartFlightRecording=\
  settings=profile,\
  filename=recording.jfr,\
  duration=60s \
  MyApp

# 分析 CPU 热点
jfr print --events jdk.CPUTimeSample recording.jfr
```

## 5. 性能影响

### 采样开销

```
配置: 10ms 采样周期

Execution Sample:  ~1% 开销
CPU-Time Sample:   ~2-3% 开销
```

### 准确性提升

```
场景: Web 服务器 (50% CPU, 50% I/O)

Execution Sample 热点:
- 50% request processing
- 30% database wait (误报)
- 20% network wait (误报)

CPU-Time Sample 热点:
- 80% request processing (准确)
- 15% JSON parsing
- 5% serialization
```
