# JEP 471: Deprecate the Memory-Access Methods in sun.misc.Unsafe

> **Status**: Final | **Type**: Feature | **Component**: HotSpot / Runtime
>
> **Lead**: Alan Bateman (Oracle)
>
> **JDK Version**: JDK 23 (2024-09)
>
> **Organization**: [Oracle](../../contributors/orgs/oracle.md)

---

## 概览

废弃 `sun.misc.Unsafe` 中的内存访问方法，引导开发者迁移到受支持的替代 API：`VarHandle` (JDK 9+) 和 `Foreign Function & Memory API` (JDK 22+)。

---

## 目标

- 废弃 `sun.misc.Unsafe` 中的堆内和堆外内存访问方法
- 在运行时发出警告，帮助识别依赖 Unsafe 的代码
- 引导迁移到标准 API

---

## 废弃的方法

### 堆内存访问

```java
// ❌ 以下方法已废弃
Unsafe.getInt(Object, long)
Unsafe.putInt(Object, long, int)
Unsafe.getObject(Object, long)
Unsafe.putObject(Object, long, Object)
// ... 及所有类型变体 (getLong, getDouble, etc.)
```

### 堆外内存访问

```java
// ❌ 以下方法已废弃
Unsafe.allocateMemory(long)
Unsafe.reallocateMemory(long, long)
Unsafe.freeMemory(long)
Unsafe.getInt(long)
Unsafe.putInt(long, int)
// ... 及所有类型变体
```

---

## 迁移方案

### 堆内存访问 → VarHandle

```java
// ❌ Unsafe 旧方式
long offset = unsafe.objectFieldOffset(MyClass.class.getDeclaredField("value"));
int val = unsafe.getInt(obj, offset);

// ✅ VarHandle 替代 (JDK 9+)
VarHandle handle = MethodHandles.lookup()
    .findVarHandle(MyClass.class, "value", int.class);
int val = (int) handle.get(obj);
```

### 堆外内存访问 → Foreign Memory API

```java
// ❌ Unsafe 旧方式
long addr = unsafe.allocateMemory(1024);
unsafe.putInt(addr, 42);
int val = unsafe.getInt(addr);
unsafe.freeMemory(addr);

// ✅ Foreign Memory API 替代 (JDK 22+)
try (Arena arena = Arena.ofConfined()) {
    MemorySegment segment = arena.allocate(1024);
    segment.set(ValueLayout.JAVA_INT, 0, 42);
    int val = segment.get(ValueLayout.JAVA_INT, 0);
}  // 自动释放
```

## 演进历程

| JDK 版本 | 状态 | 关键变化 |
|----------|------|----------|
| **JDK 9** | VarHandle 引入 | JEP 193，提供安全的堆内存访问替代 |
| **JDK 22** | FFM API 正式版 | JEP 454，提供安全的堆外内存访问替代 |
| **JDK 23** | Unsafe 方法废弃 | JEP 471，运行时发出警告 |
| **未来** | 移除 | 计划最终移除 Unsafe 内存访问方法 |

---

## 影响评估

| 受影响场景 | 影响程度 | 替代方案 |
|-----------|----------|----------|
| Netty 堆外缓冲区 | 高 | Netty 4.1.100+ 已适配 |
| LMAX Disruptor | 中 | VarHandle 替代 |
| 序列化框架 (Kryo) | 中 | 更新到兼容版本 |
| 数据库驱动 (直接内存) | 中 | Foreign Memory API |
